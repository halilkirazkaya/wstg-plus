## WSTG-CONF-13 — パス・コンフュージョン (Path Confusion)

パス・コンフュージョン (Path Confusion) は、リバースプロキシ (Reverse Proxy)、ロードバランサー (Load Balancer)、バックエンドアプリケーションサーバ (Backend Application Server) など、異なるウェブコンポーネント間における URL パスの解析および解釈の相違から発生します。攻撃者は、セミコロン、エンコードされたスラッシュ、またはドットセグメント (Dot-segment) などの特定の文字を注入することで、セキュリティフィルタを欺き、制限されたエンドポイントや静的リソースにアクセスするためにこれらの不整合を悪用します。この脆弱性は、フロントエンドがバックエンドとは異なる解釈をするパスに対してセキュリティルールを適用するため、認証バイパス (Authentication Bypass)、不正データアクセス、ウェブキャッシュポイズニング (Web Cache Poisoning) などの重大なセキュリティ障害につながる可能性があります。典型的な悪用は、テックスタック全体でリクエストルーティングと正規化ロジックが乖離するアーキテクチャの境界で発生します。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-CONF-13 |
| **CWE** | CWE-444 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *パス・コンフュージョンによって機密性の高い管理用エンドポイントの認証またはアクセス制御のバイパスが発生した場合、深刻度は「高」となります。

### アーキテクチャの異なるレイヤー間で、パス区切り文字（例: `;`, `#`, `?`）は一貫して解釈されていますか？
- [ ] はい — 全てのレイヤーでパス区切り文字が同一に正規化および解釈されている  
- [ ] いいえ — 不整合が存在するが、制限されたリソースへのアクセスには**利用できない**  
- [ ] いいえ — 相違により、攻撃者がフロントエンドのフィルタをバイパスしてバックエンドのロジックに到達することが**可能である**  

### パストラバーサル (Path Traversal) シーケンスやエンコードされた文字を使用して、アクセス制御をバイパスできますか？
- [ ] いいえ — セキュリティチェックの前に正規化が一貫して適用されている  
- [ ] はい — ドットセグメントシーケンス（例: `/admin/..;/`）を介してバイパスが**可能である**  
- [ ] はい — URL エンコードされた文字（例: `%2f`, `%2e%2e%2f`）を介してバイパスが**可能である**  

### アプリケーションはパス・コンフュージョンによるウェブキャッシュポイズニングの影響を受けますか？
- [ ] いいえ — キャッシュロジックはパスの曖昧さや追加のパス情報の影響を受けない  
- [ ] はい — マッピングの不整合により、正当なパスに対して悪意のあるコンテンツがキャッシュされる**可能性がある**  
- [ ] はい — 相対パス上書き (RPO: Relative Path Overwrite) テクニックを介して、機密情報が公開ディレクトリにキャッシュされる**可能性がある**  

### サーバは「追加のパス情報」(PathInfo) を安全に処理していますか？
- [ ] いいえ — 機能が無効化されている、または存在しない  
- [ ] はい — `PathInfo` は処理されており、セキュリティフィルタを妨害し**ない**  
- [ ] いいえ — `PathInfo` により、攻撃者がアプリケーションのルーティングロジックを混乱させるデータを追加することが**可能である**
