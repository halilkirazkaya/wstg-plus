## WSTG-INPV-15 — HTTP 分割およびリクエストスナグリング (HTTP Splitting Smuggling)

HTTP 分割 (HTTP Splitting) および HTTP リクエストスナグリング (HTTP Request Smuggling) の脆弱性は、フロントエンドプロキシとバックエンドサーバーの間で、HTTP リクエストの境界、特に `Content-Length` および `Transfer-Encoding` ヘッダーの解釈や処理に不一致がある場合に発生します。攻撃者は、曖昧なリクエストを作成することで、隠れたリクエストをバックエンドに「スナグル（密輸）」したり、CRLF シーケンスを注入してレスポンスを分割したりすることができます。これにより、キャッシュポイズニング (Cache Poisoning)、リクエストハイジャック (Request Hijacking)、またはセキュリティ制御のバイパスが引き起こされます。これらの欠陥は通常、リバースプロキシ、ロードバランサー、または CDN が使用されており、パース（解析）ロジックが一貫していない複雑な環境で顕在化します。攻撃者の視点からは、ユーザーのトラフィックの転送、機密性の高いセッショントークンの窃取、および他のユーザーのセッションのコンテキスト内での不正な操作の実行が可能になります。

| **テストステータス** | 実施なし |

### 環境は CL.TE または TE.CL の不一致によるリクエストスナグリングの影響を受けやすいですか？
- [ ] いいえ — フロントエンドとバックエンドのサーバーはリクエストの境界を一貫して処理しています。  
- [ ] はい — 不一致は存在しますが、インフラストラクチャ側の緩和策により悪用は**不可能**です。  
- [ ] はい — CL.TE または TE.CL スナグリングが**可能**であり、隠れたリクエストがバックエンドに到達します *(高)*。  
- [ ] はい — フロントエンドのフィルタをバイパスするための TE.TE（二重エンコード/難読化）が**可能**です *(緊急)*。  

### ヘッダーへの CRLF 注入により HTTP レスポンス分割を実現できますか？
- [ ] いいえ — アプリケーションはすべてのヘッダー入力において CRLF シーケンスを適切にサニタイズしています。  
- [ ] はい — CRLF シーケンスはヘッダーに反映されますが、レスポンス分割は**不可能**です。  
- [ ] はい — CRLF 注入が**可能**であり、ヘッダー注入またはキャッシュポイズニングが可能です。  

### スナグルされたリクエストを使用してセキュリティ制御（WAF/ACL）をバイパスできますか？
- [ ] いいえ — セキュリティ制御は外部リクエストとスナグルされたリクエストの両方に適用されます。  
- [ ] はい — スナグルされたリクエストは、フロントエンドの WAF ルールまたは IP ベースの ACL をバイパス**できます**。  

### 他のユーザーのセッションをハイジャックしたり、トラフィックを転送したりすることは可能ですか？
- [ ] いいえ — リクエスト/レスポンスのストリームは隔離されており、交差することはありません。  
- [ ] はい — リクエストスナグリング (Request Smuggling) が**可能**であり、他のユーザーのリクエストをキャプチャできます *(緊急)*。  
- [ ] はい — レスポンス分割 (Response Splitting) が**可能**であり、ブラウザ側でのキャッシュポイズニングまたは クロスサイトスクリプティング (XSS) が可能です。
