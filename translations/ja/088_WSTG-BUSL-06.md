## WSTG-BUSL-06 — ワークフローの回避 (Circumvention of Work Flows) のテスト

ワークフローの回避 (Workflow circumvention) とは、アプリケーションのロジックを操作して、多段階のプロセス (multi-step processes) 内で意図された順序や前提条件をバイパスすることを指します。攻撃者は、支払い確認、管理者の承認、または多要素認証 (MFA) の完了画面などの機密性の高いエンドポイント (sensitive endpoints) を特定し、必須の中間ステップをスキップしてそれらに直接アクセスしようとします。この脆弱性は、サーバー側のロジックが堅牢なステートマシン (state machine) を維持できず、ユーザーが UI 主導のパスに従うという想定に依存している場合に発生します。悪用に成功すると、不正なトランザクション、権限昇格 (privilege escalation)、本人確認メカニズムのバイパスなど、重大なビジネスロジックの不備 (business logic failures) を招く可能性があります。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-BUSL-06 |
| **CWE** | CWE-841 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *バイパスにより不正な金銭的取引、権限昇格 (privilege escalation)、または管理者アクセスが可能になる場合、深刻度は「高」となります。

### アプリケーションには多段階のビジネスプロセスが含まれていますか？
- [ ] いいえ — アプリケーションは単一リクエストのアクションのみで構成されています  
- [ ] はい — 多段階のプロセス（例：ショッピングカート、ウィザード形式のフォーム、登録など）が存在します  

### ステップの順序はサーバーによって厳格に強制されていますか？
- [ ] はい — サーバー側の状態追跡により、ステップをスキップ (skip) できないようになっています  
- [ ] はい — クライアント側の制御で順序が示されていますが、サーバーは進行状況を検証していません  
- [ ] いいえ — アプリケーションは特定の操作順序を強制していません  

### 攻撃者は、最終段階のURLやエンドポイントを直接リクエストすることで、プロセスの最終段階に到達できますか？
- [ ] いいえ — 事前の完了なしに最終ステップへの直接アクセスは不可能です  
- [ ] はい — 直接リクエストにより最終ステップ（例：`/api/v1/checkout/complete`）に到達可能です  

### アプリケーションは、前のステップからの前提条件となるデータが存在し、有効であることを確認していますか？
- [ ] はい — サーバーは続行前にすべての前のステップのデータを検証しています  
- [ ] いいえ — サーバーは、データが期待されているが検証されていないステップの「スキップ」に対して脆弱です  

### ワークフローを操作することで、MFAや本人確認などのセキュリティ制御をバイパスできますか？
- [ ] いいえ — フローの操作によって重要な制御を回避することはできません  
- [ ] はい — 検証後のステップにジャンプすることで、多要素認証 (MFA) や本人確認のバイパスが可能です
