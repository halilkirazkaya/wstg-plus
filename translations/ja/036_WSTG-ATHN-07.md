## WSTG-ATHN-07 — 脆弱なパスワードポリシーのテスト (Testing for Weak Password Policy)

脆弱なパスワードポリシーのテストでは、アカウント作成や資格情報の更新時にアプリケーションによって強制される複雑性、長さ、およびエントロピ (Entropy) の要件を評価します。不十分なポリシーは、自動化された辞書攻撃 (Dictionary Attack)、ブルートフォース (Brute Force) 試行、およびクレデンシャルスタッフィング (Credential Stuffing) に対してユーザーアカウントを脆弱にし、機密性を直接損ないます。これらの脆弱性は通常、登録エンドポイント、パスワードリセットのワークフロー、および管理用ユーザー管理パネルに存在します。攻撃者の視点からは、寛容なポリシーは、一般的なワードリストやパターンベースのクラッキングを使用して資格情報を奪取するために必要な計算コストと時間を大幅に削減します。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-ATHN-07 |
| **CWE** | CWE-521 |
| **テストステータス** | 実施なし (Not Performed) |
| **深刻度** | 中 / 低* |

> *自動化された推測を防ぐためのアカウントロックアウト (Account Lockout) またはレート制限 (Rate Limiting) メカニズムがアプリケーションに欠けている場合、深刻度は「高」になります。

* https://owasp.org/www-project-web-security-testing-guide/latest/4-Web_Application_Security_Testing/04-Authentication_Testing/07-Testing_for_Weak_Authentication_Methods

**ツール:** `Burp Suite (Intruder)`, `ZAP (Fuzzer)`, `Hydra`, `Hashcat`, `Cewl`

### パスワードの最小長はアプリケーションによって強制されていますか？
- [ ] はい — 最小長は12文字以上 *(最も安全)*  
- [ ] はい — 最小長は8文字から11文字の間  
- [ ] はい — 最小長は **8文字未満**  
- [ ] いいえ — 最小長は強制されていない  

### 複雑性の要件（大文字、小文字、数字、記号）は強制されていますか？
- [ ] はい — 複数の文字種の使用が必須であり、バイパス (Bypass) は不可能  
- [ ] はい — 文字種の使用は推奨されているが、**強制はされていない**  
- [ ] いいえ — 複雑性の要件は適用されていない  

### アプリケーションは、一般的/脆弱なパスワードや、ユーザー固有の情報を含むパスワードをブロックしますか？
- [ ] はい — 既知の脆弱なパスワード（例: "password123"）やユーザー名に基づいたパスワードは**使用できない**  
- [ ] はい — 一般的なパスワードはブロックされるが、ユーザー固有の情報（例: ユーザー名、メールアドレス）は**使用できる**  
- [ ] いいえ — 長さや複雑性の要件を満たす任意の文字列が許可される  

### 複雑性や長さの要件をクライアント側の改ざんによってバイパスできますか？
- [ ] いいえ — ポリシーは**サーバー側 (Server-side)** で厳格に強制されている  
- [ ] はい — ポリシーは JavaScript を介して強制されており、リクエストをインターセプトすることでバイパスが**可能**  

### パスワードポリシーは、実装の詳細を漏洩させることなく、ユーザーに明確に伝えられていますか？
- [ ] はい — 入力時にポリシーが表示され、失敗時には一般的なエラーメッセージが表示される  
- [ ] はい — ポリシーは表示されるが、失敗メッセージによって特定の正規表現 (Regex) やロジックの不備が判明する  
- [ ] いいえ — ポリシーは表示されず、試行錯誤による特定を余儀なくされる
