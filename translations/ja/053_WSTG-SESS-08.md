## WSTG-SESS-08 — セッションパズリング (Session Puzzling)

セッションパズリング (Session Puzzling) は、セッション変数のオーバーロード (Session Variable Overloading) とも呼ばれ、アプリケーションが異なるモジュール間で同じセッション変数を複数の目的で使用したり、ワークフローの遷移中にセッション状態を適切にクリアしなかったりした場合に発生します。攻撃者は、未認証のプロファイルプレビューや多段階の登録プロセスなどの特定のコンテキストでセッション変数に値を入力し、その後、アプリケーションが既存の変数を不当に信頼している保護された領域に遷移することで、この挙動を悪用します。これにより、アプリケーションを欺いて実際よりも高い権限状態にあると誤認させることで、認証バイパス (Authentication Bypass)、権限昇格 (Privilege Escalation)、または機密データへの不正アクセスといった重大な影響を及ぼす可能性があります。この脆弱性は、セッション管理ロジックが異なる機能コンポーネント間で一貫性なく適用されている、複雑でステートフル (Stateful) なアプリケーションで最も多く見られます。

### アプリケーションは、別々のモジュール間で異なる目的のために同じセッション変数名を使用していますか？
- [ ] いいえ — 変数名は一意であるか、スコープが分離されています  
- [ ] はい — 共有変数名が存在しますが、モジュール間で状態がクリアされます  
- [ ] はい — 共有変数名が存在し、状態がクリア**されていません**  

### 未認証のアクションが、認証済みのコンテキストで使用されるセッション変数に影響を与えることができますか？
- [ ] いいえ — セッション変数は認証に成功した**後**にのみ初期化されます  
- [ ] はい — ログイン前にセッション変数を設定できますが、ログイン後の状態には影響し**ません**  
- [ ] はい — 認証前に設定されたセッション変数が、認証後も信頼されます *(緊急)*  

### 特権レベルが変更された際に、アプリケーションはセッション識別子および関連する変数を再初期化しますか？
- [ ] はい — セッションは完全にリセットされ、新しいIDが**発行されます**  
- [ ] 部分的 — 新しいIDは発行されますが、一部のセッション変数が**保持されます**  
- [ ] いいえ — セッションIDとすべての変数は変更**されません**  

### 非特権アカウントを介してセッション変数を操作することで、管理者権限を取得できますか？
- [ ] いいえ — 権限に関する変数はユーザーによって変更**できません**  
- [ ] はい — 変数は変更可能ですが、アプリケーションがバックエンドデータベースに対してそれらを**検証します**  
- [ ] はい — 変数は変更可能であり、アプリケーションはセッション状態を暗黙的に**信頼します** *(緊急)*  

### 特定のワークフロー（例：パスワードリセット、チェックアウト）の完了直後に、セッション状態はクリアされますか？
- [ ] はい — ワークフロー用のセッション変数は完了時に破棄されます  
- [ ] いいえ — ワークフローの変数が**保持され**、他のアプリケーション領域で再利用できます
