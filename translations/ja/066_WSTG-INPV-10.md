## WSTG-INPV-10 — IMAP SMTP インジェクション (IMAP SMTP Injection)

IMAP および SMTP インジェクション (IMAP SMTP Injection) の脆弱性は、ウェブアプリケーションがメールサーバーに送信されるコマンドにユーザー入力を組み込む際、そのデータを不適切にフィルタリングすることで発生します。キャリッジリターンおよびラインフィード (CRLF) 文字を注入することで、攻撃者は本来のコマンドを終了させ、追加の受信者 (CC/BCC) の追加やメッセージ本文の改ざんといった任意のメール命令を付加することができます。これらの欠陥は、Web-to-Mail ゲートウェイ、お問い合わせフォーム、および IMAP4 や SMTP などのプロトコルを介してメールサーバーと直接通信するメールクライアントで頻繁に見られます。攻撃に成功すると、スパムの不正なリレー、機密性の高いメール内容の漏洩 (Exfiltration)、さらにはメールサーバーの完全性が全面的に侵害される可能性があります。

| **CWE** | CWE-93 |
| **テストステータス** | 未実施 |

**ツール:** `Burp Suite (Repeater/Intruder)`, `netcat`, `telnet`, `nmap`

### アプリケーションは、ユーザー指定の入力を使用して IMAP または SMTP コマンドを構築していますか？
- [ ] いいえ — アプリケーションはユーザー入力を介してメールサーバーと**やり取りしません**
- [ ] はい — アプリケーションはメールサーバーとやり取りしますが、安全な API またはライブラリを使用しています
- [ ] はい — アプリケーションはユーザーが制御可能なパラメータを使用して、生のメールコマンドを構築しています

### キャリッジリターン (`\r`) およびラインフィード (`\n`) シーケンスのインジェクションを防止するための入力バリデーションは行われていますか？
- [ ] はい — CRLF 文字は**厳格に**フィルタリング、エンコード、または拒否されています
- [ ] はい — 入力は文字の厳格な許可リスト (Allowlist) に対して検証されています
- [ ] いいえ — CRLF 文字はフィルタリングされておらず、コマンドの終端とインジェクションが可能です

### 攻撃者は `Bcc:` や `Cc:` などの追加のメールヘッダーを注入できますか？
- [ ] いいえ — 厳格な入力制限により、ヘッダーの変更は**不可能です**
- [ ] はい — CRLF シーケンスを介した追加ヘッダーの注入が**可能です**
- [ ] はい — 外部ドメインへのメールリレーが**有効**であり、悪用可能です *(致命的)*

### アプリケーションは、権限のないメールボックスにアクセスするための IMAP コマンドの操作を許可していますか？
- [ ] いいえ — アプリケーションは IMAP を使用していないか、メールボックスへのアクセスを認証済みユーザーに厳密に制限しています
- [ ] はい — IMAP コマンドインジェクション (IMAP Command Injection) は**可能**ですが、メタデータの列挙に限定されています
- [ ] はい — IMAP コマンドインジェクションにより、他のユーザーのメールの読み取りや削除が可能です

### バックエンドのメールサーバーは、コマンドパイプライニングに対して脆弱ですか？
- [ ] いいえ — メールサーバーは、バッチ処理されたコマンドや単一セッション内の複数コマンドを拒否します
- [ ] はい — メールサーバーは、単一の入力フィールドを介して注入された複数のコマンド (Command Pipelining) を処理します
