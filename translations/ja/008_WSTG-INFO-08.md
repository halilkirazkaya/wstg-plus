## WSTG-INFO-08 — Webアプリケーションフレームワークのフィンガープリント (Fingerprint Web Application Framework)

Webアプリケーションフレームワークのフィンガープリント (Fingerprint) とは、アプリケーションの構築と提供に使用されている特定のテクノロジースタックやソフトウェアバージョンを特定することを指します。攻撃者は、HTTPレスポンスヘッダー、フレームワーク固有のクッキー (Cookie)、予測可能なファイル構造、および独自のJavaScriptアーティファクトを分析し、ターゲットが React、Angular、Django、Spring などのプラットフォームを実行しているかどうかを判断します。この偵察フェーズは、既知の脆弱性 (CVE) やそのフレームワーク固有の一般的な設定の弱点に攻撃対象領域を絞り込むことができるため、非常に重要です。基盤となるアーキテクチャを理解することで、攻撃者は一般的なテストから、高度にターゲットを絞ったエクスプロイト (Exploit) 戦略へと移行できるようになります。

### HTTPレスポンスヘッダーがフレームワーク名やバージョンを露出させていますか？
- [ ] いいえ — `X-Powered-By` や `Server` などのヘッダーは存在しないか、サニタイズ (Sanitize) されています
- [ ] はい — フレームワーク名は存在しますが、バージョンは**隠されています**
- [ ] はい — フレームワーク名とバージョンの**両方が露出しています**

### フレームワーク固有のクッキーやディレクトリ構造が特定可能ですか？
- [ ] いいえ — 一般的なフレームワークのアーティファクト（例：`.js` パス、クッキー名）は難読化されているか、名前が変更されています
- [ ] はい — フレームワーク固有のクッキー（例：`JSESSIONID`, `PHPSESSID`, `csrftoken`）が**特定可能です**
- [ ] はい — ディレクトリ構造（例：`/wp-content/`, `_next/static/`）が**可視化されており**、フレームワークを裏付けています

### エラーメッセージやソースコードのコメントにフレームワークの詳細が含まれていますか？
- [ ] いいえ — エラーハンドリングは一般的であり、コメントは削除されています
- [ ] はい — フレームワーク固有のスタックトレースが**有効になっており**、レスポンス内で確認できます
- [ ] はい — HTMLソースコードにフレームワークを特定するコメントやメタデータタグが含まれています

### 特定されたフレームワークのバージョンに関連する既知の脆弱性はありますか？
- [ ] いいえ — 特定されたフレームワークは最新であり、公開されている既知の脆弱性は**ありません**
- [ ] はい — フレームワークのバージョンが古く、特定の CVE が**特定可能です**
