## WSTG-ATHZ-04 — 不適切な直接オブジェクト参照のテスト

不適切な直接オブジェクト参照 (Insecure Direct Object References (IDOR)) は、アプリケーションがリクエスト送信者の権限を確認するための認可 (Authorization) チェックを行わずに、ユーザーから提供された入力に基づいてオブジェクトへの直接アクセスを許可する場合に発生します。この脆弱性は、攻撃者が他のユーザーやシステムのデータを閲覧、変更、または削除することを可能にするため、機密性と完全性に重大な影響を及ぼします。通常、内部データベースのキー、ファイル名、またはアカウント識別子を参照する URL パラメータ、POST ボディデータ、または JSON キーに現れます。攻撃者の視点からは、整数のインクリメントや UUID のファジング (Fuzzing) などによってこれらの識別子を操作し、認可された範囲外のリソースにアクセスすることで悪用が行われます。

### アプリケーションのリクエストパラメータで直接オブジェクト識別子が使用されていますか？
- [ ] いいえ — アプリケーションは間接的な参照または暗号化されたマップを使用している *(最も安全)*  
- [ ] はい — アプリケーションは予測不可能な識別子（例：長い UUID やハッシュ）を使用している  
- [ ] はい — アプリケーションは予測可能な識別子（例：連続した整数やユーザー名）を使用している  

### オブジェクト識別子を含むすべてのリクエストに対して、サーバーサイドで認可が実行されていますか？
- [ ] はい — サーバーサイドのチェックをいかなる識別子に対してもバイパスすることは**不可能**である  
- [ ] はい — サーバーサイドのチェックは存在するが、パラメータ汚染 (Parameter Pollution) やヘッダー操作 (Header Manipulation) を通じてバイパスが**可能**である  
- [ ] いいえ — リクエストされたオブジェクトの所有権を確認するための認可チェックが**適用されていない**  

### 攻撃者が他のユーザーに属するオブジェクトにアクセスまたは変更できますか (水平的な IDOR)？
- [ ] いいえ — 他のユーザーのデータへのアクセスは**不可能**である  
- [ ] はい — 他のユーザーのデータの閲覧は**可能**だが、変更は**不可能**である  
- [ ] はい — 他のユーザーのデータの閲覧と変更の両方が**可能**である *(致命的)*  

### 管理レベルまたはシステムレベルのオブジェクトにアクセスすることは可能ですか (垂直的な IDOR)？
- [ ] いいえ — より高い権限を持つオブジェクトへのアクセスは**不可能**である  
- [ ] はい — 管理設定やシステムファイルへのアクセスが**可能**である  

### アプリケーションが検索、メタデータ、または他のエンドポイントを介してオブジェクト識別子を漏洩させていますか？
- [ ] いいえ — 識別子が意図せず**露出することはない**  
- [ ] はい — 他のユーザーやオブジェクトの識別子が、二次的なエンドポイントや公開プロファイルを通じて**列挙可能**である
