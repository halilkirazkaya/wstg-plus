## WSTG-CONF-11 — クラウドストレージのテスト (Test Cloud Storage)

クラウドストレージの設定ミス (Cloud Storage Misconfigurations) のテストでは、Amazon S3、Azure Blobs、Google Cloud Storageなどのパブリックにアクセス可能なオブジェクトストレージ (Object Storage) サービスを特定し、監査します。これらのリソースには、過剰に許可されたアクセス制御リスト (ACL) やアイデンティティおよびアクセス管理 (IAM) ポリシーによって、バックアップ、ソースコード、個人識別情報 (PII)、または設定ファイルなどの機密データが含まれていることがよくあります。攻撃者は、JavaScriptファイルやHTMLソースの偵察、およびブルートフォース (Brute Force) 手法を通じてバケット名を列挙 (Enumerate) し、データの持ち出し (Data Exfiltration) や不正なファイル変更の侵入口を探します。悪用されると、完全なデータ侵害 (Data Breaches) や、信頼されたドメインから悪意のあるファイルを配信されるといった、重大な影響を及ぼす可能性があります。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-CONF-11 |
| **CWE** | CWE-922 |
| **テストステータス** | 未実施 (Not Performed) |
| **深刻度** | 高 (High) / 緊急 (Critical)* |

> *PII、認証情報 (Credentials)、または本番環境のバックアップにアクセス可能な場合、あるいは認証なしの書き込みアクセスが有効な場合、深刻度は「緊急」となります。

### クラウドストレージのエンドポイント (バケット/コンテナ) は特定・列挙されていますか？
- [ ] いいえ — クラウドストレージのエンドポイントは使用されていない、または検出不可能です  
- [ ] はい — コード解析またはトラフィック監視を通じてストレージエンドポイントが特定されています  
- [ ] はい — 命名規則に対するブルートフォースによりストレージエンドポイントが検出されています  

### 未認証のユーザーがクラウドストレージの内容を一覧表示 (List) できますか？
- [ ] いいえ — バケットの一覧表示は**無効化**されており、403 Forbiddenまたは404 Not Foundを返します  
- [ ] はい — 一覧表示は**有効**ですが、機密ファイルは存在しません  
- [ ] はい — 一覧表示は**有効**であり、機密ファイルの名称やメタデータ (Metadata) が閲覧可能です  

### 特定されたバケットから機密ファイルをダウンロードすることは可能ですか？
- [ ] いいえ — ファイルはIAMポリシーによって保護されているか、有効な認証が必要です  
- [ ] はい — ファイルにアクセス可能ですが、推測が困難な署名付きURL (Signed URL) が必要です  
- [ ] はい — 機密ファイル (バックアップ、`.env`、PII) がダウンロードのために**パブリックにアクセス可能**です *(緊急)*  

### 未認証のファイルアップロードや変更は許可されていますか？
- [ ] いいえ — 未認証のアップロードや変更は**不可能です**  
- [ ] はい — 認証されたアップロードが必要ですが、弱いポリシー条件によりバイパス (Bypass) が**可能です**  
- [ ] はい — ファイルの未認証の書き込み、上書き、または削除が**可能です** *(緊急)*  

### ストレージエンドポイントのオリジン間リソース共有 (CORS) ポリシーは制限されていますか？
- [ ] はい — CORSは**無効**、または特定の信頼されたオリジンに制限されています  
- [ ] いいえ — CORSがワイルドカード (Wildcard) `*` オリジンで**有効**になっており、Webベースの不正アクセスを許可しています
