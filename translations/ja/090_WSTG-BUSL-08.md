## WSTG-BUSL-08 — 予期しないファイル形式のアップロードのテスト

予期しないファイル形式 (Unexpected file types) のアップロードは、アプリケーションが処理を想定していないファイルを送信することで、攻撃者がビジネスロジック (Business Logic) の制約を回避することを可能にします。これにより、リモートコード実行 (Remote Code Execution (RCE))、クロスサイトスクリプティング (Cross-Site Scripting (XSS))、またはサービス拒否 (Denial of Service (DoS)) につながる可能性があります。攻撃者は、ファイル拡張子、MIMEタイプ (MIME type)、およびマジックバイト (Magic Bytes) を変更することで、サーバー側のバリデーション (Validation) ロジックを欺き、悪意のあるコンテンツを受け入れさせることで、これらの脆弱性を調査します。これは通常、バックエンドでのバリデーションが不十分な、プロフィール画像のアップロード、ドキュメント管理システム、またはサポートチケットの添付ファイルなどで発生します。攻撃に成功すると、アップロードされたファイルが実行された場合にホストサーバーが侵害されたり、ファイルが誤った Content-Type で他のユーザーに提供された場合にクライアントサイドの攻撃につながったりする可能性があります。

### アプリケーションは、ファイルのアップロードを特定の拡張子セットに制限していますか？
- [ ] はい — 拡張子の厳格なホワイトリスト (Whitelist) が適用されており、回避は**不可能**である  
- [ ] はい — 拡張子のホワイトリストは存在するが、二重拡張子やヌルバイト (Null Bytes) によって回避が**可能**である  
- [ ] いいえ — 任意のファイル拡張子をアップロード**できる**  

### ファイルの内容は、ファイル拡張子以外にも検証されていますか？
- [ ] はい — サーバー側のロジックがマジックバイトを検証し、詳細なコンテンツ検査を実行している  
- [ ] はい — サーバー側のロジックが `Content-Type` ヘッダーのみを介して MIME タイプを検証している  
- [ ] いいえ — 拡張子のチェック後、コンテンツのバリデーションは**適用されない**  

### 攻撃者は、ウェブシェル (Web Shell) やスクリプトをアップロードすることでコードを実行できますか？
- [ ] いいえ — アップロードされたファイルは、実行不可能なディレクトリまたはストレージバケット (S3/Azure Blob) に保存されている  
- [ ] はい — ファイルはウェブからアクセス可能なディレクトリに保存されているが、サーバー設定によって実行が**無効化**されている  
- [ ] はい — アップロードされた悪意のあるスクリプトは、予測可能な URL パスを介して直接実行される**可能性がある** *(緊急)*  

### アップロードされたファイル形式を介して、XSS などのクライアントサイド攻撃が可能ですか？
- [ ] いいえ — ファイルは `Content-Disposition: attachment` および `X-Content-Type-Options: nosniff` と共に提供されている  
- [ ] はい — SVG または HTML ファイルをアップロードしてブラウザでレンダリングさせることができ、XSS につながる  
- [ ] はい — 画像のメタデータ (EXIF) が**処理され**、サニタイズ (Sanitization) されずに UI に反映される
