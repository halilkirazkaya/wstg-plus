## WSTG-CONF-09 — ファイルパーミッションのテスト

ファイルパーミッション（File Permission）のテストには、Webサーバー上のファイルやディレクトリに割り当てられたアクセスコントロール（Access Control）レベルを監査し、機密リソースが権限のないユーザーやプロセスに過度に公開されていないかを確認することが含まれます。不適切に設定されたパーミッションは、攻撃者がデータベースの認証情報を含む機密設定ファイルを読み取ったり、独自のソースコードを閲覧したり、サーバー側のスクリプトを修正してリモートコード実行（Remote Code Execution）を達成することを可能にします。この脆弱性は通常、デプロイメント（Deployment）フェーズにおいて、設定パス、ログフォルダ、アップロードリポジトリなどの重要なアプリケーションディレクトリに、デフォルトの「全ユーザー読み取り可能（World-readable）」または「全ユーザー書き込み可能（World-writable）」フラグが残されたままになることで発生します。攻撃者の視点からは、不適切に保護されたファイルを発見することは、権限昇格（Privilege Escalation）やシステム全体の侵害（System Compromise）の主要なきっかけとなります。

| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-CONF-09 |
| **CWE** | CWE-732 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *機密性の高い設定ファイル（例: `.env`, `web.config`）が読み取り可能な場合、またはWebルートへの書き込みアクセスが可能な場合、深刻度は「高」となります。

**ツール:** `ls`, `find`, `LinPeas`, `WinPeas`, `ffuf`, `dirsearch`, `Nmap`

### 機密性の高いアプリケーション設定ファイルは、不正アクセスから保護されていますか？
- [ ] はい — 設定ファイル（例: `.env`, `config.php`）はWebリクエスト経由で**アクセス不能**である  
- [ ] はい — ファイルは存在するが、アクセスは許可されたローカルユーザーのみに**制限**されている  
- [ ] いいえ — 機密設定ファイルに**アクセス可能**であり、認証情報やシークレットが漏洩している  

### 攻撃者はWebルート内のファイルやディレクトリを修正できますか？
- [ ] いいえ — すべてのWebルートディレクトリは、Webサーバーユーザーに対して**読み取り専用**である  
- [ ] はい — 書き込み可能なディレクトリが存在するが、スクリプトの実行は**無効化**されている  
- [ ] はい — 全ユーザー書き込み可能なディレクトリが存在し、スクリプトのアップロードと実行を**許可**している（致命的）  

### パーミッションの不備により、バージョン管理のメタデータやバックアップファイルが公開されていませんか？
- [ ] いいえ — `.git`, `.svn`、およびバックアップファイル（例: `.bak`, `~`）は**存在しない**か、保護されている  
- [ ] はい — メタデータディレクトリは存在するが、ディレクトリリスティング（Directory Listing）は**無効化**されている  
- [ ] はい — 機密メタデータやバックアップに**完全にアクセス可能**であり、ソースコードの復元が可能である  

### Webサーバーユーザーは最小権限の原則（Principle of Least Privilege）に基づいて動作していますか？
- [ ] はい — Webサーバープロセスは**専用の低権限**ユーザーとして実行されている  
- [ ] いいえ — Webサーバープロセスが**不要な権限**（例: `root` または `Administrator`）で実行されている  

### ログファイルは、不正な読み取りや修正から保護されていますか？
- [ ] はい — ログファイルへのアクセスは管理者ユーザーに**制限**されている  
- [ ] はい — ログファイルは読み取り可能だが、セッショントークンや個人情報（PII）は**含まれていない**  
- [ ] いいえ — ログファイルが**パブリックに読み取り可能**であり、機密性の高いトランザクションデータやユーザー情報が漏洩している
