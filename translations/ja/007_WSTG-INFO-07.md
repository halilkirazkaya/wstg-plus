## WSTG-INFO-07 — アプリケーションの実行パスのマッピング (Map Execution Paths Through Application)

実行パスのマッピングには、Webアプリケーション内のすべての到達可能なエンドポイント (Endpoints)、機能フロー、および意思決定分岐の体系的な特定が含まれます。このプロセスは、レガシーコード、デバッグインターフェース、または未文書化のAPIルートといった「ダーク」な機能が、セキュリティの精査から隠れたままにならないようにするために不可欠です。これらは多くの場合、現代的なセキュリティ制御が欠如しているためです。攻撃者は、自動化されたスパイダリング (Spidering) と手動探索 (Manual Exploration) を組み合わせてアプリケーションの構造を可視化し、その過程で不正な管理者アクセスやビジネスロジック (Business Logic) の脆弱性を発見することがよくあります。入力を特定のサーバーサイドのレスポンスに関連付けることで、テスターは堅牢なカバレッジを確保するために重点的なディープダイブテスト (Deep-dive Testing) を必要とする、機密性の高い処理ロジックをピンポイントで特定できます。

| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-INFO-07 |
| **CWE** | CWE-200 |
| **テストステータス** | 未実施 (Not Performed) |
| **深刻度** | Informational (情報) |

### 自動および手動のクローリングを使用して、アプリケーションは完全にインデックス化されていますか？
- [ ] はい — すべての可視リソースおよびリンクされたリソースの包括的なマップ作成が**完了している**  
- [ ] はい — 自動クローリングは**完了している**が、手動探索は保留中である  
- [ ] いいえ — アプリケーションはインデックス化**されていない**  

### JS解析またはディレクトリのブルートフォース (Brute Force) を通じて、隠しエンドポイントや未文書化のエンドポイントが特定されていますか？
- [ ] いいえ — サイドチャネル分析による未文書化のパスは発見されなかった  
- [ ] はい — 隠しパスが発見されたが、権限なしではアクセス**できない**  
- [ ] はい — 未文書化のエンドポイントにアクセス**可能であり**、機密性の高い機能を提供している *(緊急 / 高)*  

### 非標準のパラメータやヘッダーを介して実行パスを変更できますか？
- [ ] いいえ — `debug`、`test`、`admin` などのパラメータは実行フローに影響を**与えない**  
- [ ] はい — パラメータによって出力は変化するが、セキュリティ制御のバイパス (Bypass) は**できない**  
- [ ] はい — パスを変更するパラメータが**適用され**、意図されたロジックのバイパスが可能である  

### 多段階のビジネスロジック（例：多要素認証、チェックアウト）のフローは完全にマッピングされていますか？
- [ ] はい — 多段階プロセスにおけるすべての状態と遷移が**特定されている**  
- [ ] いいえ — 複雑な状態マシンの遷移を完全にマッピングすることが**できない**  

### APIドキュメントファイル (Swagger/OpenAPI) やクライアントサイドのマップ (Source Maps) は公開されていますか？
- [ ] いいえ — アーキテクチャマップやドキュメントファイルは公開されていない  
- [ ] はい — ドキュメントは存在するが、認証によって**保護されている**  
- [ ] はい — Swagger UI、OpenAPI JSON、または JS Source Maps が**有効になっており**、公開アクセスが可能である
