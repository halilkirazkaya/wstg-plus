## WSTG-INPV-19 — サーバーサイドリクエストフォージェリ (SSRF) のテスト

サーバーサイドリクエストフォージェリ (SSRF) は、攻撃者がウェブアプリケーションを操作して、サーバーから内部または外部のリソースに対して不正なリクエストを行わせることで発生します。URL、ホスト名、またはIPアドレスを期待するパラメータを操作することにより、攻撃者はファイアウォールやアクセス制御リスト (ACL) などのネットワークレベルの制御をバイパスし、内部ネットワークセグメントの調査、クラウドメタデータサービス (IMDS) へのアクセス、あるいは脆弱な内部APIやデータベースとの対話を行うことができます。この脆弱性は通常、画像取得、PDF生成、Webhook、またはURLによるファイルアップロードなどの機能において顕著に現れます。攻撃者の視点からは、SSRFは、隔離された環境へのピボット（足がかりの構築）、機密性の高い設定データの窃取、あるいはRedisやMemcachedなどの内部サービスとの対話を通じたリモートコード実行 (RCE) を達成するために使用される強力な手法です。

### アプリケーションはユーザーが提供したURLまたはホスト名を処理しますか？
- [ ] いいえ — 外部URLまたはホスト名を受け入れる機能はありません  
- [ ] はい — 機能は存在しますが、入力は許可リスト（Allow-list）に対して厳格に検証されています  
- [ ] はい — 機能が存在し、ユーザーが提供したURLを受け入れます  

### リクエスト先に対して検証コントロールまたは拒否リストが適用されていますか？
- [ ] はい — 信頼されたドメイン/IPの厳格な**許可リスト**が適用されています  
- [ ] はい — **拒否リスト**（Deny-list）が使用されています（例：`127.0.0.1`、`169.254.169.254` のブロック）  
- [ ] いいえ — 入力に対して**検証やフィルタリング**は適用されていません  

### エンコーディング、リダイレクト、またはDNSリバインディングを介してフィルタをバイパスすることは可能ですか？
- [ ] いいえ — フィルタは堅牢であり、リダイレクトやDNS解決を安全に処理します  
- [ ] はい — URLエンコーディングまたは代替IP形式（16進数、8進数）を使用してバイパスが**可能です**  
- [ ] はい — 内部ターゲットへの3xx HTTPリダイレクトを介してバイパスが**可能です**  
- [ ] はい — 内部IPに解決させるDNSリバインディング (DNS Rebinding) 攻撃を介してバイパスが**可能です**  

### アプリケーションは内部メタデータサービスまたはローカルインターフェースに到達できますか？
- [ ] いいえ — ローカルネットワークおよびクラウドメタデータエンドポイントには**到達できません**  
- [ ] はい — localhost (`127.0.0.1`) またはループバックサービスへのアクセスが**可能です**  
- [ ] はい — クラウドメタデータサービス（例：AWS/GCP/Azure IMDS）へのアクセスが**可能です** *(緊急)*  

### SSRFレスポンスの性質は何ですか？
- [ ] ブラインド (Blind) — レスポンスやメタデータはユーザーに返されません  
- [ ] パーシャル (Partial) — レスポンスのメタデータ（ヘッダー、サイズ、タイミング）がユーザーに返されます  
- [ ] フル (Full) — 内部リソースからの完全なレスポンスボディがユーザーに**反映されます**
