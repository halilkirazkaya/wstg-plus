## WSTG-INFO-02 — Webサーバのフィンガープリント (Web Server Fingerprinting)

Webサーバのフィンガープリント (Web Server Fingerprinting) とは、対象となるWebサーバの特定のソフトウェアの種類、バージョン、および基盤となるオペレーティングシステム (OS) を特定するプロセスです。攻撃者は、Apache、Nginx、IIS、またはその他のサーバ技術の特定のバージョンに固有の、未パッチの CVE や設定の脆弱性などの既知の脆弱性を特定するために、この偵察を実施します。これは通常、HTTP レスポンスヘッダ（例: `Server`、`X-Powered-By`）の分析、固有のプロトコル動作の調査、またはデフォルトのエラーページやファイルを通じて漏洩した情報の観察によって行われます。フィンガープリントの成功により、攻撃者は攻撃戦略を調整し、一般的なスキャンから、既知のアーキテクチャ上の欠陥に対する精度の高い攻撃へと移行することが可能になります。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-INFO-02 |
| **CWE** | CWE-200 |
| **テストステータス** | 未実施 |
| **深刻度** | 情報 (Informational) / 低 (Low)* |

> *深刻な既知の脆弱性を持つ、サポート終了 (EOL) または古いサーババージョンがフィンガープリントによって判明した場合、深刻度は「高 (High)」になります。

### HTTP レスポンスヘッダに識別文字列が含まれていますか？
- [ ] いいえ — `Server` および `X-Powered-By` ヘッダが存在しない、または汎用的な値が含まれている  
- [ ] はい — ヘッダからサーバソフトウェアは判明するが、特定のバージョン番号は判明しない  
- [ ] はい — ヘッダから特定のサーバソフトウェアと正確なバージョン番号が判明する  

### デフォルトのエラーページやシステム生成のレスポンスからサーバの詳細が漏洩していますか？
- [ ] いいえ — カスタムエラーページが使用されており、サーバ情報は開示されていない  
- [ ] はい — デフォルトのエラーページからサーバソフトウェア名および/またはバージョンが漏洩している  
- [ ] はい — エラーページからサーバの詳細、バージョン、および基盤となるオペレーティングシステム (OS) が漏洩している  

### デフォルトのファイルやディレクトリ構造を通じてサーバのバージョンが開示されていますか？
- [ ] いいえ — デフォルトのインストールファイル、マニュアル、およびテストスクリプトは削除されている  
- [ ] はい — デフォルトのファイル（例: `info.php`、`manual/`、`test.html`）が存在し、バージョンデータを開示している  

### 固有の動作分析を通じてサーバタイプを正確に推測できますか？
- [ ] いいえ — サーバのレスポンスは正規化されており、動作ベースのフィンガープリントは不可能である  
- [ ] はい — 固有の動作（例: ヘッダの順序、特定のエラーコード、または TCP/IP スタックの特性）により、サーバの特定が可能である  

### 特定されたサーババージョンは現在サポートされており、パッチが適用されていますか？
- [ ] はい — 特定されたバージョンは最新であり、ベンダーによってサポートされている  
- [ ] いいえ — 特定されたバージョンは古くなっているか、サポート終了 (EOL) であり、既知の脆弱性が存在する
