## WSTG-INFO-06 — アプリケーションのエントリポイントの特定 (Identify Application Entry Points)

アプリケーションのエントリポイント（Entry Points）の特定は、ユーザーや外部システムがアプリケーションと対話可能なすべての方法を列挙（Enumeration）し、アタックサーフェス（攻撃対象領域：Attack Surface）全体をマッピングすることを含みます。このプロセスには、すべてのURL、APIエンドポイント（API Endpoints）、パラメータ（GET、POST、クッキーベース）、HTTPヘッダー、および WebSockets や gRPC といった特殊なプロトコルの発見が含まれます。ペネトレーションテスターにとって、この段階は非常に重要です。なぜなら、文書化されていない「シャドウAPI（Shadow APIs）」、レガシーエンドポイント、または開発中に見落とされた複雑なパラメータ構造の中に脆弱性が発見されることが多いためです。徹底的な列挙により、アプリケーションのどのコンポーネントもテストされていないブラックボックスとして残らないようにし、攻撃者がシステムへの監視されていないルートを見つけることを防ぎます。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-INFO-06 |
| **CWE** | CWE-1059 |
| **Test Status** | 未実施 |
| **Severity** | インフォメーショナル (Informational) |

### アプリケーション全体で、すべての GET および POST パラメータが列挙されていますか？
- [ ] はい — 自動および手動のクローリングによる包括的な列挙が**完了しています**  
- [ ] はい — 標準的なクローリングにより一般的なパラメータのみが特定されています  
- [ ] いいえ — パラメータはほとんど**特定されていない**か、テストされていません  

### ファジング (Fuzzing) を使用して、文書化されていない、または隠しパラメータ（例：debug、admin）を探索していますか？
- [ ] いいえ — この特定のスコープにおいて、隠しパラメータの発見は**不要です**  
- [ ] はい — `Arjun` などを用いたパラメータのファジングが**実施されました**が、知見（Findings）はありませんでした  
- [ ] はい — 隠しパラメータが**発見され**、さらなるテストのためにマッピングされました  
- [ ] いいえ — 隠しパラメータの探索は**実施されませんでした**  

### 各エンドポイントに対して、サポートされているすべての HTTP メソッド（PUT、DELETE、PATCHなど）が特定されていますか？
- [ ] はい — 発見されたすべてのエンドポイントに対してメソッドの特定が**適用されています**  
- [ ] はい — 関心の高い、または機密性の高いエンドポイントに対してのみメソッドの特定が**適用されています**  
- [ ] いいえ — 標準的な GET および POST メソッドのみが**特定されています**  

### WebSockets、gRPC、カスタムヘッダーなどの非標準的なエントリポイントはマッピングされていますか？
- [ ] いいえ — アプリケーションは非標準的なプロトコルやカスタムヘッダーを**使用していません**  
- [ ] はい — すべてのプロトコル固有のエントリポイントおよびカスタムヘッダーが**特定されています**  
- [ ] いいえ — 非標準的なエントリポイントは**存在します**が、マッピングされていません  

### APIのアタックサーフェス（/v1/ や /v2/ などのバージョン管理されたエンドポイントを含む）は完全にマッピングされていますか？
- [ ] いいえ — アプリケーションは API を**公開していません**  
- [ ] はい — すべての API バージョンとエンドポイントが**特定され**、文書化されています  
- [ ] はい — 現在の商用環境の API バージョンのみが**特定されています**  
- [ ] いいえ — API エンドポイントが**未マッピングである**か、一部しか発見されていません
