## WSTG-ATHN-02 — デフォルトの認証情報のテスト (Testing for Default Credentials)

デフォルトの認証情報 (Default Credentials) のテストでは、工場出荷時設定またはベンダー提供のユーザー名とパスワードを依然として使用している管理インターフェース、サービス、またはアプリケーションコンポーネントの特定を行います。この脆弱性は、最小限の労力でシステム全体の完全な侵害、管理者アクセス、またはリモートコード実行 (Remote Code Execution; RCE) への即時の経路を提供することが多いため、攻撃者にとって優先度の高い標的となります。これは通常、既製品のソフトウェア、コンテンツ管理システム (CMS) プラットフォーム、データベース管理ツール、および IoT デバイスやネットワーク機器などの統合ハードウェアで発生します。悪用は通常、特定されたソフトウェアバージョンを公開されているデフォルトパスワードデータベースと照合するか、初期の偵察および悪用フェーズで自動化されたクレデンシャルスタッフィング (Credential Stuffing) ツールを使用することによって行われます。

### 管理用またはマネジメントインターフェースが、インターネットまたは信頼できないネットワークセグメントに公開されていますか？
- [ ] いいえ — 管理インターフェースにアクセスできません  
- [ ] はい — インターフェースは存在しますが、ネットワークレベルの制御（例：VPN、IP許可リスト）によって制限されています  
- [ ] はい — インターフェースは公開されており、認証が必要です  

### 特定されたすべてのコンポーネントにおいて、ベンダー提供のデフォルトの認証情報が変更されていますか？
- [ ] はい — すべてのコンポーネントにわたって、すべてのデフォルトの認証情報が変更されています *(最も安全)*  
- [ ] はい — メインアプリケーションの認証情報は変更されていますが、二次的なコンポーネント（例：CMS、データベース）はデフォルトのままです  
- [ ] いいえ — 1つ以上の特定可能なインターフェースでデフォルトの認証情報が有効です *(緊急)*  

### アプリケーションは、新規アカウントまたは管理者アカウントの初回ログイン時にパスワード変更を強制しますか？
- [ ] はい — 管理者アクションを実行する前に、パスワード変更が必須となっています  
- [ ] いいえ — アプリケーションはデフォルトの認証情報を無期限に使用することを許可しています  

### 自動化されたデフォルトの認証情報テストを防止するための、ロックアウトメカニズムまたはレート制限 (Rate Limiting) の制御が有効になっていますか？
- [ ] はい — 強力なレート制限またはアカウントロックアウトが適用されています  
- [ ] はい — 制御は存在しますが、ヘッダー操作または IP ローテーションによるバイパスが可能です  
- [ ] いいえ — レート制限またはロックアウトメカニズムが有効になっていません  

### サードパーティのプラグイン、ミドルウェア、または管理ツール（例：`phpMyAdmin`, `Tomcat Manager`）は一意の認証情報を使用していますか？
- [ ] いいえ — 環境内にサードパーティのコンポーネントは存在しません  
- [ ] はい — すべてのサードパーティコンポーネントは、デフォルトではない一意の認証情報を使用しています  
- [ ] いいえ — 少なくとも1つのサードパーティコンポーネントが、既知のデフォルト認証情報でアクセス可能です
