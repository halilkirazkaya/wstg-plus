## WSTG-ATHN-06 — ブラウザキャッシュの脆弱性のテスト

ブラウザキャッシュの脆弱性 (Browser Cache Weakness) は、機密情報がローカルのブラウザキャッシュに保存され、同じ物理マシンにアクセスできる権限のないユーザーによって取得可能になった場合に発生します。適切なキャッシュ制御ヘッダー (Cache Control Headers) が欠如していると、ユーザーがログアウトしたりセッションを終了したりした後も、個人情報、アカウントの詳細、セッション識別子 (Session Identifiers) などの機密データが保持される可能性があります。攻撃者や、共有端末または公共端末の後続ユーザーは、ブラウザの履歴をたどる、「戻る」ボタンを使用する、あるいはローカルのキャッシュファイルを調査することで、キャッシュされたレスポンスを窃取 (Exfiltrate) し、これを悪用することができます。認証されたコンテンツを処理するアプリケーションにおいては、機密データがディスクに書き込まれないようにするため、`Cache-Control: no-store` や `Pragma: no-cache` といった厳格なディレクティブを実装することが、セキュリティ上極めて重要です。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-ATHN-06 |
| **CWE** | CWE-525 |
| **テストステータス** | 未実施 |
| **深刻度** | 低 / 中* |

> *アプリケーションが共有/公共のキオスクから頻繁にアクセスされる場合、または非常に機密性の高い個人を特定できる情報 (PII) や保護対象保健情報 (PHI) を含む場合、深刻度は「中」となります。

### 認証済みページまたは機密ページに適切なキャッシュ制御ヘッダーが存在するか？
- [ ] はい — `Cache-Control: no-store, no-cache` および `Pragma: no-cache` が**存在し**、**正しく設定されている**  
- [ ] はい — 一部のヘッダーは存在するが `no-store` が**欠落しており**、ディスクへのキャッシュを許容している  
- [ ] いいえ — キャッシュ関連のヘッダーが**適用されておらず**、ブラウザのデフォルトの動作に依存している  

### アプリケーションはユーザー固有のデータに対して `private` ディレクティブを使用しているか？
- [ ] はい — プロキシによるキャッシュを防止するため、すべてのパーソナライズされたコンテンツに対して `Cache-Control: private` が**有効になっている**  
- [ ] いいえ — コンテンツが `public` とマークされているか、または `private` ディレクティブが不足しており、中間プロキシによって**キャッシュ可能になっている**  

### ログアウト成功後、ブラウザの「戻る」ボタンを介して機密情報にアクセス可能か？
- [ ] いいえ — ブラウザが再認証を要求するか、ページがローカルキャッシュから**読み込まれない**  
- [ ] はい — セッション終了後も「戻る」ボタンを介して機密情報が**依然として閲覧可能である**  

### HTML 以外の機密ファイル（例：PDF、CSV エクスポート、JSON API レスポンス）はキャッシュから保護されているか？
- [ ] いいえ — アプリケーションによって HTML 以外の機密ファイルが生成または処理されることはない  
- [ ] はい — すべての機密ファイルのダウンロードおよび API エンドポイントに厳格なキャッシュ制御ヘッダーが**適用されている**  
- [ ] いいえ — 機密性の高いダウンロードや API レスポンスがローカルのブラウザキャッシュや一時ディレクトリに**保存されている**  

### アプリケーションは古いデータの使用を防ぐために `Expires: 0` または過去の日付を使用しているか？
- [ ] はい — `Expires` ヘッダーが `0` または過去の日付に設定されており、ブラウザがコンテンツを即座に古いものとして扱うように**徹底されている**  
- [ ] いいえ — 機密ページにおいて `Expires` ヘッダーが**欠落している**か、将来の日付に設定されている
