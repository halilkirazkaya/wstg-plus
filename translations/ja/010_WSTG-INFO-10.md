## WSTG-INFO-10 — アプリケーションアーキテクチャのマッピング (Map Application Architecture)

アプリケーションアーキテクチャのマッピング（Map Application Architecture）は、Webアプリケーションを支える基盤技術、インフラ構成要素、およびデータフローの経路を特定するプロセスです。HTTPレスポンスヘッダー（HTTP response headers）、エラーメッセージ、クッキー（Cookie）の形式、およびファイル拡張子を分析することにより、診断員は使用されているWebサーバー、アプリケーションサーバー、データベース、およびサードパーティ製の統合サービスの構成図を再構築できます。この知識は、その後の攻撃を調整するために不可欠です。プラットフォーム固有のエクスプロイト（Exploit）を選択したり、ロードバランサーやWebアプリケーションファイアウォール（WAF）といった中間デバイスにおける潜在的なボトルネックや設定ミスを特定したりすることが可能になるためです。攻撃者はこの構造マップを利用して、一般的なスキャンから、特定のソフトウェアスタック（Software stack）とその相互接続されたコンポーネントに対する標的型の攻撃へと移行します。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-INFO-10 |
| **CWE** | CWE-200 |
| **テストステータス** | 未実施 |
| **深刻度** | 情報 / 低* |

> *内部ネットワークトポロジまたはプライベートIPアドレスが漏洩している場合、深刻度は「中（Medium）」になります。

### Webサーバーおよびアプリケーションサーバーの技術を正確に特定できますか？
- [ ] いいえ — 効果的なハードニング（Hardening）または難読化（Obfuscation）により特定できません。
- [ ] はい — 技術の種類は判明していますが、特定のバージョンは特定できません。
- [ ] はい — ヘッダーやファイル署名（File signatures）を通じて、技術と特定のバージョンが完全に公開されています（情報）。

### 通信経路において、中間デバイス（WAF、ロードバランサー、プロキシ）を検出できますか？
- [ ] いいえ — 中間デバイスは検出されないか、完全に透過的です。
- [ ] はい — タイミングや挙動から存在が疑われますが、正体は確認されていません。
- [ ] はい — 固有のヘッダーや挙動から、特定の製品（例：Cloudflare、Nginx、F5）が特定されています。

### アプリケーションは、内部ディレクトリ構造やネットワークトポロジに関する情報を漏洩していますか？
- [ ] いいえ — 内部パスやIPアドレスは公開されていません。
- [ ] はい — エラーメッセージやスタックトレース（Stack trace）で内部パスが漏洩していますが、内部IPは漏洩していません。
- [ ] はい — 内部ディレクトリ構造とプライベートIPアドレスの両方が公開されています。

### アプリケーションの挙動から、バックエンドデータベースの種類とバージョンを推測できますか？
- [ ] いいえ — データベースの詳細を特定できません。
- [ ] はい — エラーメッセージや挙動から、データベースの種類（例：PostgreSQL、MSSQL）が推測されます。
- [ ] はい — レスポンスボディまたはヘッダーに、データベースの種類とバージョンが明示的に公開されています。

### アプリケーションは、特定のクラウドサービスプロバイダーやサードパーティのコンテンツデリバリネットワーク（CDN）でホストされていますか？
- [ ] いいえ — ホスティングインフラを特定できません。
- [ ] はい — クラウドプロバイダー（AWS、Azure、GCP）は特定されていますが、特定のサービスは特定されていません。
- [ ] はい — 特定のクラウドサービス（例：S3バケット、Lambda、Azure Functions）がマッピングされ、アクセス可能です。
