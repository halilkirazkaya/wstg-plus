## WSTG-BUSL-09 — 悪意のあるファイルのアップロードテスト (Test Upload of Malicious Files)

ファイルアップロード機能は、ユーザーがサーバーにデータを送信することを可能にし、アプリケーション環境に悪意のあるコンテンツを導入するための直接的なベクターを提供します。攻撃者は、脆弱なバリデーション（検証）ロジックを悪用して、ウェブシェル（Web Shell）、マルウェア、または SVG や HTML などの特別に細工されたファイルをアップロードし、リモートコード実行 (Remote Code Execution: RCE) やクロスサイトスクリプティング (Cross-Site Scripting: XSS) を実行します。この脆弱性は、通常、プロファイル設定、ドキュメント管理システム、添付ファイル処理などの機能で見られ、サーバーがファイルの種類、内容、実行権限を適切に検証できない場合に発生します。攻撃に成功すると、システム全体の侵害、データの流出、または内部ネットワーク攻撃の踏み台（Pivot Point）としてサーバーが利用されることがよくあります。

### アプリケーションはファイルをアップロードする機能を提供していますか？
- [ ] いいえ — ファイルアップロード機能は存在しません  
- [ ] はい — 認証済みユーザー向けの機能が存在します  
- [ ] はい — 未認証（Unauthenticated）ユーザーが機能を利用可能です *(緊急)*  

### サーバー側でファイル拡張子のバリデーションが強制されていますか？
- [ ] はい — 拡張子の厳格な許可リスト（Allowlist）が適用されており、バイパス（Bypass）は不可能です  
- [ ] はい — 許可リストが使用されていますが、大文字小文字の区別や二重拡張子によるバイパスが可能です  
- [ ] はい — 拒否リスト（Blocklist）が使用されており、代替拡張子（例: `.phtml`, `.asa`）でバイパス可能です  
- [ ] いいえ — サーバー側で拡張子のバリデーションが適用されていません  

### ファイルの内容は、拡張子や MIME タイプ以外にも検証されていますか？
- [ ] はい — アプリケーションはマジックバイト（Magic Bytes）を検証し、詳細なコンテンツ検査を実行します  
- [ ] はい — アプリケーションは `Content-Type` ヘッダーのみを確認しますが、これは容易に偽装（Spoofing）可能です  
- [ ] いいえ — アプリケーションはバリデーションを完全にファイル拡張子に依存しています  

### アップロードされたファイルは実行権限のあるディレクトリに保存されていますか？
- [ ] いいえ — ファイルは専用のストレージサービス（例: S3）または実行不可能なディレクトリに保存されています  
- [ ] はい — ファイルはウェブサーバーに保存されていますが、設定により実行は無効化されています  
- [ ] はい — ファイルはウェブからアクセス可能なディレクトリに保存されており、実行が有効になっています *(緊急)*  

### ファイル名の操作によってセキュリティフィルタをバイパスできますか？
- [ ] いいえ — ファイル名はサーバーによってサニタイズ（無害化）されるか、ランダムに再生成されます  
- [ ] はい — ヌルバイト（Null Bytes）（例: `shell.php%00.jpg`）を使用してフィルタをバイパス可能です  
- [ ] はい — パストラベルサル（Path Traversal）文字（例: `../../`）を使用して、任意のディレクトリにファイルをアップロード可能です
