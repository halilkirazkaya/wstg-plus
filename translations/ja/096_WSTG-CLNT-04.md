## WSTG-CLNT-04 — クライアントサイドURLリダイレクト (Client-Side URL Redirect) のテスト

クライアントサイドURLリダイレクト (Client-Side URL Redirect) は、ウェブアプリケーションがユーザー制御の入力（通常はURLパラメータやハッシュフラグメント経由）を受け取り、それを `window.location` や `document.location.href` といったクライアントサイドのリダイレクトシンク (Sink) 内で使用する場合に発生します。この脆弱性は、主に攻撃者が巧妙なフィッシング (Phishing) キャンペーンを行うために悪用されます。被害者は最初に正規のドメインを信頼し、その後、悪意のあるサイトへ密かにリダイレクトされるためです。フィッシング以外にも、URLや `Referer` ヘッダーに値が含まれている場合に、OAuthアクセストークンやセッション識別子 (Session Identifiers) などの機密データを窃取 (Exfiltrate) するためにクライアントサイドリダイレクトが連鎖的に利用されることがよくあります。モダンなシングルページアプリケーション (SPA) において、この脆弱性はルーティングロジック、認証後の「戻り先 (return to)」パラメータ、または言語切り替え機能などで頻繁に発生します。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-CLNT-04 |
| **CWE** | CWE-601 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 (Medium)* |

> *リダイレクトがOAuthトークンやセッションIDの窃取、または認証フローにおけるセキュリティ制御のバイパスに使用できる場合、深刻度は「高 (High)」となります。

### アプリケーションは、ユーザー入力に基づいたリダイレクトを処理するためにクライアントサイドスクリプトを使用していますか？
- [ ] いいえ — リダイレクトは `3xx` HTTPステータスコードを介してサーバーサイドでのみ処理されます  
- [ ] はい — アプリケーションはURLパラメータに基づいて遷移するために `window.location` などのJavaScriptシンクを使用しています  
- [ ] はい — アプリケーションは、ユーザー提供のパスを処理するクライアントサイドフレームワークのルーター（例：React Router、Vue Router）を使用しています  

### 遷移先URLに対して入力バリデーション (Input Validation) またはホワイトリスト (Whitelist) 制御が実装されていますか？
- [ ] はい — 許可されたドメイン/パスの厳格なホワイトリストが強制されており、バイパス (Bypass) は不可能です  
- [ ] はい — バリデーションは存在しますが、脆弱な正規表現やブラックリストに依存しており、バイパスが可能です  
- [ ] いいえ — アプリケーションは任意のリダイレクト先文字列を受け入れます  

### 一般的な難読化 (Obfuscation) 手法を使用してリダイレクトロジックをバイパスできますか？
- [ ] いいえ — 制御によって、エンコードされた文字、プロトコル相対URL (`//`)、およびバックスラッシュが正しく処理されています  
- [ ] はい — URLエンコード (URL Encoding) または二重エンコードを使用してバイパスが可能です  
- [ ] はい — プロトコル相対URLまたは `@` 文字（例：`https://expected.com@attacker.com`）を使用してバイパスが可能です  
- [ ] はい — 特定のブラウザの挙動 (Quirks) やヌルバイトインジェクション (Null Byte Injection) を使用してバイパスが可能です  

### リダイレクトプロセスによって、機密データが遷移先サイトに公開されますか？
- [ ] いいえ — リダイレクト中にURLに含まれる、またはヘッダーを介して送信される機密データはありません  
- [ ] はい — 機密データ（例：セッショントークン、APIキー）が `Referer` ヘッダーを介して外部サイトに漏洩しています  
- [ ] はい — URLフラグメント (`#`) に含まれる機密データが、遷移先サイトのスクリプトからアクセス可能です  

### リダイレクトシンクを介してJavaScriptを実行すること（DOM-based XSS）は可能ですか？
- [ ] いいえ — シンクは `http` または `https` プロトコルへの遷移のみを許可しています  
- [ ] はい — リダイレクトシンクが `javascript:` または `data:` URIを許可しており、DOM-based XSS (DOMベースのクロスサイトスクリプティング) が可能です
