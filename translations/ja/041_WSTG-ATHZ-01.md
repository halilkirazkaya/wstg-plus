## WSTG-ATHZ-01 — ディレクトリトラバーサルおよびファイルインクルージョンのテスト (Testing Directory Traversal File Include)

ディレクトリトラバーサル（Directory Traversal）およびファイルインクルージョン（File Inclusion）の脆弱性は、アプリケーションが十分な検証（Validation）やサニタイズ（Sanitization）を行わずに、ユーザーが制御可能な入力を使用してファイルまたはディレクトリへのパスを構築する場合に発生します。攻撃者は `../` などのシーケンスを注入して、意図されたディレクトリの外部に移動することで、機密性の高いシステムファイル、設定データ、またはアプリケーションのソースコードにアクセスする可能性があります。ローカルファイルインクルージョン（LFI）やリモートファイルインクルージョン（RFI）が関与するより深刻なケースでは、攻撃者は悪意のあるスクリプトを読み込んだり、ログポイズニング（Log Poisoning）やPHPラッパー（PHP Wrappers）を悪用したりすることで、リモートコード実行（Remote Code Execution）を達成する可能性があります。これらの脆弱性は通常、動的なコンテンツの読み込み、テンプレートエンジン、またはサーバー側のロジックがファイルシステムパスを処理する画像取得エンドポイントで使用されるパラメータに存在します。

### パラメータはサーバー側の処理のためにファイル名またはパスを受け取りますか？
- [ ] いいえ — ファイルシステムと対話するパラメータは見当たりません  
- [ ] はい — パラメータは存在しますが、ファイル識別子の厳格な許可リスト（Allowlist）を使用しています  
- [ ] はい — パラメータは直接的なファイル名またはパスを受け取ります  

### ディレクトリトラバーサルシーケンスを防ぐために、入力はサニタイズされていますか？
- [ ] はい — 入力は厳格な許可リストに対して検証されており、トラバーサルシーケンスは**不可能です**  
- [ ] はい — 入力から `../` シーケンスを削除することでサニタイズされていますが、再帰的なバイパス（Bypass）が**可能です**  
- [ ] いいえ — パス関連の入力に対してサニタイズや検証は**適用されていません**  

### トラバーサルシーケンスを介して、制限されたディレクトリ外のファイルにアクセスすることは可能ですか？
- [ ] いいえ — アプリケーションまたはOSが、定義されたスコープ外のファイルへのアクセスを防止しています  
- [ ] はい — ドキュメントルート（Web Root）内のファイルへのアクセスが**可能です**  
- [ ] はい — 機密性の高いシステムファイル（例：`/etc/passwd`, `C:\Windows\win.ini`）へのアクセスが**可能です** *(緊急)*  

### サーバーはリモートURLのインクルード（RFI）を許可していますか？
- [ ] いいえ — リモートファイルインクルードは、サーバーまたはアプリケーションレベルで**無効化されています**  
- [ ] はい — リモートファイルをインクルードすることは**できます**が、実行は**不可能です**  
- [ ] はい — リモートファイルをサーバー上でインクルードし、実行することが**可能です** *(緊急)*  

### エンコーディングや特殊文字を使用してフィルタをバイパスできますか？
- [ ] いいえ — フィルタは、さまざまなエンコーディングやヌルバイトを効果的に処理しています  
- [ ] はい — URLエンコード、ダブルURLエンコード、または16ビットUnicodeを使用したバイパスが**可能です**  
- [ ] はい — ヌルバイト注入（`%00`）やファイルシステムラッパー（例：`php://filter`）を使用したバイパスが**可能です**
