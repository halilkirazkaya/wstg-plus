## WSTG-ERRH-02 — スタックトレースのテスト (Testing for Stack Traces)

アプリケーションが例外 (Exception) を適切に処理できない場合にスタックトレース (Stack Trace) が生成され、その結果、内部の実行状態や呼び出し階層 (Call Hierarchy) がエンドユーザーに公開されます。ペネトレーションテスターにとって、これらのトレースはアプリケーションフレームワーク、ライブラリのバージョン、内部ファイルシステムのパス、およびロジックフローを明らかにするため、偵察 (Reconnaissance) に必要な労力を大幅に削減できる非常に価値のある情報です。悪用 (Exploitation) 手法としては、不正な形式の入力、予期しないデータ型、または境界条件の違反を意図的に引き起こし、アプリケーションを不安定な状態にして基礎となるテクノロジースタックに関する情報を抽出することが含まれます。この情報の漏洩は、脆弱なサードパーティ製コンポーネントの特定や、公開されたコードパス内で発見されたロジックフラウ (Logic Flaws) に対する特定の攻撃コード (Exploit) を作成するために必要なコンテキストを提供することがよくあります。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-ERRH-02 |
| **CWE** | CWE-209 |
| **テストステータス** | 未実施 |
| **深刻度** | 低 / 中* |

> *深刻度は、スタックトレースによって機密性の高いアーキテクチャの詳細、データベースクエリ、または内部設定パラメータが明らかになる場合、「中」に上昇します。

### アプリケーションエラー時に、完全なスタックトレースがクライアントに返されますか？
- [ ] いいえ — アプリケーションは汎用的なエラーページまたはカスタムエラーメッセージを返す  
- [ ] はい — スタックトレースは**有効**だが、特定の機密性の低いモジュールのみに限定されている  
- [ ] はい — 完全なスタックトレースが**有効**であり、HTTPレスポンスボディ内で確認できる  

### エラーメッセージに機密性の高い環境情報が含まれていますか？
- [ ] いいえ — エラーメッセージに技術的または環境的な詳細は含まれていない  
- [ ] はい — メッセージによって**内部ファイルパス**またはサーバー側の**ディレクトリ構造**が漏洩している  
- [ ] はい — メッセージによって**データベーススキーマ**、**SQLクエリ**、または**サードパーティ製ライブラリのバージョン**が漏洩している  

### 入力パラメータを操作することでスタックトレースを誘発できますか？
- [ ] いいえ — 不正な形式の入力は適切に処理されるか、バリデーションによって拒否される  
- [ ] はい — **予期しないデータ型**（例：文字列が期待される場所に配列を送信する）によってトレースを誘発できる  
- [ ] はい — **ヌルバイト (Null Bytes)**、**特殊文字**、または**境界値**によってトレースが誘発される  

### グローバルエラーハンドラーがアプリケーション全体で一貫して適用されていますか？
- [ ] はい — グローバル例外ハンドラーがテストされたすべてのエンドポイントで**一貫して適用**されている  
- [ ] はい — ハンドラーは存在するが、レガシーシステム、API、または新しく追加されたエンドポイントには**適用されていない**  
- [ ] いいえ — グローバルエラーハンドリングメカニズムが**検出されず**、デフォルトのサーバーエラーが発生する
