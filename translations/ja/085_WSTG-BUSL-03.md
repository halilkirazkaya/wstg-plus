## WSTG-BUSL-03 — 整合性チェックのテスト (Test Integrity Checks)

整合性チェックのテスト (Integrity Check Testing) は、ビジネスプロセス内、あるいはクライアントとサーバー間をデータが移動する際に、アプリケーションがデータの未改ざんと信頼性を確実に維持しているかを検証することに焦点を当てます。攻撃者は、HTTP リクエスト (HTTP Requests)、隠しフィールド (Hidden Fields)、またはクッキー (Cookies) 内の製品価格、数量、ユーザー権限、取引状態などの重要なパラメータを改ざん (Tampering) することで、このロジックを標的にします。アプリケーションにサーバーサイド (Server-side) での検証や、ハッシュ化 (Hashing) または HMAC などの堅牢な整合性制御が欠けている場合、攻撃者はこれらの値を操作して、不正行為を行ったり、自身の権限を昇格 (Privilege Escalation) させたり、意図されたビジネス上の制約を回避したりする可能性があります。このテストは、金融取引、機密性の高い状態遷移 (State Transitions)、またはある段階の出力が次の段階の信頼できる入力となるマルチステップのワークフローを処理するすべてのアプリケーションにとって極めて重要です。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-BUSL-03 |
| **CWE** | CWE-353 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *整合性の不備により、金銭的な窃盗、不正な権限昇格、または永続的な記録の改ざんが可能な場合、深刻度は「高」となります。

### 機密性の高いビジネスパラメータがクライアントサイドでの改ざんにさらされていますか？
- [ ] いいえ — 機密性の高い値はサーバーサイドでのみ管理されています  
- [ ] はい — 価格、数量、ステータスフラグなどの値は公開されていますが、暗号化されています  
- [ ] はい — 隠しフィールド、URL パラメータ、またはクッキー内にプレーンテキストで値が公開されています  

### クライアントが制御するパラメータに、整合性メカニズム (HMAC、チェックサム (Checksum) など) が適用されていますか？
- [ ] はい — 堅牢な整合性チェックが適用され、すべてのリクエストで検証されています  
- [ ] はい — 整合性チェックは適用されていますが、脆弱または予測可能なアルゴリズム、あるいはハードコードされたシークレットが使用されています  
- [ ] いいえ — 機密性の高いパラメータに整合性チェックは適用されていません  

### 攻撃者によって整合性チェックを回避したり、再計算したりすることは可能ですか？
- [ ] いいえ — 署名検証が厳格に強制されており、回避は不可能です  
- [ ] はい — 署名パラメータを省略することで署名検証を回避できます  
- [ ] はい — シークレットキーまたはロジックが公開されているか脆弱なため、署名を再計算できます  

### アプリケーションは、信頼できるソースに対してデータのバックエンドでの再検証を行っていますか？
- [ ] はい — サーバーはデータベースに対してすべての値を再検証しており、回避は不可能です  
- [ ] はい — サーバーは部分的な検証を行っていますが、特定の境界条件 (Edge Case) を通じて回避が可能です  
- [ ] いいえ — サーバーはバックエンドでの検証なしにクライアントから提供されたデータを信頼しています *(重要/Critical)*  

### マルチステッププロセスの状態を改ざんすることは可能ですか？
- [ ] いいえ — セッション状態はサーバーサイドで管理されており、操作は不可能です  
- [ ] はい — 状態情報がクライアント経由で渡されており、ステップのスキップやアクションの繰り返しなどの改ざんが可能です
