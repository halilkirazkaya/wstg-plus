## WSTG-ATHN-09 — 脆弱なパスワード変更またはリセット機能のテスト

パスワード変更およびリセットのメカニズムは認証管理の極めて重要な構成要素であり、不適切に実装された場合、攻撃者によるアカウントの乗っ取り (Account Takeover) を許容する可能性があります。これらの脆弱性には、通常、予測可能なリセットトークン、ブルートフォース (Brute Force) 試行時のアカウントロックアウトの欠如、トークンの安全でない配信、または現在のパスワードを検証せずにパスワードを変更できることなどが含まれます。攻撃者は、リカバリリンクの傍受や推測、ホストヘッダーインジェクション (Host Header Injection) によるリセットメールの転送、あるいはパスワード変更後もアクセスを維持するためのセッション固定化 (Session Fixation) などを利用して、これらの欠陥を悪用します。アカウントの整合性を維持し、不正アクセスを防止するためには、これらのプロセスで強力な検証を要求し、暗号学的なランダム性を利用することが不可欠です。

### 標準的な変更時に、新しいパスワードを設定するために現在のパスワードが要求されますか？
- [ ] はい — 現在のパスワードが**要求され**、検証されている  
- [ ] はい — 現在のパスワードは**要求される**が、パラメータ汚染 (Parameter Pollution) または操作によってバイパス可能である  
- [ ] いいえ — 現在のパスワードは**要求されず**、セッションハイジャック (Session Hijacking) を通じたアカウントの乗っ取りが可能である *(高)*  

### パスワードリセットトークンは暗号学的に安全で、予測不可能ですか？
- [ ] はい — トークンは高エントロピーでランダムであり、**予測不可能**である  
- [ ] はい — トークンは生成されるが、エントロピーが低い、または予測可能なパターン（タイムスタンプベースなど）を示している  
- [ ] いいえ — トークンは**予測可能**であるか、Base64エンコードされたメールアドレス/IDのような静的なユーザーデータに基づいている *(緊急)*  

### パスワードリセットリンクはホストヘッダーインジェクションを通じてリダイレクト可能ですか？
- [ ] いいえ — アプリケーションはリンク生成において `Host` ヘッダーを無視するか、検証している  
- [ ] はい — アプリケーションはリンクの構築に `Host` または `X-Forwarded-Host` ヘッダーを使用しているが、検証は**存在する**  
- [ ] はい — ヘッダーの操作により、攻撃者が制御するドメインにリンクを**リダイレクトできる** *(高)*  

### リセットトークンには制限されたライフサイクルと即時の無効化設定がありますか？
- [ ] はい — トークンは速やかに期限切れとなり、使用後またはパスワード変更後に**直ちに**無効化される  
- [ ] はい — トークンの有効期限が長い（例：24時間以上）、または**複数回の使用**を許可している  
- [ ] いいえ — トークンに**有効期限がない**、またはパスワードが正常に変更された後も有効なままである  

### パスワードリセットおよび変更のエンドポイントにレート制限またはロックアウトは設定されていますか？
- [ ] はい — 列挙 (Enumeration) やブルートフォースを防ぐために、厳格なレート制限 (Rate Limiting) と CAPTCHA が**適用されている**  
- [ ] はい — 限定的な制御は存在するが、IPローテーションやヘッダーの偽装によりバイパスが**可能である**  
- [ ] いいえ — レート制限が**適用されておらず**、大量のアカウント列挙やトークンのブルートフォースが可能である
