## WSTG-CRYP-02 — パディング・オラクル (Padding Oracle) のテスト

パディング・オラクル脆弱性は、アプリケーションの復号プロセスにおいて、復号された暗号文（Ciphertext）のパディングが有効か無効かに関する情報が漏洩した際に発生します。攻撃者は、異なるHTTPステータスコード、特定のエラーメッセージ、またはレスポンス時間の変動といったサイドチャネル・レスポンス（Side-channel responses）を観察することで、暗号化キーを知ることなくデータを復号し、さらに任意の平文（Plaintext）を再暗号化できる可能性があります。この脆弱性は通常、暗号ブロック連鎖（CBC）モード（Cipher Block Chaining mode）でPKCS#7パディングを使用するブロック暗号（Block ciphers）を用いた、暗号化されたセッションCookie、認証トークン、またはURLパラメータにおいて顕在化します。この脆弱性をエクスプロイト（Exploit）すると、機密性（Confidentiality）が完全に失われるだけでなく、偽造された暗号ブロックの構築を通じて完全性（Integrity）の侵害を招く恐れがあります。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-CRYP-02 |
| **CWE** | CWE-209 |
| **テストステータス** | 未実施 |
| **深刻度** | 高 / 緊急* |

> *セッション管理、認証トークン、またはPII（個人を特定できる情報）の暗号化に脆弱性が存在する場合、深刻度は「緊急（Critical）」となります。

### 機密データに対してCBCモードのブロック暗号が使用されていますか？
- [ ] いいえ — アプリケーションは認証付き暗号（AES-GCM/ChaCha20-Poly1305）を使用しています
- [ ] はい — アプリケーションはブロック暗号を使用していますが、パディングは**不要**です（例：ストリーム暗号またはCTRモード）
- [ ] はい — アプリケーションはCBCモードを使用しており、パディングが**適用されています**

### サーバーはサイドチャネルを通じてパディングの有効性を漏洩させていますか？
- [ ] いいえ — サーバーは一律のエラーメッセージと一定のレスポンスタイムを返します
- [ ] はい — サーバーはパディングエラーに対して異なるHTTPステータスコード（例：200 vs 500）を返します
- [ ] はい — サーバーは特定のエラーメッセージ（例：「Invalid Padding」、「Decryption failed」）を返します
- [ ] はい — サーバーは復号失敗時に測定可能なタイミングの差を示します

### 暗号文の自動復号は可能ですか？
- [ ] いいえ — サイドチャネルがエクスプロイトに十分なほど安定していません
- [ ] はい — 暗号文の一部（最後の数ブロック）を復号**できます**
- [ ] はい — `PadBuster` などのツールを使用して、平文の完全な復元が**可能です**

### 攻撃者はビット反転 (Bit-flipping) を実行したり、有効な暗号文を偽造したりできますか？
- [ ] いいえ — 復号の**前**に完全性チェック（HMAC）が検証されます
- [ ] はい — 完全性チェックは存在しませんが、ペイロード (Payload) の構築は実行不可能です
- [ ] はい — 任意の暗号文の構築が**可能**であり、権限昇格やデータインジェクションにつながる可能性があります
