## WSTG-CONF-03 — 機密情報に関するファイル拡張子の処理のテスト (Test File Extensions Handling for Sensitive Information)

ファイル拡張子の処理に関するテストでは、Webサーバーやアプリケーションサーバーが、制限されるべき、あるいは実行されるべきファイルを提供してしまうことで、機密情報を漏洩させていないかを特定します。攻撃者は、特定のハンドラーマッピングの欠如によりWebルートに残され、プレーンテキストとして提供される可能性のあるバックアップファイル、設定ファイル、ソースコードの断片（例：`.bak`、`.old`、`.env`、`.inc`）を頻繁に探索します。この脆弱性は、データベースの認証情報、APIキー、内部のビジネスロジック (Business Logic) の露出を招き、インフラストラクチャへのさらなる深刻な攻撃を容易にする可能性があるため重要です。これらの露出は、通常、公開ディレクトリ、アップロードフォルダ、またはサーバー上の誤った MIMEタイプ (MIME-type) の設定を介して発生します。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-CONF-03 |
| **CWE** | CWE-552 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *認証情報を含む設定ファイルや完全なソースコードがアクセス可能な場合、深刻度は「高」となります。

### 機密性の高いバックアップまたは一時ファイルの拡張子（例：`.bak`, `.old`, `.swp`, `~`）にアクセス可能ですか？
- [ ] いいえ — サーバーは一般的なバックアップ拡張子に対して 403 Forbidden または 404 Not Found を返します  
- [ ] はい — サーバーはバックアップファイルのダウンロードを許可していますが、それらに機密データは**含まれていません**  
- [ ] はい — サーバーは機密データやソースコードを含むバックアップファイルのダウンロードを許可しています *(高)*  

### サーバーは環境設定ファイルやシステム設定ファイル（例：`.env`, `.config`, `.yml`, `.ini`）を露出させていますか？
- [ ] いいえ — 制限された設定ファイルにはアクセス**できず**、403/404 を返します  
- [ ] はい — 機密性の高い設定ファイルにアクセス**可能**であり、内部のシークレットや認証情報を開示しています  

### 代替拡張子を付加することでソースコードを取得できますか（例：`.php.txt`, `.jsp.old`, `.aspx.bak`）?
- [ ] いいえ — 拡張子の操作に関わらず、アプリケーションコードがプレーンテキストとしてレンダリングされることは**ありません**  
- [ ] はい — 付加された拡張子に対するハンドラーがサーバーに存在しないため、ソースコードが露出しています  

### 管理用ディレクトリまたはメタデータディレクトリ（例：`.git/`, `.svn/`, `.DS_Store`）へのパブリックアクセスはブロックされていますか？
- [ ] いいえ — これらのディレクトリ/ファイルは Web ルートに存在しません  
- [ ] はい — サーバー側のリライトルールまたは権限設定により、アクセスは**不可能です**  
- [ ] はい — メタデータディレクトリにアクセス**可能**であり、完全なソースコードの復元が可能です  

### サーバーは複数の拡張子を持つファイル（例：`file.php.jpg`）のリクエストをどのように処理しますか？
- [ ] いいえ — サーバーは内部拡張子のセキュリティを正しく優先するか、リクエストをブロックします  
- [ ] はい — サーバーはファイルを最初の拡張子として処理しますが、実行のためのバイパス (Bypass) は**不可能です**  
- [ ] はい — サーバーは最終的な拡張子を無視するため、コード実行またはソースコードの露出が**可能です**
