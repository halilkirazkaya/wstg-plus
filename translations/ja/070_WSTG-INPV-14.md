## WSTG-INPV-14 — 潜伏型脆弱性 (Incubated Vulnerabilities) のテスト

潜伏型脆弱性は、持続型 (Persistent) または 二次的 (Second-order) 脆弱性とも呼ばれ、悪意のあるペイロード (Payload) がアプリケーションによって「コールド」状態で保存され、後に別のコンテキストで実行または処理されることで発生します。この攻撃パターンは通常、共有データベースやファイルシステムからデータを取得する際に、適切な再バリデーションや出力エンコーディング (Output Encoding) を行わないバックエンドシステム、管理コンソール、または自動レポート作成ツールを標的にします。ペネトレーションテスター (Pentester) は、入力ポイント（「インキュベーター」）から、そのデータが最終的にレンダリングされたり、他のコンポーネントや二次的なアプリケーションによって利用されたりするすべての場所まで、ユーザー入力のライフサイクルを追跡する必要があります。攻撃の成功は、格納型クロスサイトスクリプティング (Stored XSS) による管理者セッションのハイジャックや、内部レポートモジュールにおける二次的 SQL インジェクション (Second-order SQL Injection) によるデータ漏洩 (Data Exfiltration) など、重大な影響を及ぼすことがよくあります。

### ユーザーが制御する入力が、他のユーザーやプロセスによって後で取得されるために保存されるエンドポイントは存在しますか？
- [ ] いいえ — アプリケーションは後で使用するためにユーザー入力を保存しません
- [ ] はい — 入力はデータベースフィールド、ログファイル、または設定設定に保存されます

### データが永続ストレージ層に書き込まれる前に、入力バリデーションまたはサニタイジング (Sanitization) が適用されていますか？
- [ ] はい — 入力ポイントで厳格なホワイトリストによるバリデーションが適用されています
- [ ] はい — 一般的なサニタイジングは適用されていますが、エンコーディングや非標準文字によるバイパスが可能です
- [ ] いいえ — 入力はバリデーションされていない未加工の形式で保存されます

### 保存されたデータが管理者用インターフェースや二次的なインターフェースでレンダリングされる際、適切にエンコードまたはサニタイジングされていますか？
- [ ] はい — データがレンダリングされるすべての場所で、コンテキストに応じた出力エンコーディングが適用されています
- [ ] はい — 一部の場所ではエンコーディングが適用されていますが、他の場所（内部管理ダッシュボードなど）では欠落しています
- [ ] いいえ — データはエンコーディングやサニタイジングなしでレンダリングされており、ペイロードの実行が可能です

### データベースに保存されたペイロードが、バックエンドのバッチプロセスや帯域外 (Out-of-band) 監視システムに影響を与える可能性はありますか？
- [ ] いいえ — バックエンドプロセスは安全な解析メソッドまたはパラメータ化されたクエリを使用しています
- [ ] はい — 保存されたペイロードがバックエンドのロジックでインタラクションを引き起こす可能性があります（例：CSV インジェクション (CSV Injection)、ログにおけるコマンドインジェクション (Command Injection)）
- [ ] はい — 保存されたペイロードが攻撃者の制御するサーバーへの帯域外 (OOB) リクエストをトリガーする可能性があります
