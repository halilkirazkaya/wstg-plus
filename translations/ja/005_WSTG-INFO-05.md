## WSTG-INFO-05 — 情報漏洩に関するウェブページコンテンツの確認 (Review Webpage Content for Information Leakage)

ウェブページコンテンツの確認（Review Webpage Content）には、HTML、JavaScript、およびCSSファイルを含むアプリケーションのレスポンスを手動および自動で検査し、ユーザーに意図せず公開されている機密情報を特定する作業が含まれます。攻撃者は、開発者のコメント、非表示のフォームフィールド、内部サーバーパス、ハードコードされたAPIキー（Hardcoded API keys）、およびデバッグ情報など、その後の悪用フェーズに役立つ情報を求めて、クライアント側のソースコードを精査します。この情報漏洩（Information Leakage）は多くの場合、本番環境への移行前に開発者がメタデータやデバッグコードを削除し忘れることで発生し、ロジックの欠陥やバックエンドのインフラ構成の詳細を露呈させる可能性があります。攻撃者の視点からは、これによりセキュリティアラートを発生させたり、サーバーのバックエンドロジックと直接対話したりすることなく、アプリケーションの内部構造をマッピングし、見落とされたエントリポイント（Entry point）を発見するための低労力な手法が提供されます。

| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-INFO-05 |
| **CWE** | CWE-200 |
| **テストステータス** | 未実施 (Not Performed) |
| **深刻度** | 低 (Low) / 中 (Medium)* |

> *機密性の高いハードコードされた認証情報、秘密鍵、または内部環境変数が発見された場合、深刻度は「高（High）」になります。

### ソースコード内に機密情報を含む開発者のコメントが存在しますか？
- [ ] いいえ — コメントが存在しない、または機密性のない一般的なコメントのみが発見された
- [ ] はい — コメントは存在するが、機密性の高いロジックやデータは含まれていない
- [ ] はい — コメントによって内部パス、SQLクエリ、または管理者向けの指示が**公開されている**
- [ ] はい — コメントによって認証情報や機密性の高い開発者メモが**公開されている** *(高)*

### 非表示のHTML入力フィールド (Hidden HTML input fields) に機密データやステータス情報が含まれていますか？
- [ ] いいえ — 非表示フィールドが見つからない、または機密性のないトークンのみが含まれている
- [ ] はい — 非表示フィールドが状態管理に使用されているが、**暗号化**または**署名**されている
- [ ] はい — 非表示フィールドに**プレーンテキスト**のパスワード、ユーザーロール、または内部IDが含まれている

### JavaScriptファイル内にハードコードされたシークレット、APIキー、または内部IPアドレスが存在しますか？
- [ ] いいえ — JSファイル内に機密性の高い文字列やシークレットは見つからない
- [ ] はい — JSファイルに公開APIキーが含まれているが、**適切に**制限されている
- [ ] はい — JSファイルに**保護されていない**APIキー、内部IP、またはプライベートなエンドポイントが含まれている
- [ ] はい — JSファイルに**ハードコードされた**管理者認証情報や秘密鍵が含まれている *(致命的)*

### アプリケーションのソース内でフレームワークのバージョンや内部ファイルパスが公開されていますか？
- [ ] いいえ — フレームワーク情報やファイルパスは**ミニファイ (Minification)** または **難読化 (Obfuscation)** されている
- [ ] はい — フレームワーク名やバージョンは**可視化**されているが、パッチは適用済みである
- [ ] はい — 内部ファイルパスやサーバーの絶対パスがソースコード内で**露出している**

### ミニファイまたは難読化の欠如により、ソースコードが漏洩しやすい状態になっていますか？
- [ ] いいえ — 本番環境のコードは**完全にミニファイ**されており、コメントも削除されている
- [ ] はい — コードはミニファイされているが、ソース内にコメントが**残っている**
- [ ] はい — ソースマップ（`.map` ファイル）が**パブリックにアクセス可能**であり、ソースコードの完全な復元が可能である
