## WSTG-CLNT-05 — CSSインジェクション (CSS Injection) のテスト

CSSインジェクション (CSS Injection) は、アプリケーションがユーザー供給の入力を、適切なサニタイズ (Sanitization) やエスケープ (Escaping) を行わずに Cascading Style Sheets (CSS) に反映させることで発生します。単なる見た目の問題として捉えられがちですが、攻撃者は属性セレクター (Attribute Selectors) や `background-image` プロパティを使用して外部リクエストをトリガーし、CSRFトークンやセッションIDなどの機密データを外部送信 (Exfiltration) することが可能です。この脆弱性は通常、ユーザーの好み（カスタムテーマやフォントの色など）が `<style>` ブロックやインラインの `style` 属性内に反映されるエンドポイントで発生します。攻撃者の視点からは、この脆弱性の悪用により、UI レッドレッシング (UI Redressing) や不正なレイアウト変更を通じたフィッシング、あるいは厳格なコンテンツセキュリティポリシー (CSP) によって JavaScript ベースの攻撃がブロックされる環境下での隠密なデータ窃取につながる可能性があります。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-CLNT-05 |
| **CWE** | CWE-74 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *インジェクションポイントにより CSRFトークンなどの機密属性の外部送信が可能である場合、または機密性の高いワークフローにおいて UI レッドレッシングに使用できる場合、深刻度は「高」となります。

### アプリケーションは CSS コンテキスト内でユーザー制御の入力を反映していますか？
- [ ] いいえ — 入力は style タグ、属性、または CSS ファイル内に反映されていません。
- [ ] はい — 入力は反映されていますが、安全な英数字のみに厳格に制限されています。
- [ ] はい — 入力は `style` 属性または `<style>` ブロック内に反映されています。

### CSS 固有のメタ文字や関数は適切にサニタイズされていますか？
- [ ] はい — `{`, `}`, `:` などの文字や `url()` などの関数は**適切にエスケープ**されています。
- [ ] はい — サニタイズは実施されていますが、エンコーディングや CSS コメントにより**バイパスが可能**です。
- [ ] いいえ — CSS メタ文字に対してサニタイズが適用されていません。

### CSS セレクターを使用したデータの外部送信は可能ですか？
- [ ] いいえ — セレクターによって外部リクエストをトリガーしたりデータを漏洩させたりすることは**できません**。
- [ ] はい — 属性セレクターと `background-image` を介して、部分的なデータの外部送信が**可能**です。
- [ ] はい — 自動化されたブルートフォース (Brute Force) 手法を用いて、機密トークンの完全な外部送信が**可能**です。

### コンテンツセキュリティポリシー (CSP) は CSS インジェクションのリスクを軽減していますか？
- [ ] はい — `style-src` が 'self' に制限されており、`img-src` または `connect-src` が**制限されています**。
- [ ] はい — CSP は存在しますが、'unsafe-inline' を使用しているか、ワイルドカードによる外部ドメインを許可しています。
- [ ] いいえ — CSS を介した外部リソースの読み込みを防止するための CSP が実装されていません。

### この脆弱性は UI レッドレッシングやフィッシングに使用される可能性がありますか？
- [ ] いいえ — インジェクションのスコープが限定的すぎて、レイアウトを大幅に変更することができません。
- [ ] はい — UI の修正が**可能**であり、悪意のある要素のオーバーレイを許容します。
- [ ] はい — ページレイアウトの完全な制御が**可能**であり、非常に説得力の高いフィッシング攻撃を助長します。
