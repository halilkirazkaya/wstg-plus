## WSTG-ATHN-05 — 脆弱な「パスワードを記憶する」機能のテスト

「パスワードを記憶する」(Remember Me) または持続的ログイン機能 (Persistent Login) は、クライアント側にトークンまたは認証情報を保存することで、ブラウザの再起動後もユーザーの認証状態を維持するように設計されています。この機能の実装が不適切である場合（平文のパスワード (Plaintext Password)、弱いハッシュ、または低エントロピーなトークンをクッキー (Cookie) に保存するなど）、クロスサイトスクリプティング (XSS)、物理的アクセス、またはネットワーク傍受を通じて、ユーザーはアカウントの乗っ取り (Account Takeover) のリスクにさらされます。ペネトレーションテスター (Pentester) は、これらの持続的トークンのエントロピー、保存メカニズムに適用されているセキュリティフラグ、およびセッションのライフサイクルを評価し、持続的アクセスの利便性が、多要素認証 (MFA) やセッションの無効化といった重要なセキュリティ制御をバイパスしていないことを確認します。悪用されるケースでは、多くの場合、これらの長期間有効なトークンを収集することで、元の認証情報を必要とせずにアカウントへの不正アクセスが行われます。

| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-ATHN-05 |
| **CWE** | CWE-522 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 (Medium) / 高 (High)* |

> *平文の認証情報やソルトなしのハッシュが持続的クッキーに保存されている場合、またはトークンが多要素認証 (MFA) をバイパスする場合は、深刻度は「高 (High)」になります。

### アプリケーションは「パスワードを記憶する」または「ログイン状態を維持する」機能を実装していますか？
- [ ] いいえ — アプリケーションに機能は存在しません
- [ ] はい — 機能が存在し、暗号学的に強力で予測不可能なトークンを使用しています
- [ ] はい — 機能は存在しますが、予測可能または低エントロピーなトークンを使用しています *(中)*
- [ ] はい — 機能が存在し、平文の認証情報または Base64 エンコードされたパスワードを保存しています *(高)*

### 持続的クッキーにセキュリティフラグが正しく適用されていますか？
- [ ] はい — `HttpOnly`、`Secure`、および `SameSite` フラグが適用されています
- [ ] はい — 一部のフラグは適用されていますが、`HttpOnly` が欠落しています
- [ ] はい — 一部のフラグは適用されていますが、HTTPS 通信において `Secure` が欠落しています
- [ ] いいえ — 持続的クッキーにセキュリティフラグが適用されていません

### 手動ログアウト後も「パスワードを記憶する」トークンは有効なままですか？
- [ ] いいえ — ログアウト直後にサーバー側でトークンが無効化されます
- [ ] はい — トークンは有効なままですが、機密性の高い操作にはパスワードが必要です
- [ ] はい — トークンは有効なままで、再認証なしでフルアカウントアクセスが可能です *(中)*

### パスワード変更時に持続的セッションは終了しますか？
- [ ] はい — ユーザーがパスワードを変更すると、すべての持続的トークンが無効化されます
- [ ] いいえ — パスワードのリセットまたは変更が行われた後も、「パスワードを記憶する」トークンが有効なままです *(高)*

### 次回以降のアクセス時に、持続的トークンによって多要素認証 (MFA) がバイパスされますか？
- [ ] いいえ — 「パスワードを記憶する」トークンが存在する場合でも MFA が要求されます
- [ ] はい — MFA はバイパスされますが、トークンは特定のデバイスフィンガープリントに関連付けられています
- [ ] はい — 任意のデバイスからの持続的クッキーのみを使用して、MFA が完全にバイパスされます *(高)*
