## WSTG-SESS-09 — セッションハイジャック (Session Hijacking) のテスト

セッションハイジャック (Session Hijacking) は、攻撃者が有効なセッション識別子 (SID) を奪取、予測、または固定することで、ユーザーのアクティブなセッションに対して不正なアクセスを行う際に発生します。この脆弱性は通常、不十分なトランスポート層セキュリティ、クッキー (Cookie) のセキュリティフラグの欠如、または攻撃者が認証を完全にバイパスすることを可能にする予測可能な SID 生成アルゴリズムによって顕在化します。攻撃者の視点では、悪用に成功すると、認証情報を必要とせずに被害者の完全ななりすましが可能になります。これは多くの場合、ネットワークスニッフィング、クロスサイトスクリプティング (XSS)、またはセッションフィクセーション (Session Fixation) 攻撃を通じて達成されます。

### セッション識別子は転送中に保護されていますか？
- [ ] はい — `Secure` フラグが有効化されており、HSTS が厳格に適用されている
- [ ] はい — `Secure` フラグは有効化されているが、HSTS が無効化されている
- [ ] いいえ — `Secure` フラグが適用されておらず、暗号化されていないチャネルを介した SID の傍受が可能である *(高)*

### セッション識別子はクライアント側のスクリプトアクセスから保護されていますか？
- [ ] はい — `HttpOnly` フラグがすべてのセッション関連クッキーに適用されている
- [ ] いいえ — `HttpOnly` フラグが適用されておらず、XSS による SID の漏洩が可能である *(緊急)*

### アプリケーションはクライアント属性へのセッションバインディングを実装していますか？
- [ ] はい — セッションがクライアント属性 (IP/User-Agent) に紐付けられており、再利用は不可能である
- [ ] はい — セッションバインディング (Session Binding) は存在するが、ヘッダーのスプーフィング (Spoofing) によりバイパスが可能である
- [ ] いいえ — セッションバインディングが存在せず、SID は任意のソースから使用された場合に有効である

### セッション識別子は予測を防ぐのに十分なランダム性を持っていますか？
- [ ] はい — SID は暗号論的擬似乱数生成器 (CSPRNG) を使用している
- [ ] いいえ — SID の長さが不十分、または予測可能なシーケンスやパターンに従っている

### 攻撃の機会を制限するために同時セッションが管理されていますか？
- [ ] いいえ — アプリケーションは 1 ユーザーにつき単一のアクティブなセッションを厳格に適用している
- [ ] はい — 複数の同時セッションが有効化されているが、監視されている
- [ ] はい — 異なるデバイスや場所から無制限の同時セッションが可能である
