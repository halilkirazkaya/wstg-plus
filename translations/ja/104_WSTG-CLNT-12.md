## WSTG-CLNT-12 — ブラウザストレージのテスト (Test Browser Storage)

ブラウザストレージのテストでは、アプリケーションが LocalStorage、SessionStorage、IndexedDB、WebSQL などのクライアント側メカニズムをどのように利用してデータを永続化しているかを分析します。PII (個人を特定できる情報)、認証トークン、またはセッション識別子を含む機密情報が安全でない状態で保存されている場合、攻撃者が クロスサイトスクリプティング (XSS) を成功させたり、ユーザーのデバイスへの物理的なアクセスを得たりすると、情報漏洩が発生する可能性があります。ペネトレーションテスター（侵入テスト担当者）は、データがクリアテキスト (Cleartext) で保存されていないか、ログアウト後もアクセス可能なままになっていないか、およびデータ漏洩を防ぐためにストレージのライフサイクルが適切に管理されているかを確認する必要があります。攻撃者の視点では、これらのストレージメカニズムは、セッションハイジャック (Session Hijacking) や長期的なアカウントの維持を可能にする状態保持情報を奪取するための格好のターゲットとなります。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-CLNT-12 |
| **CWE** | CWE-922 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *セッショントークンや機密性の高い PII が LocalStorage に保存され、XSS を介してアクセス可能な場合、深刻度は「高」になります。

**ツール:** `Browser DevTools`, `Storage Explorer`, `Burp Suite`, `OWASP ZAP`

### アプリケーションは機密情報をブラウザストレージに保存していますか？
- [ ] いいえ — アプリケーションはブラウザストレージを使用していない、または機密性のない UI 状態のみを保存している。
- [ ] はい — 機密データは保存されているが、堅牢なクライアント側の暗号化技術を使用して暗号化されている。
- [ ] はい — 機密データ（トークン、PII、シークレット）がクリアテキストで保存されている（中）。

### 保存されたデータは、許可されていないスクリプトからアクセス可能ですか（XSS のリスク）？
- [ ] いいえ — データは HttpOnly 属性の Cookie（ブラウザストレージではない）に保存されている、またはストレージが利用されていない。
- [ ] はい — LocalStorage/SessionStorage が使用されており、あらゆる XSS ペイロード (Payload) を介してデータに完全にアクセスできる（高）。

### ユーザーのログアウト時に、ブラウザストレージは適切に消去されますか？
- [ ] はい — ログアウトプロセス中に、アプリケーションに関連するすべてのストレージが明示的に消去される。
- [ ] いいえ — ストレージはログアウト後も保持されるが、機密性の高いセッションデータは含まれていない。
- [ ] いいえ — 認証トークンまたは PII がセッション終了後もストレージに残っている（中）。

### アプリケーションは大規模または機密性の高いデータセットに IndexedDB または WebSQL を使用していますか？
- [ ] いいえ — これらのストレージメカニズムは使用されていない。
- [ ] はい — データは保存されているが、DOM にレンダリングされる前に適切にサニタイズ (Sanitize) されている。
- [ ] はい — 機密データセットが IndexedDB/WebSQL 構造内にクリアテキストで保存されている。

### 保存されたデータの整合性を改ざんして、アプリケーションのロジックに影響を与えることは可能ですか？
- [ ] いいえ — アプリケーションは、ストレージからデータを処理する前に、データの検証または署名を行っている。
- [ ] はい — ストレージの値（ユーザーロール、フラグなど）を変更することが可能であり、その結果、アプリケーションの動作が変化する。
