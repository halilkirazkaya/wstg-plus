## WSTG-CONF-02 — アプリケーションプラットフォーム設定のテスト

アプリケーションプラットフォーム設定のテスト（Application Platform Configuration Testing）には、基盤となるウェブサーバ、アプリケーションサーバ、およびフレームワークの設定を監査し、一般的なエクスプロイト（Exploit）に対して堅牢化（Hardening）されているかを確認することが含まれます。攻撃者は、デフォルトの認証情報（Default Credentials）、パッチが適用されていないプラットフォームの脆弱性、および公開された管理インターフェースを積極的に探し出し、不正アクセスやコード実行を試みます。設定ミス（Misconfigurations）は、機密性の高い環境変数や内部システムパスの漏洩、あるいは攻撃対象領域（Attack Surface）を拡大させる不要なサンプルアプリケーションの存在につながることがよくあります。専門的な観点からは、適切に設定されていないプラットフォームは、ターゲット環境への初期アクセスを達成するための可能性が高いエントリーポイントとなります。

| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-CONF-02 |
| **CWE** | CWE-16 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *管理インターフェースがデフォルトの認証情報でアクセス可能な場合、またはサンプルスクリプトによってリモートコード実行（Remote Code Execution - RCE）が可能な場合、深刻度は「高」となります。

**ツール:** `Nmap`, `Nikto`, `WhatWeb`, `Wappalyzer`, `Burp Suite`, `Curl`

### プラットフォーム上でデフォルトの認証情報やアカウントが有効になっていますか？
- [ ] いいえ — すべてのデフォルトアカウントが無効化（Disabled）されているか、パスワードが変更されています。
- [ ] はい — デフォルトアカウントは存在しますが、外部ネットワークからはアクセスできません。
- [ ] はい — デフォルトアカウントが有効であり、インターネット経由でアクセス可能です（緊急：Critical）。

### プラットフォームがヘッダーやエラーページを通じて機密性の高いバージョン情報を漏洩させていませんか？
- [ ] いいえ — バージョン文字列は隠蔽されているか、一般的なものです。
- [ ] はい — 詳細なバージョン情報がHTTPヘッダー（例：`Server`, `X-Powered-By`）で公開されています。
- [ ] はい — 詳細なエラーメッセージに完全なスタックトレース（Stack Traces）や内部システムパスが含まれています。

### 管理インターフェースが適切に制限されていますか？
- [ ] いいえ — 管理インターフェースは公開されていません。
- [ ] はい — インターフェースは存在しますが、IP許可リスト（IP Allowlisting）または多要素認証（Multi-Factor Authentication - MFA）によって制限されています。
- [ ] はい — インターフェース（例：`/admin`, `/manager`, `/console`）が単一要素認証でパブリックにアクセス可能です。

### 本番サーバ上にサンプルファイル、スクリプト、またはドキュメントが存在しますか？
- [ ] いいえ — すべての不要なファイルは削除されています。
- [ ] はい — サンプルファイルやドキュメントは存在しますが、機密情報は漏洩していません。
- [ ] はい — サンプルスクリプト（例：`info.php`, `examples/`）が存在し、直接的な攻撃ベクトル（Attack Vector）となっています。

### サーバが安全でないHTTPメソッドを許可するように設定されていませんか？
- [ ] いいえ — `GET` と `POST` のみが有効化されています。
- [ ] はい — `PUT`、`DELETE`、`TRACE` などの潜在的にリスクのあるメソッド（Insecure HTTP Methods）が有効ですが、制限されています。
- [ ] はい — リスクのあるメソッドが有効であり、不正なファイル改ざんや認証情報の窃取が可能です。
