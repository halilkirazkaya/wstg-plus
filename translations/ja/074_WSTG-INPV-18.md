## WSTG-INPV-18 — サーバサイドテンプレートインジェクション (Server-Side Template Injection) のテスト

サーバサイドテンプレートインジェクション (Server-Side Template Injection (SSTI)) は、アプリケーションが適切なサニタイズ (Sanitization) やエスケープ (Escaping) を行わずに、ユーザー制御の入力をサーバサイドのテンプレートエンジンに埋め込むことで発生します。これにより、攻撃者は悪意のあるテンプレートディレクティブを注入できるようになり、それらがサーバ上で実行されることで、最終的にリモートコード実行 (Remote Code Execution (RCE)) を通じたシステム全体の完全な侵害につながる可能性があります。SSTI は通常、動的なメール生成、プロフィールページ、通知システムなど、Jinja2、Twig、FreeMarker といったエンジンを使用する機能で顕在化します。攻撃者の視点からは、この脆弱性はテンプレートエンジン固有のオブジェクトやメソッドを利用することで、機密性の高い環境変数の漏洩、内部ファイルの読み取り、またはシステムコマンドの実行に悪用されることが一般的です。

### アプリケーションはユーザー入力を処理するためにサーバサイドテンプレートエンジンを使用していますか？
- [ ] いいえ — アプリケーションは動的コンテンツにサーバサイドテンプレートを使用していません  
- [ ] はい — テンプレートは使用されていますが、ユーザー入力はテンプレートディレクティブに埋め込まれていません  
- [ ] はい — 適切なサニタイズなしにユーザー入力がテンプレートに埋め込まれています  

### 入力フィールドに注入された基本的な算術式は評価されますか？
- [ ] いいえ — `{{7*7}}` や `${7*7}` などのペイロード (Payload) は、そのままの文字列として反映されます  
- [ ] はい — 式が評価され（例：`49` が返される）、テンプレートエンジンが脆弱であることが確認されました  

### テンプレートエンジンの組み込みオブジェクトやメソッドにアクセスできますか？
- [ ] いいえ — 危険なオブジェクト、メソッド、または設定へのアクセスは制限されているか、サンドボックス (Sandbox) 化されています  
- [ ] はい — 機密データを漏洩させるために、組み込みオブジェクト（例：`self`、`config`、`__class__`）にアクセス可能です  
- [ ] はい — システムコールや OS ライブラリを介したリモートコード実行 (RCE) が可能です  

### 危険なディレクティブの実行を防止するサンドボックスやホワイトリストのメカニズムは存在しますか？
- [ ] はい — 厳格なホワイトリスト (Allowlist) または安全なサンドボックスが導入されており、バイパス (Bypass) は不可能です  
- [ ] はい — サンドボックスは存在しますが、特定のエンジン依存のペイロードによってバイパスが可能です  
- [ ] いいえ — テンプレートエンジンにサンドボックスやホワイトリストは適用されていません
