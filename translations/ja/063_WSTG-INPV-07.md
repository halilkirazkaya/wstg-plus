## WSTG-INPV-07 — XMLインジェクション

XMLインジェクション (XML Injection) は、アプリケーションがXMLメタ文字 (XML meta-characters) に対して適切な検証 (Validation)、サニタイズ (Sanitization)、またはエスケープ (Escaping) を行わずに、ユーザー提供のデータをXMLドキュメントやストリームに組み込んだ場合に発生します。特定のXMLタグや構造要素を注入することで、攻撃者はドキュメントのロジックを操作し、認証メカニズムをバイパスし、あるいはバックエンドで処理されるデータの整合性を妨害することが可能になります。この脆弱性は、SOAPベースのウェブサービス、XMLを受け入れるREST API、SAMLアサーション (SAML assertions)、およびXML設定ファイルやレポートを動的に生成するアプリケーションで一般的に見られます。攻撃者の視点では、意図されたデータコンテキストから脱却してドキュメント構造を変更し、最終的に不正な権限昇格 (Privilege escalation) や意図しないビジネスロジック (Business logic) の実行を引き起こすことが目的となります。

### アプリケーションは、ユーザーからのXML形式の入力を受け入れ、処理しますか？
- [ ] いいえ — アプリケーションはXML入力を処理しません
- [ ] はい — XML入力はAPI (SOAP/REST) を介して処理されます
- [ ] はい — XMLはファイルアップロード (SVG, DOCX など) を介して処理されます

### XMLメタ文字（例: `<`, `>`, `&`, `'`, `"`）は適切にエスケープまたはサニタイズされていますか？
- [ ] はい — すべてのメタ文字が**適切に**エスケープまたはサニタイズされています *(最も安全)*
- [ ] はい — 一部のフィルタリングは適用されていますが、エンコーディングを使用してバイパスが**可能**です
- [ ] いいえ — 生の入力がサニタイズ**なし**でXML構造に直接埋め込まれています

### すべてのXML入力に対して、サーバー側のスキーマ検証 (Schema validation)（XSD/DTD）が強制されていますか？
- [ ] はい — 厳格なスキーマ検証が**有効**であり、構造的な変更を防止しています
- [ ] はい — 検証は**有効**ですが、データフィールド内へのタグ注入は防止されてい**ません**
- [ ] いいえ — スキーマ検証が**無効**であるか、実装されていません

### 攻撃者は新しいXMLタグを注入して、ドキュメントの構造やロジックを変更できますか？
- [ ] いいえ — 入力に関わらず構造は維持されます
- [ ] はい — タグの注入は可能ですが、ビジネスロジックを変更することは**できません**
- [ ] はい — タグの注入が**可能**であり、ロジックのバイパスやデータの改ざんが許容されます *(重大)*

### CDATAセクションやXMLコメントを使用して、XMLパーサーを操作できますか？
- [ ] いいえ — パーサーはCDATAやコメントマーカーを正しく処理するか、拒否します
- [ ] はい — `<![CDATA[...]]>` や `<!-- ... -->` の注入が**可能**であり、フィルタの隠蔽やバイパスに使用できます
