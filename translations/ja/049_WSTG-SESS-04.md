## WSTG-SESS-04 — 露出したセッション変数のテスト

露出したセッション変数は、機密性の高いセッション識別子 (Session Identifiers) や状態関連のデータが、URLクエリ文字列 (URL Query Strings)、Refererヘッダー (Referer Headers)、またはシステムログなどの安全でないチャネルを介して送信されるときに発生します。このような露出は、識別子がブラウザ履歴にキャッシュされたり、中間プロキシ (Intermediate Proxies) によってログに記録されたり、Refererヘッダーを介してサードパーティのサイトに漏えいしたりするため、セッションハイジャック (Session Hijacking) のリスクを大幅に高めます。ペネトレーションテスター (Pentesters) は、すべての送信リクエストを監視し、不注意なデータ漏えいがないかアプリケーションログやサーバー構成を検査することで、これらの露出を特定します。現実の世界では、攻撃者は共有マシンから有効なセッショントークンを収集したり、トラフィックパターンを分析したりすることで、これらの漏えいを悪用し、認証 (Authentication) をバイパスしてユーザーアカウントへの不正アクセスを取得します。


| 項目 | 値 |
|---|---|
| **WSTG ID** | WSTG-SESS-04 |
| **CWE** | CWE-598 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *露出した変数が、即座のアカウント乗っ取りを可能にするセッショントークンである場合、深刻度は「高」となります。

### セッション識別子がURLクエリ文字列で送信されていますか？
- [ ] いいえ — セッション識別子はURLクエリ文字列に含まれていません  
- [ ] はい — 識別子は含まれていますが、セッションが短期間であるか、リスクが低いです  
- [ ] はい — 一意のセッションIDがURLクエリ文字列に含まれています *(重要)*  

### アプリケーションがRefererヘッダーを通じてセッショントークンを漏えいさせていますか？
- [ ] いいえ — `Referrer-Policy` が漏えいを防止しているか、サードパーティへのリンクが存在しません  
- [ ] はい — セッショントークンがRefererヘッダーを介してサードパーティのドメインに送信されています  

### セッション変数がサーバー側またはアプリケーションのログに保存されていますか？
- [ ] いいえ — セッション変数はマスクされているか、ログから除外されています  
- [ ] はい — セッション変数がアプリケーションまたはウェブサーバーのログにプレーンテキストで記録されています  

### アプリケーションは状態変更操作にGETリクエストを使用していますか？
- [ ] いいえ — アプリケーションは機密性の高い操作に `POST` またはその他の非べき等メソッドを使用しています  
- [ ] はい — `GET` が使用されており、セッションデータがブラウザ履歴や中間プロキシにキャッシュされる原因となっています  

### セッション関連データのキャッシュを防止するように `Cache-Control` ヘッダーが構成されていますか？
- [ ] はい — 機密性の高いレスポンスに `Cache-Control: no-store` (または同様の設定) が適用されています  
- [ ] はい — 制御は実施されていますが、ブラウザの例外的なケースを通じてバイパスが可能です  
- [ ] いいえ — セッションデータを含む機密性の高いレスポンスがブラウザによってキャッシュされる可能性があります
