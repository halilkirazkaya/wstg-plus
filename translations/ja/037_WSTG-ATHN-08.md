## WSTG-ATHN-08 — 脆弱な秘密の質問の回答に関するテスト

脆弱な秘密の質問の回答に関するテストでは、通常、パスワードリカバリや多要素認証 (MFA) フロー中に使用されるユーザーの本人確認のための知識ベース認証 (KBA) メカニズムを評価します。秘密の質問は、オープンソースインテリジェンス (OSINT) やソーシャルエンジニアリングを通じて容易に発見できる静的で非秘密の情報に依存することが多く、不正なアカウント乗っ取り (Account Takeover) につながる可能性があるため、このテストは重要です。これらの脆弱性は、通常、ユーザーが設定済みの質問に対して回答を提供する「パスワードを忘れた場合」や「アカウントのロック解除」モジュールで発生します。攻撃者の視点からは、多くのユーザーが「母親の旧姓は？」や「出身高校は？」といった一般的で公開情報から検証可能な質問に対して予測可能な回答を提供するため、自動化されたブルートフォース (Brute Force) や標的型調査の格好の標的となります。


| フィールド | 値 |
|---|---|
| **WSTG ID** | WSTG-ATHN-08 |
| **CWE** | CWE-640 |
| **テストステータス** | 未実施 |
| **深刻度** | 中 / 高* |

> *秘密の質問が、追加の検証なしに完全なパスワードリセットおよびアカウント乗っ取りを可能にする唯一の要件である場合、深刻度は「高」となります。

### アプリケーションは本人確認やパスワードリカバリのために秘密の質問を実装していますか？
- [ ] いいえ — 秘密の質問は認証やリカバリに**使用されていない**  
- [ ] はい — 秘密の質問がオプションの二要素目として**使用されている**  
- [ ] はい — 秘密の質問がプライマリまたは唯一のリカバリメカニズムとして**使用されている** *(高リスク)*  

### 秘密の質問は固定リストから選択されますか、それともユーザー定義ですか？
- [ ] いいえ — 質問は完全にユーザー定義であり、高いエントロピーを必要とする  
- [ ] はい — 質問は固定されているが、OSINTで発見不可能なオプションが含まれている  
- [ ] はい — 質問は一般的でOSINTにより発見可能なトピックの固定リストから選択されている *(脆弱)*  

### 秘密の質問の回答試行に対して、レート制限 (Rate Limiting) またはアカウントロックアウトが適用されていますか？
- [ ] はい — ブルートフォースを防止するために、厳格なレート制限およびアカウントロックアウトが**適用されている**  
- [ ] はい — レート制限は**適用されている**が、IPローテーションやヘッダー操作による回避が**可能である**  
- [ ] いいえ — 回答試行に対して**レート制限が強制されていない**  

### アプリケーションは回答の内容に対して複雑性や検証を強制していますか？
- [ ] はい — 検証により一般的な単語の使用が禁止され、回答の複雑性と一意性が確保されている  
- [ ] はい — 基本的な検証は存在するが、一般的なワードリストや辞書攻撃によって回避される**可能性がある**  
- [ ] いいえ — **検証は行われていない**。単純な回答や1文字の回答が**許可されている**  

### 秘密の質問のチャレンジをロジックの欠陥によって回避することは可能ですか？
- [ ] いいえ — リカバリフローを続行するには、有効な回答が厳格に要求される  
- [ ] はい — HTTPレスポンスコードやセッションパラメータを操作することで、リカバリフローを**回避できる**  
- [ ] はい — 回答がページのソースコードや隠しフィールドに反映されている
