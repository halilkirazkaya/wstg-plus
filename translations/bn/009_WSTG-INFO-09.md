## WSTG-INFO-09 — ওয়েব অ্যাপ্লিকেশন ফিঙ্গারপ্রিন্টিং (Fingerprint Web Application)

একটি ওয়েব অ্যাপ্লিকেশন ফিঙ্গারপ্রিন্টিং (Fingerprint Web Application) হলো একটি নির্দিষ্ট ফ্রেমওয়ার্ক (Framework), কন্টেন্ট ম্যানেজমেন্ট সিস্টেম (CMS), বা এর পেছনের টেকনোলজি স্ট্যাক (Technology stack) এবং সেটির সংশ্লিষ্ট ভার্সন শনাক্ত করার প্রক্রিয়া। এই রিকনাসেন্স (Reconnaissance) পর্যায়টি অত্যন্ত গুরুত্বপূর্ণ কারণ এটি একজন আক্রমণকারীকে শনাক্তকৃত কম্পোনেন্টগুলোকে পরিচিত দুর্বলতা (CVEs) এবং পাবলিক এক্সপ্লয়েট (Exploits)-এর সাথে মানচিত্রায়ন করতে সাহায্য করে, যা অ্যাটাক সারফেস (Attack surface)-কে উল্লেখযোগ্যভাবে কমিয়ে আনে। তথ্যগুলো সাধারণত HTTP response headers, অনন্য ফাইল পাথ (File paths), কুকি নেম (Cookie names), HTML source code metadata, এবং স্ট্যাটিক অ্যাসেট (Static assets)-এর ক্রিপ্টোগ্রাফিক হ্যাশ থেকে সংগ্রহ করা হয়। টেকনোলজি স্ট্যাক সঠিকভাবে নির্ধারণের মাধ্যমে একজন আক্রমণকারী সাধারণ স্বয়ংক্রিয় স্ক্যানিং থেকে নির্দিষ্ট ভার্সন-ভিত্তিক দুর্বলতাগুলোর লক্ষ্যবস্তু এক্সপ্লোরেশনে (Exploitation) যেতে পারেন।

### HTTP response headers কি টেকনোলজি স্ট্যাক বা ভার্সন নম্বর প্রকাশ করছে?
- [ ] না — সংবেদনশীল হেডারগুলো সরিয়ে ফেলা হয়েছে অথবা সেগুলোতে সাধারণ (Generic) মান রয়েছে  
- [ ] হ্যাঁ — ডিফল্ট হেডার যেমন `X-Powered-By`, `Server`, বা `X-AspNet-Version` উপস্থিত রয়েছে  
- [ ] হ্যাঁ — হেডারগুলো ওয়েব সার্ভার বা অ্যাপ্লিকেশন ফ্রেমওয়ার্কের নির্দিষ্ট এবং পুরনো ভার্সন প্রকাশ করছে  

### অ্যাপ্লিকেশন ফ্রেমওয়ার্ক বা CMS কি অনন্য ফাইল পাথ বা নামকরণের প্রথার মাধ্যমে শনাক্ত করা সম্ভব?
- [ ] না — ফাইল পাথগুলো অবফাসকেটেড (Obfuscated) এবং টেকনোলজি প্রকাশ করে না  
- [ ] হ্যাঁ — সাধারণ ডিরেক্টরি স্ট্রাকচার (যেমন: `/wp-content/`, `/_next/`, `/node_modules/`) দৃশ্যমান  
- [ ] হ্যাঁ — অনন্য ফাইল যেমন `robots.txt`, `humans.txt`, বা `sitemap.xml`-এ টেকনোলজি-নির্দিষ্ট মেটাডেটা রয়েছে  

### HTML সোর্স কোড বা স্ট্যাটিক অ্যাসেটগুলোর মধ্যে কি নির্দিষ্ট ভার্সন নম্বর প্রকাশ করা হয়েছে?
- [ ] না — কমেন্ট বা অ্যাসেট প্যারামিটারে কোনো ভার্সন তথ্য পাওয়া যায়নি  
- [ ] হ্যাঁ — HTML কমেন্ট বা মেটা ট্যাগ (যেমন: `<meta name="generator" content="...">`) উপস্থিত রয়েছে  
- [ ] হ্যাঁ — CSS/JS ফাইল নেমের সাথে ভার্সন স্ট্রিং যুক্ত করা হয়েছে (যেমন: `jquery.js?v=1.12.4`) এবং এটি ডিজেবল করা সম্ভব নয়  

### ডিফল্ট এরর পেজ বা ম্যানেজমেন্ট ইন্টারফেস কি সফটওয়্যার এনভায়রনমেন্ট প্রকাশ করে?
- [ ] না — অ্যাপ্লিকেশন সব স্ট্যাটাস কোডের জন্য কাস্টম এবং সাধারণ এরর পেজ (Error pages) ব্যবহার করে  
- [ ] হ্যাঁ — ওয়েব সার্ভার বা ফ্রেমওয়ার্কের ডিফল্ট এরর পেজগুলো এনাবল করা আছে এবং ভার্সন ডেটা লিক করছে  
- [ ] হ্যাঁ — অ্যাডমিনিস্ট্রেটিভ লগইন পোর্টাল (যেমন: `/wp-admin`, `/phpmyadmin`) অ্যাক্সেসযোগ্য এবং শনাক্তযোগ্য  

### কুকিগুলো কি সেশন ম্যানেজমেন্ট ফ্রেমওয়ার্ক প্রকাশ করছে?
- [ ] না — সেশন কুকিগুলো সাধারণ বা কাস্টম নাম ব্যবহার করে  
- [ ] হ্যাঁ — টেকনোলজি-নির্দিষ্ট কুকি নেম (যেমন: `PHPSESSID`, `JSESSIONID`, `ASPSESSIONID`) ব্যবহৃত হচ্ছে
