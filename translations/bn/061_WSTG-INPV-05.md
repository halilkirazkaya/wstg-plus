## WSTG-INPV-05 — SQL ইনজেকশন (SQL Injection)

SQL ইনজেকশন (SQL Injection) ঘটে তখন যখন ব্যবহারকারীর প্রদান করা ইনপুট (Input) সঠিক প্যারামিটারাইজেশন (Parameterization) বা স্যানিটাইজেশন (Sanitization) ছাড়াই SQL কুয়েরিতে (SQL Queries) অন্তর্ভুক্ত করা হয়, যা আক্রমণকারীকে কুয়েরি লজিক পরিবর্তন করার সুযোগ দেয়। এর সফল প্রয়োগের মাধ্যমে অননুমোদিত ডেটা এক্সফিল্ট্রেশন (Data exfiltration), অথেন্টিকেশন বাইপাস (Authentication bypass), ডেটা পরিবর্তন এবং কিছু ক্ষেত্রে ডাটাবেস ফিচার যেমন `xp_cmdshell` বা `LOAD_FILE()` এর মাধ্যমে রিমোট কোড এক্সিকিউশন (Remote code execution) হতে পারে। এই দুর্বলতা সাধারণত লগইন ফর্ম, সার্চ ফাংশনালিটি, REST API প্যারামিটার, HTTP হেডার এবং ব্যবহারকারী-নিয়ন্ত্রিত ইনপুট থেকে ডাইনামিক SQL কুয়েরি তৈরি করে এমন যেকোনো এন্ডপয়েন্টে (Endpoint) দেখা যায়। একজন আক্রমণকারীর দৃষ্টিকোণ থেকে, একটি ইনজেকশন পয়েন্ট (Injection point) শনাক্ত করা হলো ডাটাবেসের পূর্ণ নিয়ন্ত্রণ নেওয়া এবং ইনফ্রাস্ট্রাকচারের মধ্যে ল্যাটারাল মুভমেন্টের (Lateral movement) প্রাথমিক পথ।

### ব্যবহারকারী-নিয়ন্ত্রিত প্যারামিটারগুলো কি নিরাপদ ডাটাবেস অ্যাক্সেস মেথড ব্যবহার করে প্রসেস করা হয়?
- [ ] হ্যাঁ — অ্যাপ্লিকেশন একচেটিয়াভাবে ORM বা প্যারামিটারাইজড কুয়েরি (Parameterized Queries) ব্যবহার করে এবং বাইপাস করা **সম্ভব নয়**  
- [ ] হ্যাঁ — প্যারামিটারাইজড কুয়েরি ব্যবহার করা হয়েছে কিন্তু এজ কেসগুলোর (যেমন: `ORDER BY` ক্লজ) মাধ্যমে বাইপাস করা **সম্ভব**  
- [ ] না — প্যারামিটারাইজেশন ছাড়াই SQL কুয়েরি তৈরির জন্য র স্ট্রিং কনক্যাটিনেশন (Raw string concatenation) ব্যবহার করা হয়েছে  

### SQL ইনজেকশনের মাধ্যমে কি অথেন্টিকেশন মেকানিজম (Authentication mechanism) বাইপাস করা সম্ভব?
- [ ] না — লগইন প্যারামিটারগুলো ইনজেকশনের জন্য ঝুঁকিপূর্ণ নয়  
- [ ] হ্যাঁ — টটোলজি-ভিত্তিক (Tautology-based) পেলোড (Payload) ব্যবহার করে অথেন্টিকেশন বাইপাস (Authentication bypass) **সম্ভব** (যেমন: `' OR 1=1 --`)  

### ইন-ব্যান্ড (In-band) বা আউট-অফ-ব্যান্ড (Out-of-band) টেকনিকের মাধ্যমে কি ডেটা এক্সফিল্ট্রেশন সম্ভব?
- [ ] না — কোনো ডেটা এক্সফিল্ট্রেশন শনাক্ত করা হয়নি  
- [ ] হ্যাঁ — UNION-ভিত্তিক বা এরর-ভিত্তিক (Error-based) এক্সফিল্ট্রেশন **সম্ভব**  
- [ ] হ্যাঁ — ব্লাইন্ড বা অন্ধ (Boolean বা Time-based) এক্সফিল্ট্রেশন **সম্ভব**  
- [ ] হ্যাঁ — আউট-অফ-ব্যান্ড (DNS/HTTP) এক্সফিল্ট্রেশন **সম্ভব**  

### অ্যাপ্লিকেশন ফাংশনগুলো কি সেকেন্ড-অর্ডার SQL ইনজেকশন (Second-order SQL Injection) দুর্বলতা প্রদর্শন করে?
- [ ] না — সংরক্ষিত ডেটা পরবর্তী কুয়েরির জন্য রিট্রিভ করার সময় নিরাপদে হ্যান্ডেল করা হয়  
- [ ] হ্যাঁ — ব্যবহারকারীর ইনপুট সংরক্ষণ করা হয় এবং পরে ভ্যালিডেশন ছাড়াই একটি অনিরাপদ SQL কুয়েরিতে ব্যবহার করা হয়  

### ডাটাবেস সার্ভিস অ্যাকাউন্টের প্রিভিলেজ (Privileges) কি প্রয়োজনীয় ন্যূনতম সীমায় সীমাবদ্ধ?
- [ ] হ্যাঁ — সার্ভিস অ্যাকাউন্টের ন্যূনতম প্রিভিলেজ (Least privilege) আছে (যেমন: নির্দিষ্ট টেবিল/অ্যাকশনে সীমাবদ্ধ)  
- [ ] না — সার্ভিস অ্যাকাউন্টের এলিভেটেড প্রিভিলেজ (Elevated privileges) আছে (যেমন: `DROP TABLE`, `FILE` প্রিভিলেজ, অথবা `xp_cmdshell` **সক্রিয়** করা আছে)
