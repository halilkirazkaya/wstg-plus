## WSTG-ERRH-01 — ত্রুটিপূর্ণ এরর হ্যান্ডলিং (Improper Error Handling) পরীক্ষা করা

ত্রুটিপূর্ণ এরর হ্যান্ডলিং (Improper Error Handling) তখনই ঘটে যখন একটি ওয়েব অ্যাপ্লিকেশন তার এরর রেসপন্সের মাধ্যমে সংবেদনশীল প্রযুক্তিগত তথ্য—যেমন স্ট্যাক ট্রেস (Stack Traces), ডেটাবেস স্কিমা (Database Schema) তথ্য, বা ইন্টারনাল ফাইল পাথ—প্রকাশ করে দেয়। আক্রমণকারীরা ত্রুটিপূর্ণ ইনপুট জমা দিয়ে, অস্তিত্বহীন রিসোর্স অ্যাক্সেস করে, বা সার্ভার-সাইড এক্সেপশন (Exception) তৈরির মাধ্যমে এই এররগুলো ঘটায় যাতে অ্যাপ্লিকেশনের অন্তর্নিহিত আর্কিটেকচার বোঝা যায় এবং পরবর্তী শোষণের (Exploitation) জন্য সম্ভাব্য ভেক্টরগুলো চিহ্নিত করা যায়। এই তথ্যের ফাঁস (Information Leakage) প্রায়ই আরও মারাত্মক আক্রমণের পূর্বসূরি হিসেবে কাজ করে, যেমন SQL Injection বা Path Traversal, কারণ এটি আক্রমণকারীকে সুনির্দিষ্ট এনভায়রনমেন্ট স্পেসিফিকেশন প্রদান করে। একটি সুরক্ষিত ইমপ্লিমেন্টেশনে ব্যবহারকারীকে সাধারণ বন্ধুত্বপূর্ণ মেসেজ প্রদান করা উচিত এবং বিস্তারিত ডায়াগনস্টিক তথ্য শুধুমাত্র সুরক্ষিত, সার্ভার-সাইড লগে সংরক্ষণ করা উচিত।

### অ্যাপ্লিকেশনটি কি আনহ্যান্ডলড এক্সেপশনগুলোর জন্য একটি জেনেরিক গ্লোবাল এরর হ্যান্ডলার ব্যবহার করে?
- [ ] হ্যাঁ — জেনেরিক এরর পেজগুলো **সক্রিয় (Enabled)** এবং **কোনো** প্রযুক্তিগত তথ্য প্রকাশ করে না  
- [ ] হ্যাঁ — কাস্টম এরর পেজ ব্যবহার করা হয়েছে কিন্তু নির্দিষ্ট হেডারের মাধ্যমে তথ্য ফাঁস (Leakage) **সম্ভব**  
- [ ] না — ডিফল্ট সার্ভার এরর পেজ (যেমন, Tomcat, IIS, Nginx) **দৃশ্যমান**  

### ত্রুটিপূর্ণ ইনপুট বা বাউন্ডারি কেসের (Boundary Cases) মাধ্যমে কি প্রযুক্তিগত তথ্য এনিউমারেট (Enumerate) করা সম্ভব?
- [ ] না — লগিংয়ের জন্য ইউনিক রেফারেন্স আইডি সহ এররগুলো সুন্দরভাবে হ্যান্ডেল করা হয়  
- [ ] হ্যাঁ — রেসপন্সে স্ট্যাক ট্রেস বা ডেটাবেস কুয়েরি **প্রকাশ করা হয়েছে**  
- [ ] হ্যাঁ — ইন্টারনাল ফাইল পাথ, এনভায়রনমেন্ট ভেরিয়েবল (Environment Variables), বা সার্ভার ভার্সন স্ট্রিং **প্রকাশ করা হয়েছে**  

### API রেসপন্সগুলো কি ভারবোস (Verbose) এরর অবজেক্ট বা ডিবাগ তথ্য ফাঁস করছে?
- [ ] না — API-গুলো স্ট্যান্ডার্ড এরর কোড এবং স্যানিটাইজড JSON মেসেজ প্রদান করে  
- [ ] হ্যাঁ — API-গুলো রেসপন্স বডিতে ভারবোস ডিবাগ অবজেক্ট বা সম্পূর্ণ এক্সেপশন ডিটেইলস প্রদান করে  
- [ ] হ্যাঁ — JSON রেসপন্সের `details` বা `exception` ফিল্ডে স্ট্যাক ট্রেস অন্তর্ভুক্ত রয়েছে  

### এরর ঘটলে অ্যাপ্লিকেশনটি কি ভিন্নভাবে (টাইম-বেজড বা কন্টেন্ট-বেজড) আচরণ করে?
- [ ] না — এররের ধরন যাই হোক না কেন রেসপন্স সিগনেচার এবং টাইমিং একই থাকে  
- [ ] হ্যাঁ — ডিফারেনশিয়াল রেসপন্সগুলো ভ্যালিড বনাম ইনভ্যালিড স্টেট (যেমন, ইউজার এনিউমারেশন - User Enumeration) খুঁজে পেতে ব্যবহার করা **যেতে পারে**
