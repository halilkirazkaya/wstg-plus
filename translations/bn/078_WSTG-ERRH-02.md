## WSTG-ERRH-02 — স্ট্যাক ট্রেস পরীক্ষা (Testing for Stack Traces)

অ্যাপ্লিকেশন যখন কোনো এক্সেপশন (Exception) সুন্দরভাবে পরিচালনা করতে ব্যর্থ হয়, তখন স্ট্যাক ট্রেস (Stack Traces) তৈরি হয়, যার ফলে অভ্যন্তরীণ এক্সিকিউশন স্টেট (Execution state) এবং কল হায়ারার্কি (Call hierarchy) শেষ ব্যবহারকারীর কাছে প্রকাশ হয়ে যায়। একজন পেনিট্রেশন টেস্টার (Penetration Tester)-এর জন্য এই ট্রেসগুলো অত্যন্ত মূল্যবান, কারণ এগুলো অ্যাপ্লিকেশনের ফ্রেমওয়ার্ক (Framework), লাইব্রেরি ভার্সন (Library versions), অভ্যন্তরীণ ফাইল সিস্টেম পাথ (Internal file system paths) এবং লজিক ফ্লো প্রকাশ করে, যা রিকনেসান্স (Reconnaissance)-এর জন্য প্রয়োজনীয় প্রচেষ্টাকে উল্লেখযোগ্যভাবে হ্রাস করে। এক্সপ্লয়েটেশন (Exploitation)-এর মধ্যে ম্যালফরমড ইনপুট (Malformed input), অপ্রত্যাশিত ডেটা টাইপ বা বাউন্ডারি কন্ডিশন (Boundary condition) লঙ্ঘনের মাধ্যমে ইচ্ছাকৃতভাবে ত্রুটি তৈরি করা অন্তর্ভুক্ত, যাতে অ্যাপ্লিকেশনটিকে একটি অস্থিতিশীল অবস্থায় নিয়ে গিয়ে অন্তর্নিহিত টেকনোলজি স্ট্যাক (Technology stack) সম্পর্কে তথ্য সংগ্রহ করা যায়। এই তথ্য ফাঁসের ফলে প্রায়শই দুর্বল থার্ড-পার্টি কম্পোনেন্ট (Third-party components) শনাক্ত করতে বা প্রকাশিত কোড পাথের মধ্যে থাকা লজিক ফ্লজ (Logic flaws) এর জন্য নির্দিষ্ট এক্সপ্লয়েট (Exploit) তৈরি করতে প্রয়োজনীয় তথ্য পাওয়া যায়।


| ক্ষেত্র | মান |
|---|---|
| **WSTG ID** | WSTG-ERRH-02 |
| **CWE** | CWE-209 |
| **টেস্টের অবস্থা** | সম্পন্ন হয়নি (Not Performed) |
| **গুরুত্ব (Severity)** | নিম্ন (Low) / মাধ্যম (Medium)* |

### অ্যাপ্লিকেশন এররের ক্ষেত্রে কি সম্পূর্ণ স্ট্যাক ট্রেস ক্লায়েন্টের কাছে ফেরত পাঠানো হয়?
- [ ] না — অ্যাপ্লিকেশন জেনেরিক এরর পেজ (Generic error pages) বা কাস্টম এরর মেসেজ প্রদান করে  
- [ ] হ্যাঁ — স্ট্যাক ট্রেস সক্রিয় (Enabled), তবে শুধুমাত্র নির্দিষ্ট, সংবেদনশীল নয় এমন মডিউলের জন্য  
- [ ] হ্যাঁ — সম্পূর্ণ স্ট্যাক ট্রেস সক্রিয় এবং HTTP রেসপন্স বডিতে দৃশ্যমান  

### এরর মেসেজ কি সংবেদনশীল এনভায়রনমেন্টাল ইনফরমেশন (Environmental information) প্রকাশ করে?
- [ ] না — এরর মেসেজে কোনো প্রযুক্তিগত বা পরিবেশগত বিবরণ নেই  
- [ ] হ্যাঁ — মেসেজগুলো অভ্যন্তরীণ ফাইল পাথ (Internal file paths) বা সার্ভার-সাইড ডিরেক্টরি স্ট্রাকচার প্রকাশ করে  
- [ ] হ্যাঁ — মেসেজগুলো ডেটাবেস স্কিমা (Database schemas), এসকিউএল কোয়েরি (SQL queries) বা থার্ড-পার্টি লাইব্রেরি ভার্সন প্রকাশ করে  

### ইনপুট প্যারামিটার (Input parameters) ম্যানিপুলেট করার মাধ্যমে কি স্ট্যাক ট্রেস ট্রিগার করা যায়?
- [ ] না — ম্যালফরমড ইনপুট সুন্দরভাবে পরিচালনা করা হয় বা ভ্যালিডেশনের মাধ্যমে প্রত্যাখ্যান করা হয়  
- [ ] হ্যাঁ — অপ্রত্যাশিত ডেটা টাইপ (Unexpected data types) দ্বারা ট্রেস ট্রিগার করা যেতে পারে (যেমন, স্ট্রিং প্রত্যাশিত জায়গায় একটি অ্যারে সাবমিট করা)  
- [ ] হ্যাঁ — নাল বাইট (Null bytes), বিশেষ ক্যারেক্টার (Special characters) বা বাউন্ডারি ভ্যালু (Boundary values) দ্বারা ট্রেস ট্রিগার হয়  

### অ্যাপ্লিকেশনে কি সব জায়গায় একটি গ্লোবাল এরর হ্যান্ডলার (Global error handler) ধারাবাহিকভাবে প্রয়োগ করা হয়েছে?
- [ ] হ্যাঁ — একটি গ্লোবাল এক্সেপশন হ্যান্ডলার সমস্ত পরীক্ষিত এন্ডপয়েন্টে (Endpoints) ধারাবাহিকভাবে প্রয়োগ করা হয়েছে  
- [ ] হ্যাঁ — একটি হ্যান্ডলার বিদ্যমান কিন্তু লিগ্যাসি (Legacy), API বা নতুন যোগ করা এন্ডপয়েন্টে প্রয়োগ করা হয়নি  
- [ ] না — কোনো গ্লোবাল এরর হ্যান্ডলিং মেকানিজম শনাক্ত করা যায়নি, যার ফলে ডিফল্ট সার্ভার এরর দেখা দিচ্ছে
