## WSTG-ATHZ-01 — ডাইরেক্টরি ট্র্যাভার্সাল এবং ফাইল ইনক্লুশন পরীক্ষা (Testing Directory Traversal File Include)

ডাইরেক্টরি ট্র্যাভার্সাল (Directory Traversal) এবং ফাইল ইনক্লুশন (File Inclusion) দুর্বলতাগুলো তখন দেখা দেয় যখন একটি অ্যাপ্লিকেশন পর্যাপ্ত ভ্যালিডেশন (Validation) বা স্যানিটাইজেশন (Sanitization) ছাড়াই ফাইল বা ডাইরেক্টরির পাথ (Path) তৈরির জন্য ব্যবহারকারী-নিয়ন্ত্রিত ইনপুট ব্যবহার করে। আক্রমণকারীরা `../` এর মতো সিকোয়েন্স ইনজেক্ট করার মাধ্যমে এই ত্রুটিগুলোর সুযোগ নেয় যাতে উদ্দিষ্ট ডাইরেক্টরির বাইরে নেভিগেট করা যায়, যার ফলে সংবেদনশীল সিস্টেম ফাইল, কনফিগারেশন ডেটা বা অ্যাপ্লিকেশনের সোর্স কোড অ্যাক্সেস করা সম্ভব হতে পারে। লোকাল ফাইল ইনক্লুশন (LFI) বা রিমোট ফাইল ইনক্লুশন (RFI) সম্পর্কিত আরও গুরুতর ক্ষেত্রে, একজন আক্রমণকারী ক্ষতিকারক স্ক্রিপ্ট অন্তর্ভুক্ত করে বা লগ পয়জনিং (Log Poisoning) এবং পিএইচপি র‍্যাপারস (PHP wrappers) ব্যবহার করে রিমোট কোড এক্সিকিউশন (Remote Code Execution - RCE) অর্জন করতে পারে। এই দুর্বলতাগুলো সাধারণত ডায়নামিক কন্টেন্ট লোডিং (Dynamic content loading), টেমপ্লেট ইঞ্জিন (Template engines) বা ইমেজ রিট্রিভাল এন্ডপয়েন্টের (Endpoints) প্যারামিটারগুলোতে থাকে যেখানে সার্ভার-সাইড লজিক (Server-side logic) ফাইল সিস্টেম পাথ হ্যান্ডেল করে।

### প্যারামিটারগুলো কি সার্ভার-সাইড প্রসেসিংয়ের জন্য ফাইলের নাম বা পাথ গ্রহণ করে?
- [ ] না — কোনো প্যারামিটার ফাইল সিস্টেমের সাথে ইন্টারঅ্যাক্ট করে বলে মনে হচ্ছে না  
- [ ] হ্যাঁ — প্যারামিটার বিদ্যমান কিন্তু ফাইল আইডেন্টিফায়ারের একটি কঠোর অ্যালাওলিস্ট (Allowlist) ব্যবহার করে  
- [ ] হ্যাঁ — প্যারামিটার সরাসরি ফাইলের নাম বা পাথ গ্রহণ করে  

### ডাইরেক্টরি ট্র্যাভার্সাল সিকোয়েন্স প্রতিরোধ করতে ইনপুট কি স্যানিটাইজ করা হয়েছে?
- [ ] হ্যাঁ — ইনপুট একটি কঠোর অ্যালাওলিস্টের বিপরীতে যাচাই করা হয় এবং ট্র্যাভার্সাল সিকোয়েন্স **সম্ভব নয়**  
- [ ] হ্যাঁ — `../` সিকোয়েন্স সরানোর মাধ্যমে ইনপুট স্যানিটাইজ করা হয়েছে, কিন্তু রিকার্সিভ বাইপাস (Recursive bypass) **সম্ভব**  
- [ ] না — পাথ-সম্পর্কিত ইনপুটে কোনো স্যানিটাইজেশন বা ভ্যালিডেশন (Validation) **প্রয়োগ করা হয়নি**  

### ট্র্যাভার্সাল সিকোয়েন্সের মাধ্যমে সীমাবদ্ধ ডাইরেক্টরির বাইরের ফাইলগুলো কি অ্যাক্সেস করা সম্ভব?
- [ ] না — অ্যাপ্লিকেশন বা অপারেটিং সিস্টেম (OS) নির্ধারিত স্কোপের বাইরের ফাইলে অ্যাক্সেস প্রতিরোধ করে  
- [ ] হ্যাঁ — ওয়েব রুটের (Web root) ভেতরের ফাইলগুলো অ্যাক্সেস করা **সম্ভব**  
- [ ] হ্যাঁ — সংবেদনশীল সিস্টেম ফাইল (যেমন, `/etc/passwd`, `C:\Windows\win.ini`) অ্যাক্সেস করা **সম্ভব** *(গুরুতর)*  

### সার্ভার কি রিমোট ইউআরএল (RFI) অন্তর্ভুক্ত করার অনুমতি দেয়?
- [ ] না — সার্ভার বা অ্যাপ্লিকেশন স্তরে রিমোট ফাইল ইনক্লুশন **নিষ্ক্রিয়** করা আছে  
- [ ] হ্যাঁ — রিমোট ফাইল অন্তর্ভুক্ত করা **যেতে পারে** কিন্তু এক্সিকিউশন **সম্ভব নয়**  
- [ ] হ্যাঁ — রিমোট ফাইল সার্ভারে অন্তর্ভুক্ত এবং এক্সিকিউট করা **যেতে পারে** *(গুরুতর)*  

### এনকোডিং বা বিশেষ ক্যারেক্টার ব্যবহার করে কি ফিল্টার বাইপাস করা সম্ভব?
- [ ] না — ফিল্টারগুলো কার্যকরভাবে বিভিন্ন এনকোডিং এবং নাল বাইট (Null bytes) হ্যান্ডেল করে  
- [ ] হ্যাঁ — ইউআরএল এনকোডিং (URL encoding), ডাবল ইউআরএল এনকোডিং, বা ১৬-বিট ইউনিকোড ব্যবহার করে বাইপাস **সম্ভব**  
- [ ] হ্যাঁ — নাল বাইট ইনজেকশন (`%00`) বা ফাইল সিস্টেম র‍্যাপারস (যেমন, `php://filter`) ব্যবহার করে বাইপাস **সম্ভব**
