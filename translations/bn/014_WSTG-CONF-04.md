## WSTG-CONF-04 — সংবেদনশীল তথ্যের জন্য পুরনো ব্যাকআপ এবং অপ্রাসঙ্গিক ফাইলসমূহ পর্যালোচনা (Review Old Backup and Unreferenced Files for Sensitive Information)

পুরনো ব্যাকআপ (Backup) এবং অপ্রাসঙ্গিক ফাইল পর্যালোচনা করার অর্থ হলো ওয়েব সার্ভারে থাকা ভুলে যাওয়া, অস্থায়ী বা লুকানো ফাইলগুলো চিহ্নিত করা যা জনসাধারণের ব্যবহারের জন্য নয়। এই ফাইলগুলোতে প্রায়শই সোর্স কোড ব্যাকআপ (Source Code Backups) (`.zip`, `.bak`), টেক্সট এডিটর সোয়াপ ফাইল (Text Editor Swap Files) (`.swp`, `~`), অথবা সোর্স কন্ট্রোল মেটাডেটা (Source Control Metadata) (`.git`, `.svn`) থাকে যা সংবেদনশীল তথ্য ফাঁস করতে পারে। আক্রমণকারীরা সাধারণ নামকরণের নিয়মাবলী ব্রুট ফোর্স (Brute Force) করার জন্য স্বয়ংক্রিয় ওয়ার্ডলিস্ট (Wordlists) এবং ডিসকভারি টুলস (Discovery Tools) ব্যবহার করে, যার উদ্দেশ্য হলো ডাটাবেস ক্রেডেনশিয়াল (Database Credentials), হার্ডকোডেড API কী (API Keys), অথবা এমন লজিক বের করা যা পরবর্তী শোষণে (Exploitation) সহায়তা করে। এই দুর্বলতা সাধারণত ম্যানুয়াল সার্ভার রক্ষণাবেক্ষণ বা ত্রুটিপূর্ণ CI/CD পাইপলাইন থেকে উদ্ভূত হয় যা প্রোডাকশন ওয়েব রুটকে (Production Web Root) স্যানিটাইজ (Sanitize) করতে ব্যর্থ হয়।

### ওয়েব সার্ভারে কি ডিরেক্টরি লিস্টিং (Directory Listings) সক্রিয় আছে?
- [ ] না — ডিরেক্টরি লিস্টিং নিষ্ক্রিয় (Disabled) এবং 403 Forbidden বা কাস্টম ত্রুটি প্রদান করে  
- [ ] হ্যাঁ — অ-সংবেদনশীল ডিরেক্টরিগুলোতে ডিরেক্টরি লিস্টিং সক্রিয়  
- [ ] হ্যাঁ — সোর্স কোড বা সংবেদনশীল ফাইল থাকা ডিরেক্টরিগুলোতে ডিরেক্টরি লিস্টিং সক্রিয় *(গুরুতর/Critical)*  

### সাধারণ এক্সটেনশন (যেমন: .bak, .old, .save) যুক্ত ব্যাকআপ ফাইলগুলো কি খুঁজে পাওয়া যায়?
- [ ] না — সাধারণ ব্যাকআপ এক্সটেনশনগুলো খুঁজে পাওয়া যায়নি অথবা সার্ভার কনফিগারেশনের মাধ্যমে ব্লক করা আছে  
- [ ] হ্যাঁ — ব্যাকআপ ফাইল বিদ্যমান কিন্তু সেগুলোতে কোনো সংবেদনশীল তথ্য নেই  
- [ ] হ্যাঁ — কনফিগারেশন বা সোর্স কোডের ব্যাকআপ ফাইলগুলো অ্যাক্সেসযোগ্য (Accessible) *(উচ্চ/High)*  

### সোর্স কন্ট্রোল ডিরেক্টরি (যেমন: .git, .svn) বা মেটাডেটা কি উন্মুক্ত (Exposed)?
- [ ] না — সোর্স কন্ট্রোল মেটাডেটা উপস্থিত নেই অথবা সঠিকভাবে ব্লক করা আছে  
- [ ] হ্যাঁ — মেটাডেটা বিদ্যমান কিন্তু পুরো রিপোজিটরি (Repository) পুনর্গঠন করা সম্ভব নয়  
- [ ] হ্যাঁ — উন্মুক্ত মেটাডেটার মাধ্যমে সম্পূর্ণ সোর্স কোড রিপোজিটরি এক্সফিল্ট্রেট (Exfiltrate) করা সম্ভব  

### ওয়েব রুটে কি অ্যাপ্লিকেশনের কম্প্রেসড আর্কাইভ (Compressed Archives) (যেমন: .zip, .tar.gz, .rar) বিদ্যমান?
- [ ] না — ব্রুট ফোর্স বা এনামেরেশন (Enumeration) এর মাধ্যমে কোনো সংবেদনশীল আর্কাইভ ফাইল পাওয়া যায়নি  
- [ ] হ্যাঁ — আর্কাইভ পাওয়া গেছে কিন্তু সেগুলো পাসওয়ার্ড সুরক্ষিত অথবা পাবলিক অ্যাসেট ধারণ করে  
- [ ] হ্যাঁ — অ্যাপ্লিকেশন সোর্স বা ডেটা সম্বলিত আনএনক্রিপ্টেড (Unencrypted) আর্কাইভগুলো জনসম্মুখে অ্যাক্সেসযোগ্য  

### অ্যাপ্লিকেশনটি কি টেক্সট এডিটর বা IDE দ্বারা তৈরি অস্থায়ী ফাইল (Temporary Files) ফাঁস করে?
- [ ] না — .swp, ~, বা .DS_Store এর মতো অস্থায়ী ফাইলগুলো উপস্থিত নেই  
- [ ] হ্যাঁ — অ-সংবেদনশীল অস্থায়ী ফাইলগুলো উপস্থিত আছে  
- [ ] হ্যাঁ — অস্থায়ী ফাইলগুলো সোর্স কোডের অংশ বা অভ্যন্তরীণ পাথ (Internal Paths) প্রকাশ করে
