## WSTG-INFO-05 — তথ্য ফাঁসের (Information Leakage) জন্য ওয়েবপেজের বিষয়বস্তু পর্যালোচনা

ওয়েবপেজের বিষয়বস্তু পর্যালোচনার মধ্যে অ্যাপ্লিকেশন রেসপন্স (Response), যার মধ্যে HTML, JavaScript এবং CSS ফাইল অন্তর্ভুক্ত, সেগুলোর ম্যানুয়াল এবং স্বয়ংক্রিয় পরিদর্শন করা হয় যাতে ব্যবহারকারীদের কাছে অনিচ্ছাকৃতভাবে উন্মোচিত সংবেদনশীল তথ্য (Sensitive Information) সনাক্ত করা যায়। আক্রমণকারীরা ডেভেলপার কমেন্ট (Developer Comments), হিডেন ফর্ম ফিল্ড (Hidden form fields), ইন্টারনাল সার্ভার পাথ (Internal server paths), হার্ডকোডেড এপিআই কী (Hardcoded API keys) এবং ডিবাগিং তথ্যের (Debugging information) জন্য ক্লায়েন্ট-সাইড সোর্স কোড (Client-side source code) নিবিড়ভাবে পরীক্ষা করে, যা পরবর্তী এক্সপ্লয়টেশন (Exploitation) পর্যায়ে সহায়ক হতে পারে। এই ধরনের ফাঁস সাধারণত ঘটে যখন ডেভেলপাররা প্রোডাকশনে (Production) যাওয়ার আগে মেটাডেটা (Metadata) বা ডিবাগিং কোড মুছে ফেলতে ভুলে যান, যা সম্ভাব্যভাবে লজিক ফ্লজ (Logic flaws) বা ব্যাকএন্ড ইনফ্রাস্ট্রাকচার ডিটেইলস প্রকাশ করে। একজন আক্রমণকারীর দৃষ্টিকোণ থেকে, এটি সিকিউরিটি অ্যালার্ট (Security alerts) ট্রিগার না করে বা সার্ভারের ব্যাকএন্ড লজিকের (Backend logic) সাথে ইন্টারঅ্যাক্ট না করেই অ্যাপ্লিকেশনের অভ্যন্তরীণ কাঠামো ম্যাপ করার এবং উপেক্ষা করা এন্ট্রি পয়েন্টগুলো (Entry points) খুঁজে বের করার একটি সহজ পদ্ধতি প্রদান করে।

### সোর্স কোডে কি সংবেদনশীল তথ্য সম্বলিত ডেভেলপার কমেন্ট উপস্থিত আছে?
- [ ] না — কোনো কমেন্ট নেই অথবা শুধুমাত্র সাধারণ, অ-সংবেদনশীল কমেন্ট পাওয়া গেছে  
- [ ] হ্যাঁ — কমেন্ট আছে কিন্তু তাতে কোনো সংবেদনশীল লজিক বা ডেটা নেই  
- [ ] হ্যাঁ — কমেন্টগুলো ইন্টারনাল পাথ, SQL কুয়েরি (SQL queries) বা প্রশাসনিক নির্দেশাবলী প্রকাশ করে  
- [ ] হ্যাঁ — কমেন্টগুলো ক্রেডেনশিয়াল বা সংবেদনশীল ডেভেলপার নোট প্রকাশ করে *(উচ্চ)*  

### হিডেন HTML ইনপুট ফিল্ডগুলোতে কি সংবেদনশীল ডেটা বা স্টেট ইনফরমেশন (State information) আছে?
- [ ] না — কোনো হিডেন ফিল্ড পাওয়া যায়নি অথবা সেগুলোতে শুধুমাত্র অ-সংবেদনশীল টোকেন রয়েছে  
- [ ] হ্যাঁ — হিডেন ফিল্ডগুলো স্টেটের জন্য ব্যবহৃত হয় কিন্তু সেগুলো এনক্রিপ্টেড (Encrypted) বা সাইনড (Signed) করা  
- [ ] হ্যাঁ — হিডেন ফিল্ডগুলোতে প্লেইনটেক্সট (Plaintext) পাসওয়ার্ড, ইউজার রোল বা ইন্টারনাল আইডি রয়েছে  

### JavaScript ফাইলগুলোতে কি হার্ডকোডেড সিক্রেট (Secrets), এপিআই কী (API keys), অথবা ইন্টারনাল আইপি (IP) অ্যাড্রেস উপস্থিত আছে?
- [ ] না — JS ফাইলগুলোতে কোনো সংবেদনশীল স্ট্রিং বা সিক্রেট পাওয়া যায়নি  
- [ ] হ্যাঁ — JS ফাইলগুলোতে সঠিক বিধিনিষেধসহ পাবলিক এপিআই কী রয়েছে  
- [ ] হ্যাঁ — JS ফাইলগুলোতে অরক্ষিত (Unprotected) এপিআই কী, ইন্টারনাল আইপি, বা প্রাইভেট এন্ডপয়েন্ট (Endpoint) রয়েছে  
- [ ] হ্যাঁ — JS ফাইলগুলোতে হার্ডকোডেড প্রশাসনিক ক্রেডেনশিয়াল বা প্রাইভেট কী রয়েছে *(ক্রিটিক্যাল)*  

### অ্যাপ্লিকেশনটি কি সোর্স কোডে ফ্রেমওয়ার্ক ভার্সন বা ইন্টারনাল ফাইল পাথ প্রকাশ করে?
- [ ] না — ফ্রেমওয়ার্ক তথ্য এবং ফাইল পাথ মিনিফাইড (Minified) বা অবফাসকেটেড (Obfuscated) করা  
- [ ] হ্যাঁ — ফ্রেমওয়ার্কের নাম এবং ভার্সন দৃশ্যমান কিন্তু প্যাচ (Patch) করা  
- [ ] হ্যাঁ — সোর্স কোডে ইন্টারনাল ফাইল পাথ বা অ্যাবসলিউট সার্ভার পাথ উন্মোচিত হয়েছে  

### মিনিফিকেশন (Minification) বা অবফাসকেশনের (Obfuscation) অভাবের কারণে কি সোর্স কোডটি ফাঁস হওয়ার ঝুঁকিতে রয়েছে?
- [ ] না — প্রোডাকশন কোড সম্পূর্ণ মিনিফাইড এবং কমেন্টগুলো মুছে ফেলা হয়েছে  
- [ ] হ্যাঁ — কোড মিনিফাইড কিন্তু সোর্স কোডে কমেন্টগুলো রয়ে গেছে  
- [ ] হ্যাঁ — সোর্স ম্যাপ (`.map` ফাইল) জনসমক্ষে অ্যাক্সেসযোগ্য, যা সোর্স কোডের পূর্ণ পুনর্গঠনের অনুমতি দেয়
