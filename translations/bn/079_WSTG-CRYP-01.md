## WSTG-CRYP-01 — দূর্বল ট্রান্সপোর্ট লেয়ার সিকিউরিটি পরীক্ষা (Testing for Weak Transport Layer Security)

দূর্বল ট্রান্সপোর্ট লেয়ার সিকিউরিটি পরীক্ষা করার মধ্যে ট্রানজিটের সময় ডেটার গোপনীয়তা এবং অখণ্ডতা নিশ্চিত করতে SSL/TLS-এর বাস্তবায়ন এবং কনফিগারেশন বিশ্লেষণ করা অন্তর্ভুক্ত। আক্রমণকারীরা ম্যান-ইন-দ্য-মিডল (Man-in-the-Middle - MitM) আক্রমণ পরিচালনার জন্য পুরানো প্রোটোকল (SSLv2, TLS 1.0), দূর্বল সাইফার স্যুট (Cipher Suites - NULL, RC4, বা anonymous), এবং অবৈধ বা দূর্বলভাবে স্বাক্ষরিত সার্টিফিকেটের মতো ভুল কনফিগারেশনগুলিকে লক্ষ্য করে। এই পরীক্ষাটি সাধারণত অ্যাপ্লিকেশনের ওয়েব সার্ভার বা লোড ব্যালেন্সার এন্ডপয়েন্টগুলিকে লক্ষ্য করে যেখানে এনক্রিপ্টেড যোগাযোগ স্থাপন করা হয়। সফল এক্সপ্লয়েট (Exploit) একজন আক্রমণকারীকে সংবেদনশীল ডেটা ইন্টারসেপ্ট করতে, ইউজার সেশন হাইজ্যাক (Hijack) করতে বা প্রোটোকল ডাউনগ্রেড (Downgrade) বা ট্রাফিক ডিক্রিপশনের মাধ্যমে সংযোগগুলিকে অনিরাপদ অবস্থায় নামিয়ে আনতে অনুমতি দেয়।


| ক্ষেত্র | মান |
|---|---|
| **WSTG ID** | WSTG-CRYP-01 |
| **CWE** | CWE-326 |
| **পরীক্ষার স্থিতি** | সম্পন্ন করা হয়নি |
| **গুরুত্ব (Severity)** | উচ্চ / মাঝারি* |

> *গুরুত্ব উচ্চ (High) হবে যদি সংবেদনশীল ডেটা (PII, ক্রেডেনশিয়াল, সেশন টোকেন) স্থানান্তরিত হয়; সাধারণ তথ্যগত ট্রাফিকের জন্য মাঝারি (Medium)।

### পুরানো বা অনিরাপদ TLS/SSL প্রোটোকলগুলি কি সমর্থিত (Supported)?
- [ ] না — শুধুমাত্র TLS 1.2 এবং TLS 1.3 সক্রিয় (Enabled)  
- [ ] হ্যাঁ — TLS 1.0 বা TLS 1.1 সক্রিয় (Enabled)  
- [ ] হ্যাঁ — SSLv2 বা SSLv3 সক্রিয় (Enabled) *(ক্রিটিক্যাল)*  

### সার্ভার দ্বারা কি দূর্বল বা অনিরাপদ সাইফার স্যুট (Cipher Suites) গ্রহণ করা হয়?
- [ ] না — শুধুমাত্র শক্তিশালী, আধুনিক সাইফার (যেমন: AES-GCM, ChaCha20) সমর্থিত (Supported)  
- [ ] হ্যাঁ — দূর্বল এনক্রিপশন যুক্ত সাইফার (যেমন: 3DES, RC4) বা কম বিট-লেন্থ সমর্থিত (Supported)  
- [ ] হ্যাঁ — NULL, anonymous (ADH), বা এক্সপোর্ট সাইফার সমর্থিত (Supported) *(ক্রিটিক্যাল)*  

### সার্ভার সার্টিফিকেট কি বৈধ এবং বিশ্বস্ত?
- [ ] হ্যাঁ — সার্টিফিকেট বৈধ, ডোমেনের সাথে মিল আছে এবং একটি বিশ্বস্ত CA (Certificate Authority) দ্বারা স্বাক্ষরিত  
- [ ] না — সার্টিফিকেটের মেয়াদ শেষ (Expired) বা দূর্বল সিগনেচার অ্যালগরিদম (যেমন: SHA-1) ব্যবহার করা হয়েছে  
- [ ] না — সার্টিফিকেটটি সেলফ-সাইনড (Self-signed) বা ডোমেন নেম অমিল (Mismatch) ঘটেছে  

### সার্ভার কি সিকিউর রিনিগোসিয়েশন (Secure Renegotiation) সমর্থন করে এবং ডাউনগ্রেড (Downgrade) আক্রমণ প্রতিরোধ করে?
- [ ] হ্যাঁ — সিকিউর রিনিগোসিয়েশন সমর্থিত এবং TLS Fallback SCSV সক্রিয় আছে  
- [ ] না — সিকিউর রিনিগোসিয়েশন সমর্থিত নয়, যা সম্ভাব্য MitM ইনজেকশনের সুযোগ দেয়  
- [ ] না — সার্ভারটি POODLE বা ROBOT আক্রমণের জন্য ঝুঁকিপূর্ণ  

### HTTP স্ট্রিক্ট ট্রান্সপোর্ট সিকিউরিটি (HTTP Strict Transport Security - HSTS) কি সঠিকভাবে বাস্তবায়িত হয়েছে?

| হেডার | উপস্থিত | অনুপস্থিত |
|--------|:-------:|:-------:|
| `Strict-Transport-Security` |  |  |

- [ ] হ্যাঁ — দীর্ঘ `max-age` এবং `includeSubDomains` সক্রিয় (Enabled) সহ HSTS হেডার উপস্থিত আছে  
- [ ] হ্যাঁ — HSTS হেডার উপস্থিত কিন্তু `includeSubDomains` অনুপস্থিত বা এতে সংক্ষিপ্ত `max-age` রয়েছে  
- [ ] না — HSTS হেডার উপস্থিত নেই
