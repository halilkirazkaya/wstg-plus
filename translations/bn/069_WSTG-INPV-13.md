## WSTG-INPV-13 — Format String Injection

ফরম্যাট স্ট্রিং ইনজেকশন (Format String Injection) তখন ঘটে যখন একটি ওয়েব অ্যাপ্লিকেশন ব্যবহারকারীর নিয়ন্ত্রিত ইনপুটকে সরাসরি একটি ভ্যারিয়াডিক ফাংশন (variadic function), যেমন C এর `printf` ফ্যামিলি বা লো-লেভেল লগিং র‍্যাপারের ফরম্যাট স্ট্রিং আর্গুমেন্টে পাস করে। যদিও আধুনিক ম্যানেজড-কোড (managed-code) ওয়েব ফ্রেমওয়ার্কগুলোতে এটি কম দেখা যায়, তবুও লিগ্যাসি সিজিআই (CGI) অ্যাপ্লিকেশন, নেটিভ এক্সটেনশন বা লো-লেভেল সিস্টেম লাইব্রেরির সাথে ইন্টারঅ্যাক্ট করা এজ-কেস লগিং সিস্টেমে এই দুর্বলতাটি অত্যন্ত গুরুত্বপূর্ণ। আক্রমণকারীরা `%x` বা `%p` এর মতো ফরম্যাট স্পেসিফায়ার সরবরাহ করে স্ট্যাক (Stack) থেকে সংবেদনশীল মেমরি অ্যাড্রেস এবং ডেটা লিক করে, অথবা `%n` স্পেসিফায়ার ব্যবহার করে যথেচ্ছ মেমরি রাইট (arbitrary memory writes) সম্পন্ন করে। সফলভাবে এক্সপ্লয়েট (Exploit) করার ফলে সাধারণত রিমোট কোড এক্সিকিউশন (Remote Code Execution - RCE) এর মাধ্যমে সম্পূর্ণ সিস্টেম কম্প্রোমাইজ হতে পারে অথবা অন্ততপক্ষে প্রসেস ক্র্যাশের মাধ্যমে একটি প্রভাবশালী ডিনায়েল-অফ-সার্ভিস (Denial-of-Service - DoS) অবস্থার সৃষ্টি হতে পারে।

### অ্যাপ্লিকেশন কি নেটিভ কোড বা লিগ্যাসি CGI ইন্টারফেসের মাধ্যমে ব্যবহারকারীর ইনপুট প্রসেস করে?
- [ ] No — অ্যাপ্লিকেশনটি সম্পূর্ণভাবে ম্যানেজড কোড ফ্রেমওয়ার্কের (যেমন: JVM, CLR) উপর ভিত্তি করে তৈরি  
- [ ] Yes — অ্যাপ্লিকেশনটি JNI, নেটিভ C/C++ এক্সটেনশন বা লিগ্যাসি বাইনারি CGI ব্যবহার করে যেখানে ফরম্যাট স্ট্রিং দুর্বলতা থাকতে **পারে**  

### ব্যবহারকারী সরবরাহকৃত ইনপুট কি ফরম্যাট-অ্যাওয়্যার (format-aware) ফাংশনগুলোর প্রাথমিক আর্গুমেন্ট হিসেবে পাস করা হয়?
- [ ] No — ইনপুটগুলো ডেটা আর্গুমেন্ট হিসেবে পাস করা হয় এবং ফরম্যাট স্ট্রিংগুলো **স্ট্যাটিক (static)** এবং **হার্ডকোডেড (hardcoded)**  
- [ ] Yes — ব্যবহারকারীর ইনপুট সরাসরি ফরম্যাট স্ট্রিং আর্গুমেন্টের সাথে সংযুক্ত করা হয়, তবে স্যানিটাইজেশন (Sanitization) **প্রয়োগ করা হয়েছে**  
- [ ] Yes — ব্যবহারকারীর ইনপুট সরাসরি ফরম্যাট স্ট্রিং হিসেবে ব্যবহৃত হয় এবং কোনো স্যানিটাইজেশন **প্রয়োগ করা হয়নি**  

### ফরম্যাট স্পেসিফায়ার ব্যবহার করে কি মেমরি কন্টেন্ট লিক করা সম্ভব?
- [ ] No — ইনপুট যাচাই করা হয় এবং `%p`, `%x`, বা `%s` এর মতো স্পেসিফায়ারগুলো **প্রসেস করা হয় না**  
- [ ] Yes — `%p` বা `%x` স্পেসিফায়ার সরবরাহ করলে স্ট্যাক বা হিপ (Heap) মেমরি অ্যাড্রেস প্রতিফলিত হয়  
- [ ] Yes — ফরম্যাট স্পেসিফায়ারের মাধ্যমে সংবেদনশীল ডেটা (যেমন: পয়েন্টার, স্ট্যাক কুকিজ) এক্সফিল্ট্রেট (Exfiltrate) করা **সম্ভব**  

### `%n` স্পেসিফায়ার ব্যবহার করে কি অ্যাপ্লিকেশনটি ক্র্যাশ বা ম্যানিপুলেট করা সম্ভব?
- [ ] No — মেমরি রাইট করার অনুমতি দেয় এমন স্পেসিফায়ারগুলো কম্পাইলার বা এনভায়রনমেন্ট দ্বারা **নিষ্ক্রিয় (disabled)** বা ব্লক করা আছে  
- [ ] Yes — যখন `%s%s%s%s` বা অনুরূপ স্ট্রিং প্রদান করা হয় তখন অ্যাপ্লিকেশনটি ক্র্যাশ (DoS) করে  
- [ ] Yes — `%n` এর মাধ্যমে যথেচ্ছ মেমরি রাইট **সম্ভব**, যা সম্ভাব্যভাবে রিমোট কোড এক্সিকিউশন (RCE) এর দিকে পরিচালিত করতে পারে  

### আধুনিক বাইনারি প্রোটেকশন (ASLR, DEP, Stack Canaries) কি এই ইনজেকশনের মাধ্যমে বাইপাস করা সম্ভব?
- [ ] No — এনভায়রনমেন্টটি হার্ডেনড (Hardened) এবং মেমরি লিকেজ প্রোটেকশন বাইপাস করার জন্য ব্যবহার করা **যাবে না**  
- [ ] Yes — ফরম্যাট স্ট্রিং ইনজেকশন ব্যবহার করে বেস অ্যাড্রেস লিক করা **সম্ভব**, যা ASLR/DEP বাইপাস করাকে **সম্ভব** করে তোলে
