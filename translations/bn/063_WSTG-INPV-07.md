## WSTG-INPV-07 — এক্সএমএল ইনজেকশন (XML Injection)

এক্সএমএল ইনজেকশন (XML Injection) ঘটে যখন একটি অ্যাপ্লিকেশন XML মেটা-ক্যারেক্টারগুলোর যথাযথ ভ্যালিডেশন (Validation), স্যানিটাইজেশন (Sanitization) বা এস্কেপিং (Escaping) ছাড়াই ব্যবহারকারী-প্রদত্ত ডেটাকে একটি XML ডকুমেন্ট বা স্ট্রিমে যুক্ত করে। নির্দিষ্ট XML ট্যাগ বা স্ট্রাকচারাল এলিমেন্ট ইনজেক্ট করার মাধ্যমে, একজন আক্রমণকারী বা অ্যাটাকার (Attacker) ডকুমেন্টের লজিক ম্যানিপুলেট করতে পারে, অথেন্টিকেশন মেকানিজম (Authentication mechanisms) বাইপাস করতে পারে অথবা ব্যাকএন্ড দ্বারা প্রক্রিয়াজাত ডেটার ইন্টিগ্রিটি (Integrity) নষ্ট করতে পারে। এই ভালনারেবিলিটি (Vulnerability) সাধারণত SOAP-ভিত্তিক ওয়েব সার্ভিস, XML গ্রহণকারী REST APIs, SAML অ্যাসারশন (SAML assertions) এবং অ্যাপ্লিকেশনগুলোতে পাওয়া যায় যা ডাইনামিকভাবে XML কনফিগারেশন ফাইল বা রিপোর্ট তৈরি করে। একজন অ্যাটাকারের দৃষ্টিকোণ থেকে, লক্ষ্য হলো ডকুমেন্টের কাঠামো পরিবর্তনের জন্য নির্ধারিত ডেটা কনটেক্সট (Context) থেকে বেরিয়ে আসা, যা অননুমোদিত প্রিভিলেজ এসক্যালেশন (Privilege escalation) বা অনাকাঙ্ক্ষিত বিজনেস লজিক (Business logic) কার্যকর করার দিকে পরিচালিত করতে পারে।

### অ্যাপ্লিকেশনটি কি ব্যবহারকারীর কাছ থেকে XML-ফরম্যাট করা ইনপুট গ্রহণ এবং প্রসেস করে?
- [ ] না — অ্যাপ্লিকেশন XML ইনপুট প্রসেস করে না  
- [ ] হ্যাঁ — XML ইনপুট APIs (SOAP/REST) এর মাধ্যমে প্রসেস করা হয়  
- [ ] হ্যাঁ — XML ফাইল আপলোডের (SVG, DOCX ইত্যাদি) মাধ্যমে প্রসেস করা হয়  

### XML মেটা-ক্যারেক্টারগুলো (যেমন: `<`, `>`, `&`, `'`, `"`) কি সঠিকভাবে এস্কেপ বা স্যানিটাইজ করা হয়েছে?
- [ ] হ্যাঁ — সমস্ত মেটা-ক্যারেক্টার সঠিকভাবে এস্কেপ বা স্যানিটাইজ করা হয়েছে *(সবচেয়ে নিরাপদ)*  
- [ ] হ্যাঁ — কিছু ফিল্টারিং প্রয়োগ করা হয়েছে তবে এনকোডিং ব্যবহার করে বাইপাস (Bypass) করা সম্ভব  
- [ ] না — র (Raw) ইনপুট স্যানিটাইজেশন ছাড়াই সরাসরি XML স্ট্রাকচারে এম্বেড করা হয়েছে  

### সমস্ত XML ইনপুটের জন্য কি সার্ভার-সাইড স্কিমা ভ্যালিডেশন (XSD/DTD) এনফোর্স করা হয়েছে?
- [ ] হ্যাঁ — কঠোর স্কিমা ভ্যালিডেশন (Schema validation) চালু করা হয়েছে এবং এটি কাঠামোগত পরিবর্তন প্রতিরোধ করে  
- [ ] হ্যাঁ — ভ্যালিডেশন চালু আছে কিন্তু ডেটা ফিল্ডের মধ্যে ট্যাগ ইনজেকশন প্রতিরোধ করে না  
- [ ] না — স্কিমা ভ্যালিডেশন নিষ্ক্রিয় করা হয়েছে বা ইমপ্লিমেন্ট করা হয়নি  

### একজন অ্যাটাকার কি ডকুমেন্টের কাঠামো বা লজিক পরিবর্তন করার জন্য নতুন XML ট্যাগ ইনজেক্ট করতে পারে?
- [ ] না — ইনপুট যাই হোক না কেন কাঠামো অক্ষত থাকে  
- [ ] হ্যাঁ — ট্যাগ ইনজেক্ট করা যেতে পারে কিন্তু বিজনেস লজিক পরিবর্তন করা যায় না  
- [ ] হ্যাঁ — ট্যাগ ইনজেকশন সম্ভব এবং এটি লজিক বাইপাস বা ডেটা মডিফিকেশনের সুযোগ দেয় *(গুরুতর)*  

### সি-ডেটা সেকশন (CDATA sections) বা XML কমেন্টস ব্যবহার করে কি XML পার্সারকে ম্যানিপুলেট করা সম্ভব?
- [ ] না — পার্সার (Parser) সঠিকভাবে CDATA/কমেন্ট মার্কারগুলো হ্যান্ডেল করে বা রিজেক্ট করে  
- [ ] হ্যাঁ — ফিল্টার লুকানোর বা বাইপাস করার জন্য `<![CDATA[...]]>` বা `<!-- ... -->` ইনজেকশন করা সম্ভব
