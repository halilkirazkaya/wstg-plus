## WSTG-APIT-02 — এপিআই ব্রোকেন অবজেক্ট লেভেল অথোরাইজেশন (API Broken Object Level Authorization - BOLA)

ব্রোকেন অবজেক্ট লেভেল অথোরাইজেশন (Broken Object Level Authorization - BOLA), যা ইনসিকিউর ডাইরেক্ট অবজেক্ট রেফারেন্স (Insecure Direct Object Reference - IDOR) নামেও পরিচিত, তখন ঘটে যখন একটি API এটি যাচাই করতে ব্যর্থ হয় যে কোনও ব্যবহারকারীর একটি আইডি (ID) দ্বারা চিহ্নিত নির্দিষ্ট রিসোর্স (Resource) অ্যাক্সেস বা ম্যানিপুলেট করার যথাযথ অনুমতি আছে কি না। আক্রমণকারীরা অন্যান্য ব্যবহারকারীদের ডেটা অ্যাক্সেস করার জন্য রিকোয়েস্ট পাথ (Request paths), কোয়েরি প্যারামিটার (Query parameters), বা জেএসওএন বডি (JSON bodies)-তে আইডেন্টিফায়ার—যেমন নিউমেরিক আইডি (Numeric IDs) বা ইউইউআইডি (UUIDs)—সিস্টেম্যাটিকভাবে ইনুমারেট বা অনুমান করার মাধ্যমে এর সুযোগ নেয়। আধুনিক API সিকিউরিটির ক্ষেত্রে এই ভালনারেবিলিটিটি (Vulnerability) সবচেয়ে সাধারণ এবং প্রভাবশালী সমস্যা, যা ব্যাপক হারে ডেটা এক্সফিল্ট্রেশন (Data exfiltration), অননুমোদিত পরিবর্তন বা সম্পূর্ণ অ্যাকাউন্ট টেকওভার (Account takeover)-এর দিকে পরিচালিত করতে পারে। একজন আক্রমণকারীর দৃষ্টিকোণ থেকে লক্ষ্য হলো এমন এন্ডপয়েন্ট (Endpoints) শনাক্ত করা যা অবজেক্ট আইডেন্টিফায়ার গ্রহণ করে এবং পরীক্ষা করা যে সার্ভার-সাইড অথোরাইজেশন লজিক (Authorization logic) অনুপস্থিত কি না বা সেই আইডিগুলো পরিবর্তনের মাধ্যমে বাইপাস (Bypass) করা যায় কি না।

### API রিকোয়েস্টের মধ্যে অবজেক্ট আইডেন্টিফায়ারগুলো কি অনুমানযোগ্য বা গণনাযোগ্য (Enumerable)?
- [ ] না — আইডেন্টিফায়ারগুলো দীর্ঘ, র‍্যান্ডম এবং ক্রিপ্টোগ্রাফিকভাবে সুরক্ষিত (যেমন: UUIDv4)  
- [ ] হ্যাঁ — আইডেন্টিফায়ারগুলো ক্রমিক সংখ্যা বা সিকোয়েন্সিয়াল ইন্টিজার (যেমন: `101`, `102`)  
- [ ] হ্যাঁ — আইডেন্টিফায়ারগুলো একটি অনুমানযোগ্য প্যাটার্ন অনুসরণ করে অথবা পাবলিক তথ্য থেকে প্রাপ্ত  

### API কি প্রতিটি রিকোয়েস্টের জন্য অবজেক্টের মালিকানার সার্ভার-সাইড ভ্যালিডেশন সম্পন্ন করে?
- [ ] হ্যাঁ — প্রতিটি রিকোয়েস্টের জন্য অথোরাইজেশন চেক প্রয়োগ করা হয় এবং বাইপাস করা **সম্ভব নয়**  
- [ ] হ্যাঁ — চেকগুলো প্রয়োগ করা হয় কিন্তু প্যারামিটার পলিউশন (Parameter pollution) বা মেথড টানেলিং (Method tunneling)-এর মাধ্যমে বাইপাস করা **সম্ভব**  
- [ ] না — অ্যাপ্লিকেশনটি নির্দিষ্ট রিসোর্সের মালিকানা যাচাই না করেই শুধুমাত্র ব্যবহারকারী অথেন্টিকেটেড হওয়ার ওপর নির্ভর করে  

### আইডেন্টিফায়ার পরিবর্তনের মাধ্যমে কি অন্য ব্যবহারকারীর রিসোর্স অ্যাক্সেস বা পরিবর্তন করা সম্ভব?
- [ ] না — অন্য ব্যবহারকারীর রিসোর্সে অননুমোদিত অ্যাক্সেস **সম্ভব নয়**  
- ** [ ] হ্যাঁ — অননুমোদিত **রিড (Read)** অ্যাক্সেস (Horizontal BOLA) **সম্ভব**  
- [ ] হ্যাঁ — অননুমোদিত **পরিবর্তন (Modification)** বা **ডিলিট (Deletion)** (Horizontal BOLA) **সম্ভব**  

### API কি আইডি (ID) প্রতিস্থাপনের মাধ্যমে অ্যাডমিনিস্ট্রেটিভ বা সিস্টেম-লেভেল অবজেক্ট অ্যাক্সেস করার অনুমতি দেয়?
- [ ] না — অ্যাডমিনিস্ট্রেটিভ রিসোর্সগুলো সেকেন্ডারি অথোরাইজেশন লেয়ার দ্বারা সুরক্ষিত  
- [ ] হ্যাঁ — সিস্টেম-লেভেল রিসোর্স অ্যাক্সেস বা পরিবর্তন করা (Vertical BOLA) **সম্ভব**  

### আইডেন্টিফায়ারটিকে রিকোয়েস্টের অন্য অংশে সরিয়ে নিয়ে কি অথোরাইজেশন চেক বাইপাস করা সম্ভব?
- [ ] না — প্যারামিটারের অবস্থান নির্বিশেষে অথোরাইজেশন লজিক সামঞ্জস্যপূর্ণ থাকে  
- [ ] হ্যাঁ — আইডি ইউআরএল পাথ (URL path) থেকে জেএসওএন বডি (JSON body) বা হেডারে সরিয়ে নিলে বাইপাস করা **সম্ভব**  
- [ ] হ্যাঁ — যখন একাধিক আইডি প্রদান করা হয় এবং সার্ভার অননুমোদিত আইডিটি প্রসেস করে তখন বাইপাস করা **সম্ভব**
