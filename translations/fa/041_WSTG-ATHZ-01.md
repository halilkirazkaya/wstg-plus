## WSTG-ATHZ-01 — تست پیمایش دایرکتوری و فراخوانی فایل (Testing Directory Traversal File Include)

آسیب‌پذیری‌های پیمایش دایرکتوری (Directory Traversal) و فراخوانی فایل (File Inclusion) زمانی رخ می‌دهند که یک اپلیکیشن از ورودی‌های قابل کنترل توسط کاربر برای ساختن مسیر فایل‌ها یا دایرکتوری‌ها بدون اعتبار‌سنجی (Validation) یا پاک‌سازی (Sanitization) کافی استفاده می‌کند. نفوذگران با تزریق رشته‌هایی مانند `../` برای خروج از دایرکتوری تعیین‌شده، از این نقص‌ها سوءاستفاده می‌کنند تا به‌صورت بالقوه به فایل‌های حساس سیستم، داده‌های پیکربندی یا سورس کد (Source Code) اپلیکیشن دسترسی پیدا کنند. در موارد شدیدتر که شامل فراخوانی محلی فایل (LFI) یا فراخوانی از راه دور فایل (RFI) می‌شود، مهاجم ممکن است با فراخوانی اسکریپت‌های مخرب یا بهره‌گیری از آلوده‌سازی لاگ (Log poisoning) و رپرهای پی‌اچ‌پی (PHP wrappers)، به اجرای کد از راه دور (Remote Code Execution) دست یابد. این آسیب‌پذیری‌ها معمولاً در پارامترهای مورد استفاده برای بارگذاری داینامیک محتوا، موتورهای قالب‌ساز (Template Engines) یا نقاط انتهایی (Endpoints) بازیابی تصویر که منطق سمت سرور در آن‌ها مسیرهای فایل‌سیستم را مدیریت می‌کند، وجود دارند.


| فیلد | مقدار |
|---|---|
| **شناسه WSTG** | WSTG-ATHZ-01 |
| **CWE** | CWE-22 |
| **وضعیت تست** | انجام نشده |
| **سطح اهمیت** | بالا / حیاتی (High / Critical) |

**ابزارها:** `Burp Suite`, `FFUF`, `DotDotPwn`, `LFI Suite`, `Wfuzz`

### آیا پارامترها نام فایل یا مسیرها را برای پردازش سمت سرور می‌پذیرند؟
- [ ] خیر — هیچ پارامتری برای تعامل با فایل‌سیستم مشاهده نشد
- [ ] بله — پارامترها وجود دارند اما از یک لیست مجاز (Allowlist) سخت‌گیرانه برای شناسه‌های فایل استفاده می‌کنند
- [ ] بله — پارامترها نام فایل یا مسیرهای مستقیم را می‌پذیرند

### آیا ورودی برای جلوگیری از رشته‌های پیمایش دایرکتوری پاک‌سازی شده است؟
- [ ] بله — ورودی در برابر یک لیست مجاز سخت‌گیرانه اعتبارسنجی شده و استفاده از رشته‌های پیمایش **امکان‌پذیر نیست**
- [ ] بله — ورودی با حذف رشته‌های `../` پاک‌سازی می‌شود، اما دور زدن بازگشتی (Recursive Bypass) **امکان‌پذیر است**
- [ ] خیر — هیچ پاک‌سازی یا اعتبارسنجی روی ورودی‌های مرتبط با مسیر **اعمال نشده است**

### آیا دسترسی به فایل‌های خارج از دایرکتوری محدود شده از طریق رشته‌های پیمایش امکان‌پذیر است؟
- [ ] خیر — اپلیکیشن یا سیستم‌عامل از دسترسی به فایل‌های خارج از محدوده تعریف شده جلوگیری می‌کند
- [ ] بله — دسترسی به فایل‌های داخل ریشه وب (Web root) **امکان‌پذیر است**
- [ ] بله — دسترسی به فایل‌های حساس سیستم (مانند `/etc/passwd` یا `C:\Windows\win.ini`) **امکان‌پذیر است** *(حیاتی)*

### آیا سرور اجازه فراخوانی یو‌آر‌ال‌های از راه دور (RFI) را می‌دهد؟
- [ ] خیر — فراخوانی فایل از راه دور در سطح سرور/اپلیکیشن **غیرفعال است**
- [ ] بله — فایل‌های از راه دور **می‌توانند** فراخوانی شوند اما اجرا **امکان‌پذیر نیست**
- [ ] بله — فایل‌های از راه دور **می‌توانند** فراخوانی و روی سرور اجرا شوند *(حیاتی)*

### آیا امکان دور زدن فیلترها با استفاده از کدگذاری یا کاراکترهای خاص وجود دارد؟
- [ ] خیر — فیلترها به‌طور موثری انواع کدگذاری‌ها و بایت‌های پوچ (Null bytes) را مدیریت می‌کنند
- [ ] بله — دور زدن فیلتر با استفاده از کدگذاری یو‌آر‌ال (URL encoding)، کدگذاری دوبل (Double URL encoding) یا یونیکد ۱۶ بیتی **امکان‌پذیر است**
- [ ] بله — دور زدن فیلتر با استفاده از تزریق بایت پوچ (`%00`) یا رپرهای فایل‌سیستم (مانند `php://filter`) **امکان‌پذیر است**
