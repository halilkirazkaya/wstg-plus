## WSTG-ATHN-06 — تست ضعف حافظه پنهان مرورگر (Browser Cache Weakness)

ضعف حافظه پنهان مرورگر (Browser Cache Weakness) زمانی رخ می‌دهد که اطلاعات حساس در حافظه پنهان محلی مرورگر ذخیره شده و توسط یک کاربر غیرمجاز که به همان سیستم فیزیکی دسترسی دارد، قابل بازیابی باشد. عدم وجود سرآیندهای (Headers) کنترل حافظه پنهان مناسب، اجازه می‌دهد داده‌های بالقوه حساس مانند اطلاعات شخصی، جزئیات حساب یا شناسه‌های نشست (Session Identifiers) پس از خروج کاربر یا بستن نشست، باقی بمانند. مهاجمان یا کاربران بعدی در ترمینال‌های اشتراکی یا عمومی می‌توانند با پیمایش در تاریخچه مرورگر، استفاده از دکمه "Back" یا بررسی فایل‌های حافظه پنهان محلی برای استخراج پاسخ‌های ذخیره شده، از این ضعف سوءاستفاده کنند. اطمینان از پیاده‌سازی دستورالعمل‌های سخت‌گیرانه مانند `Cache-Control: no-store` و `Pragma: no-cache` برای هر برنامه‌ای که محتوای احراز هویت شده را مدیریت می‌کند، جهت اطمینان از اینکه داده‌های حساس هرگز روی دیسک نوشته نمی‌شوند، حیاتی است.


| فیلد | مقدار |
|---|---|
| **شناسه WSTG** | WSTG-ATHN-06 |
| **CWE** | CWE-525 |
| **وضعیت تست** | انجام نشده |

> *اگر برنامه به طور مکرر از کیوسک‌های اشتراکی/عمومی مورد دسترسی قرار گیرد یا حاوی داده‌های شناسایی شخصی (PII) یا اطلاعات سلامت (PHI) بسیار حساس باشد، شدت به "متوسط" تغییر می‌کند.

### آیا سرآیندهای کنترل حافظه پنهان مناسب در صفحات احراز هویت شده یا حساس وجود دارند؟
- [ ] بله — `Cache-Control: no-store, no-cache` و `Pragma: no-cache` **موجود** و **به‌درستی پیکربندی شده‌اند**  
- [ ] بله — برخی سرآیندها موجود هستند اما `no-store` **مفقود** است، که اجازه ذخیره‌سازی روی دیسک را می‌دهد  
- [ ] خیر — هیچ سرآیند مرتبط با حافظه پنهان **اعمال نشده است** و به رفتار پیش‌فرض مرورگر اتکا شده است  

### آیا برنامه از دستورالعمل `private` برای داده‌های مختص کاربر استفاده می‌کند؟
- [ ] بله — `Cache-Control: private` برای تمام محتواهای شخصی‌سازی شده جهت جلوگیری از ذخیره‌سازی در پروکسی (Proxy Caching) **فعال است**  
- [ ] خیر — محتوا به عنوان `public` علامت‌گذاری شده یا فاقد دستورالعمل `private` است، که آن را توسط پروکسی‌های واسط **قابل ذخیره‌سازی** می‌کند  

### آیا اطلاعات حساس پس از خروج موفقیت‌آمیز، از طریق دکمه "Back" مرورگر قابل دسترسی است؟
- [ ] خیر — مرورگر درخواست احراز هویت مجدد می‌دهد یا صفحه از حافظه پنهان محلی **بارگذاری نمی‌شود**  
- [ ] بله — اطلاعات حساس پس از پایان نشست، همچنان از طریق دکمه بازگشت **قابل مشاهده است**  

### آیا فایل‌های غیر HTML حساس (مانند PDFها، خروجی‌های CSV، پاسخ‌های API از نوع JSON) در برابر ذخیره‌سازی در حافظه پنهان محافظت می‌شوند؟
- [ ] خیر — هیچ فایل غیر HTML حساسی توسط برنامه تولید یا مدیریت نمی‌شود  
- [ ] بله — سرآیندهای کنترل حافظه پنهان سخت‌گیرانه برای تمام دانلودهای فایل‌های حساس و نقاط انتهایی (Endpoints) API **اعمال شده‌اند**  
- [ ] خیر — دانلودهای حساس یا پاسخ‌های API در حافظه پنهان محلی مرورگر یا دایرکتوری‌های موقت **ذخیره می‌شوند**  

### آیا برنامه از `Expires: 0` یا تاریخی در گذشته برای جلوگیری از استفاده از داده‌های منقضی شده (Stale Data) استفاده می‌کند؟
- [ ] بله — سرآیند `Expires` روی `0` یا یک تاریخ قدیمی تنظیم شده است که **تضمین می‌کند** مرورگر محتوا را بلافاصله منقضی شده تلقی کند  
- [ ] خیر — سرآیند `Expires` **مفقود** است یا برای صفحات حساس روی تاریخی در آینده تنظیم شده است
