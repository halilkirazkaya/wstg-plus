## WSTG-INFO-10 — نقشه‌برداری از معماری اپلیکیشن (Map Application Architecture)

نقشه‌برداری از معماری اپلیکیشن (Application Architecture) شامل شناسایی تکنولوژی‌های زیربنایی، اجزای زیرساخت و مسیرهای جریان داده (Data Flow) است که از اپلیکیشن وب پشتیبانی می‌کنند. با تحلیل هدرهای پاسخ HTTP (HTTP response headers)، پیام‌های خطا، فرمت کوکی‌ها و پسوندهای فایل، تست‌کننده‌ها می‌توانند شماتیکی از وب سرورها (Web Servers)، اپلیکیشن سرورها (Application Servers)، پایگاه‌های داده (Databases) و ادغام‌های شخص ثالث (Third-party Integrations) در حال استفاده را بازسازی کنند. این دانش برای سفارشی‌سازی حملات بعدی ضروری است، زیرا امکان انتخاب اکسپلویت‌های (Exploits) خاص پلتفرم و شناسایی گلوگاه‌های احتمالی یا دستگاه‌های واسط با پیکربندی اشتباه مانند بارتوزیع‌کننده‌ها (Load Balancers) و فایروال‌های اپلیکیشن وب (WAF) را فراهم می‌کند. یک نفوذگر از این نقشه ساختاری برای حرکت از اسکن‌های عمومی به سمت بهره‌جویی هدفمند از پشته نرم‌افزاری (Software Stack) خاص و اجزای متصل به آن استفاده می‌کند.

### آیا تکنولوژی‌های وب سرور و اپلیکیشن سرور را می‌توان به طور دقیق شناسایی کرد؟
- [ ] خیر — به دلیل مقاوم‌سازی (Hardening) موثر یا مبهم‌سازی (Obfuscation)، شناسایی **امکان‌پذیر نیست**  
- [ ] بله — نوع تکنولوژی مشخص است اما نسخه‌های دقیق را **نمی‌توان** تعیین کرد  
- [ ] بله — تکنولوژی و نسخه‌های دقیق از طریق هدرها یا امضای فایل‌ها (File Signatures) به طور کامل فاش شده‌اند *(Informational)*  

### آیا دستگاه‌های واسط (WAF، Load Balancers، Proxies) در مسیر ارتباطی قابل شناسایی هستند؟
- [ ] خیر — هیچ واسطی شناسایی نشد یا آن‌ها کاملاً شفاف (Transparent) هستند  
- [ ] بله — وجود آن‌ها از طریق زمان‌بندی یا رفتار مشکوک است، اما هویت آن‌ها تأیید **نشده است**  
- [ ] بله — محصولات خاص (مانند Cloudflare، Nginx، F5) از طریق هدرهای منحصربه‌فرد یا رفتار شناسایی شده‌اند  

### آیا اپلیکیشن اطلاعاتی درباره ساختار دایرکتوری داخلی یا توپولوژی شبکه (Network Topology) نشت می‌دهد؟
- [ ] خیر — مسیرهای داخلی و IPها فاش **نشده‌اند**  
- [ ] بله — مسیرهای داخلی در پیام‌های خطا یا Stack Traces نشت کرده‌اند، اما IPهای داخلی فاش **نشده‌اند**  
- [ ] بله — هم ساختارهای دایرکتوری داخلی و هم آدرس‌های IP خصوصی فاش شده‌اند  

### آیا نوع و نسخه پایگاه داده (Database) پشتیبان از رفتار اپلیکیشن قابل استنتاج است؟
- [ ] خیر — جزئیات پایگاه داده **قابل تعیین نیست**  
- [ ] بله — نوع پایگاه داده (مانند PostgreSQL، MSSQL) از طریق پیام‌های خطا یا رفتار سیستم استنتاج شده است  
- [ ] بله — نوع و نسخه پایگاه داده به طور صریح در بدنه پاسخ‌ها یا هدرها فاش شده است  

### آیا اپلیکیشن روی ارائه‌دهندگان سرویس ابری (Cloud Service Providers) یا CDNهای شخص ثالث خاصی میزبانی می‌شود؟
- [ ] خیر — زیرساخت میزبانی **قابل شناسایی نیست**  
- [ ] بله — ارائه‌دهنده ابری (AWS، Azure، GCP) شناسایی شده است اما سرویس‌های خاص **مشخص نیستند**  
- [ ] بله — سرویس‌های ابری خاص (مانند S3 buckets، Lambda، Azure Functions) نقشه‌برداری شده و قابل دسترس هستند
