## WSTG-CRYP-01 — تست امنیت ضعیف لایه انتقال

تست امنیت ضعیف لایه انتقال (Transport Layer Security) شامل تحلیل پیاده‌سازی و پیکربندی SSL/TLS برای اطمینان از محرمانگی (Confidentiality) و یکپارچگی (Integrity) داده‌ها در حین انتقال است. مهاجمان پیکربندی‌های نادرست مانند پروتکل‌های منسوخ (SSLv2، TLS 1.0)، مجموعه‌های رمزنگاری (Cipher suites) ضعیف (مانند NULL، RC4 یا ناشناس) و گواهی‌های نامعتبر یا دارای امضای ضعیف را برای اجرای حملات مرد میانی (Man-in-the-Middle - MitM) هدف قرار می‌دهند. این تست معمولاً نقاط پایانی (Endpoints) وب‌سرور یا بارتوزکن (Load Balancer) برنامه را که در آن ارتباطات رمزنگاری‌شده برقرار می‌شود، هدف قرار می‌دهد. بهره‌برداری (Exploit) موفقیت‌آمیز به مهاجم اجازه می‌دهد تا داده‌های حساس را شنود کند، نشست‌های کاربر (User sessions) را برباید یا از طریق تنزل رتبه پروتکل (Protocol downgrade) یا رمزگشایی ترافیک، اتصالات را به حالت ناامن درآورد.


| فیلد | مقدار |
|---|---|
| **شناسه WSTG** | WSTG-CRYP-01 |
| **CWE** | CWE-326 |
| **وضعیت تست** | انجام نشده |
| **شدت** | بالا / متوسط* |

### آیا پروتکل‌های TLS/SSL منسوخ یا ناامن پشتیبانی می‌شوند؟
- [ ] خیر — فقط TLS 1.2 و TLS 1.3 **فعال** هستند  
- [ ] بله — TLS 1.0 یا TLS 1.1 **فعال** هستند  
- [ ] بله — SSLv2 یا SSLv3 **فعال** هستند *(بحرانی)*  

### آیا مجموعه‌های رمزنگاری ضعیف یا ناامن توسط سرور پذیرفته می‌شوند؟
- [ ] خیر — فقط رمزنگاری‌های مدرن و قوی (مانند AES-GCM، ChaCha20) **پشتیبانی می‌شوند**  
- [ ] بله — رمزنگاری‌های با رمزگذاری ضعیف (مانند 3DES، RC4) یا طول کلید کم **پشتیبانی می‌شوند**  
- [ ] بله — رمزنگاری‌های NULL، ناشناس (ADH) یا Export **پشتیبانی می‌شوند** *(بحرانی)*  

### آیا گواهی سرور معتبر و قابل اعتماد (Trusted) است؟
- [ ] بله — گواهی معتبر است، با دامنه مطابقت دارد و توسط یک **مرجع صدور گواهی (CA)** معتبر امضا شده است  
- [ ] خیر — گواهی **منقضی شده** است یا از یک **الگوریتم امضای ضعیف** (مانند SHA-1) استفاده می‌کند  
- [ ] خیر — گواهی **خود-امضا (Self-signed)** است یا **عدم تطابق** نام دامنه رخ داده است  

### آیا سرور از بازهماهنگی امن (Secure Renegotiation) پشتیبانی کرده و از حملات تنزل رتبه جلوگیری می‌کند؟
- [ ] بله — بازهماهنگی امن **پشتیبانی می‌شود** و TLS Fallback SCSV **فعال** است  
- [ ] خیر — بازهماهنگی امن **پشتیبانی نمی‌شود**، که پتانسیل تزریق MitM را فراهم می‌کند  
- [ ] خیر — سرور در برابر حملات **POODLE** یا **ROBOT** آسیب‌پذیر است  

### آیا امنیت انتقال سخت‌گیرانه HTTP (HSTS) به درستی پیاده‌سازی شده است؟

| هدر | موجود | ناموجود |
|--------|:-------:|:-------:|
| `Strict-Transport-Security` |  |  |

- [ ] بله — هدر HSTS با یک `max-age` طولانی موجود است و `includeSubDomains` **فعال** می‌باشد  
- [ ] بله — هدر HSTS موجود است اما `includeSubDomains` در آن نیست یا دارای **max-age کوتاه** است  
- [ ] خیر — هدر HSTS **موجود نیست**
