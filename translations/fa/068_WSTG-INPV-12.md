## WSTG-INPV-12 — Command Injection

تزریق دستور (Command Injection) زمانی رخ می‌دهد که یک اپلیکیشن داده‌های ناامن ارائه شده توسط کاربر، مانند ورودی‌های فرم، هدرهای HTTP (HTTP headers) یا کوکی‌ها (cookies) را به یک شل سیستم (system shell) منتقل می‌کند. این آسیب‌پذیری به مهاجم اجازه می‌دهد تا دستورات دلخواه سیستم‌عامل (arbitrary OS commands) را بر روی سرور، معمولاً با امتیازات فرآیند اپلیکیشن تحت وب، اجرا کند. از دیدگاه مهاجم، این مورد با استفاده از فراکاراکترهای شل (shell metacharacters) مانند نقطه‌ویرگول (;)، پایپ (|) یا بک‌تیک (`) برای زنجیره‌سازی دستورات مخرب به جریان اجرای (execution flow) مورد نظر، مورد سوءاستفاده قرار می‌گیرد. پیامد آن اغلب فاجعه‌بار است و منجر به نفوذ کامل به سیستم (system compromise)، استخراج غیرمجاز داده‌ها (data exfiltration) یا حرکت جانبی (lateral movement) در شبکه داخلی می‌شود.


| فیلد | مقدار |

| **WSTG ID** | WSTG-INPV-12 |
| **CWE** | CWE-78 |
| **وضعیت تست** | انجام نشده |

### آیا اپلیکیشن دستورات سطح سیستم یا فایل‌های اجرایی شل را فراخوانی می‌کند؟
- [ ] خیر — اپلیکیشن با شل سیستم تعامل **ندارد**  
- [ ] بله — اپلیکیشن از APIهای داخلی یا کتابخانه‌های سطح بالا برای وظایف سیستم استفاده می‌کند  
- [ ] بله — اپلیکیشن مستقیماً دستورات شل را از طریق فراخوانی‌های سیستم مانند `system()`، `exec()` یا `popen()` اجرا می‌کند  

### آیا ورودی کاربر قبل از ارسال به فراخوانی‌های سیستم، اعتبارسنجی یا پاکسازی می‌شود؟
- [ ] بله — لیست مجاز (allow-listing) دقیق و اعتبارسنجی ورودی **اعمال شده است**  
- [ ] بله — پاکسازی (sanitization) یا گریزدهی (escaping) استفاده شده است اما کنار گذاشتن (bypass) آن **امکان‌پذیر است**  
- [ ] خیر — ورودی کاربر **بدون** اعتبارسنجی مستقیماً به شل ارسال می‌شود  

### آیا می‌توان از فراکاراکترهای شل برای دستکاری جریان اجرای دستور استفاده کرد؟
- [ ] خیر — فراکاراکترها به درستی فیلتر یا گریزدهی شده‌اند  
- [ ] بله — اجرای درون‌باند (in-band) که در آن خروجی دستور در پاسخ (response) منعکس می‌شود، **امکان‌پذیر است**  
- [ ] بله — اجرای کور (blind execution) که در آن هیچ خروجی منعکس نمی‌شود، **امکان‌پذیر است**  

### آیا استخراج داده‌ها به صورت خارج از باند (OOB) از میزبان امکان‌پذیر است؟
- [ ] خیر — سرور هیچ خروجی (egress) ندارد یا سیگنال‌های OOB (مانند DNS/HTTP) شکست می‌خورند  
- [ ] بله — استخراج داده‌ها از طریق سیگنال‌های خارج از باند (out-of-band) مبتنی بر DNS یا HTTP **امکان‌پذیر است**  

### آیا فرآیند اپلیکیشن با امتیازات سیستمی بالا اجرا می‌شود؟
- [ ] خیر — اپلیکیشن با یک حساب کاربری سرویس اختصاصی و کم‌امتیاز اجرا می‌شود  
- [ ] بله — اپلیکیشن به عنوان `root`، `Administrator` یا یک کاربر با امتیاز بالا اجرا می‌شود *(حیاتی)*
