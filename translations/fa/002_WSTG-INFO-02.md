## WSTG-INFO-02 — شناسایی اثر انگشت وب‌سرور (Fingerprint Web Server)

شناسایی اثر انگشت وب‌سرور (Web server fingerprinting) فرآیند تعیین نوع نرم‌افزار، نسخه و سیستم‌عامل زیربنایی یک وب‌سرور هدف است. مهاجمان این شناسایی (Reconnaissance) را برای یافتن آسیب‌پذیری‌های شناخته‌شده، مانند CVEهای وصله‌نشده یا ضعف‌های پیکربندی، که مختص نسخه‌های خاصی از Apache، Nginx، IIS یا سایر فناوری‌های سرور هستند، انجام می‌دهند. این امر معمولاً از طریق تحلیل هدرهای پاسخ HTTP (مانند `Server` و `X-Powered-By`)، بررسی رفتارهای منحصربه‌فرد پروتکل یا مشاهده اطلاعات نشت‌یافته از طریق صفحات خطا و فایل‌های پیش‌فرض انجام می‌شود. شناسایی موفق اثر انگشت به مهاجم اجازه می‌دهد استراتژی اکسپلویت (Exploitation) خود را شخصی‌سازی کرده و از اسکن‌های عمومی به سمت حملاتی با احتمال موفقیت بالا علیه نقص‌های معماری شناخته‌شده حرکت کند.


| فیلد | مقدار |
|---|---|
| **WSTG ID** | WSTG-INFO-02 |
| **CWE** | CWE-200 |
| **وضعیت تست** | انجام نشده |
| **سطح اهمیت** | اطلاعاتی / کم* |

> *اگر شناسایی اثر انگشت، نسخه‌ای از سرور را نشان دهد که قدیمی یا پایان عمر (EOL) بوده و دارای آسیب‌پذیری‌های بحرانی شناخته‌شده باشد، سطح اهمیت به "بالا" (High) تغییر می‌یابد.

**منابع:**
* https://owasp.org/www-project-web-security-testing-guide/latest/4-Web_Application_Security_Testing/01-Information_Gathering/02-Fingerprint_Web_Server  
* https://hacktricks.wiki/en/pentesting-web/web-vulnerabilities-methodology.html  
* https://portswigger.net/web-security/information-disclosure  

**ابزارها:** `curl`, `Nmap`, `WhatWeb`, `Wappalyzer`, `Nikto`, `Netcat`

### آیا رشته‌های شناسایی در هدرهای پاسخ HTTP وجود دارند؟
- [ ] خیر — هدرهای `Server` و `X-Powered-By` وجود ندارند یا حاوی مقادیر عمومی (Generic) هستند  
- [ ] بله — هدرها نرم‌افزار سرور را نشان می‌دهند اما شماره نسخه دقیق را خیر  
- [ ] بله — هدرها نرم‌افزار سرور و شماره نسخه‌های دقیق را فاش می‌کنند  

### آیا صفحات خطای پیش‌فرض یا پاسخ‌های تولیدشده توسط سیستم، جزئیات سرور را نشت می‌دهند؟
- [ ] خیر — از صفحات خطای سفارشی (Custom error pages) استفاده شده است و اطلاعات سرور را فاش نمی‌کنند  
- [ ] بله — صفحات خطای پیش‌فرض، نام و/یا نسخه نرم‌افزار سرور را نشت می‌دهند  
- [ ] بله — صفحات خطا جزئیات سرور، نسخه و سیستم‌عامل زیربنایی را فاش می‌کنند  

### آیا نسخه سرور از طریق فایل‌های پیش‌فرض یا ساختارهای دایرکتوری فاش می‌شود؟
- [ ] خیر — فایل‌های نصب پیش‌فرض، دفترچه‌های راهنما و اسکریپت‌های تست حذف شده‌اند  
- [ ] بله — فایل‌های پیش‌فرض (مانند `info.php` ، `manual/` ، `test.html`) موجود هستند و داده‌های نسخه را فاش می‌کنند  

### آیا نوع سرور را می‌توان از طریق تحلیل رفتار منحصربه‌فرد به‌طور دقیق استنباط کرد؟
- [ ] خیر — پاسخ‌های سرور عادی‌سازی (Normalized) شده‌اند و شناسایی اثر انگشت مبتنی بر رفتار امکان‌پذیر نیست  
- [ ] بله — رفتارهای منحصربه‌فرد (مانند ترتیب هدرها، کدهای خطای خاص یا ویژگی‌های پشته TCP/IP) شناسایی سرور را امکان‌پذیر می‌سازند  

### آیا نسخه سرور شناسایی‌شده در حال حاضر پشتیبانی می‌شود و وصله شده است؟
- [ ] بله — نسخه شناسایی‌شده به‌روز است و توسط توسعه‌دهنده پشتیبانی می‌شود  
- [ ] خیر — نسخه شناسایی‌شده قدیمی یا پایان عمر (EOL) است و دارای آسیب‌پذیری‌های شناخته‌شده می‌باشد
