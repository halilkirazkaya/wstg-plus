## WSTG-INPV-07 — تزریق اکس‌ام‌ال (XML Injection)

تزریق اکس‌ام‌ال (XML Injection) زمانی رخ می‌دهد که یک اپلیکیشن داده‌های ارائه‌شده توسط کاربر را بدون اعتبار‌سنجی (Validation)، پاک‌سازی (Sanitization) یا فراردهی (Escaping) مناسبِ نویسه‌های فراداده XML، در یک سند یا جریان XML ادغام کند. با تزریق تگ‌های خاص XML یا عناصر ساختاری، یک مهاجم می‌تواند منطق سند را دست‌کاری کرده، مکانیزم‌های احراز هویت را دور بزند یا در یکپارچگی داده‌های پردازش‌شده توسط بک‌اند (Backend) اختلال ایجاد کند. این آسیب‌پذیری معمولاً در سرویس‌های وب مبتنی بر SOAP، رابط‌های برنامه‌نویسی کاربردی (REST API) که XML می‌پذیرند، اظهارنامه‌های SAML و اپلیکیشن‌هایی که فایل‌های پیکربندی یا گزارش‌های XML را به صورت پویا تولید می‌کنند، یافت می‌شود. از دیدگاه یک مهاجم، هدف این است که از بستر داده‌ای مورد نظر خارج شده تا ساختار سند را تغییر دهد، که به‌طور بالقوه منجر به ارتقای سطح دسترسی (Privilege Escalation) غیرمجاز یا اجرای منطق کسب‌وکار (Business Logic) ناخواسته می‌شود.

### آیا اپلیکیشن ورودی با قالب XML را از کاربر دریافت و پردازش می‌کند؟
- [ ] خیر — اپلیکیشن ورودی XML را پردازش **نمی‌کند**  
- [ ] بله — ورودی XML از طریق APIها (SOAP/REST) پردازش می‌شود  
- [ ] بله — XML از طریق آپلود فایل (SVG، DOCX و غیره) پردازش می‌شود  

### آیا نویسه‌های فراداده XML (مانند `<`, `>`, `&`, `'`, `"`) به درستی فراردهی یا پاک‌سازی می‌شوند؟
- [ ] بله — تمامی نویسه‌های فراداده به طور **مناسب** فراردهی یا پاک‌سازی می‌شوند *(بیشترین امنیت)*  
- [ ] بله — برخی فیلترها اعمال شده‌اند اما دور زدن آن‌ها با استفاده از کدگذاری (Encoding) **امکان‌پذیر است**  
- [ ] خیر — ورودی خام به صورت مستقیم و **بدون** پاک‌سازی در ساختارهای XML درج می‌شود  

### آیا اعتبار‌سنجی طرح‌واره (Schema Validation) در سمت سرور (XSD/DTD) برای تمامی ورودی‌های XML اجباری است؟
- [ ] بله — اعتبار‌سنجی سخت‌گیرانه طرح‌واره **فعال است** و از تغییرات ساختاری جلوگیری می‌کند  
- [ ] بله — اعتبار‌سنجی **فعال است** اما از تزریق تگ (Tag Injection) در فیلدهای داده جلوگیری **نمی‌کند**  
- [ ] خیر — اعتبار‌سنجی طرح‌واره **غیرفعال است** یا پیاده‌سازی نشده است  

### آیا یک مهاجم می‌تواند تگ‌های جدید XML را جهت تغییر ساختار یا منطق سند تزریق کند؟
- [ ] خیر — ساختار سند صرف‌نظر از ورودی، دست‌نخورده باقی می‌ماند  
- [ ] بله — تگ‌ها قابل تزریق هستند اما **نمی‌توانند** منطق کسب‌وکار را تغییر دهند  
- [ ] بله — تزریق تگ **امکان‌پذیر است** و اجازه دور زدن منطق یا تغییر داده‌ها را می‌دهد *(بحرانی)*  

### آیا امکان دست‌کاری پارسر XML با استفاده از بخش‌های CDATA یا کامنت‌های XML وجود دارد؟
- [ ] خیر — پارسر به درستی نشانگرهای CDATA/comment را مدیریت یا رد می‌کند  
- [ ] بله — تزریق `<![CDATA[...]]>` یا `<!-- ... -->` برای پنهان‌سازی یا دور زدن فیلترها **امکان‌پذیر است**
