## WSTG-INPV-09 — تست برای تزریق XPath (XPath Injection)

تزریق ایکس‌پت (XPath Injection) زمانی رخ می‌دهد که یک برنامه از اطلاعات ارائه‌شده توسط کاربر برای ساختن یک پرس‌وجوی XPath جهت داده‌های XML استفاده می‌کند، که به مهاجم اجازه می‌دهد در منطق پرس‌وجو اختلال ایجاد کند. با تزریق ساختارهای خاصی مانند کوتیشن‌های تکی (Single quotes)، کروشه‌ها و عملگرهای منطقی، یک مهاجم می‌تواند در ساختار سند XML پیمایش کرده، احراز هویت (Authentication) را دور بزند یا گره‌های داده‌ای حساس را استخراج (Exfiltrate) کند. این آسیب‌پذیری معمولاً در وب‌سرویس‌ها (Web Services)، رابط‌های مدیریت پیکربندی و سیستم‌های قدیمی که به جای پایگاه‌های داده سنتی SQL به مخازن داده مبتنی بر XML متکی هستند، یافت می‌شود. از دیدگاه یک مهاجم، این مورد اغلب با استفاده از تکنیک‌های مبتنی بر منطق بولی (Boolean-based) یا مبتنی بر خطا (Error-based) برای نقشه‌برداری سیستماتیک درخت XML و بازیابی مقادیر پنهان از بک‌اند (Backend) مورد سوءاستفاده قرار می‌گیرد.

### آیا ورودی‌های کاربر که برای ساخت پرس‌وجوهای XPath استفاده می‌شوند، به درستی پاکسازی یا پارامتریک شده‌اند؟
- [ ] بله — ورودی‌ها در پرس‌وجوهای XPath استفاده نمی‌شوند یا به طور دقیق پارامتریک (Parameterized) شده‌اند  
- [ ] بله — اعتبارسنجی/کدگذاری (Encoding) ورودی قدرتمندی اعمال شده و دور زدن (Bypass) آن امکان‌پذیر نیست  
- [ ] بله — برخی اعتبارسنجی‌ها اعمال شده است اما دور زدن با استفاده از کاراکترهای کدگذاری‌شده امکان‌پذیر است  
- [ ] خیر — ورودی کاربر مستقیماً به پرس‌وجوهای XPath متصل (Concatenated) شده است *(حیاتی)*  

### آیا برنامه جزئیات ساختاری XML/XPath را از طریق پیام‌های خطا فاش می‌کند؟
- [ ] خیر — صفحات خطای عمومی نمایش داده می‌شوند و هیچ جزئیات XML نشت (Leak) نمی‌کند  
- [ ] بله — پیام‌های خطا وجود پردازش XML را نشان می‌دهند اما هیچ جزئیاتی از مسیر (Path) فاش نمی‌شود  
- [ ] بله — پیام‌های خطای پرجزئیات (Verbose)، سینتکس XPath، نام گره‌ها یا مسیرهای فایل را فاش می‌کنند  

### آیا دور زدن منطق احراز هویت یا سطح دسترسی با استفاده از منطق XPath امکان‌پذیر است؟
- [ ] خیر — منطق احراز هویت به XML/XPath وابسته نیست یا به صورت امن پیاده‌سازی شده است  
- [ ] بله — بررسی‌های ورود یا مجوزها را می‌توان با استفاده از پی‌لودهای (Payloads) مبتنی بر منطق مانند `' or 1=1 or '` دور زد  

### آیا ساختار سند XML یا داده‌های آن از طریق تزریق کور قابل استخراج است؟
- [ ] خیر — برنامه در برابر استنتاج (Inference) مبتنی بر منطق بولی یا مبتنی بر زمان آسیب‌پذیر نیست  
- [ ] بله — نام گره‌ها و داده‌ها را می‌توان با استفاده از استنتاج مبتنی بر منطق بولی (Boolean-based inference) استخراج کرد  
- [ ] بله — پیمایش کامل درخت XML و استخراج داده‌ها امکان‌پذیر است
