## WSTG-INFO-05 — بررسی محتوای صفحات وب برای نشت اطلاعات (Information Leakage)

بررسی محتوای صفحات وب شامل بازرسی دستی و خودکار پاسخ‌های برنامه، از جمله فایل‌های HTML، JavaScript و CSS، به منظور شناسایی اطلاعات حساس است که به طور ناخواسته در معرض دید کاربران قرار گرفته‌اند. مهاجمان کدهای منبع سمت کاربر (Client-side source code) را برای یافتن کامنت‌های توسعه‌دهندگان (Developer comments)، فیلدهای پنهان فرم (Hidden form fields)، مسیرهای داخلی سرور، کلیدهای API هاردکد شده (Hardcoded API keys) و اطلاعات عیب‌یابی (Debugging information) که می‌تواند در مراحل بعدی بهره‌جویی (Exploitation) کمک کند، به دقت بررسی می‌کنند. این نشت اطلاعات (Information Leakage) اغلب زمانی رخ می‌دهد که توسعه‌دهندگان پیش از انتقال به محیط عملیاتی (Production)، حذف متاداده‌ها (Metadata) یا کدهای عیب‌یابی را فراموش می‌کنند که این امر به طور بالقوه نقص‌های منطقی یا جزئیات زیرساخت بک‌اند (Backend infrastructure) را فاش می‌کند. از دیدگاه یک مهاجم، این یک روش کم‌هزینه برای نقشه‌برداری از ساختار داخلی برنامه و کشف نقاط ورودی نادیده گرفته شده بدون ایجاد هشدارهای امنیتی یا تعامل با منطق بک‌اند سرور است.


| فیلد | مقدار |
|---|---|
| **WSTG ID** | WSTG-INFO-05 |
| **CWE** | CWE-200 |
| **وضعیت تست** | انجام نشده |
| **شدت** | Low / Medium* |

> *در صورت کشف اعتبارنامه‌های حساس هاردکد شده، کلیدهای خصوصی یا متغیرهای محیطی داخلی، شدت به High تغییر می‌یابد.

### آیا کامنت‌های توسعه‌دهندگان حاوی اطلاعات حساس در کد منبع وجود دارد؟
- [ ] خیر — هیچ کامنتی یافت نشد یا فقط کامنت‌های عمومی و غیرحساس مشاهده شد  
- [ ] بله — کامنت‌ها وجود دارند اما حاوی هیچ منطق یا داده حساسی **نیستند**  
- [ ] بله — کامنت‌ها مسیرهای داخلی، کوئری‌های SQL یا دستورالعمل‌های مدیریتی را **فاش می‌کنند**  
- [ ] بله — کامنت‌ها اعتبارنامه‌ها یا یادداشت‌های حساس توسعه‌دهنده را **فاش می‌کنند** *(High)*  

### آیا فیلدهای ورودی پنهان HTML حاوی داده‌های حساس یا اطلاعات وضعیت (State) هستند؟
- [ ] خیر — فیلد پنهانی یافت نشد یا فقط حاوی توکن‌های غیرحساس هستند  
- [ ] بله — فیلدهای پنهان برای وضعیت استفاده می‌شوند اما **رمزنگاری شده** یا **امضا شده** (Signed) هستند  
- [ ] بله — فیلدهای پنهان حاوی رمزهای عبور **متن ساده** (Plaintext)، نقش‌های کاربری یا شناسه‌های داخلی هستند  

### آیا اسرار هاردکد شده، کلیدهای API یا آدرس‌های IP داخلی در فایل‌های JavaScript وجود دارند؟
- [ ] خیر — هیچ رشته حساس یا رازی در فایل‌های JS یافت نشد  
- [ ] بله — فایل‌های JS حاوی کلیدهای عمومی API با محدودیت‌های **مناسب** هستند  
- [ ] بله — فایل‌های JS حاوی کلیدهای API **بدون محافظت**، IPهای داخلی یا نقاط پایانی (Endpoints) خصوصی هستند  
- [ ] بله — فایل‌های JS حاوی اعتبارنامه‌های مدیریتی **هاردکد شده** یا کلیدهای خصوصی هستند *(Critical)*  

### آیا برنامه نسخه‌های فریم‌ورک یا مسیرهای داخلی فایل را در سورس‌کد فاش می‌کند؟
- [ ] خیر — اطلاعات فریم‌ورک و مسیرهای فایل **Minified** یا **Obfuscated** شده‌اند  
- [ ] بله — نام و نسخه فریم‌ورک‌ها **قابل مشاهده** هستند اما وصله شده‌اند  
- [ ] بله — مسیرهای داخلی فایل یا مسیرهای مطلق سرور در کد منبع **افشا شده‌اند**  

### آیا کد منبع به دلیل عدم فشرده‌سازی (Minification) یا درهم‌سازی مستعد نشت اطلاعات است؟
- [ ] خیر — کد محیط عملیاتی **به‌طور کامل Minified** شده و کامنت‌ها حذف شده‌اند  
- [ ] بله — کد فشرده شده است اما کامنت‌ها در کد منبع **باقی مانده‌اند**  
- [ ] بله — نقشه‌های منبع (`.map` files) به‌صورت **عمومی در دسترس** هستند که اجازه بازسازی کامل کد منبع را می‌دهند
