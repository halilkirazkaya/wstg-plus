## WSTG-INPV-20 — Mass Assignment

انتساب انبوه (Mass Assignment) که با نام‌های ارسال بیش از حد (Overposting) یا دی‌سریال‌سازی ناامن پارامترها (Insecure Deserialization of parameters) نیز شناخته می‌شود، زمانی رخ می‌دهد که یک اپلیکیشن وب به‌طور خودکار پارامترهای ورودی در درخواست HTTP را بدون فیلتر کردن مناسب ویژگی‌ها، به ویژگی‌های اشیاء داخلی متصل می‌کند. این آسیب‌پذیری در فریم‌ورک‌های مدرن MVC و رابط‌های برنامه‌نویسی RESTful API که داده‌های JSON، XML یا فرم‌های کدگذاری شده مستقیماً به مدل‌های داده در سمت اپلیکیشن تبدیل می‌شوند، رایج است. مهاجمان با تزریق پارامترهای اضافی و فهرست‌نشده در یک درخواست — مانند `isAdmin`، `role` یا `account_balance` — از این رفتار برای اصلاح فیلدهای حساسی که توسعه‌دهندگان قصد قرار دادن آن‌ها در کنترل کاربر را نداشتند، بهره‌برداری می‌کنند. بهره‌برداری موفق معمولاً منجر به ارتقای سطح دسترسی (Privilege Escalation)، اصلاح غیرمجاز داده‌ها یا دور زدن جریان‌های منطق کسب‌وکار (Business Logic) حساس می‌گردد.


| فیلد | مقدار |

| **شناسه WSTG** | WSTG-INPV-20 |
| **CWE** | CWE-915 |
| **وضعیت تست** | انجام نشده |

### آیا اپلیکیشن از اتصال خودکار (Automatic Binding) برای پارامترهای ورودی درخواست استفاده می‌کند؟
- [ ] خیر — پارامترها به‌صورت دستی به متغیرهای خاص یا اشیاء امن نگاشت می‌شوند  
- [ ] بله — اتصال خودکار استفاده می‌شود اما لیست‌های مجاز (White-lists) سخت‌گیرانه یا اشیاء انتقال داده (DTO) اعمال می‌گردند  
- [ ] بله — اتصال خودکار استفاده می‌شود و ویژگی‌های داخلی حساس **قابل** تغییر هستند  

### آیا امکان تزریق موفقیت‌آمیز ویژگی‌های مدیریتی یا مرتبط با سطح دسترسی وجود دارد؟
- [ ] خیر — پارامترهای تزریق شده نادیده گرفته می‌شوند، حذف می‌شوند یا باعث خطای اعتبارسنجی می‌گردند  
- [ ] بله — پارامترهای اضافی پذیرفته می‌شوند اما بر وضعیت شیء در سمت سرور (Backend) تأثیری **ندارند**  
- [ ] بله — تزریق پارامترهایی مانند `is_admin`، `role` یا `permissions` **با موفقیت** سطح دسترسی را ارتقا می‌دهد *(حیاتی)*  

### آیا اصلاح منطق حساس کسب‌وکار یا فیلدهای مالی از طریق Overposting امکان‌پذیر است؟
- [ ] خیر — فیلدهایی مانند `price`، `balance` یا `status` در برابر انتساب انبوه محافظت می‌شوند  
- [ ] بله — ویژگی‌های حساس کسب‌وکار با افزودن آن‌ها به بدنه درخواست JSON یا فرم، **قابل** اصلاح هستند  

### آیا اپلیکیشن ساختارهای شیء داخلی را که حدس زدن پارامترها را تسهیل می‌کند، فاش می‌سازد؟
- [ ] خیر — پاسخ‌ها فقط شامل فیلدهای عمومی در نظر گرفته شده هستند و دسترسی به مستندات محدود شده است  
- [ ] بله — فیلدهای اشیاء داخلی در پاسخ‌های GET قابل مشاهده هستند، که کشف پارامترها را **امکان‌پذیر** می‌سازد  
- [ ] بله — مستندات API (مانند Swagger/OpenAPI) به‌طور صریح ویژگی‌های داخلی را که **قابل** انتساب هستند، فهرست کرده است
