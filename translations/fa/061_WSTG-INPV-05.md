## WSTG-INPV-05 — تزریق اس‌کیوال (SQL Injection)

تزریق اس‌کیوال (SQL Injection) زمانی رخ می‌دهد که ورودی‌های ارائه شده توسط کاربر بدون پارامتری‌سازی (Parameterization) یا پاک‌سازی (Sanitization) مناسب در کوئری‌های SQL گنجانده شوند و به مهاجم اجازه دهند منطق کوئری را تغییر دهد. بهره‌برداری (Exploitation) موفقیت‌آمیز می‌تواند منجر به استخراج غیرمجاز داده‌ها (Data Exfiltration)، دور زدن احراز هویت (Authentication Bypass)، تغییر داده‌ها و در برخی موارد اجرای کد از راه دور (Remote Code Execution - RCE) از طریق ویژگی‌های پایگاه داده مانند `xp_cmdshell` یا `LOAD_FILE()` شود. این آسیب‌پذیری معمولاً فرم‌های ورود، قابلیت جستجو، پارامترهای API، هدرهای HTTP و هر نقطه‌ی نهایی (Endpoint) که کوئری‌های SQL پویا را از ورودی‌های تحت کنترل کاربر می‌سازد، تحت تأثیر قرار می‌دهد. از دیدگاه مهاجم، شناسایی یک نقطه تزریق، دروازه اصلی برای نفوذ کامل به پایگاه داده و احتمال حرکت جانبی (Lateral Movement) در زیرساخت است.

### آیا پارامترهای تحت کنترل کاربر با استفاده از روش‌های امن دسترسی به پایگاه داده پردازش می‌شوند؟
- [ ] بله — اپلیکیشن منحصراً از ORM یا کوئری‌های پارامتری استفاده می‌کند و دور زدن آن‌ها **امکان‌پذیر نیست**  
- [ ] بله — از کوئری‌های پارامتری استفاده شده است اما دور زدن آن‌ها در موارد خاص (مانند عبارات `ORDER BY`) **امکان‌پذیر است**  
- [ ] خیر — برای ساخت کوئری‌های SQL از اتصال مستقیم رشته‌ها (Raw String Concatenation) **بدون** پارامتری‌سازی استفاده شده است  

### آیا مکانیسم احراز هویت از طریق تزریق اس‌کیوال قابل دور زدن است؟
- [ ] خیر — پارامترهای ورود نسبت به تزریق **آسیب‌پذیر نیستند**  
- [ ] بله — دور زدن احراز هویت با استفاده از پیلودهای مبتنی بر توتولوژی (Tautology-based Payloads) (مانند `' OR 1=1 --`) **امکان‌پذیر است**  

### آیا استخراج داده‌ها از طریق تکنیک‌های درون‌باندی یا برون‌باندی امکان‌پذیر است؟
- [ ] خیر — هیچ موردی از استخراج داده شناسایی نشد  
- [ ] بله — استخراج داده مبتنی بر UNION یا مبتنی بر خطا (Error-based) **امکان‌پذیر است**  
- [ ] بله — استخراج داده کور (Blind) (منطقی یا مبتنی بر زمان) **امکان‌پذیر است**  
- [ ] بله — استخراج داده برون‌باندی (Out-of-band - OOB) از طریق DNS/HTTP **امکان‌پذیر است**  

### آیا عملکردهای اپلیکیشن دارای آسیب‌پذیری‌های تزریق اس‌کیوال مرتبه دوم هستند؟
- [ ] خیر — داده‌های ذخیره شده هنگام بازیابی برای کوئری‌های بعدی به صورت ایمن مدیریت می‌شوند  
- [ ] بله — ورودی کاربر ذخیره شده و بعداً در یک کوئری SQL ناامن **بدون** اعتبارسنجی استفاده می‌شود  

### آیا امتیازات حساب کاربری سرویس پایگاه داده به حداقل سطح دسترسی لازم محدود شده است؟
- [ ] بله — حساب کاربری سرویس (Service Account) دارای **حداقل سطح دسترسی** (Least Privilege) است (مثلاً محدود به جداول/عملیات‌های خاص)  
- [ ] خیر — حساب کاربری سرویس دارای امتیازات بالا است (مثلاً `DROP TABLE` ، دسترسی‌های `FILE` یا فعال بودن `xp_cmdshell`)
