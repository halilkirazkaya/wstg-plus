## WSTG-BUSL-09 — تست آپلود فایل‌های مخرب (Test Upload of Malicious Files)

قابلیت آپلود فایل (File Upload) به کاربران اجازه می‌دهد تا داده‌ها را به سرور ارسال کنند، که این امر یک بردار مستقیم برای ورود محتوای مخرب به محیط برنامه فراهم می‌کند. مهاجمان از منطق اعتبارسنجی (Validation Logic) ضعیف برای آپلود وب‌شل (Web Shell)، بدافزار (Malware) یا فایل‌های خاص طراحی شده — مانند SVG یا HTML — جهت دستیابی به اجرای کد از راه دور (Remote Code Execution - RCE) یا اسکریپت‌نویسی بین‌سایتی (Cross-Site Scripting - XSS) سوءاستفاده می‌کنند. این آسیب‌پذیری معمولاً در ویژگی‌هایی مانند تنظیمات پروفایل، سیستم‌های مدیریت اسناد و مدیریت‌کننده‌های پیوست یافت می‌شود؛ جایی که سرور در بررسی صحیح نوع فایل، محتوا و مجوزهای اجرا (Execution Permissions) باز می‌ماند. بهره‌برداری (Exploit) موفق اغلب منجر به نفوذ کامل به سیستم، استخراج داده‌ها (Data Exfiltration) یا استفاده از سرور به عنوان یک نقطه محوری (Pivot Point) برای حملات شبکه داخلی می‌گردد.

### آیا برنامه قابلیتی برای آپلود فایل ارائه می‌دهد؟
- [ ] خیر — هیچ قابلیت آپلود فایلی وجود ندارد  
- [ ] بله — قابلیت برای کاربران احراز هویت شده وجود دارد  
- [ ] بله — قابلیت برای کاربران **احراز هویت نشده** در دسترس است *(بحرانی)*  

### آیا اعتبارسنجی پسوند فایل در سمت سرور (Server-side) اعمال می‌شود؟
- [ ] بله — یک لیست سفید (Allowlist) سخت‌گیرانه از پسوندها **اعمال شده** و دور زدن آن **امکان‌پذیر نیست**  
- [ ] بله — یک لیست سفید استفاده شده است اما دور زدن آن از طریق حساسیت به حروف بزرگ و کوچک یا پسوندهای دوگانه (Double Extensions) **امکان‌پذیر است**  
- [ ] بله — یک لیست سیاه (Blocklist) استفاده شده است که **می‌توان** آن را با پسوندهای جایگزین (مانند `.phtml` یا `.asa`) دور زد  
- [ ] خیر — هیچ اعتبارسنجی پسوندی در سمت سرور **اعمال نشده است**  

### آیا محتوای فایل فراتر از پسوند یا MIME type اعتبارسنجی می‌شود؟
- [ ] بله — برنامه بایت‌های جادویی (Magic Bytes) را بررسی کرده و بازرسی عمیق محتوا را انجام می‌دهد  
- [ ] بله — برنامه فقط هدر `Content-Type` را بررسی می‌کند که به راحتی **قابل جعل (Spoof)** است  
- [ ] خیر — برنامه برای اعتبارسنجی کاملاً به پسوند فایل متکی است  

### آیا فایل‌های آپلود شده در دایرکتوری با مجوزهای اجرا (Execution Permissions) ذخیره می‌شوند؟
- [ ] خیر — فایل‌ها در یک سرویس ذخیره‌سازی اختصاصی (مانند S3) یا یک دایرکتوری غیرقابل اجرا ذخیره می‌شوند  
- [ ] بله — فایل‌ها روی وب‌سرور ذخیره می‌شوند اما اجرا از طریق پیکربندی **غیرفعال شده است**  
- [ ] بله — فایل‌ها در دایرکتوری قابل دسترسی از طریق وب ذخیره می‌شوند و اجرا **فعال است** *(بحرانی)*  

### آیا فیلترهای امنیتی از طریق دستکاری نام فایل (Filename Manipulation) قابل دور زدن هستند؟
- [ ] خیر — نام فایل‌ها توسط سرور پاکسازی (Sanitize) شده یا به صورت تصادفی بازتولید می‌شوند  
- [ ] بله — فیلترها را **می‌توان** با استفاده از بایت‌های نال (Null Bytes) دور زد (مانند `shell.php%00.jpg`)  
- [ ] بله — از کاراکترهای پیمایش مسیر (Path Traversal) (مانند `../../`) **می‌توان** برای آپلود فایل‌ها در دایرکتوری‌های دلخواه استفاده کرد
