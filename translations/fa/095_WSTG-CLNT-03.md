## WSTG-CLNT-03 — تزریق HTML (HTML Injection)

تزریق HTML (HTML Injection) زمانی رخ می‌دهد که یک اپلیکیشن در پاک‌سازی (Sanitize) ورودی‌های ارائه‌شده توسط کاربر پیش از رندر (Render) کردن آن‌ها در مرورگر ناتوان باشد؛ این امر به مهاجم اجازه می‌دهد تا تگ‌های HTML دلخواه را به مدل شیء سند (DOM) صفحه وب تزریق کند. اگرچه این آسیب‌پذیری مشابه اسکریپت‌نویسی بین‌سایتی (XSS) است، اما هدف اصلی در تزریق HTML، دستکاری ظاهر بصری صفحه یا تسهیل حملات فیشینگ (Phishing) از طریق تزریق فرم‌های مخرب و محتوای فریبنده است. مهاجمان پارامترهایی را که در رابط کاربری (UI) منعکس می‌شوند، مانند عبارات جستجو، فیلدهای پروفایل کاربری یا پیام‌های خطا، هدف قرار می‌دهند تا قربانیان را برای افشای اعتبارنامه‌های (Credentials) حساس یا تعامل با لینک‌های غیرمجاز شخص ثالث فریب دهند. این آسیب‌پذیری ریسک قابل توجهی برای یکپارچگی رابط کاربری اپلیکیشن محسوب می‌شود و می‌تواند پیش‌درامدی برای حملات پیچیده‌تر مهندسی اجتماعی یا حملات مرتبط با نشست (Session) باشد.

### آیا ورودی‌های تحت کنترل کاربر بدون کدگذاری صحیح HTML در DOM منعکس می‌شوند؟
- [ ] خیر — تمام ورودی‌های منعکس‌شده پیش از رندر شدن به‌طور دقیق کدگذاری HTML (HTML-encoded) می‌شوند  
- [ ] بله — برخی کاراکترها کدگذاری می‌شوند اما دور زدن (Bypass) آن‌ها برای تگ‌های خاص امکان‌پذیر است  
- [ ] بله — ورودی به‌صورت خام (Raw) منعکس می‌شود و تزریق HTML امکان‌پذیر است  

### آیا امکان تزریق المان‌های ساختاری HTML برای تغییر چیدمان صفحه وجود دارد؟
- [ ] خیر — اپلیکیشن تگ‌های ساختاری مانند `<div>`، `<table>` یا `<iframe>` را فیلتر یا حذف می‌کند  
- [ ] بله — المان‌های ساختاری قابل تزریق هستند اما تأثیر قابل توجهی بر رابط کاربری ندارند  
- [ ] بله — تخریب ظاهری (Defacement) قابل توجه رابط کاربری از طریق تگ‌های تزریق شده امکان‌پذیر است  

### آیا تزریق المان‌های فیشینگ کاربردی مانند فرم‌ها یا لینک‌های مخرب امکان‌پذیر است؟
- [ ] خیر — تگ‌های `<form>`، `<input>` و `<a>` به‌طور مؤثری مسدود یا پاک‌سازی می‌شوند  
- [ ] بله — لینک‌های مخرب برای هدایت کاربران به سایت‌های خارجی قابل تزریق هستند  
- [ ] بله — المان‌های کاربردی `<form>` برای سرقت اعتبارنامه‌ها قابل تزریق هستند *(متوسط)*  

### آیا هدرهای امنیتی یا سیاست‌های امنیت محتوا (CSP) اثرات تزریق را کاهش می‌دهند؟
- [ ] بله — یک CSP محدودکننده برقرار است و `form-action` یا `frame-src` اعمال شده است  
- [ ] خیر — هدرهای امنیتی وجود دارند اما CSP غیرفعال است یا شامل موارد ناامنی مانند unsafe-inline یا کاراکترهای عمومی (Wildcards) است  
- [ ] خیر — هیچ CSP یا هدر امنیتی مرتبطی فعال نیست
