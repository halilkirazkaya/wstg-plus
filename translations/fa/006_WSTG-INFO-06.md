## WSTG-INFO-06 — شناسایی نقاط ورود اپلیکیشن (Identify Application Entry Points)

شناسایی نقاط ورود اپلیکیشن (Identify Application Entry Points) شامل نگاشت کل سطح حمله (Attack Surface) از طریق احصاء تمامی راه‌های ممکنی است که یک کاربر یا سیستم خارجی می‌تواند با اپلیکیشن تعامل داشته باشد. این فرآیند شامل کشف تمامی URLها، نقاط پایانی API (API Endpoints)، پارامترها (Parameters) در متدهای GET و POST و مبتنی بر کوکی (Cookie)، هدرهای HTTP (HTTP Headers) و پروتکل‌های تخصصی مانند وب‌ساکت (WebSockets) یا gRPC است. برای یک تست‌کننده نفوذ، این مرحله حیاتی است؛ زیرا آسیب‌پذیری‌ها (Vulnerabilities) اغلب در APIهای سایه (Shadow APIs) مستند نشده، نقاط پایانی قدیمی (Legacy Endpoints) یا ساختارهای پیچیده پارامتری که در طول توسعه نادیده گرفته شده‌اند، یافت می‌شوند. احصاء کامل تضمین می‌کند که هیچ جزئی از اپلیکیشن به صورت یک جعبه سیاه (Black Box) آزمایش‌نشده باقی نماند، و در نتیجه از یافتن مسیرهای نظارت‌نشده به سیستم توسط مهاجمان جلوگیری می‌شود.

### آیا تمامی پارامترهای GET و POST در سراسر اپلیکیشن احصاء شده‌اند؟
- [ ] بله — احصاء جامع از طریق خزیدن (Crawling) خودکار و دستی تکمیل شده است
- [ ] بله — تنها پارامترهای رایج از طریق خزیدن استاندارد شناسایی شده‌اند
- [ ] خیر — پارامترها عمدتاً شناسایی نشده یا آزمایش نشده‌اند

### آیا پارامترهای مستند نشده یا مخفی (مانند debug، admin) با استفاده از فازینگ (Fuzzing) جستجو شده‌اند؟
- [ ] خیر — کشف پارامترهای مخفی برای این محدوده (Scope) خاص ضروری نیست
- [ ] بله — فازینگ پارامتر (مثلاً با استفاده از `Arjun`) انجام شده و یافته‌ای نداشته است
- [ ] بله — پارامترهای مخفی کشف و برای تست‌های بیشتر نگاشت شده‌اند
- [ ] خیر — کشف پارامترهای مخفی انجام نشده است

### آیا تمامی متدهای پشتیبانی شده HTTP (مانند PUT، DELETE، PATCH و غیره) برای هر نقطه پایانی شناسایی شده‌اند؟
- [ ] بله — شناسایی متدها برای تمامی نقاط پایانی کشف شده اعمال شده است
- [ ] بله — شناسایی متدها فقط برای نقاط پایانی حساس یا با اهمیت بالا اعمال شده است
- [ ] خیر — فقط متدهای استاندارد GET و POST شناسایی شده‌اند

### آیا نقاط ورود غیر استاندارد مانند WebSockets، gRPC یا هدرهای سفارشی نگاشت شده‌اند؟
- [ ] خیر — اپلیکیشن از پروتکل‌های غیر استاندارد یا هدرهای سفارشی استفاده نمی‌کند
- [ ] بله — تمامی نقاط ورود وابسته به پروتکل و هدرهای سفارشی شناسایی شده‌اند
- [ ] خیر — نقاط ورود غیر استاندارد وجود دارند اما نگاشت نشده‌اند

### آیا سطح API (شامل نقاط پایانی نسخه‌بندی شده مانند /v1/ یا /v2/) به طور کامل نگاشت شده است؟
- [ ] خیر — اپلیکیشن هیچ API ای را در معرض قرار نداده است
- [ ] بله — تمامی نسخه‌ها و نقاط پایانی API شناسایی و مستند شده‌اند
- [ ] بله — تنها نسخه فعلی API عملیاتی شناسایی شده است
- [ ] خیر — نقاط پایانی API نگاشت نشده باقی مانده‌اند یا فقط به صورت جزئی کشف شده‌اند
