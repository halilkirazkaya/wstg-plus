## WSTG-CLNT-05 — تست تزریق CSS (CSS Injection)

تزریق CSS (CSS Injection) زمانی رخ می‌دهد که یک اپلیکیشن به ورودی‌های ارائه‌شده توسط کاربر اجازه می‌دهد تا بدون پاک‌سازی (Sanitization) یا گریزدهی (Escaping) مناسب، بر شیوه‌نامه‌های آبشاری (CSS) یک صفحه وب تأثیر بگذارند. در حالی که این موضوع اغلب به عنوان یک مشکل ظاهری تصور می‌شود، مهاجمان می‌توانند با استفاده از انتخابگرهای ویژگی (Attribute Selectors) و ویژگی‌های background-image برای تحریک درخواست‌های خارجی، از تزریق CSS جهت خروج غیرمجاز داده‌های حساس (Exfiltration) مانند توکن‌های CSRF یا شناسه‌های نشست (Session IDs) بهره‌برداری کنند. این آسیب‌پذیری معمولاً در نقاط پایانی (Endpoints) رخ می‌دهد که ترجیحات کاربر (مانند تم‌های سفارشی یا رنگ قلم) در داخل بلاک‌های `<style>` یا ویژگی‌های `style` درون‌خطی (Inline) منعکس می‌شوند. از دیدگاه مهاجم، بهره‌برداری موفق می‌تواند منجر به تغییر ظاهر رابط کاربری (UI Redressing)، فیشینگ (Phishing) از طریق تغییر غیرمجاز چیدمان، یا خروج مخفیانه داده‌ها در محیط‌هایی شود که سیاست‌های امنیت محتوا (CSP) سخت‌گیرانه ممکن است مانع حملات مبتنی بر جاوااسکریپت شوند.


| فیلد | مقدار |
|---|---|
| **شناسه WSTG** | WSTG-CLNT-05 |
| **CWE** | CWE-74 |
| **وضعیت تست** | انجام نشده |
| **شدت** | متوسط / بالا* |

> *در صورتی که نقطه تزریق اجازه خروج ویژگی‌های حساسی مانند توکن‌های CSRF را بدهد یا بتوان از آن برای تغییر ظاهر رابط کاربری در فرآیندهای حساس استفاده کرد، شدت آن به «بالا» تغییر می‌کند.

### آیا اپلیکیشن ورودی‌های تحت کنترل کاربر را در محیط‌های CSS منعکس می‌کند؟
- [ ] خیر — ورودی در تگ‌های style، ویژگی‌ها یا فایل‌های CSS منعکس نمی‌شود  
- [ ] بله — ورودی منعکس می‌شود اما دقیقاً به مقادیر الفبایی-عددی ایمن محدود شده است  
- [ ] بله — ورودی در یک ویژگی `style` یا بلاک `<style>` منعکس می‌شود  

### آیا فرانویسه‌ها (Metacharacters) و توابع مخصوص CSS به‌درستی پاک‌سازی می‌شوند؟
- [ ] بله — کاراکترهایی مانند `{` ، `}` ، `:` و توابعی مانند `url()` به‌درستی گریزدهی (Escaped) شده‌اند  
- [ ] بله — پاک‌سازی انجام شده است اما امکان گذر (Bypass) از طریق کدگذاری یا کامنت‌های CSS وجود دارد  
- [ ] خیر — هیچ پاک‌سازی روی فرانویسه‌های CSS اعمال نشده است  

### آیا خروج غیرمجاز داده‌ها با استفاده از انتخابگرهای CSS امکان‌پذیر است؟
- [ ] خیر — انتخابگرها نمی‌توانند باعث ارسال درخواست‌های خارجی یا نشت داده شوند  
- [ ] بله — خروج جزئی داده‌ها از طریق انتخابگرهای ویژگی و `background-image` امکان‌پذیر است  
- [ ] بله — خروج کامل توکن‌های حساس با استفاده از تکنیک‌های خودکار Brute Force امکان‌پذیر است  

### آیا یک سیاست امنیت محتوا (CSP) ریسک تزریق CSS را کاهش می‌دهد؟
- [ ] بله — `style-src` به 'self' محدود شده و `img-src` یا `connect-src` محدود شده است  
- [ ] بله — CSP وجود دارد اما از 'unsafe-inline' استفاده می‌کند یا دامنه‌های خارجی با استفاده از Wildcard مجاز هستند  
- [ ] خیر — هیچ CSP برای جلوگیری از بارگذاری منابع خارجی از طریق CSS پیاده‌سازی نشده است  

### آیا می‌توان از این آسیب‌پذیری برای انجام تغییر ظاهر رابط کاربری (UI Redressing) یا فیشینگ استفاده کرد؟
- [ ] خیر — محدوده تزریق برای تغییر قابل توجه چیدمان بسیار محدود است  
- [ ] بله — تغییر رابط کاربری امکان‌پذیر است و اجازه قرار دادن المان‌های مخرب روی صفحه را می‌دهد  
- [ ] بله — کنترل کامل چیدمان صفحه امکان‌پذیر است که انجام حملات فیشینگ بسیار متقاعدکننده را تسهیل می‌کند
