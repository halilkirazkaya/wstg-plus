## WSTG-INFO-08 — شناسایی چارچوب برنامه تحت وب (Fingerprint Web Application Framework)

شناسایی (Fingerprinting) چارچوب یک برنامه تحت وب شامل تعیین پشته فناوری (Technology Stack) خاص و نسخه‌های نرم‌افزاری است که برای ساخت و ارائه برنامه استفاده شده‌اند. مهاجمان سرآیندهای پاسخ HTTP (HTTP response headers)، کوکی‌های (Cookies) اختصاصی چارچوب، ساختارهای فایل قابل پیش‌بینی و آرتیفکت‌های (Artifacts) منحصر‌به‌فرد جاوااسکریپت (JavaScript) را برای تعیین اینکه آیا هدف از پلتفرم‌هایی مانند React، Angular، Django یا Spring استفاده می‌کند، تحلیل می‌کنند. این مرحله شناسایی (Reconnaissance) حیاتی است زیرا به تست‌کننده اجازه می‌دهد سطح حمله را به آسیب‌پذیری‌های شناخته‌شده (CVEs) و نقاط ضعف رایج در پیکربندی (Configuration) که مختص آن چارچوب است، محدود کند. با درک معماری (Architecture) زیربنایی، یک مهاجم می‌تواند از تست عمومی به سمت استراتژی‌های بهره‌برداری (Exploitation) کاملاً هدفمند حرکت کند.

### آیا سرآیندهای پاسخ HTTP نام یا نسخه چارچوب را فاش می‌کنند؟
- [ ] خیر — سرآیندهایی مانند `X-Powered-By` یا `Server` وجود ندارند یا پاکسازی (Sanitize) شده‌اند  
- [ ] بله — نام چارچوب موجود است اما نسخه **مخفی است**  
- [ ] بله — نام چارچوب و نسخه آن **هر دو فاش شده‌اند**  

### آیا کوکی‌های اختصاصی چارچوب یا ساختارهای دایرکتوری قابل شناسایی هستند؟
- [ ] خیر — آرتیفکت‌های رایج چارچوب (مانند مسیرهای `.js` یا نام کوکی‌ها) مبهم‌سازی (Obfuscate) شده یا تغییر نام یافته‌اند  
- [ ] بله — کوکی‌های اختصاصی چارچوب (مانند `JSESSIONID`، `PHPSESSID` یا `csrftoken`) قابل شناسایی **هستند**  
- [ ] بله — ساختارهای دایرکتوری (مانند `/wp-content/` یا `_next/static/`) قابل مشاهده **بوده** و چارچوب را تایید می‌کنند  

### آیا پیام‌های خطا یا کامنت‌های کد منبع جزئیات چارچوب را افشا می‌کنند؟
- [ ] خیر — مدیریت خطا به صورت عمومی انجام شده و کامنت‌ها حذف شده‌اند  
- [ ] بله — ردپای پشته (Stack Trace) اختصاصی چارچوب **فعال بوده** و در پاسخ‌ها قابل مشاهده است  
- [ ] بله — کد منبع HTML شامل کامنت‌ها یا تگ‌های متادیتا (Metadata) است که چارچوب را شناسایی می‌کنند  

### آیا آسیب‌پذیری‌های شناخته‌شده‌ای با نسخه چارچوب شناسایی‌شده در ارتباط است؟
- [ ] خیر — چارچوب شناسایی‌شده به‌روز است و **هیچ آسیب‌پذیری** عمومی شناخته‌شده‌ای ندارد  
- [ ] بله — نسخه چارچوب قدیمی است و آسیب‌پذیری‌های (CVEs) خاصی **قابل شناسایی** هستند
