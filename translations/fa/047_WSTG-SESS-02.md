## WSTG-SESS-02 — بررسی ویژگی‌های کوکی‌ها (Testing for Cookies Attributes)

مدیریت نشست (Session Management) اغلب به کوکی‌های (Cookies) HTTP برای حفظ وضعیت متکی است، که این امر ویژگی‌های امنیتی این کوکی‌ها را به هدف اصلی مهاجمان تبدیل می‌کند. با حذف یا پیکربندی نادرست ویژگی‌هایی مانند `HttpOnly` ،`Secure` و `SameSite` برنامه‌ها توکن‌های نشست را در معرض سرقت از طریق تزریق اسکریپت از طریق سایت (Cross-Site Scripting - XSS)، شنود در کانال‌های رمزگذاری نشده، یا حملات جعل درخواست میان‌سایتی (Cross-Site Request Forgery - CSRF) قرار می‌دهند. آزمون‌گران نفوذ (Pentesters) این پرچم‌ها (Flags) را بررسی می‌کنند تا تعیین کنند آیا یک مهاجم می‌تواند شناسه‌های نشست را از مدل شیء سند (Document Object Model - DOM) مرورگر استخراج کند یا مرورگر را مجبور به ارسال آن‌ها در درخواست‌های میان‌منشأ (Cross-origin requests) نماید. پیکربندی صحیح ویژگی‌ها یک اقدام اساسی در دفاع در عمق (Defense-in-depth) برای اطمینان از باقی ماندن توکن‌های حساس در بسترهای مجاز و امن است.

### آیا پرچم `HttpOnly` برای کوکی‌های حساس نشست اعمال شده است؟
- [ ] بله — برای **تمامی** کوکی‌های حساس نشست اعمال شده است *(بیشترین امنیت)*  
- [ ] بله — فقط برای برخی از کوکی‌های نشست اعمال شده است  
- [ ] خیر — پرچم **اعمال نشده است**، که اجازه دسترسی از طریق اسکریپت‌های سمت کلاینت را می‌دهد *(بحرانی)*  

### آیا پرچم `Secure` برای کوکی‌های منتقل شده در بستر HTTPS اجبار شده است؟
- [ ] بله — برای **تمامی** کوکی‌های منتقل شده در بستر TLS اعمال شده است  
- [ ] خیر — پرچم **اعمال نشده است**، که اجازه می‌دهد کوکی‌ها در بستر HTTP به صورت متن ساده (Plaintext) ارسال شوند  

### آیا ویژگی `SameSite` برای کاهش مخاطرات CSRF استفاده شده است؟
- [ ] بله — برای کوکی‌های نشست روی `Strict` یا `Lax` تنظیم شده است  
- [ ] بله — روی `None` تنظیم شده اما پرچم `Secure` وجود دارد  
- [ ] خیر — ویژگی **مفقود است** یا روی `None` **بدون** پرچم `Secure` تنظیم شده است  

### آیا ویژگی‌های `Domain` و `Path` به حداقل محدوده لازم محدود شده‌اند؟
- [ ] بله — به زیردامنه (Subdomain) و مسیر اپلیکیشن **خاص** محدود شده است  
- [ ] خیر — محدوده بسیار **گسترده** است (مثلاً روی یک دامنه والد یا ریشه `/` تنظیم شده)، که سطح حمله (Attack Surface) را افزایش می‌دهد  

### آیا کوکی‌های حساس نشست از طریق ویژگی‌های `Expires` یا `Max-Age` به درستی منقضی می‌شوند؟
- [ ] بله — تاریخ‌های انقضای مناسب تنظیم شده است  
- [ ] خیر — کوکی‌ها دائمی (Persistent) با طول عمر بیش از حد **طولانی** هستند  
- [ ] خیر — کوکی‌ها فقط برای نشست (Session-only) هستند اما **فاقد** اجبار برای زمان استراحت (Timeout) در سمت سرور می‌باشند
