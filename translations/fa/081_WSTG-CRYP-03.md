## WSTG-CRYP-03 — تست انتقال اطلاعات حساس از طریق کانال‌های رمزنگاری نشده

انتقال داده‌های حساس از طریق کانال‌های رمزنگاری نشده، مانند پروتکل انتقال ابرمتن ساده (HTTP)، اطلاعات را در معرض شنود و تغییر از طریق حملات مرد میانی (Man-in-the-Middle - MITM) قرار می‌دهد. این آسیب‌پذیری معمولاً زمانی رخ می‌دهد که برنامه‌های تحت وب در اجبار به استفاده از امنیت لایه انتقال (Transport Layer Security - TLS) در تمامی نقاط انتهایی (Endpoints) ناتوان باشند؛ این موضوع به‌ویژه در مؤلفه‌های قدیمی (Legacy)، فرم‌های ورود، یا در طول فرآیند انتقال اولیه از HTTP به HTTPS مشاهده می‌شود. از دیدگاه مهاجم، این وضعیت اجازه شنود غیرفعال (Passive Sniffing) اعتبارنامه‌های احراز هویت، توکن‌های نشست (Session Tokens) و اطلاعات هویتی حساس (Personally Identifiable Information - PII) را در بخش‌های ناامن یا به مخاطره افتاده شبکه، مانند Wi-Fi عمومی، می‌دهد. اطمینان از اینکه تمام داده‌های حساس در یک تونل TLS مستحکم کپسوله‌سازی شده‌اند، برای حفظ محرمانگی و یکپارچگی تعاملات کاربر و جلوگیری از سرقت نشست (Session Hijacking) حیاتی است.

### آیا برنامه اجازه برقراری ارتباط از طریق HTTP رمزنگاری نشده را برای نقاط انتهایی حساس می‌دهد؟
- [ ] خیر — تمام ترافیک برنامه به طور سخت‌گیرانه از طریق HTTPS اجبار می‌شود *(امن‌ترین حالت)*  
- [ ] بله — صفحات غیرحساس از طریق HTTP در دسترس هستند اما نقاط انتهایی حساس **در دسترس نیستند**  
- [ ] بله — نقاط انتهایی حساس (login، checkout، profile) از طریق HTTP رمزنگاری نشده **در دسترس هستند** *(ریسک بالا)*  

### آیا بازهدایت (Redirection) سراسری از HTTP به HTTPS وجود دارد؟
- [ ] بله — برنامه برای تمام درخواست‌های HTTP، یک بازهدایت 301/302 به HTTPS انجام می‌دهد  
- [ ] بله — بازهدایت تنها برای نقاط انتهایی حساسِ خاص انجام می‌شود  
- [ ] خیر — هیچ بازهدایتی از HTTP به HTTPS **اعمال نشده است**  

### آیا امنیت انتقال سخت‌گیرانه ابرمتن (HTTP Strict Transport Security - HSTS) پیاده‌سازی شده است؟
- [ ] بله — HSTS با یک `max-age` طولانی **فعال است** و شامل دستورالعمل‌های `includeSubDomains` و `preload` می‌باشد  
- [ ] بله — HSTS **فعال است** اما فاقد دستورالعمل‌های `includeSubDomains` یا `preload` می‌باشد  
- [ ] خیر — HSTS روی سرور برنامه **فعال نیست**

### آیا برنامه داده‌های حساس را از طریق کانال‌های رمزنگاری نشده به APIهای شخص ثالث منتقل می‌کند؟
- [ ] خیر — تمام فراخوانی‌های API خارجی از طریق تونل‌های رمزنگاری شده (HTTPS) انجام می‌شوند  
- [ ] بله — برخی داده‌های تله‌متری (Telemetry) غیرحساس از طریق HTTP ارسال می‌شوند  
- [ ] بله — کلیدهای API، اطلاعات PII یا اعتبارنامه‌ها از طریق HTTP رمزنگاری نشده به شخص ثالث **ارسال می‌شوند**
