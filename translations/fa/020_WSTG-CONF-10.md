## WSTG-CONF-10 — تصاحب زیردامنه (Subdomain Takeover)

تصاحب زیردامنه (Subdomain Takeover) زمانی رخ می‌دهد که یک رکورد DNS (معمولاً CNAME، اما گاهی رکوردهای A یا MX) به یک منبع از کار افتاده یا غیرموجود در یک ارائه‌دهنده ابر شخص ثالث (Third-party Cloud Provider) یا سرویس میزبانی اشاره می‌کند. مهاجمان این رکوردهای DNS معلق (Dangling DNS records) را با جستجوی پیام‌های خطای خاص از ارائه‌دهندگانی مانند AWS، GitHub یا Azure شناسایی می‌کنند که نشان‌دهنده رها شدن آن منبع است. با ثبت نام منبع رها شده در پلتفرم ارائه‌دهنده، مهاجم کنترل کامل زیردامنه را به دست می‌آورد. این امر به آن‌ها اجازه می‌دهد محتوای مخرب میزبانی کنند، کوکی‌های نشست (Session Cookies) مربوط به دامنه اصلی را استخراج نمایند و محافظت‌های سیاست امنیت محتوا (CSP) یا CORS را دور بزنند. این آسیب‌پذیری بسیار خطرناک است زیرا از اعتمادی که به ساختار دامنه قانونی سازمان وجود دارد، برای تسهیل حملات فیشینگ (Phishing) و سرقت داده‌ها با تأثیر بالا استفاده می‌کند.

| فیلد | مقدار |
|---|---|
| **شناسه WSTG** | WSTG-CONF-10 |
| **شناسه CWE** | CWE-1329 |
| **وضعیت تست** | انجام نشده |
| **شدت** | بالا / حیاتی* |

> *در صورتی که بتوان از زیردامنه برای سرقت کوکی‌های نشست از دامنه اصلی یا دور زدن ریدایرکت‌های SSO/OAuth استفاده کرد، شدت به "حیاتی" تغییر می‌یابد.

**ابزارها:** `Subjack`, `SubOver`, `dnscan`, `Amass`, `Nuclei`, `dig`, `nslookup`

### آیا اپلیکیشن از سرویس‌های میزبانی شخص ثالث یا ابری از طریق زیردامنه‌ها استفاده می‌کند؟
- [ ] خیر — تمام زیردامنه‌ها به فضای IP تحت کنترل سازمان اشاره دارند  
- [ ] بله — زیردامنه‌ها به ارائه‌دهندگان شخص ثالث (S3، GitHub Pages، Heroku و غیره) اشاره دارند  

### آیا رکورد DNS معلقی (Dangling DNS) وجود دارد که به منابع غیرموجود اشاره کند؟
- [ ] خیر — تمام رکوردهای DNS شناسایی‌شده به منابع فعال و تحت کنترل ختم می‌شوند  
- [ ] بله — رکوردهای CNAME یا A برای منابعی وجود دارند که دیگر در سمت ارائه‌دهنده **وجود ندارند**  
- [ ] بله — رکوردهای DNS به NXDOMAIN یا صفحات خطای اختصاصی ارائه‌دهنده (مانند "NoSuchBucket") ختم می‌شوند  

### آیا منبع معلق شناسایی‌شده با موفقیت قابل تصاحب است؟
- [ ] خیر — ارائه‌دهنده دارای محافظت‌هایی در برابر تصاحب نام‌های رها شده است یا تأیید حساب کاربری (Account Verification) الزامی است  
- [ ] بله — نام منبع **می‌تواند** توسط هر کاربری در پلتفرم شخص ثالث ثبت شود  

### آیا دامنه اصلی از کوکی‌های دارای محدوده "Domain" استفاده می‌کند که ممکن است به خطر بیفتند؟
- [ ] خیر — کوکی‌ها دقیقاً به زیردامنه‌های خاص محدود شده‌اند یا از فلگ‌های `Host-Only` استفاده می‌کنند  
- [ ] بله — کوکی‌های حساس (شناسه‌های نشست، JWTها) در محدوده دامنه والد تعریف شده‌اند و **می‌توانند** از طریق یک زیردامنه ربوده شده استخراج شوند  

### آیا می‌توان از زیردامنه برای دور زدن هدرهای امنیتی یا جریان‌های احراز هویت استفاده کرد؟
- [ ] خیر — هیچ وابستگی امنیتی به زیردامنه‌های شناسایی‌شده وجود ندارد  
- [ ] بله — زیردامنه در دستورالعمل‌های `script-src` یا `connect-src` در CSP در لیست سفید قرار دارد  
- [ ] بله — زیردامنه یک URI بازگشتی (Redirect URI) معتبر برای پیکربندی‌های OAuth یا SSO است
