## WSTG-INFO-07 — ترسیم مسیرهای اجرا در کل برنامه (Map Execution Paths Through Application)

فرآیند ترسیم مسیرهای اجرا (Execution Paths) شامل شناسایی سیستماتیک تمام نقاط انتهایی (Endpoints) قابل دسترس، جریان‌های عملکردی و شاخه‌های تصمیم‌گیری در یک برنامه وب است. این فرآیند برای اطمینان از اینکه هیچ «قابلیت پنهانی» (Dark Functionality) مانند کدهای قدیمی (Legacy code)، رابط‌های دیباگ (Debug interfaces) یا مسیرهای API مستند نشده، از دید بررسی‌های امنیتی پنهان نمی‌ماند، حیاتی است؛ چرا که این موارد اغلب فاقد کنترل‌های امنیتی مدرن هستند. مهاجمان با ترکیبی از خزش خودکار (Automated spidering) و جستجوی دستی، ساختار برنامه را بصری‌سازی می‌کنند و اغلب در طول این فرآیند، آسیب‌پذیری‌هایی مانند دسترسی مدیریتی غیرمجاز یا نقص‌های منطق کسب‌وکار (Business logic flaws) را کشف می‌کنند. تست‌کنندگان با تطبیق ورودی‌ها با پاسخ‌های خاص سمت سرور (Server-side)، می‌توانند منطق پردازشی حساسی را که نیازمند تست عمیق و هدفمند برای پوشش کامل امنیتی است، شناسایی کنند.

### آیا برنامه به‌طور کامل با استفاده از خزش (Crawling) خودکار و دستی ایندکس شده است؟
- [ ] بله — نقشه جامع تمامی منابع قابل مشاهده و لینک‌شده **تکمیل شده است**  
- [ ] بله — خزش خودکار **تکمیل شده** اما جستجوی دستی در انتظار انجام است  
- [ ] خیر — برنامه هنوز ایندکس **نشده است**  

### آیا نقاط انتهایی (Endpoints) مخفی یا مستند نشده از طریق تحلیل JS یا حملات جستجوی فراگیر (Brute Force) دایرکتوری شناسایی شده‌اند؟
- [ ] خیر — هیچ مسیر مستند نشده‌ای از طریق تحلیل کانال جانبی (Side-channel analysis) کشف نشد  
- [ ] بله — مسیرهای پنهان کشف شدند اما **بدون** داشتن مجوز (Authorization) قابل دسترسی نیستند  
- [ ] بله — نقاط انتهایی مستند نشده **قابل دسترسی هستند** و قابلیت‌های حساسی را ارائه می‌دهند *(حیاتی / بالا)*  

### آیا مسیرهای اجرا از طریق پارامترها یا هدرهای (Headers) غیراستاندارد قابل تغییر هستند؟
- [ ] خیر — پارامترهایی مانند `debug` ،`test` یا `admin` بر جریان اجرا **تأثیری ندارند**  
- [ ] بله — پارامترها خروجی را تغییر می‌دهند اما کنترل‌های امنیتی را دور **نمی‌زنند**  
- [ ] بله — پارامترهای تغییردهنده مسیر **اعمال می‌شوند** و اجازه می‌دهند منطق در نظر گرفته شده دور زده شود  

### آیا جریان منطق کسب‌وکار چندمرحله‌ای (مانند احراز هویت چندعاملی یا فرآیند پرداخت) به‌طور کامل ترسیم شده است؟
- [ ] بله — تمام وضعیت‌ها و انتقالات در فرآیندهای چندمرحله‌ای **شناسایی شده‌اند**  
- [ ] خیر — انتقالات پیچیده در ماشین وضعیت (State machine) به‌طور کامل قابل ترسیم **نیستند**  

### آیا فایل‌های مستندات API (مانند Swagger/OpenAPI) یا نقشه‌های سمت کلاینت (Source Maps) در معرض دید هستند؟
- [ ] خیر — هیچ نقشه معماری یا فایل مستنداتی به‌طور عمومی در دسترس نیست  
- [ ] بله — مستندات موجود هستند اما با احراز هویت (Authentication) **محافظت می‌شوند**  
- [ ] بله — فایل‌های Swagger UI ،OpenAPI JSON یا JS Source Maps **فعال هستند** و به‌طور عمومی قابلیت دسترسی دارند
