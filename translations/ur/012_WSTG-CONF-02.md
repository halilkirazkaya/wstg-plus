## WSTG-CONF-02 — ایپلیکیشن پلیٹ فارم کنفیگریشن کی جانچ (Test Application Platform Configuration)

ایپلیٹیکیشن پلیٹ فارم کنفیگریشن کی جانچ میں بنیادی ویب سرور، ایپلیکیشن سرور، اور فریم ورک کی سیٹنگز کا آڈٹ (Audit) شامل ہے تاکہ اس بات کو یقینی بنایا جا سکے کہ وہ عام ایکسپلائٹس (Exploits) کے خلاف محفوظ ہیں۔ حملہ آور غیر مجاز رسائی حاصل کرنے یا کوڈ پر عمل درآمد کرنے کے لیے ڈیفالٹ کریڈنشلز (Default Credentials)، بغیر پیچ والے پلیٹ فارم کی کمزوریوں (Unpatched Platform Vulnerabilities)، اور کھلے ہوئے ایڈمنسٹریٹو انٹرفیسز (Administrative Interfaces) کو فعال طور پر تلاش کرتے ہیں۔ غلط کنفیگریشنز (Misconfigurations) اکثر حساس ماحول کے تغیرات (Environment Variables)، اندرونی سسٹم پاتھس (Internal System Paths)، یا غیر ضروری نمونہ ایپلیکیشنز (Sample Applications) کی موجودگی کا باعث بنتی ہیں جو حملے کی سطح (Attack Surface) کو بڑھا دیتی ہیں۔ پیشہ ورانہ نقطہ نظر سے، ایک ناقص کنفیگر شدہ پلیٹ فارم ٹارگٹ ماحول تک ابتدائی رسائی (Initial Access) حاصل کرنے کے لیے ایک اعلیٰ امکانی انٹری پوائنٹ کے طور پر کام کرتا ہے۔


| فیلڈ | ویلیو |
|---|---|
| **WSTG ID** | WSTG-CONF-02 |
| **CWE** | CWE-16 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا (Not Performed) |
| **سنگینی (Severity)** | درمیانی (Medium) / اعلیٰ (High)* |

> *سنگینی اس صورت میں اعلیٰ (High) ہو جاتی ہے اگر ایڈمنسٹریٹو انٹرفیسز ڈیفالٹ کریڈنشلز کے ساتھ قابل رسائی ہوں یا اگر نمونہ اسکرپٹس ریموٹ کوڈ ایگزیکیوشن (Remote Code Execution - RCE) کی اجازت دیں۔

### کیا پلیٹ فارم پر ڈیفالٹ کریڈنشلز یا اکاؤنٹس فعال ہیں؟
- [ ] نہیں — تمام ڈیفالٹ اکاؤنٹس غیر فعال (Disabled) ہیں یا پاس ورڈ تبدیل کر دیے گئے ہیں  
- [ ] ہاں — ڈیفالٹ اکاؤنٹس موجود ہیں لیکن بیرونی نیٹ ورک سے قابل رسائی نہیں ہیں  
- [ ] ہاں — ڈیفالٹ اکاؤنٹس فعال ہیں اور انٹرنیٹ کے ذریعے قابل رسائی ہیں *(انتہائی تشویشناک / Critical)*  

### کیا پلیٹ فارم ہیڈرز یا ایرر پیجز کے ذریعے حساس ورژن کی معلومات ظاہر کر رہا ہے؟
- [ ] نہیں — ورژن اسٹرنگز (Version strings) پوشیدہ ہیں یا عمومی ہیں  
- [ ] ہاں — HTTP ہیڈرز (مثلاً `Server`, `X-Powered-By`) میں تفصیلی ورژن کی معلومات ظاہر کی گئی ہیں  
- [ ] ہاں — فل اسٹیک ٹریسز (Full stack traces) اور اندرونی سسٹم پاتھس ورلوز (Verbose) ایرر میسجز میں ظاہر کیے گئے ہیں  

### کیا ایڈمنسٹریٹو یا مینجمنٹ انٹرفیسز پر مناسب پابندیاں عائد ہیں؟
- [ ] نہیں — ایڈمنسٹریٹو انٹرفیسز کھلے (Exposed) نہیں ہیں  
- [ ] ہاں — انٹرفیسز موجود ہیں لیکن IP الاؤ لسٹنگ (IP allowlisting) یا ملٹی فیکٹر آتھنٹیکیشن (Multi-Factor Authentication - MFA) کے ذریعے محدود ہیں  
- [ ] ہاں — انٹرفیسز (مثلاً `/admin`, `/manager`, `/console`) سنگل فیکٹر آتھنٹیکیشن (Single-factor auth) کے ساتھ عوامی طور پر قابل رسائی ہیں  

### کیا پروڈکشن سرور پر نمونہ فائلیں، اسکرپٹس یا دستاویزات موجود ہیں؟
- [ ] نہیں — تمام غیر ضروری فائلیں ہٹا دی گئی ہیں  
- [ ] ہاں — نمونہ فائلیں یا دستاویزات موجود ہیں لیکن حساس معلومات ظاہر نہیں کرتیں  
- [ ] ہاں — نمونہ اسکرپٹس (مثلاً `info.php`, `examples/`) موجود ہیں اور حملے کا براہ راست راستہ (Attack Vector) فراہم کرتی ہیں  

### کیا سرور غیر محفوظ HTTP میتھڈز کو سپورٹ کرنے کے لیے کنفیگر کیا گیا ہے؟
- [ ] نہیں — صرف `GET` اور `POST` فعال (Enabled) ہیں  
- [ ] ہاں — ممکنہ طور پر خطرناک میتھڈز جیسے کہ `PUT`, `DELETE` یا `TRACE` فعال ہیں لیکن محدود ہیں  
- [ ] ہاں — خطرناک میتھڈز فعال ہیں اور غیر مجاز فائل میں تبدیلی یا کریڈنشلز کی چوری کی اجازت دیتے ہیں
