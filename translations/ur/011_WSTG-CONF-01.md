## WSTG-CONF-01 — نیٹ ورک انفراسٹرکچر کنفیگریشن کی جانچ (Test Network Infrastructure Configuration)

نیٹ ورک انفراسٹرکچر کنفیگریشن کی جانچ (Network Infrastructure Configuration Testing) میں ان بنیادی سرور اور نیٹ ورک کے اجزاء میں کمزوریوں کی نشاندہی شامل ہے جو ویب ایپلی کیشن (Web Application) کو سپورٹ کرتے ہیں۔ حملہ آور غلط کنفیگر شدہ سروسز، پرانے پروٹوکولز اور غیر ضروری اوپن پورٹس (Open Ports) کو نشانہ بناتے ہیں تاکہ وہ رسائی حاصل کر سکیں، نیٹ ورک میں لیٹرل موومنٹ (Lateral Movement) کر سکیں، یا جاسوسی (Reconnaissance) کر سکیں۔ عام مشاہدات میں غیر محفوظ TLS ورژنز، تفصیلی سروس بینرز (Service Banners)، اور ایسے ایڈمنسٹریٹو انٹرفیسز (Administrative Interfaces) کا کھلا ہونا شامل ہے جنہیں صرف اندرونی نیٹ ورکس تک محدود ہونا چاہیے۔ انفراسٹرکچر کی سطح پر ان خامیوں کا فائدہ اٹھا کر، ایک مخالف ڈیٹا کی رازداری (Confidentiality) کو نقصان پہنچا سکتا ہے یا ہوسٹ آپریٹنگ سسٹم (Host Operating System) تک غیر مجاز رسائی حاصل کر سکتا ہے۔


| فیلڈ (Field) | ویلیو (Value) |
|---|---|
| **WSTG ID** | WSTG-CONF-01 |
| **CWE** | CWE-16 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا (Not Performed) |
| **شدت** | کم / درمیانی (Low / Medium) |

### کیا ایپلی کیشن ہوسٹ پر کوئی غیر ضروری اوپن پورٹس یا سروسز کھلی ہیں؟
- [ ] نہیں — صرف ضروری پورٹس (مثلاً 443) **قابل رسائی** ہیں  
- [ ] جی ہاں — اضافی سروسز کھلی ہیں لیکن وہ **محفوظ** کنفیگریشن پر عمل پیرا ہیں  
- [ ] جی ہاں — غیر ضروری سروسز (مثلاً FTP، Telnet، SMB) عوامی طور پر **دستیاب** ہیں  

### کیا سروس بینرز یا ہیڈرز سے ورژن کی حساس معلومات افشا ہو رہی ہیں؟
- [ ] نہیں — بینرز ہٹا دیے گئے ہیں یا عام قسم کے ہیں  
- [ ] جی ہاں — بینرز سرور سافٹ ویئر (مثلاً Apache/nginx) کو ظاہر کرتے ہیں لیکن ورژن نمبر **نہیں**  
- [ ] جی ہاں — تفصیلی بینرز مخصوص سافٹ ویئر ورژن ظاہر کرتے ہیں، جو **استحصال (Exploitation)** میں مدد دیتے ہیں  

### کیا TLS کنفیگریشن جدید حفاظتی بہترین طریقوں (Security Best Practices) کے مطابق ہے؟
- [ ] جی ہاں — صرف TLS 1.2/1.3 مضبوط سائفر سوئٹس (Cipher Suites) کے ساتھ **فعال** ہیں  
- [ ] جی ہاں — پرانے پروٹوکولز (TLS 1.0/1.1) **فعال** ہیں  
- [ ] نہیں — کمزور سائفرز یا غیر محفوظ پروٹوکولز (SSLv2/v3) **فعال** ہیں  

### کیا ایڈمنسٹریٹو یا مینجمنٹ انٹرفیسز صرف مجاز نیٹ ورکس تک محدود ہیں؟
- [ ] جی ہاں — انٹرفیسز (مثلاً SSH، RDP، کنٹرول پینلز) عوامی انٹرنیٹ سے **قابل رسائی نہیں** ہیں  
- [ ] نہیں — انٹرفیسز عوامی طور پر **قابل رسائی** ہیں لیکن ملٹی فیکٹر آتھنٹیکیشن (Multi-Factor Authentication) درکار ہے  
- [ ] نہیں — انٹرفیسز سنگل فیکٹر آتھنٹیکیشن یا ڈیفالٹ کریڈینشلز (Default Credentials) کے ساتھ عوامی طور پر **قابل رسائی** ہیں
