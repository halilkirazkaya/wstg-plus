## WSTG-INPV-04 — Testing for HTTP Parameter Pollution (ایچ ٹی ٹی پی پیرامیٹر پولوشن کی جانچ)

ایچ ٹی ٹی پی پیرامیٹر پولوشن (HTTP Parameter Pollution - HPP) تب ہوتا ہے جب ایک ایپلیکیشن ایک ہی نام کے متعدد HTTP پیرامیٹرز وصول کرتی ہے اور انہیں غیر مستقل یا غیر محفوظ طریقے سے پروسیس کرتی ہے۔ ڈپلیکیٹ پیرامیٹرز فراہم کر کے، ایک حملہ آور ایپلیکیشن کی اندرونی منطق (internal logic) میں ہیرا پھیری کر سکتا ہے، جس سے ویب ایپلیکیشن فائر والز (Web Application Firewalls - WAF)، ان پٹ ویلیڈیشن (Input validation) فلٹرز، یا ایکسیس کنٹرول (Access control) میکانزم کو ممکنہ طور پر بائی پاس کیا جا سکتا ہے۔ یہ کمزوری (vulnerability) مکمل طور پر بنیادی ویب سرور یا ایپلیکیشن فریم ورک پر منحصر ہے، جو تکرار شدہ پیرامیٹرز میں سے پہلے، آخری، یا ان کے جڑے ہوئے ورژن (concatenated version) کو منتخب کر سکتا ہے۔ اس کا استحصال (Exploitation) عام طور پر ان اینڈ پوائنٹس کو نشانہ بناتا ہے جو پیرامیٹرز کو بیک اینڈ APIs، ڈیٹا بیس کوئریز، یا یو آر ایل ری ڈائریکشن (URL redirection) میکانزم کو منتقل کرتے ہیں، جس کے اثرات کراس سائٹ اسکرپٹنگ (Cross-site scripting - XSS) سے لے کر غیر مجاز ڈیٹا ایکس فلٹریشن (Data exfiltration) تک ہو سکتے ہیں۔

### ایپلیکیشن ایک ہی نام کے متعدد HTTP پیرامیٹرز کو کیسے ہینڈل کرتی ہے؟
- [ ] نہیں — ڈپلیکیٹ پیرامیٹرز کو سرور مسترد (rejected) کر دیتا ہے یا نظر انداز کر دیتا ہے  
- [ ] ہاں — سرور مستقل طور پر صرف پہلے یا آخری وقوع (occurrence) کو منتخب کرتا ہے  
- [ ] ہاں — سرور ویلیوز کو جوڑ (concatenates) دیتا ہے، جس سے انجکشن (injection) کا امکان پیدا ہو سکتا ہے  

### کیا سیکیورٹی کنٹرولز جیسے WAF یا ان پٹ فلٹر کو بائی پاس کرنے کے لیے HPP کا فائدہ اٹھایا جا سکتا ہے؟
- [ ] نہیں — سیکیورٹی کنٹرولز پیرامیٹر کے تمام وقوعات کا صحیح طریقے سے معائنہ کرتے ہیں  
- [ ] ہاں — ایک WAF یا فلٹر صرف پہلے وقوع کا معائنہ کرتا ہے، جس سے ایک بدنیتی پر مبنی دوسرے وقوع کو گزرنے کی اجازت مل جاتی ہے  
- [ ] ہاں — ویلیوز کو جوڑنا (concatenation) حملہ آور کو ایک پے لوڈ (payload) کو متعدد پیرامیٹرز میں تقسیم کرنے کی اجازت دیتا ہے تاکہ پتہ لگانے (detection) سے بچا جا سکے  

### کیا ایپلیکیشن مناسب سینیٹائزیشن (sanitization) کے بغیر رسپانس میں آلودہ (polluted) پیرامیٹرز کی عکاسی کرتی ہے؟
- [ ] نہیں — UI میں عکاسی سے پہلے پیرامیٹرز کو سینیٹائز یا انکوڈ کیا جاتا ہے  
- [ ] ہاں — پولوشن کے نتیجے میں مواد کی عکاسی ہوتی ہے، لیکن سینیٹائزیشن لاگو کی گئی ہے  
- [ ] ہاں — پولوشن کے نتیجے میں مواد کی عکاسی ہوتی ہے اور سینیٹائزیشن لاگو نہیں ہے، جس کی وجہ سے XSS یا لاجک ایررز پیدا ہوتے ہیں  

### کیا اندرونی API کالز یا بیک اینڈ سسٹمز کو منتقل کیے گئے پیرامیٹرز پولوشن کے لیے حساس ہیں؟
- [ ] نہیں — بیک اینڈ درخواستیں محفوظ، غیر متحرک (non-dynamic) تعمیراتی طریقے استعمال کرتی ہیں  
- [ ] ہاں — HPP حملہ آور کو بیک اینڈ درخواست کے ڈھانچے میں پیرامیٹرز داخل (inject) کرنے یا اوور رائٹ (overwrite) کرنے کی اجازت دیتا ہے  

### جب GET بمقابلہ POST درخواستوں میں پیرامیٹرز کو آلودہ کیا جاتا ہے تو کیا ایپلیکیشن مختلف رویہ ظاہر کرتی ہے؟
- [ ] نہیں — تمام HTTP طریقوں (methods) میں رویہ یکساں رہتا ہے  
- [ ] ہاں — صرف GET پیرامیٹرز پولوشن کے لیے حساس ہیں  
- [ ] ہاں — صرف POST (body) پیرامیٹرز پولوشن کے لیے حساس ہیں
