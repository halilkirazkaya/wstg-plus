## WSTG-ATHZ-01 — ڈائریکٹری ٹریورسل فائل انکلوژن کی ٹیسٹنگ (Testing Directory Traversal File Include)

ڈائریکٹری ٹریورسل (Directory Traversal) اور فائل انکلوژن (File Inclusion) کی کمزوریاں اس وقت پیدا ہوتی ہیں جب کوئی ایپلیکیشن فائلوں یا ڈائریکٹریوں کے راستے (paths) بنانے کے لیے صارف کے زیرِ کنٹرول ان پٹ کو بغیر کسی مناسب توثیق (validation) یا صفائی (sanitization) کے استعمال کرتی ہے۔ حملہ آور `../` جیسے سیکوئنسز انجیکٹ کر کے ان خامیوں کا فائدہ اٹھاتے ہیں تاکہ وہ مطلوبہ ڈائریکٹری سے باہر نکل سکیں، جس کے نتیجے میں حساس سسٹم فائلوں، کنفیگریشن ڈیٹا، یا ایپلیکیشن کے سورس کوڈ تک رسائی ممکن ہو سکتی ہے۔ لوکل فائل انکلوژن (LFI) یا ریموٹ فائل انکلوژن (RFI) پر مشتمل زیادہ سنگین معاملات میں، ایک حملہ آور بدنیتی پر مبنی اسکرپٹس شامل کر کے یا لاگ پوائزننگ (log poisoning) اور PHP wrappers کا فائدہ اٹھا کر ریموٹ کوڈ ایگزیکیوشن (Remote Code Execution - RCE) حاصل کر سکتا ہے۔ یہ کمزوریاں عام طور پر ان پیرامیٹرز میں پائی جاتی ہیں جو متحرک مواد کی لوڈنگ، ٹیمپلیٹ انجن (template engines)، یا امیج ریٹریول اینڈ پوائنٹس (image retrieval endpoints) کے لیے استعمال ہوتے ہیں جہاں سرور سائیڈ لاجک فائل سسٹم کے پاتھ کو سنبھالتا ہے۔


| فیلڈ | ویلیو |
|---|---|
| **WSTG ID** | WSTG-ATHZ-01 |
| **CWE** | CWE-22 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا (Not Performed) |
| **شدت** | ہائی / کریٹیکل (High / Critical) |

### کیا پیرامیٹرز سرور سائیڈ پروسیسنگ کے لیے فائل کے نام یا پاتھ قبول کرتے ہیں؟
- [ ] نہیں — کوئی بھی پیرامیٹر فائل سسٹم کے ساتھ تعامل (interact) کرتا ہوا نظر نہیں آتا  
- [ ] ہاں — پیرامیٹرز موجود ہیں لیکن وہ فائل کی شناخت کرنے والے عناصر کی ایک سخت الاؤ لسٹ (allowlist) استعمال کرتے ہیں  
- [ ] ہاں — پیرامیٹرز براہ راست فائل کے نام یا پاتھ قبول کرتے ہیں  

### کیا ڈائریکٹری ٹریورسل سیکوئنسز کو روکنے کے لیے ان پٹ کو صاف (sanitize) کیا گیا ہے؟
- [ ] ہاں — ان پٹ کی توثیق ایک سخت الاؤ لسٹ کے ذریعے کی جاتی ہے اور ٹریورسل سیکوئنسز **ممکن نہیں ہیں**  
- [ ] ہاں — ان پٹ سے `../` سیکوئنسز کو ہٹا کر اسے صاف کیا جاتا ہے، لیکن ریکرسیو بائی پاس (recursive bypass) **ممکن ہے**  
- [ ] نہیں — پاتھ سے متعلقہ ان پٹ پر کوئی صفائی یا توثیق **لاگو نہیں ہے**  

### کیا ٹریورسل سیکوئنسز کے ذریعے محدود ڈائریکٹری سے باہر کی فائلوں تک رسائی ممکن ہے؟
- [ ] نہیں — ایپلیکیشن یا آپریٹنگ سسٹم مقررہ دائرہ کار سے باہر کی فائلوں تک رسائی کو روکتا ہے  
- [ ] ہاں — ویب روٹ (web root) کے اندر کی فائلوں تک رسائی **ممکن ہے**  
- [ ] ہاں — حساس سسٹم فائلوں (مثلاً `/etc/passwd` یا `C:\Windows\win.ini`) تک رسائی **ممکن ہے** *(کریٹیکل)*  

### کیا سرور ریموٹ URLs کی شمولیت (RFI) کی اجازت دیتا ہے؟
- [ ] نہیں — سرور یا ایپلیکیشن کی سطح پر ریموٹ فائل انکلوژن **غیر فعال (disabled)** ہے  
- [ ] ہاں — ریموٹ فائلیں شامل **کی جا سکتی ہیں** لیکن ان کا ایگزیکیوشن (execution) **ممکن نہیں ہے**  
- [ ] ہاں — ریموٹ فائلیں شامل **کی جا سکتی ہیں** اور سرور پر ایگزیکیوٹ بھی کی جا سکتی ہیں *(کریٹیکل)*  

### کیا انکوڈنگ یا خصوصی حروف کا استعمال کر کے فلٹرز کو بائی پاس کیا جا سکتا ہے؟
- [ ] نہیں — فلٹرز مختلف قسم کی انکوڈنگز اور نل بائٹس (null bytes) کو مؤثر طریقے سے سنبھالتے ہیں  
- [ ] ہاں — یو آر ایل انکوڈنگ (URL encoding)، ڈبل یو آر ایل انکوڈنگ، یا 16-bit Unicode استعمال کر کے بائی پاس **ممکن ہے**  
- [ ] ہاں — نل بائٹ انجیکشن (null byte injection - `%00`) یا فائل سسٹم ریپرز (مثلاً `php://filter`) استعمال کر کے بائی پاس **ممکن ہے**
