## WSTG-IDNT-01 — کردار کی تعریفوں کی جانچ (Test Role Definitions)

کردار کی تعریفوں کی جانچ (Test Role Definitions) میں ایپلی کیشن کے اندر متعین کردہ مختلف صارفی کرداروں (User Roles) اور اجازت کی سطحوں (Permission Levels) کی شناخت اور دستاویز سازی شامل ہے تاکہ یہ یقینی بنایا جا سکے کہ وہ منطقی طور پر الگ ہیں اور کم ترین مراعات کے اصول (Principle of Least Privilege) کی پیروی کرتے ہیں۔ ایک حملہ آور ایپلی کیشن کا تجزیہ کرتا ہے تاکہ اعلیٰ مراعات یافتہ کرداروں بمقابلہ معیاری صارفی کرداروں کا نقشہ بنا سکے، اور اجازت کی حدود میں کسی بھی ابہام یا غیر دستاویزی انتظامی افعال (Administrative Functions) کو تلاش کر سکے۔ ان کرداروں کی شناخت کرنا مراعات میں اضافے (Privilege Escalation) کے راستوں کو دریافت کرنے کا پہلا قدم ہے، کیونکہ کردار کی تعریفوں میں تضادات اکثر حساس ڈیٹا یا انتظامی خصوصیات تک غیر مجاز رسائی (Unauthorized Access) کا باعث بنتے ہیں۔ یہ مرحلہ عام طور پر ابتدائی ریکی (Initial Reconnaissance) اور کاروباری منطق کی میپنگ (Business Logic Mapping) کے دوران انجام پاتا ہے، جس میں صارفی پروفائلز، انتظامی ڈیش بورڈز، اور API کی اجازت کے ڈھانچے پر توجہ مرکوز کی جاتی ہے۔

|---|---|

| **CWE** | CWE-285 |
| **جانچ کی صورتحال** | انجام نہیں دیا گیا (Not Performed) |
| **سنگینی** | کم / درمیانی (Low / Medium) |

### کیا ایپلی کیشن یا اس کی دستاویزات میں کرداروں کو واضح طور پر بیان اور دستاویزی شکل دی گئی ہے؟
- [ ] جی ہاں — کردار مکمل طور پر دستاویزی ہیں اور کم ترین مراعات (Least Privilege) کے اصول کی پیروی کرتے ہیں  
- [ ] جی ہاں — کردار دستاویزی ہیں لیکن ان میں اجازتیں ایک دوسرے پر حاوی (Overlapping permissions) ہیں  
- [ ] نہیں — کرداروں کی کوئی رسمی دستاویز سازی یا تعریفیں موجود نہیں ہیں  

### کیا انتظامی اور معیاری صارفی افعال کے درمیان واضح فرق موجود ہے؟
- [ ] جی ہاں — انتظامی افعال (Administrative functions) معیاری صارفی ویوز سے مکمل طور پر الگ تھلگ ہیں  
- [ ] جی ہاں — علیحدگی موجود ہے لیکن انتظامی اینڈ پوائنٹس (Endpoints) قابل قیاس یا اندازہ لگانے کے قابل ہیں  
- [ ] نہیں — انتظامی اور معیاری افعال ایک ہی انٹرفیس کے اندر ملے جلے ہیں  

### کیا ایپلی کیشن کردار پر مبنی رسائی کنٹرول (Role-Based Access Control - RBAC) کے لیے ایک مرکزی طریقہ کار استعمال کرتی ہے؟
- [ ] جی ہاں — مرکزی RBAC یا ABAC لاگو ہے اور مستقل طور پر نافذ کیا گیا ہے  
- [ ] جی ہاں — غیر مرکزی کنٹرولز موجود ہیں لیکن مختلف ماڈیولز میں غیر مستقل طور پر لاگو ہیں  
- [ ] نہیں — رسائی کنٹرول کی منطق (Access control logic) انفرادی صفحات یا اجزاء میں ہارڈ کوڈڈ (Hardcoded) ہے  

### کیا سسٹم میں غیر دستاویزی یا پوشیدہ کردار موجود ہیں؟
- [ ] نہیں — تمام دریافت شدہ کردار دستاویزی فعالیت کے مطابق ہیں  
- [ ] جی ہاں — پوشیدہ یا "شیڈو" (Shadow) کردار (مثلاً `super_admin`, `support`, `test`) موجود ہیں  

### کیا ایک کم مراعات یافتہ صارف دوسرے صارفین کی اجازتوں یا کرداروں کی فہرست (Enumerate) حاصل کر سکتا ہے؟
- [ ] نہیں — صارفین دیگر اداروں کے کرداروں یا اجازتوں کو دیکھ یا شمار (Enumerate) نہیں کر سکتے  
- [ ] جی ہاں — صارفین پروفائل صفحات، میٹا ڈیٹا (Metadata)، یا API کے جوابات کے ذریعے کرداروں کو شمار کر سکتے ہیں *(درمیانی)*
