## WSTG-INFO-05 — معلومات کے اخراج (Information Leakage) کے لیے ویب پیج کے مواد کا جائزہ لیں

ویب پیج کے مواد کا جائزہ لینے میں ایپلی کیشن کے جوابات بشمول HTML، JavaScript، اور CSS فائلوں کا دستی اور خودکار معائنہ شامل ہے تاکہ صارفین کے سامنے غیر ارادی طور پر ظاہر ہونے والی حساس معلومات (Sensitive Information) کی نشاندہی کی جا سکے۔ حملہ آور کلائنٹ سائیڈ سورس کوڈ (Client-side source code) میں ڈویلپر کے تبصروں، پوشیدہ فارم فیلڈز، اندرونی سرور پاتھس، ہارڈ کوڈڈ API کیز (Hardcoded API keys)، اور ڈی بگنگ معلومات (Debugging information) کو غور سے دیکھتے ہیں جو حملے کے اگلے مراحل میں مددگار ثابت ہو سکتی ہیں۔ معلومات کا یہ اخراج اکثر اس وقت ہوتا ہے جب ڈویلپرز پروڈکشن (Production) پر منتقل ہونے سے پہلے میٹا ڈیٹا (Metadata) یا ڈی بگنگ کوڈ کو ہٹانا بھول جاتے ہیں، جو ممکنہ طور پر منطقی خامیاں (Logic flaws) یا بیک اینڈ انفراسٹرکچر کی تفصیلات ظاہر کر سکتے ہیں۔ حملہ آور کے نقطہ نظر سے، یہ ایپلی کیشن کے اندرونی ڈھانچے کا نقشہ بنانے اور سیکیورٹی الرٹس کو متحرک کیے بغیر یا سرور کی بیک اینڈ لاجک (Backend logic) کے ساتھ براہ راست تعلق بنائے بغیر نظر انداز کیے گئے داخلے کے مقامات (Entry points) کو دریافت کرنے کا ایک کم محنت والا طریقہ فراہم کرتا ہے۔


| فیلڈ | ویلیو |
|---|---|
| **WSTG ID** | WSTG-INFO-05 |
| **CWE** | CWE-200 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا |
| **شدت (Severity)** | کم / درمیانی* |

> *شدت ہائی (High) ہو جاتی ہے اگر حساس ہارڈ کوڈڈ کریڈینشلز (Hardcoded credentials)، پرائیویٹ کیز (Private keys)، یا اندرونی انوائرمنٹ ویری ایبلز (Environment variables) دریافت ہوں۔

### کیا سورس کوڈ میں حساس معلومات پر مشتمل ڈویلپر کے تبصرے (Developer comments) موجود ہیں؟
- [ ] نہیں — کوئی تبصرے نہیں ملے یا صرف عمومی، غیر حساس تبصرے ملے  
- [ ] ہاں — تبصرے موجود ہیں لیکن ان میں کوئی حساس لاجک یا ڈیٹا **نہیں** ہے  
- [ ] ہاں — تبصرے اندرونی پاتھس، SQL کوئریز، یا انتظامی ہدایات کو **ظاہر** کرتے ہیں  
- [ ] ہاں — تبصرے کریڈینشلز یا حساس ڈویلپر نوٹس کو **ظاہر** کرتے ہیں *(شدید - High)*  

### کیا پوشیدہ HTML ان پٹ فیلڈز میں حساس ڈیٹا یا اسٹیٹ (State) کی معلومات موجود ہیں؟
- [ ] نہیں — کوئی پوشیدہ فیلڈز نہیں ملی یا ان میں صرف غیر حساس ٹوکنز موجود ہیں  
- [ ] ہاں — پوشیدہ فیلڈز اسٹیٹ کے لیے استعمال ہوتی ہیں لیکن وہ **انکرپٹڈ (Encrypted)** یا **سائنڈ (Signed)** ہیں  
- [ ] ہاں — پوشیدہ فیلڈز میں **پلین ٹیکسٹ (Plaintext)** پاس ورڈز، صارف کے کردار (User roles)، یا اندرونی IDs موجود ہیں  

### کیا JavaScript فائلوں میں ہارڈ کوڈڈ سیکیریٹس، API کیز، یا اندرونی IP ایڈریسز موجود ہیں؟
- [ ] نہیں — JS فائلوں میں کوئی حساس سٹرنگز یا سیکیریٹس (Secrets) نہیں ملے  
- [ ] ہاں — JS فائلوں میں پبلک API کیز موجود ہیں جن پر **مناسب** پابندیاں لاگو ہیں  
- [ ] ہاں — JS فائلوں میں **غیر محفوظ** API کیز، اندرونی IPs، یا پرائیویٹ اینڈ پوائنٹس موجود ہیں  
- [ ] ہاں — JS فائلوں میں **ہارڈ کوڈڈ** انتظامی کریڈینشلز یا پرائیویٹ کیز موجود ہیں *(انتہائی شدید - Critical)*  

### کیا ایپلی کیشن سورس میں فریم ورک ورژن یا اندرونی فائل پاتھ ظاہر کرتی ہے؟
- [ ] نہیں — فریم ورک کی معلومات اور فائل پاتھ **منی فائیڈ (Minified)** یا **اوبفسکیٹڈ (Obfuscated)** ہیں  
- [ ] ہاں — فریم ورک کے نام اور ورژن **واضح** ہیں لیکن پیچ (Patch) شدہ ہیں  
- [ ] ہاں — سورس کوڈ میں اندرونی فائل پاتھ یا سرور کے مکمل پاتھ **ظاہر** ہو رہے ہیں  

### کیا منی فیکیشن (Minification) یا اوبفسکیشن (Obfuscation) کی کمی کی وجہ سے سورس کوڈ معلومات کے اخراج کا شکار ہے؟
- [ ] نہیں — پروڈکشن کوڈ **مکمل طور پر منی فائیڈ** ہے اور تبصروں سے پاک ہے  
- [ ] ہاں — کوڈ منی فائیڈ ہے لیکن سورس میں تبصرے **باقی** ہیں  
- [ ] ہاں — سورس میپس (`.map` فائلیں) **عوامی طور پر دستیاب** ہیں، جو سورس کوڈ کی مکمل تشکیل نو (Reconstruction) کی اجازت دیتے ہیں
