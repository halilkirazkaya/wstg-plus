## WSTG-INPV-19 — Server-Side Request Forgery (SSRF) کی جانچ پڑتال

سرور سائیڈ ریکوئسٹ فورجری (Server-Side Request Forgery - SSRF) تب واقع ہوتی ہے جب ایک حملہ آور ویب ایپلیکیشن کو سرور سے اندرونی یا بیرونی وسائل (resources) کے لیے غیر مجاز درخواستیں (unauthorized requests) بھیجنے پر مجبور کر سکتا ہے۔ ایسے پیرامیٹرز (parameters) میں تبدیلی کر کے جو URLs، ہوسٹ نیمز (hostnames)، یا IP ایڈریسز کی توقع کرتے ہیں، ایک حملہ آور نیٹ ورک کی سطح کے کنٹرولز جیسے کہ فائر والز (firewalls) اور ACLs کو بائی پاس (bypass) کر سکتا ہے تاکہ اندرونی نیٹ ورک کے حصوں کی چھان بین کر سکے، کلاؤڈ میٹا ڈیٹا سروسز (Cloud Metadata Services - IMDS) تک رسائی حاصل کر سکے، یا کمزور اندرونی APIs اور ڈیٹا بیسز کے ساتھ تعامل (interact) کر سکے۔ یہ کمزوری عام طور پر ان فیچرز میں ظاہر ہوتی ہے جن میں امیج فیچنگ (image fetching)، پی ڈی ایف جنریشن (PDF generation)، ویب ہکس (webhooks)، یا URL کے ذریعے فائل اپ لوڈز شامل ہوتے ہیں۔ ایک حملہ آور کے نقطہ نظر سے، SSRF ایک طاقتور ذریعہ (primitive) ہے جس کا استعمال الگ تھلگ ماحول میں داخل ہونے، حساس کنفیگریشن ڈیٹا نکالنے (exfiltrate)، یا ممکنہ طور پر Redis یا Memcached جیسی اندرونی سروسز کے ساتھ تعامل کے ذریعے ریموٹ کوڈ ایگزیکیوشن (Remote Code Execution - RCE) حاصل کرنے کے لیے کیا جاتا ہے۔

### کیا ایپلیکیشن صارف کے فراہم کردہ URLs یا ہوسٹ نیمز کو پروسیس کرتی ہے؟
- [ ] نہیں — کوئی بھی فیچر بیرونی URLs یا ہوسٹ نیمز قبول نہیں کرتا  
- [ ] جی ہاں — فیچر موجود ہے لیکن ان پٹ کی ایک ایلاؤ لسٹ (allow-list) کے خلاف **سختی سے توثیق کی جاتی ہے**  
- [ ] جی ہاں — فیچر موجود ہے اور صارف کے فراہم کردہ URLs قبول کرتا ہے  

### کیا مطلوبہ منزل (destination) پر ویلیڈیشن کنٹرولز یا بلیک لسٹس لاگو ہیں؟
- [ ] جی ہاں — قابلِ بھروسہ ڈومینز/IPs کی ایک سخت **ایلاؤ لسٹ (allow-list)** نافذ کی گئی ہے  
- [ ] جی ہاں — ایک **ڈینائے لسٹ (deny-list)** استعمال کی جاتی ہے (مثلاً `127.0.0.1` یا `169.254.169.254` کو بلاک کرنا)  
- [ ] نہیں — ان پٹ پر **کوئی ویلیڈیشن** یا فلٹرنگ لاگو نہیں ہے  

### کیا انکوڈنگ، ری ڈائریکٹس، یا DNS ری بائنڈنگ کے ذریعے فلٹرز کو بائی پاس کرنا ممکن ہے؟
- [ ] نہیں — فلٹرز مضبوط ہیں اور ری ڈائریکٹس/DNS ریزولوشن کو محفوظ طریقے سے ہینڈل کرتے ہیں  
- [ ] جی ہاں — URL انکوڈنگ یا متبادل IP فارمیٹس (hex, octal) کا استعمال کرتے ہوئے **بائی پاس ممکن ہے**  
- [ ] جی ہاں — اندرونی اہداف کی طرف 3xx HTTP ری ڈائریکٹس کے ذریعے **بائی پاس ممکن ہے**  
- [ ] جی ہاں — اندرونی IPs تک ریزولوشن کے لیے **DNS ری بائنڈنگ (DNS rebinding)** حملوں کے ذریعے بائی پاس ممکن ہے  

### کیا ایپلیکیشن اندرونی میٹا ڈیٹا سروسز یا لوکل انٹرفیسز تک پہنچ سکتی ہے؟
- [ ] نہیں — لوکل نیٹ ورک اور کلاؤڈ میٹا ڈیٹا اینڈ پوائنٹس **قابلِ رسائی نہیں ہیں**  
- [ ] جی ہاں — لوکل ہوسٹ (`127.0.0.1`) یا لوپ بیک سروسز تک رسائی **ممکن ہے**  
- [ ] جی ہاں — کلاؤڈ میٹا ڈیٹا سروسز (مثلاً AWS/GCP/Azure IMDS) تک رسائی **ممکن ہے** *(انتہائی اہم / Critical)*  

### SSRF رسپانس (response) کی نوعیت کیا ہے؟
- [ ] بلائنڈ (Blind) — صارف کو کوئی رسپانس یا میٹا ڈیٹا واپس نہیں کیا جاتا  
- [ ] جزوی (Partial) — رسپانس میٹا ڈیٹا (headers, size, timing) صارف کو واپس کیا جاتا ہے  
- [ ] مکمل (Full) — اندرونی وسیلے سے مکمل رسپانس باڈی صارف کو **دکھائی دیتی ہے (Reflected)**
