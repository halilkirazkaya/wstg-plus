## WSTG-CONF-07 — HTTP Strict Transport Security (HSTS) کی جانچ

ایچ ٹی ٹی پی اسٹرکٹ ٹرانسپورٹ سیکیورٹی (HTTP Strict Transport Security - HSTS) ایک سیکیورٹی میکانزم ہے جو ویب سرور کو براؤزرز کو یہ بتانے کی اجازت دیتا ہے کہ اسے صرف HTTPS کے ذریعے ہی رسائی حاصل ہونی چاہیے، جس سے پروٹوکول ڈاؤن گریڈ حملوں (Protocol Downgrade Attacks) اور کوکی ہائی جیکنگ (Cookie Hijacking) کو روکا جا سکتا ہے۔ ایک انکرپٹڈ کنکشن (Encrypted Connection) کو لازمی قرار دے کر، HSTS مین ان دی مڈل (Man-in-the-Middle - MitM) حملوں جیسے کہ SSLStrip کو کم کرتا ہے، جہاں ایک حملہ آور TLS میں منتقلی کو روکنے کے لیے ابتدائی غیر محفوظ HTTP درخواست کو درمیان میں روکتا ہے۔ حملہ آور کے نقطہ نظر سے، اس ہیڈر (Header) کی عدم موجودگی یا `max-age` کی مختصر مدت ابتدائی کلیئر ٹیکسٹ ہینڈ شیک (Cleartext Handshake) کے دوران حساس سیشن ٹوکنز (Session Tokens) یا اسناد (Credentials) حاصل کرنے کا موقع فراہم کرتی ہے۔ یہ ٹیسٹ تمام حساس ایپلیکیشن اینڈ پوائنٹس (Endpoints) پر `Strict-Transport-Security` ہیڈر کی موجودگی، مدت اور دائرہ کار کی تصدیق پر مرکوز ہے۔


| فیلڈ | قدر |
|---|---|
| **WSTG ID** | WSTG-CONF-07 |
| **CWE** | CWE-319 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا |
| **شدت** | کم / درمیانی* |

> *شدت درمیانی (Medium) ہو جاتی ہے اگر ایپلیکیشن PII، سیشن ٹوکنز، یا اسناد کو ہینڈل کرتی ہے، کیونکہ HSTS کی کمی MitM حملوں کی کامیابی کی شرح کو بڑھا دیتی ہے۔

### کیا HTTPS جوابات میں `Strict-Transport-Security` ہیڈر موجود ہے؟
- [ ] ہاں — ہیڈر تمام حساس اینڈ پوائنٹس پر **موجود ہے**  
- [ ] ہاں — ہیڈر **موجود ہے** لیکن صرف لینڈنگ پیج پر  
- [ ] نہیں — ہیڈر ایپلیکیشن سے مکمل طور پر **غائب ہے**  

### کیا `max-age` ڈائریکٹو کو کافی مدت کے لیے سیٹ کیا گیا ہے؟
- [ ] ہاں — `max-age` ایک سال یا اس سے زیادہ (مثلاً `31536000`) پر سیٹ ہے  
- [ ] ہاں — `max-age` سیٹ ہے لیکن **بہت مختصر** ہے (مثلاً 6 ماہ سے کم)  
- [ ] نہیں — `max-age` ڈائریکٹو **غائب ہے** یا `0` پر سیٹ ہے *(انتہائی اہم)*  

### کیا HSTS پالیسی تمام ذیلی ڈومینز (Subdomains) کا احاطہ کرتی ہے؟
- [ ] ہاں — `includeSubDomains` ڈائریکٹو **لاگو ہے**  
- [ ] نہیں — `includeSubDomains` ڈائریکٹو **غائب ہے**، جس سے ذیلی ڈومینز ڈاؤن گریڈ کے خطرے سے دوچار ہو جاتے ہیں  
- [ ] نہیں — یہ خصوصیت لاگو نہیں ہوتی کیونکہ کوئی ذیلی ڈومین موجود نہیں ہے  

### کیا ایپلیکیشن HSTS Preloading کے لیے رجسٹرڈ ہے؟
- [ ] ہاں — `preload` ڈائریکٹو **موجود ہے** اور ڈومین HSTS پری لوڈ لسٹ میں شامل ہے  
- [ ] ہاں — `preload` ڈائریکٹو **موجود ہے** لیکن ڈومین ابھی تک پری لوڈ لسٹ میں **نہیں** ہے  
- [ ] نہیں — `preload` ڈائریکٹو **غائب ہے**، جو پہلے وزٹ پر MitM کے لیے ایک موقع فراہم کرتا ہے  

### کیا HSTS ہیڈر غلطی سے پلین ٹیکسٹ (Plaintext) HTTP پر بھیجا جا رہا ہے؟
- [ ] نہیں — ہیڈر **صرف** محفوظ HTTPS کنکشنز پر بھیجا جاتا ہے  
- [ ] ہاں — ہیڈر HTTP پر **بھیجا جاتا ہے**، جسے براؤزرز نظر انداز کر دیتے ہیں اور یہ کنفیگریشن کی تفصیلات کو لیک کر سکتا ہے
