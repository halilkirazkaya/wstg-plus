## WSTG-CLNT-05 — سی ایس ایس انجیکشن کی جانچ (Testing for CSS Injection)

سی ایس ایس انجیکشن (CSS Injection) تب ہوتا ہے جب کوئی ایپلیکیشن صارف کی فراہم کردہ ان پٹ (Input) کو ویب پیج کی کاسکیڈنگ اسٹائل شیٹس (Cascading Style Sheets - CSS) پر مناسب سینیٹائزیشن (Sanitization) یا اسکیپنگ (Escaping) کے بغیر اثر انداز ہونے کی اجازت دیتی ہے۔ اگرچہ اسے اکثر محض ایک ظاہری مسئلہ سمجھا جاتا ہے، حملہ آور ایٹریبیوٹ سلیکٹرز (Attribute selectors) اور بیک گراؤنڈ امیج پراپرٹیز (Background-image properties) کا استعمال کر کے بیرونی ریکوسٹس (External requests) کو متحرک کر سکتے ہیں، جس کے ذریعے سی ایس آر ایف ٹوکنز (CSRF tokens) یا سیشن آئی ڈیز (Session IDs) جیسے حساس ڈیٹا کا اخراج (Exfiltration) کیا جا سکتا ہے۔ یہ کمزوری عام طور پر ان اینڈ پوائنٹس (Endpoints) پر پائی جاتی ہے جہاں صارف کی ترجیحات (مثلاً کسٹم تھیمز، فونٹ کے رنگ) `<style>` بلاکس یا اِن لائن `style` ایٹریبیوٹس کے اندر ظاہر ہوتی ہیں۔ حملہ آور کے نقطہ نظر سے، اس کا کامیاب استحصال یو آئی ری ڈریسنگ (UI redressing)، غیر مجاز لے آؤٹ میں تبدیلی کے ذریعے فشنگ (Phishing)، یا ان ماحولوں میں خاموشی سے ڈیٹا کے اخراج کا سبب بن سکتا ہے جہاں سخت کنٹینٹ سیکیورٹی پالیسیز (Content Security Policies - CSP) بصورت دیگر جاوا اسکرپٹ پر مبنی حملوں (JavaScript-based attacks) کو روک سکتی ہیں۔


| فیلڈ | ویلیو |
|---|---|
| **WSTG ID** | WSTG-CLNT-05 |
| **CWE** | CWE-74 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا |
| **شدت** | درمیانی / زیادہ* |

> *شدت اس صورت میں "زیادہ" (High) ہو جاتی ہے اگر انجیکشن پوائنٹ حساس ایٹریبیوٹس جیسے CSRF ٹوکنز کے اخراج کی اجازت دیتا ہو یا اگر اسے حساس ورک فلوز میں UI ری ڈریسنگ کے لیے استعمال کیا جا سکے۔

### کیا ایپلیکیشن سی ایس ایس (CSS) کے سیاق و سباق میں صارف کے زیرِ کنٹرول ان پٹ کو ظاہر کرتی ہے؟
- [ ] نہیں — ان پٹ اسٹائل ٹیگز، ایٹریبیوٹس، یا CSS فائلوں میں ظاہر نہیں ہوتی  
- [ ] ہاں — ان پٹ ظاہر ہوتی ہے لیکن سختی سے صرف محفوظ حروف اور اعداد (Alphanumeric values) تک محدود ہے  
- [ ] ہاں — ان پٹ `style` ایٹریبیوٹ یا `<style>` بلاک کے اندر ظاہر ہوتی ہے  

### کیا CSS کے مخصوص میٹا کریکٹرز (Metacharacters) اور فنکشنز کو مناسب طریقے سے سینیٹائز کیا گیا ہے؟
- [ ] ہاں — کریکٹرز جیسے کہ `{`, `}`, `:` اور فنکشنز جیسے `url()` کو مناسب طریقے سے اسکیپ (Properly escaped) کیا گیا ہے  
- [ ] ہاں — سینیٹائزیشن موجود ہے لیکن انکوڈنگ یا CSS کمنٹس کے ذریعے اس سے بچنا (Bypass) ممکن ہے  
- [ ] نہیں — CSS میٹا کریکٹرز پر کوئی سینیٹائزیشن لاگو نہیں کی گئی ہے  

### کیا CSS سلیکٹرز کا استعمال کرتے ہوئے ڈیٹا کا اخراج (Exfiltration) ممکن ہے؟
- [ ] نہیں — سلیکٹرز بیرونی ریکوسٹس کو متحرک نہیں کر سکتے اور نہ ہی ڈیٹا لیک کر سکتے ہیں  
- [ ] ہاں — ایٹریبیوٹ سلیکٹرز اور `background-image` کے ذریعے جزوی ڈیٹا کا اخراج ممکن ہے  
- [ ] ہاں — خودکار بروٹ فورس (Brute-force) تکنیکوں کا استعمال کرتے ہوئے حساس ٹوکنز کا مکمل اخراج ممکن ہے  

### کیا کنٹینٹ سیکیورٹی پالیسی (CSP) سی ایس ایس انجیکشن کے خطرے کو کم کرتی ہے؟
- [ ] ہاں — `style-src` صرف 'self' تک محدود ہے اور `img-src` یا `connect-src` پر پابندی عائد ہے  
- [ ] ہاں — CSP موجود ہے لیکن 'unsafe-inline' استعمال کرتی ہے یا وائلڈ کارڈ بیرونی ڈومینز کی اجازت دیتی ہے  
- [ ] نہیں — CSS کے ذریعے بیرونی وسائل لوڈ کرنے سے روکنے کے لیے کوئی CSP نافذ نہیں کی گئی ہے  

### کیا اس کمزوری کو یو آئی ری ڈریسنگ (UI redressing) یا فشنگ (Phishing) کے لیے استعمال کیا جا سکتا ہے؟
- [ ] نہیں — انجیکشن کا دائرہ کار لے آؤٹ میں نمایاں تبدیلی کے لیے بہت محدود ہے  
- [ ] ہاں — یو آئی (UI) میں تبدیلی ممکن ہے، جس سے بدنیتی پر مبنی عناصر (Malicious elements) کو اوپر رکھنے (Overlay) کی اجازت ملتی ہے  
- [ ] ہاں — پورے پیج کے لے آؤٹ پر مکمل کنٹرول حاصل کرنا ممکن ہے، جو انتہائی قائل کرنے والے فشنگ حملوں میں مددگار ثابت ہوتا ہے
