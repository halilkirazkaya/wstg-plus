## WSTG-ERRH-01 — غیر مناسب ایرر ہینڈلنگ (Improper Error Handling) کی جانچ

غیر مناسب ایرر ہینڈلنگ (Improper Error Handling) اس وقت ہوتی ہے جب کوئی ویب ایپلی کیشن (Web Application) اپنے ایرر ریسپانسز (Error Responses) کے ذریعے حساس تکنیکی تفصیلات—جیسے اسٹیک ٹریسز (Stack Traces)، ڈیٹا بیس اسکیمہ (Database Schema) کی معلومات، یا اندرونی فائل پاتھس (Internal File Paths)—ظاہر کرتی ہے۔ حملہ آور (Attackers) غیر درست ان پٹ (Malformed Input) جمع کر کے، غیر موجود ذرائع تک رسائی حاصل کر کے، یا سرور سائیڈ ایکسیپشنز (Server-side Exceptions) پیدا کر کے ان ایررز کو متحرک کرتے ہیں تاکہ ایپلی کیشن کے بنیادی ڈھانچے کا نقشہ تیار کیا جا سکے اور مزید استحصال (Exploitation) کے لیے ممکنہ ویکٹرز کی نشاندہی کی جا سکے۔ معلومات کا یہ اخراج اکثر حملہ آور کو عین مطابق انوائرمنٹ کی تفصیلات فراہم کر کے زیادہ شدید حملوں، بشمول ایس کیو ایل انجیکشن (SQL Injection) یا پاتھ ٹریورسل (Path Traversal)، کے پیش خیمہ کے طور پر کام کرتا ہے۔ ایک محفوظ نفاذ کے لیے ضروری ہے کہ وہ عام صارف دوست پیغامات فراہم کرے جبکہ تفصیلی تشخیص (Diagnostics) کو صرف محفوظ، سرور سائیڈ لاگز میں محفوظ کرے۔

### کیا ایپلی کیشن غیر نمٹائے گئے استثناء (Unhandled Exceptions) کے لیے ایک عمومی، عالمی ایرر ہینڈلر (Global Error Handler) استعمال کرتی ہے؟
- [ ] جی ہاں — عمومی ایرر پیجز (Generic Error Pages) فعال ہیں اور کوئی تکنیکی تفصیل ظاہر نہیں کرتے  
- [ ] جی ہاں — کسٹم ایرر پیجز استعمال کیے گئے ہیں لیکن مخصوص ہیڈرز (Headers) کے ذریعے معلومات کا اخراج (Leakage) ممکن ہے  
- [ ] نہیں — ڈیفالٹ سرور ایرر پیجز (مثلاً Tomcat، IIS، Nginx) نظر آتے ہیں  

### کیا غیر درست ان پٹ (Malformed Input) یا باؤنڈری کیسز (Boundary Cases) کے ذریعے تکنیکی معلومات کا شمار (Enumeration) کیا جا سکتا ہے؟
- [ ] نہیں — ایررز کو بہتر طریقے سے ہینڈل کیا گیا ہے اور لاگنگ کے لیے منفرد ریفرنس IDs فراہم کی گئی ہیں  
- [ ] جی ہاں — ریسپانسز میں اسٹیک ٹریسز (Stack Traces) یا ڈیٹا بیس کوئریز (Database Queries) ظاہر ہوتی ہیں  
- [ ] جی ہاں — اندرونی فائل پاتھس، انوائرمنٹ ویری ایبلز (Environment Variables)، یا سرور ورژن اسٹرنگز ظاہر ہوتی ہیں  

### کیا اے پی آئی (API) ریسپانسز میں تفصیلی ایرر آبجیکٹس (Verbose Error Objects) یا ڈی بگ معلومات کا اخراج ہو رہا ہے؟
- [ ] نہیں — اے پی آئیز معیاری ایرر کوڈز اور محفوظ (Sanitized) JSON پیغامات واپس کرتی ہیں  
- [ ] جی ہاں — اے پی آئیز ریسپانس باڈی میں تفصیلی ڈی بگ آبجیکٹس یا مکمل ایکسیپشن کی تفصیلات واپس کرتی ہیں  
- [ ] جی ہاں — اسٹیک ٹریسز JSON ریسپانس کے `details` یا `exception` فیلڈز میں شامل ہیں  

### کیا ایررز کی صورت میں ایپلی کیشن کا برتاؤ (ٹائم بیسڈ (Time-based) یا کنٹینٹ بیسڈ (Content-based)) مختلف ہوتا ہے؟
- [ ] نہیں — ایرر کی قسم سے قطع نظر ریسپانس سگنیچرز اور ٹائمنگ مستقل رہتی ہے  
- [ ] جی ہاں — مختلف ریسپانسز کو درست بمقابلہ غیر درست حالتوں (مثلاً یوزر اینومریشن (User Enumeration)) کی نشاندہی کے لیے استعمال کیا جا سکتا ہے
