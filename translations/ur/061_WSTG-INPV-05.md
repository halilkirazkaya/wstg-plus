## WSTG-INPV-05 — SQL Injection (ایس کیو ایل انجیکشن)

ایس کیو ایل انجیکشن (SQL Injection) تب ہوتا ہے جب صارف کی فراہم کردہ ان پٹ (user-supplied input) کو مناسب پیرامیٹرائزیشن (parameterization) یا سینیٹائزیشن (sanitization) کے بغیر SQL کیوریز میں شامل کیا جاتا ہے، جس سے حملہ آوروں کو کیوری کی منطق میں ہیرا پھیری کرنے کی اجازت ملتی ہے۔ کامیاب ایکسپلائٹیشن (exploitation) کے نتیجے میں غیر مجاز ڈیٹا ایکس فلٹریشن (data exfiltration)، آتھنٹیکیشن بائی پاس (authentication bypass)، ڈیٹا میں تبدیلی، اور بعض صورتوں میں ڈیٹا بیس فیچرز جیسے کہ `xp_cmdshell` یا `LOAD_FILE()` کے ذریعے ریموٹ کوڈ ایگزیکیوشن (remote code execution) ہو سکتا ہے۔ یہ کمزوری (vulnerability) عام طور پر لاگ ان فارمز، سرچ فنکشنلٹی، REST API پیرامیٹرز، HTTP ہیڈرز، اور کسی بھی ایسے اینڈ پوائنٹ (endpoint) کو متاثر کرتی ہے جو صارف کے زیرِ کنٹرول ان پٹ سے ڈائنامک ایس کیو ایل کیوریز (dynamic SQL queries) بناتا ہے۔ حملہ آور کے نقطہ نظر سے، انجیکشن پوائنٹ کی شناخت کرنا پورے ڈیٹا بیس تک رسائی اور انفراسٹرکچر (infrastructure) کے اندر ممکنہ لیٹرل موومنٹ (lateral movement) کا بنیادی راستہ ہے۔

### کیا صارف کے زیرِ کنٹرول پیرامیٹرز کو محفوظ ڈیٹا بیس رسائی کے طریقوں کا استعمال کرتے ہوئے پروسیس کیا جاتا ہے؟
- [ ] جی ہاں — ایپلی کیشن خصوصی طور پر ORM یا پیرامیٹرائزڈ کیوریز (parameterized queries) استعمال کرتی ہے اور بائی پاس **ممکن نہیں** ہے  
- [ ] جی ہاں — پیرامیٹرائزڈ کیوریز استعمال ہوتی ہیں لیکن مخصوص صورتوں (مثلاً `ORDER BY` کلازز) کے ذریعے بائی پاس **ممکن** ہے  
- [ ] نہیں — پیرامیٹرائزیشن کے بغیر SQL کیوریز بنانے کے لیے را اسٹرنگ کنکیٹینیشن (raw string concatenation) استعمال کی جاتی ہے  

### کیا ایس کیو ایل انجیکشن کے ذریعے آتھنٹیکیشن میکانزم کو بائی پاس کیا جا سکتا ہے؟
- [ ] نہیں — لاگ ان پیرامیٹرز انجیکشن کے لیے **غیر محفوظ نہیں** ہیں  
- [ ] جی ہاں — ٹوٹولوجی پر مبنی پے لوڈز (tautology-based payloads) (مثلاً `' OR 1=1 --`) کا استعمال کرتے ہوئے آتھنٹیکیشن بائی پاس **ممکن** ہے  

### کیا اِن بینڈ (in-band) یا آؤٹ آف بینڈ (out-of-band) تکنیکوں کے ذریعے ڈیٹا ایکس فلٹریشن ممکن ہے؟
- [ ] نہیں — ڈیٹا ایکس فلٹریشن کی شناخت نہیں ہوئی  
- [ ] جی ہاں — UNION-based یا error-based ایکس فلٹریشن **ممکن** ہے  
- [ ] جی ہاں — بلائنڈ (Boolean یا Time-based) ایکس فلٹریشن **ممکن** ہے  
- [ ] جی ہاں — آؤٹ آف بینڈ (DNS/HTTP) ایکس فلٹریشن **ممکن** ہے  

### کیا ایپلی کیشن کے فنکشنز سیکنڈ آرڈر (second-order) ایس کیو ایل انجیکشن کی کمزوریاں ظاہر کرتے ہیں؟
- [ ] نہیں — اسٹور شدہ ڈیٹا کو بعد کی کیوریز کے لیے نکالتے وقت محفوظ طریقے سے ہینڈل کیا جاتا ہے  
- [ ] جی ہاں — صارف کا ان پٹ اسٹور کیا جاتا ہے اور بعد میں بغیر کسی توثیق (validation) کے غیر محفوظ SQL کیوری میں استعمال کیا جاتا ہے  

### کیا ڈیٹا بیس سروس اکاؤنٹ کی مراعات مطلوبہ حد تک کم سے کم رکھی گئی ہیں؟
- [ ] جی ہاں — سروس اکاؤنٹ کے پاس **کم سے کم مراعات** (least privilege) ہیں (مثلاً صرف مخصوص ٹیبلز/ایکشنز تک محدود ہے)  
- [ ] نہیں — سروس اکاؤنٹ کے پاس اعلیٰ مراعات ہیں (مثلاً `DROP TABLE` ،`FILE` کے حقوق، یا `xp_cmdshell` **فعال** ہے)
