## WSTG-CRYP-01 — کمزور ٹرانسپورٹ لیئر سیکیورٹی کی جانچ (Testing for Weak Transport Layer Security)

کمزور ٹرانسپورٹ لیئر سیکیورٹی (Transport Layer Security - TLS) کی جانچ میں SSL/TLS کے نفاذ اور کنفیگریشن کا تجزیہ شامل ہے تاکہ ٹرانزٹ کے دوران ڈیٹا کی رازداری (Confidentiality) اور سالمیت (Integrity) کو یقینی بنایا جا سکے۔ حملہ آور مین ان دی مڈل (Man-in-the-Middle - MitM) حملے کرنے کے لیے غلط کنفیگریشنز جیسے کہ فرسودہ پروٹوکولز (مثلاً SSLv2، TLS 1.0)، کمزور سائفر سویٹس (Cipher Suites) (جیسے NULL، RC4، یا گمنام)، اور غلط یا کمزور طور پر سائن کیے گئے سرٹیفکیٹس کو نشانہ بناتے ہیں۔ یہ ٹیسٹ عام طور پر ایپلی کیشن کے ویب سرور یا لوڈ بیلنسر اینڈ پوائنٹس (Endpoints) کو نشانہ بناتا ہے جہاں انکرپٹڈ مواصلات قائم کیے جاتے ہیں۔ کامیاب استحصال (Exploitation) کے ذریعے ایک حملہ آور حساس ڈیٹا کو درمیان میں روک سکتا ہے، صارف کے سیشنز کو ہائی جیک کر سکتا ہے، یا پروٹوکول ڈاؤن گریڈ (Protocol Downgrade) یا ٹریفک ڈی کرپشن (Decryption) کے ذریعے کنکشنز کو غیر محفوظ حالتوں میں لا سکتا ہے۔


| فیلڈ | ویلیو |
|---|---|
| **WSTG ID** | WSTG-CRYP-01 |
| **CWE** | CWE-326 |
| **ٹیسٹ کی صورتحال** | انجام نہیں دیا گیا |
| **شدت** | زیادہ / درمیانی* |

> *شدت "زیادہ" (High) ہو گی اگر حساس ڈیٹا (PII، اسناد (Credentials)، سیشن ٹوکنز) منتقل کیا جا رہا ہو؛ عام معلوماتی ٹریفک کے لیے "درمیانی" (Medium)۔

### کیا فرسودہ یا غیر محفوظ TLS/SSL پروٹوکولز سپورٹڈ ہیں؟
- [ ] نہیں — صرف TLS 1.2 اور TLS 1.3 **فعال (Enabled)** ہیں  
- [ ] جی ہاں — TLS 1.0 یا TLS 1.1 **فعال** ہیں  
- [ ] جی ہاں — SSLv2 یا SSLv3 **فعال** ہیں *(انتہائی تشویشناک - Critical)*  

### کیا سرور کے ذریعے کمزور یا غیر محفوظ سائفر سویٹس قبول کیے جاتے ہیں؟
- [ ] نہیں — صرف مضبوط اور جدید سائفرز (مثلاً AES-GCM، ChaCha20) **سپورٹڈ** ہیں  
- [ ] جی ہاں — کمزور انکرپشن والے سائفرز (مثلاً 3DES، RC4) یا کم بٹ لینتھ (Bit-length) والے سائفرز **سپورٹڈ** ہیں  
- [ ] جی ہاں — NULL، گمنام (ADH)، یا ایکسپورٹ سائفرز **سپورٹڈ** ہیں *(انتہائی تشویشناک)*  

### کیا سرور سرٹیفکیٹ درست اور قابل اعتماد ہے؟
- [ ] جی ہاں — سرٹیفکیٹ درست ہے، ڈومین سے مطابقت رکھتا ہے، اور ایک **قابل اعتماد CA** سے سائن شدہ ہے  
- [ ] نہیں — سرٹیفکیٹ کی **میعاد ختم (Expired)** ہو چکی ہے یا یہ **کمزور سگنیچر ایلگوریدم** (مثلاً SHA-1) استعمال کرتا ہے  
- [ ] نہیں — سرٹیفکیٹ **سیلف سائنڈ (Self-signed)** ہے یا ڈومین نام کی **عدم مطابقت (Mismatch)** پائی جاتی ہے  

### کیا سرور محفوظ ری نیگوشی ایشن (Secure Renegotiation) کو سپورٹ کرتا ہے اور ڈاؤن گریڈ حملوں کو روکتا ہے؟
- [ ] جی ہاں — محفوظ ری نیگوشی ایشن **سپورٹڈ** ہے اور TLS Fallback SCSV **فعال** ہے  
- [ ] نہیں — محفوظ ری نیگوشی ایشن **سپورٹڈ نہیں** ہے، جو ممکنہ طور پر MitM انجکشن کی اجازت دے سکتا ہے  
- [ ] نہیں — سرور **POODLE** یا **ROBOT** حملوں کا شکار ہو سکتا ہے  

### کیا HTTP سٹرکٹ ٹرانسپورٹ سیکیورٹی (HSTS) مناسب طریقے سے نافذ ہے؟

| ہیڈر | موجود | غائب |
|--------|:-------:|:-------:|
| `Strict-Transport-Security` |  |  |

- [ ] جی ہاں — HSTS ہیڈر طویل `max-age` اور `includeSubDomains` کے **فعال** ہونے کے ساتھ موجود ہے  
- [ ] جی ہاں — HSTS ہیڈر موجود ہے لیکن `includeSubDomains` غائب ہے یا اس کی **max-age کم** ہے  
- [ ] نہیں — HSTS ہیڈر **موجود نہیں** ہے
