## WSTG-INPV-07 — XML Injection (XML انجیکشن)

ایکس ایم ایل انجیکشن (XML Injection) اس وقت ہوتا ہے جب کوئی ایپلیکیشن صارف کے فراہم کردہ ڈیٹا کو XML دستاویز یا اسٹریم میں شامل کرتی ہے جبکہ XML میٹا کریکٹرز (Meta-characters) کی درست طریقے سے توثیق (Validation)، سینیٹائزیشن (Sanitization)، یا اسکیپنگ (Escaping) نہیں کی جاتی۔ مخصوص XML ٹیگز یا ساختی عناصر انجیکٹ کر کے، ایک حملہ آور (Attacker) دستاویز کی منطق کو تبدیل کر سکتا ہے، تصدیقی میکانزم (Authentication mechanisms) کو بائی پاس کر سکتا ہے، یا بیک اینڈ (Backend) کے ذریعے پروسیس کیے جانے والے ڈیٹا کی سالمیت (Integrity) میں مداخلت کر سکتا ہے۔ یہ کمزوری (Vulnerability) عام طور پر SOAP پر مبنی ویب سروسز، REST APIs جو XML قبول کرتے ہیں، SAML assertions، اور ایسی ایپلیکیشنز میں پائی جاتی ہے جو متحرک طور پر XML کنفیگریشن فائلز یا رپورٹیں تیار کرتی ہیں۔ ایک حملہ آور کے نقطہ نظر سے، مقصد مطلوبہ ڈیٹا کانٹیکسٹ (Data context) سے باہر نکل کر دستاویز کے ڈھانچے کو تبدیل کرنا ہے، جو ممکنہ طور پر غیر مجاز استحقاق میں اضافے (Privilege Escalation) یا غیر ارادی بزنس لاجک (Business Logic) کے نفاذ کا باعث بن سکتا ہے۔

### کیا ایپلیکیشن صارف سے XML فارمیٹ میں ان پٹ قبول اور پروسیس کرتی ہے؟
- [ ] نہیں — ایپلیکیشن XML ان پٹ کو پروسیس **نہیں** کرتی  
- [ ] ہاں — XML ان پٹ APIs (SOAP/REST) کے ذریعے پروسیس کیا جاتا ہے  
- [ ] ہاں — XML فائل اپ لوڈز (SVG, DOCX وغیرہ) کے ذریعے پروسیس کیا جاتا ہے  

### کیا XML میٹا کریکٹرز (مثلاً `<`, `>`, `&`, `'`, `"`) کو صحیح طریقے سے اسکیپ یا سینیٹائز کیا گیا ہے؟
- [ ] ہاں — تمام میٹا کریکٹرز کو **درست** طریقے سے اسکیپ یا سینیٹائز کیا گیا ہے *(سب سے زیادہ محفوظ)*  
- [ ] ہاں — کچھ فلٹرنگ لاگو ہے لیکن انکوڈنگ کا استعمال کرتے ہوئے بائی پاس (Bypass) **ممکن ہے**  
- [ ] نہیں — خام ان پٹ کو بغیر کسی سینیٹائزیشن کے براہ راست XML ڈھانچے میں شامل کیا جاتا ہے  

### کیا تمام XML ان پٹس کے لیے سرور سائیڈ سکیما توثیق (Schema Validation - XSD/DTD) نافذ ہے؟
- [ ] ہاں — سخت سکیما توثیق **فعال** ہے اور ساختی تبدیلیوں کو روکتی ہے  
- [ ] ہاں — توثیق **فعال** ہے لیکن ڈیٹا فیلڈز کے اندر ٹیگ انجیکشن کو **نہیں** روکتی  
- [ ] نہیں — سکیما توثیق **غیر فعال** ہے یا نافذ نہیں کی گئی  

### کیا ایک حملہ آور دستاویز کے ڈھانچے یا منطق کو تبدیل کرنے کے لیے نئے XML ٹیگز انجیکٹ کر سکتا ہے؟
- [ ] نہیں — ان پٹ سے قطع نظر ڈھانچہ برقرار رہتا ہے  
- [ ] ہاں — ٹیگز انجیکٹ کیے جا سکتے ہیں لیکن بزنس لاجک کو تبدیل **نہیں** کر سکتے  
- [ ] ہاں — ٹیگ انجیکشن **ممکن ہے** اور لاجک بائی پاس یا ڈیٹا میں تبدیلی کی اجازت دیتا ہے *(انتہائی سنگین)*  

### کیا CDATA سیکشنز یا XML تبصروں (Comments) کا استعمال کرتے ہوئے XML پارسر (Parser) کے ساتھ ہیرا پھیری کی جا سکتی ہے؟
- [ ] نہیں — پارسر صحیح طریقے سے CDATA/تبصرہ مارکرز کو ہینڈل کرتا ہے یا مسترد کر دیتا ہے  
- [ ] ہاں — فلٹرز کو چھپانے یا بائی پاس کرنے کے لیے `<![CDATA[...]]>` یا `<!-- ... -->` کا انجیکشن **ممکن ہے**
