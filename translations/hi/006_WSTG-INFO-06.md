## WSTG-INFO-06 — एप्लिकेशन एंट्री पॉइंट्स की पहचान करना (Identify Application Entry Points)

एप्लिकेशन एंट्री पॉइंट्स (Application Entry Points) की पहचान करने में उन सभी संभावित तरीकों की मैपिंग करके पूरे अटैक सरफेस (Attack Surface) का विवरण तैयार करना शामिल है जिनके माध्यम से एक उपयोगकर्ता या बाहरी सिस्टम एप्लिकेशन के साथ इंटरैक्ट कर सकता है। इस प्रक्रिया में सभी URL, API एंडपॉइंट्स (API Endpoints), पैरामीटर्स (GET, POST, कुकी-आधारित), HTTP हेडर (HTTP Headers), और वेबसोकेट्स (WebSockets) या gRPC जैसे विशेष प्रोटोकॉल की खोज करना शामिल है। एक पेनेट्रेशन टेस्टर (Penetration Tester) के लिए, यह चरण अत्यंत महत्वपूर्ण है क्योंकि अक्सर अनियंत्रित "शैडो" API (Shadow APIs), लेगेसी एंडपॉइंट्स (Legacy Endpoints), या जटिल पैरामीटर संरचनाओं में कमजोरियां (Vulnerabilities) पाई जाती हैं जिन्हें विकास (Development) के दौरान नजरअंदाज कर दिया गया था। विस्तृत गणना (Enumeration) यह सुनिश्चित करती है कि एप्लिकेशन का कोई भी घटक एक बिना परीक्षण वाला ब्लैक बॉक्स (Black Box) न रहे, जिससे हमलावरों को सिस्टम में बिना निगरानी वाले रास्ते खोजने से रोका जा सके।

### क्या पूरे एप्लिकेशन में सभी GET और POST पैरामीटर्स की गणना (Enumeration) की गई है?
- [ ] हाँ — स्वचालित और मैन्युअल क्रॉलिंग के माध्यम से व्यापक गणना **पूरी हो चुकी है**
- [ ] हाँ — मानक क्रॉलिंग के माध्यम से केवल सामान्य पैरामीटर्स की पहचान की गई है
- [ ] नहीं — पैरामीटर्स काफी हद तक **अज्ञात** या बिना परीक्षण के हैं

### क्या फज़िंग (Fuzzing) का उपयोग करके अघोषित या छिपे हुए पैरामीटर्स (जैसे, debug, admin) की खोज की गई है?
- [ ] नहीं — इस विशिष्ट कार्यक्षेत्र (Scope) के लिए छिपे हुए पैरामीटर की खोज **आवश्यक नहीं है**
- [ ] हाँ — पैरामीटर फज़िंग (जैसे, `Arjun` का उपयोग करके) **किया गया है** और कोई परिणाम नहीं मिला
- [ ] हाँ — छिपे हुए पैरामीटर्स **खोजे गए** और आगे के परीक्षण के लिए मैप किए गए
- [ ] नहीं — छिपे हुए पैरामीटर की खोज **नहीं की गई**

### क्या प्रत्येक एंडपॉइंट के लिए सभी समर्थित HTTP मेथड्स (PUT, DELETE, PATCH, आदि) की पहचान की गई है?
- [ ] हाँ — मेथड डिस्कवरी (Method Discovery) सभी खोजे गए एंडपॉइंट्स पर **लागू की गई है**
- [ ] हाँ — मेथड डिस्कवरी केवल उच्च-रुचि या संवेदनशील एंडपॉइंट्स पर **लागू की गई है**
- [ ] नहीं — केवल मानक GET और POST मेथड्स की **पहचान की गई है**

### क्या वेबसोकेट्स (WebSockets), gRPC, या कस्टम हेडर जैसे गैर-मानक एंट्री पॉइंट्स मैप किए गए हैं?
- [ ] नहीं — एप्लिकेशन गैर-मानक प्रोटोकॉल या कस्टम हेडर का **उपयोग नहीं करता है**
- [ ] हाँ — सभी प्रोटोकॉल-विशिष्ट एंट्री पॉइंट्स और कस्टम हेडर की **पहचान की गई है**
- [ ] नहीं — गैर-मानक एंट्री पॉइंट्स **मौजूद हैं** लेकिन उन्हें मैप नहीं किया गया है

### क्या API सरफेस (वर्जन वाले एंडपॉइंट्स जैसे /v1/ या /v2/ सहित) को पूरी तरह से मैप किया गया है?
- [ ] नहीं — एप्लिकेशन कोई API **एक्सपोज़ नहीं करता है**
- [ ] हाँ — सभी API वर्जन और एंडपॉइंट्स की **पहचान की गई है** और उन्हें प्रलेखित (Documented) किया गया है
- [ ] हाँ — केवल वर्तमान प्रोडक्शन API वर्जन की **पहचान की गई है**
- [ ] नहीं — API एंडपॉइंट्स **अनमैप्ड** हैं या केवल आंशिक रूप से खोजे गए हैं
