## WSTG-INPV-19 — सर्वर-साइड रिक्वेस्ट फोर्जिंग (Server-Side Request Forgery - SSRF) के लिए परीक्षण

सर्वर-साइड रिक्वेस्ट फोर्जिंग (SSRF) तब होती है जब कोई हमलावर (attacker) वेब एप्लिकेशन को सर्वर से आंतरिक (internal) या बाहरी (external) संसाधनों के लिए अनधिकृत अनुरोध करने के लिए प्रभावित कर सकता है। URL, होस्टनाम (hostnames), या IP एड्रेस की अपेक्षा करने वाले मापदंडों (parameters) में हेरफेर करके, एक हमलावर नेटवर्क-स्तर के नियंत्रणों जैसे कि फायरवॉल (firewalls) और ACL को दरकिनार (bypass) कर सकता है ताकि आंतरिक नेटवर्क सेगमेंट की जांच की जा सके, क्लाउड मेटाडेटा सेवाओं (IMDS) तक पहुँचा जा सके, या असुरक्षित आंतरिक API और डेटाबेस के साथ इंटरैक्ट किया जा सके। यह भेद्यता (vulnerability) आमतौर पर इमेज फेचिंग (image fetching), PDF जनरेशन (PDF generation), वेबहुक (webhooks), या URL के माध्यम से फ़ाइल अपलोड जैसी सुविधाओं में प्रकट होती है। एक हमलावर के दृष्टिकोण से, SSRF एक शक्तिशाली प्रिमिटिव (primitive) है जिसका उपयोग आइसोलेटेड वातावरण में जाने (pivot), संवेदनशील कॉन्फ़िगरेशन डेटा को निकालने (exfiltrate), या संभावित रूप से Redis या Memcached जैसी आंतरिक सेवाओं के साथ बातचीत के माध्यम से रिमोट कोड एक्जीक्यूशन (Remote Code Execution - RCE) प्राप्त करने के लिए किया जा सकता है।

### क्या एप्लिकेशन उपयोगकर्ता द्वारा आपूर्ति किए गए URL या होस्टनाम को प्रोसेस करता है?
- [ ] नहीं — कोई भी सुविधा बाहरी URL या होस्टनाम स्वीकार नहीं करती है  
- [ ] हाँ — सुविधा मौजूद है लेकिन इनपुट को एक अनुमति-सूची (allow-list) के विरुद्ध कड़ाई से मान्य (strictly validated) किया जाता है  
- [ ] हाँ — सुविधा मौजूद है और उपयोगकर्ता द्वारा आपूर्ति किए गए URL स्वीकार करती है  

### क्या अनुरोधित गंतव्य (destination) पर सत्यापन नियंत्रण या ब्लैकलिस्ट लागू की गई हैं?
- [ ] हाँ — विश्वसनीय डोमेन/IP की एक सख्त अनुमति-सूची (allow-list) लागू की गई है  
- [ ] हाँ — एक डेनाइ-लिस्ट (deny-list) का उपयोग किया जाता है (जैसे, `127.0.0.1`, `169.254.169.254` को ब्लॉक करना)  
- [ ] नहीं — इनपुट पर कोई सत्यापन (validation) या फ़िल्टरिंग (filtering) लागू नहीं की गई है  

### क्या एन्कोडिंग, रीडायरेक्ट, या DNS रिबाइंडिंग के माध्यम से फ़िल्टर को बायपास करना संभव है?
- [ ] नहीं — फ़िल्टर मजबूत हैं और रीडायरेक्ट (redirects)/DNS रेजोल्यूशन को सुरक्षित रूप से संभालते हैं  
- [ ] हाँ — URL एन्कोडिंग (encoding) या वैकल्पिक IP स्वरूपों (hex, octal) का उपयोग करके बायपास संभव है  
- [ ] हाँ — आंतरिक लक्ष्यों के लिए 3xx HTTP रीडायरेक्ट के माध्यम से बायपास संभव है  
- [ ] हाँ — आंतरिक IP पर रिज़ॉल्यूशन के लिए DNS रिबाइंडिंग (DNS rebinding) हमलों के माध्यम से बायपास संभव है  

### क्या एप्लिकेशन आंतरिक मेटाडेटा सेवाओं या स्थानीय इंटरफेस तक पहुँच सकता?
- [ ] नहीं — स्थानीय नेटवर्क और क्लाउड मेटाडेटा एंडपॉइंट पहुँच योग्य नहीं हैं  
- [ ] हाँ — लोकलहोस्ट (`127.0.0.1`) या लूपबैक सेवाओं तक पहुँच संभव है  
- [ ] हाँ — क्लाउड मेटाडेटा सेवाओं (जैसे, AWS/GCP/Azure IMDS) तक पहुँच संभव है *(क्रिटिकल)*  

### SSRF प्रतिक्रिया (response) की प्रकृति क्या है?
- [ ] ब्लाइंड (Blind) — उपयोगकर्ता को कोई प्रतिक्रिया या मेटाडेटा वापस नहीं किया जाता है  
- [ ] आंशिक (Partial) — प्रतिक्रिया मेटाडेटा (headers, size, timing) उपयोगकर्ता को वापस किया जाता है  
- [ ] पूर्ण (Full) — आंतरिक संसाधन से प्राप्त पूर्ण प्रतिक्रिया मुख्य भाग (response body) उपयोगकर्ता को दिखाई (reflected) देता है
