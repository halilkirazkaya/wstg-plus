## WSTG-APIT-02 — एपीआई ब्रोकन ऑब्जेक्ट लेवल ऑथराइजेशन (API Broken Object Level Authorization)

ब्रोकन ऑब्जेक्ट लेवल ऑथराइजेशन (Broken Object Level Authorization - BOLA), जिसे इनसिक्योर डायरेक्ट ऑब्जेक्ट रेफरेंस (Insecure Direct Object Reference - IDOR) के रूप में भी जाना जाता है, तब होता है जब कोई एपीआई (API) यह सत्यापित करने में विफल रहता है कि उपयोगकर्ता के पास आईडी (ID) द्वारा पहचाने गए किसी विशिष्ट संसाधन (Resource) तक पहुँचने या उसमें हेरफेर करने की उचित अनुमति है या नहीं। हमलावर अनुरोध पथ (Request paths), क्वेरी मापदंडों (Query parameters), या JSON बॉडी (JSON body) में पहचानकर्ताओं—जैसे संख्यात्मक आईडी या UUID—की व्यवस्थित रूप से गणना या अनुमान लगाकर अन्य उपयोगकर्ताओं के डेटा तक पहुँच प्राप्त करके इसका फायदा उठाते हैं। यह भेद्यता आधुनिक API सुरक्षा में सबसे आम और प्रभावशाली समस्या है, जो संभावित रूप से बड़े पैमाने पर डेटा चोरी (Data exfiltration), अनधिकृत संशोधन, या पूर्ण अकाउंट टेकओवर (Account takeover) का कारण बन सकती है। हमलावर के दृष्टिकोण से, लक्ष्य उन एंडपॉइंट्स (Endpoints) की पहचान करना है जो ऑब्जेक्ट पहचानकर्ताओं को स्वीकार करते हैं और यह परीक्षण करना है कि क्या सर्वर-साइड प्रमाणीकरण तर्क (Authorization logic) गायब है या उन ID में हेरफेर करके उसे बायपास (Bypass) किया जा सकता है।

### क्या एपीआई अनुरोधों के भीतर ऑब्जेक्ट पहचानकर्ता (Object identifiers) पूर्वानुमेय या गणना योग्य हैं?
- [ ] नहीं — पहचानकर्ता लंबे, यादृच्छिक (Random) और क्रिप्टोग्राफिक रूप से सुरक्षित हैं (जैसे, UUIDv4)  
- [ ] हाँ — पहचानकर्ता क्रमिक पूर्णांक (Sequential integers) हैं (जैसे, `101`, `102`)  
- [ ] हाँ — पहचानकर्ता एक पूर्वानुमेय पैटर्न का पालन करते हैं या सार्वजनिक जानकारी से प्राप्त होते हैं  

### क्या एपीआई प्रत्येक अनुरोध के लिए ऑब्जेक्ट स्वामित्व का सर्वर-साइड सत्यापन (Server-side validation) करता है?
- [ ] हाँ — प्रत्येक अनुरोध के लिए प्रमाणीकरण जांच लागू की जाती है और बायपास **संभव नहीं है**  
- [ ] हाँ — जांच लागू की जाती है लेकिन पैरामीटर पॉल्यूशन (Parameter pollution) या मेथड टनलिंग (Method tunneling) के माध्यम से बायपास **संभव है**  
- [ ] नहीं — एप्लिकेशन विशिष्ट संसाधन स्वामित्व की जांच किए बिना केवल उपयोगकर्ता के प्रमाणित होने पर निर्भर करता है  

### क्या पहचानकर्ता को बदलकर किसी अन्य उपयोगकर्ता के संसाधन तक पहुँचना या उसे संशोधित करना संभव है?
- [ ] नहीं — अन्य उपयोगकर्ताओं के संसाधनों तक अनधिकृत पहुँच **संभव नहीं है**  
- [ ] हाँ — अनधिकृत **पहुँच/पढ़ना** (Horizontal BOLA) **संभव है**  
- [ ] हाँ — अनधिकृत **संशोधन** या **हटाना** (Horizontal BOLA) **संभव है**  

### क्या एपीआई आईडी (ID) को बदलकर प्रशासनिक या सिस्टम-स्तरीय ऑब्जेक्ट्स तक पहुँचने की अनुमति देता है?
- [ ] नहीं — प्रशासनिक संसाधन माध्यमिक प्रमाणीकरण परतों (Secondary authorization layers) द्वारा सुरक्षित हैं  
- [ ] हाँ — सिस्टम-स्तरीय संसाधनों तक पहुँचना या उनमें संशोधन करना (Vertical BOLA) **संभव है**  

### क्या पहचानकर्ता को अनुरोध के किसी दूसरे हिस्से में ले जाकर प्रमाणीकरण जांच को बायपास किया जा सकता है?
- [ ] नहीं — पैरामीटर के स्थान की परवाह किए बिना प्रमाणीकरण तर्क सुसंगत है  
- [ ] हाँ — बायपास **संभव है** जब ID को URL पथ से हटाकर JSON बॉडी या हेडर (Headers) में ले जाया जाता है  
- [ ] हाँ — बायपास **संभव है** जब कई ID प्रदान की जाती हैं और सर्वर अनधिकृत ID को प्रोसेस करता है
