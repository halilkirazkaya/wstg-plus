## WSTG-CONF-04 — संवेदनशील जानकारी के लिए पुराने बैकअप और असंदर्भित फ़ाइलों की समीक्षा करें (Review Old Backup and Unreferenced Files for Sensitive Information)

पुराने बैकअप (Backup) और असंदर्भित फ़ाइलों (Unreferenced Files) की समीक्षा करने में वेब सर्वर पर भूली हुई, अस्थायी या छिपी हुई फ़ाइलों की पहचान करना शामिल है जो सार्वजनिक पहुँच के लिए नहीं हैं। इन फ़ाइलों में अक्सर सोर्स कोड बैकअप (Source code backups) जैसे कि `.zip`, `.bak`, टेक्स्ट एडिटर स्वैप फ़ाइलें (Text editor swap files) जैसे कि `.swp`, `~`, या सोर्स कंट्रोल मेटाडेटा (Source control metadata) जैसे कि `.git`, `.svn` शामिल होते हैं जो संवेदनशील जानकारी लीक कर सकते हैं। हमलावर (Attackers) डेटाबेस क्रेडेंशियल्स (Database credentials), हार्डकोडेड API keys, या लॉजिक जो आगे के शोषण (Exploitation) को सुविधाजनक बनाता है, उन्हें निकालने के उद्देश्य से सामान्य नामकरण परंपराओं को ब्रूट-फ़ोर्स (Brute Force) करने के लिए स्वचालित वर्डलिस्ट (Wordlists) और डिस्कवरी टूल्स (Discovery tools) का उपयोग करते हैं। यह भेद्यता (Vulnerability) आमतौर पर मैन्युअल सर्वर रखरखाव या त्रुटिपूर्ण CI/CD पाइपलाइनों (Pipelines) से उत्पन्न होती है जो प्रोडक्शन वेब रूट (Production web root) को सैनिटाइज़ (Sanitize) करने में विफल रहती हैं।

### क्या वेब सर्वर पर डायरेक्टरी लिस्टिंग (Directory listing) सक्षम है?
- [ ] नहीं — डायरेक्टरी लिस्टिंग अक्षम (Disabled) है और 403 Forbidden या कस्टम त्रुटि लौटाती है  
- [ ] हाँ — गैर-संवेदनशील निर्देशिकाओं (Directories) पर डायरेक्टरी लिस्टिंग सक्षम है  
- [ ] हाँ — सोर्स कोड या संवेदनशील फ़ाइलों वाली निर्देशिकाओं पर डायरेक्टरी लिस्टिंग सक्षम है *(गंभीर/Critical)*  

### क्या सामान्य एक्सटेंशन (जैसे कि .bak, .old, .save) वाली बैकअप फ़ाइलें खोजने योग्य हैं?
- [ ] नहीं — सामान्य बैकअप एक्सटेंशन नहीं मिले या सर्वर कॉन्फ़िगरेशन द्वारा ब्लॉक किए गए हैं  
- [ ] हाँ — बैकअप फ़ाइलें मौजूद हैं लेकिन उनमें संवेदनशील जानकारी नहीं है  
- [ ] हाँ — कॉन्फ़िगरेशन या सोर्स कोड के लिए बैकअप फ़ाइलें सुलभ हैं *(उच्च/High)*  

### क्या सोर्स कंट्रोल डायरेक्टरी (जैसे कि .git, .svn) या मेटाडेटा उजागर (Exposed) हैं?
- [ ] नहीं — सोर्स कंट्रोल मेटाडेटा मौजूद नहीं है या सही ढंग से ब्लॉक किया गया है  
- [ ] हाँ — मेटाडेटा मौजूद है लेकिन पूर्ण रिपॉजिटरी (Repository) को पुनर्गठित नहीं किया जा सकता है  
- [ ] हाँ — उजागर मेटाडेटा के माध्यम से संपूर्ण सोर्स कोड रिपॉजिटरी को निकाला (Exfiltrate) जा सकता है  

### क्या वेब रूट में एप्लिकेशन के कंप्रेस्ड आर्काइव्स (Compressed archives) (जैसे कि .zip, .tar.gz, .rar) मौजूद हैं?
- [ ] नहीं — ब्रूट-फ़ोर्स या इन्यूमरेशन (Enumeration) के माध्यम से कोई संवेदनशील आर्काइव फ़ाइलें नहीं मिलीं  
- [ ] हाँ — आर्काइव्स मिले हैं लेकिन वे पासवर्ड से सुरक्षित हैं या उनमें सार्वजनिक संपत्तियां (Public assets) हैं  
- [ ] हाँ — एप्लिकेशन सोर्स या डेटा वाले अनएन्क्रिप्टेड आर्काइव्स (Unencrypted archives) सार्वजनिक रूप से सुलभ हैं  

### क्या एप्लिकेशन टेक्स्ट एडिटर या IDE द्वारा बनाई गई अस्थायी फ़ाइलों (Temporary files) को लीक करता है?
- [ ] नहीं — `.swp`, `~`, या `.DS_Store` जैसी अस्थायी फ़ाइलें मौजूद नहीं हैं  
- [ ] हाँ — गैर-संवेदनशील अस्थायी फ़ाइलें मौजूद हैं  
- [ ] हाँ — अस्थायी फ़ाइलें सोर्स कोड के स्निपेट्स (Snippets) या आंतरिक पथ (Internal paths) को प्रकट करती हैं
