## WSTG-CRYP-03 — असुरक्षित चैनलों के माध्यम से भेजी गई संवेदनशील जानकारी का परीक्षण (Testing for Sensitive Information Sent Via Unencrypted Channels)

सादे HTTP जैसे असुरक्षित चैनलों (unencrypted channels) के माध्यम से संवेदनशील डेटा (sensitive data) प्रसारित करना, सूचनाओं को मैन-इन-द-मिडल (Man-in-the-Middle - MITM) हमलों के माध्यम से इंटरसेप्शन और संशोधन के प्रति संवेदनशील बनाता है। यह भेद्यता (vulnerability) आमतौर पर तब होती है जब वेब एप्लिकेशन सभी एंडपॉइंट्स (endpoints) पर ट्रांसपोर्ट लेयर सिक्योरिटी (Transport Layer Security - TLS) को लागू करने में विफल होते हैं, विशेष रूप से लेगेसी घटकों (legacy components), लॉगिन फॉर्मों में, या HTTP से HTTPS में प्रारंभिक संक्रमण (transition) के दौरान। एक हमलावर के दृष्टिकोण से, यह सार्वजनिक वाई-फाई जैसे समझौता किए गए या अविश्वसनीय नेटवर्क सेगमेंट पर प्रमाणीकरण क्रेडेंशियल्स (authentication credentials), सेशन टोकन (session tokens) और व्यक्तिगत पहचान योग्य जानकारी (Personally Identifiable Information - PII) की पैसिव स्निफिंग (passive sniffing) की अनुमति देता है। यह सुनिश्चित करना कि सभी संवेदनशील डेटा एक मजबूत TLS टनल के भीतर समाहित है, उपयोगकर्ता इंटरैक्शन की गोपनीयता (confidentiality) और अखंडता (integrity) बनाए रखने और सेशन हाईजैकिंग (session hijacking) को रोकने के लिए मौलिक है।

### क्या एप्लिकेशन संवेदनशील एंडपॉइंट्स के लिए असुरक्षित HTTP पर संचार की अनुमति देता है?
- [ ] नहीं — सभी एप्लिकेशन ट्रैफ़िक को सख्ती से HTTPS पर बाध्य किया जाता है *(सबसे सुरक्षित)*  
- [ ] हाँ — गैर-संवेदनशील पेज HTTP पर सुलभ हैं लेकिन संवेदनशील एंडपॉइंट्स **नहीं हैं**  
- [ ] हाँ — संवेदनशील एंडपॉइंट्स (लॉगिन, चेकआउट, प्रोफाइल) असुरक्षित HTTP पर सुलभ **हैं** *(उच्च जोखिम)*  

### क्या HTTP से HTTPS तक वैश्विक रीडायरेक्शन (global redirection) लागू है?
- [ ] हाँ — एप्लिकेशन सभी HTTP अनुरोधों के लिए HTTPS पर 301/302 रीडायरेक्ट करता है  
- [ ] हाँ — रीडायरेक्शन केवल विशिष्ट संवेदनशील एंडपॉइंट्स के लिए किया जाता है  
- [ ] नहीं — HTTP से HTTPS तक कोई रीडायरेक्शन **लागू नहीं है**  

### क्या HTTP स्ट्रिक्ट ट्रांसपोर्ट सिक्योरिटी (HTTP Strict Transport Security - HSTS) लागू किया गया है?
- [ ] हाँ — HSTS एक लंबी `max-age` के साथ **सक्षम** है और इसमें `includeSubDomains` और `preload` डायरेक्टिव्स शामिल हैं  
- [ ] हाँ — HSTS **सक्षम** है लेकिन इसमें `includeSubDomains` या `preload` डायरेक्टिव्स की कमी है  
- [ ] नहीं — एप्लिकेशन सर्वर पर HSTS **सक्षम नहीं है**

### क्या एप्लिकेशन असुरक्षित चैनलों के माध्यम से तीसरे पक्ष के API (third-party APIs) को संवेदनशील डेटा प्रसारित करता है?
- [ ] नहीं — सभी बाहरी API कॉल एन्क्रिप्टेड (HTTPS) टनल के माध्यम से किए जाते हैं  
- [ ] हाँ — कुछ गैर-संवेदनशील टेलीमेट्री HTTP के माध्यम से भेजी जाती है  
- [ ] हाँ — API की (keys), PII, या क्रेडेंशियल्स असुरक्षित HTTP के माध्यम से तीसरे पक्षों को भेजे जाते **हैं**
