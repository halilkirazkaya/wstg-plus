## WSTG-INPV-07 — XML इंजेक्शन (XML Injection)

XML इंजेक्शन (XML Injection) तब होता है जब एक एप्लिकेशन उपयोगकर्ता द्वारा दिए गए डेटा को उचित सत्यापन (validation), परिशोधन (sanitization), या XML मेटा-कैरेक्टर्स (meta-characters) की एस्केपिंग (escaping) के बिना XML दस्तावेज़ या स्ट्रीम में शामिल करता है। विशिष्ट XML टैग या संरचनात्मक तत्वों को इंजेक्ट करके, एक हमलावर दस्तावेज़ के लॉजिक में हेरफेर कर सकता है, प्रमाणीकरण तंत्र को बाईपास कर सकता है, या बैकएंड द्वारा संसाधित डेटा की अखंडता में हस्तक्षेप कर सकता है। यह भेद्यता (vulnerability) आमतौर पर SOAP-आधारित वेब सेवाओं, XML स्वीकार करने वाले REST API, SAML असर्शन्स (assertions), और उन एप्लिकेशनों में पाई जाती है जो गतिशील रूप से XML कॉन्फ़िगरेशन फ़ाइलें या रिपोर्ट तैयार करते हैं। एक हमलावर के दृष्टिकोण से, लक्ष्य दस्तावेज़ की संरचना को बदलने के लिए इच्छित डेटा संदर्भ से बाहर निकलना होता है, जिससे संभावित रूप से अनधिकृत विशेषाधिकार उन्नयन (privilege escalation) या अनपेक्षित बिजनेस लॉजिक (business logic) का निष्पादन हो सकता है।

### क्या एप्लिकेशन उपयोगकर्ता से XML-फॉर्मेट में इनपुट स्वीकार और संसाधित (process) करता है?
- [ ] नहीं — एप्लिकेशन XML इनपुट को संसाधित **नहीं** करता है  
- [ ] हाँ — XML इनपुट को APIs (SOAP/REST) के माध्यम से संसाधित किया जाता है  
- [ ] हाँ — XML को फ़ाइल अपलोड (SVG, DOCX, आदि) के माध्यम से संसाधित किया जाता है  

### क्या XML मेटा-कैरेक्टर्स (जैसे, `<`, `>`, `&`, `'`, `"`) को ठीक से एस्केप या परिशोधित किया गया है?
- [ ] हाँ — सभी मेटा-कैरेक्टर्स को **उचित रूप से** एस्केप या परिशोधित किया गया है *(सबसे सुरक्षित)*  
- [ ] हाँ — कुछ फ़िल्टरिंग लागू की गई है लेकिन एन्कोडिंग का उपयोग करके बाईपास (bypass) **संभव है**  
- [ ] नहीं — कच्चे इनपुट (raw input) को बिना परिशोधन के सीधे XML संरचनाओं में **एम्बेड** किया गया है  

### क्या सभी XML इनपुट के लिए सर्वर-साइड स्कीमा वैलिडेशन (XSD/DTD) लागू किया गया है?
- [ ] हाँ — सख्त स्कीमा वैलिडेशन **सक्षम (enabled)** है और संरचनात्मक परिवर्तनों को रोकता है  
- [ ] हाँ — वैलिडेशन **सक्षम** है लेकिन डेटा फ़ील्ड के भीतर टैग इंजेक्शन (tag injection) को **नहीं** रोकता है  
- [ ] नहीं — स्कीमा वैलिडेशन **अक्षम (disabled)** है या लागू नहीं किया गया है  

### क्या कोई हमलावर दस्तावेज़ की संरचना या लॉजिक को बदलने के लिए नए XML टैग इंजेक्ट कर सकता है?
- [ ] नहीं — इनपुट के बावजूद संरचना बरकरार रहती है  
- [ ] हाँ — टैग इंजेक्ट किए जा सकते हैं लेकिन बिजनेस लॉजिक को **नहीं** बदल सकते  
- [ ] हाँ — टैग इंजेक्शन **संभव है** और लॉजिक बाईपास या डेटा संशोधन की अनुमति देता है *(महत्वपूर्ण)*  

### क्या XML पार्सर (parser) को CDATA सेक्शन या XML कमेंट्स का उपयोग करके मैनिपुलेट किया जा सकता है?
- [ ] नहीं — पार्सर CDATA/कमेंट मार्कर्स को सही ढंग से संभालता है या अस्वीकार करता है  
- [ ] हाँ — फ़िल्टर को छिपाने या बाईपास करने के लिए `<![CDATA[...]]>` या `<!-- ... -->` का इंजेक्शन **संभव है**
