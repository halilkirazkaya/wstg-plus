## WSTG-ATHZ-01 — Тестирование обхода каталогов и включения файлов (Testing Directory Traversal File Include)

Уязвимости обхода каталогов (Directory Traversal) и включения файлов (File Inclusion) возникают, когда приложение использует контролируемые пользователем входные данные для формирования путей к файлам или каталогам без достаточной валидации (validation) или очистки (sanitization). Злоумышленники эксплуатируют эти недостатки, внедряя такие последовательности, как `../`, для перемещения за пределы целевого каталога, что потенциально позволяет получить доступ к конфиденциальным системным файлам, конфигурационным данным или исходному коду приложения. В более серьезных случаях, связанных с локальным включением файлов (Local File Inclusion, LFI) или удаленным включением файлов (Remote File Inclusion, RFI), злоумышленник может добиться удаленного выполнения кода (Remote Code Execution, RCE) путем включения вредоносных скриптов или использования отравления логов (log poisoning) и PHP-оберток (PHP wrappers). Эти уязвимости обычно находятся в параметрах, используемых для динамической загрузки контента, в шаблонизаторах (template engines) или эндпоинтах (endpoints) получения изображений, где серверная логика обрабатывает пути файловой системы.

### Принимают ли параметры имена файлов или пути для обработки на стороне сервера?
- [ ] Нет — параметры не взаимодействуют с файловой системой  
- [ ] Да — параметры существуют, но используют строгий белый список (allowlist) идентификаторов файлов  
- [ ] Да — параметры принимают прямые имена файлов или пути  

### Очищаются ли входные данные для предотвращения последовательностей обхода каталогов?
- [ ] Да — входные данные проверяются на соответствие строгому белому списку, и последовательности обхода **невозможны**  
- [ ] Да — входные данные очищаются путем удаления последовательностей `../`, но возможен рекурсивный обход (recursive bypass)  
- [ ] Нет — очистка или валидация **не применяются** к входным данным, связанным с путями  

### Возможен ли доступ к файлам за пределами ограниченного каталога с помощью последовательностей обхода?
- [ ] Нет — приложение или ОС предотвращают доступ к файлам вне определенной области  
- [ ] Да — доступ к файлам внутри корневого каталога веб-сервера (web root) **возможен**  
- [ ] Да — доступ к конфиденциальным системным файлам (например, `/etc/passwd`, `C:\Windows\win.ini`) **возможен** *(Критическая)*  

### Позволяет ли сервер включать удаленные URL-адреса (RFI)?
- [ ] Нет — удаленное включение файлов **отключено** на уровне сервера или приложения  
- [ ] Да — удаленные файлы **могут** быть включены, но выполнение **невозможно**  
- [ ] Да — удаленные файлы **могут** быть включены и выполнены на сервере *(Критическая)*  

### Можно ли обойти фильтры с помощью кодирования или специальных символов?
- [ ] Нет — фильтры эффективно обрабатывают различные кодировки и нулевые байты  
- [ ] Да — обход (bypass) **возможен** с использованием URL-кодирования, двойного URL-кодирования или 16-битного Unicode  
- [ ] Да — обход **возможен** с использованием инъекции нулевого байта (null byte injection) (`%00`) или оберток файловой системы (например, `php://filter`)
