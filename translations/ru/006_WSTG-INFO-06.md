## WSTG-INFO-06 — Идентификация точек входа в приложение (Identify Application Entry Points)

Идентификация точек входа в приложение (Identify Application Entry Points) включает в себя составление карты всей поверхности атаки (Attack Surface) путем перечисления всех возможных способов взаимодействия пользователя или внешней системы с приложением. Этот процесс включает в себя обнаружение всех URL-адресов, конечных точек API (API endpoints), параметров (GET, POST, на основе куки (cookie-based)), HTTP-заголовков и специализированных протоколов, таких как WebSockets или gRPC. Для тестировщика на проникновение (penetration tester) этот этап является жизненно важным, так как уязвимости часто обнаруживаются в недокументированных «теневых» API (shadow APIs), устаревших эндпоинтах (legacy endpoints) или сложных структурах параметров, которые были упущены из виду при разработке. Тщательное перечисление гарантирует, что ни один компонент приложения не останется непротестированным «черным ящиком» (black box), тем самым предотвращая поиск злоумышленниками неконтролируемых маршрутов в систему.
