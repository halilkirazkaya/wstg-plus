## WSTG-CONF-09 — Тестирование прав доступа к файлам (Test File Permission)

Тестирование прав доступа к файлам включает в себя аудит уровней контроля доступа (Access Control), назначенных файлам и каталогам на веб-сервере, чтобы убедиться, что конфиденциальные ресурсы не являются избыточно доступными для неавторизованных пользователей или процессов. Неправильно настроенные права доступа могут позволить злоумышленникам читать конфиденциальные файлы конфигурации, содержащие учетные данные базы данных, просматривать проприетарный исходный код или изменять серверные скрипты для выполнения произвольного кода (Remote Code Execution, RCE). Эта уязвимость обычно возникает на этапе развертывания (deployment), когда флаги «доступен для чтения всем» (world-readable) или «доступен для записи всем» (world-writable) по умолчанию остаются в критически важных каталогах приложения, таких как пути конфигурации, папки логов или репозитории загрузок. С точки зрения злоумышленника, обнаружение ненадлежащим образом защищенного файла часто служит основным катализатором для повышения привилегий (Privilege Escalation) или полной компрометации системы.


| Поле | Значение |
|---|---|
| **WSTG ID** | WSTG-CONF-09 |
| **CWE** | CWE-732 |
| **Статус теста** | Не выполнен |
| **Уровень риска** | Средний / Высокий* |

> *Уровень риска становится высоким, если конфиденциальные файлы конфигурации (например, `.env`, `web.config`) доступны для чтения или если возможна запись в корневую директорию веб-сервера (Web Root).

### Защищены ли конфиденциальные файлы конфигурации приложения от несанкционированного доступа?
- [ ] Да — файлы конфигурации (например, `.env`, `config.php`) **недоступны** через веб-запросы  
- [ ] Да — файлы присутствуют, но доступ **ограничен** только авторизованными локальными пользователями  
- [ ] Нет — конфиденциальные файлы конфигурации **доступны** и раскрывают учетные данные или секреты  

### Может ли злоумышленник изменять файлы или каталоги в корневой директории веб-сервера?
- [ ] Нет — все каталоги в корне веб-сервера доступны **только для чтения** пользователю веб-сервера  
- [ ] Да — существуют каталоги с правами на запись, но выполнение скриптов **отключено**  
- [ ] Да — существуют каталоги, доступные для записи всем, и они **разрешают** загрузку и выполнение скриптов *(Критично)*  

### Доступны ли метаданные систем контроля версий или файлы резервных копий из-за недостаточно строгих прав доступа?
- [ ] Нет — `.git`, `.svn` и файлы резервных копий (например, `.bak`, `~`) **отсутствуют** или защищены  
- [ ] Да — каталоги метаданных существуют, но просмотр содержимого директории (Directory Listing) **отключен**  
- [ ] Да — конфиденциальные метаданные или резервные копии **полностью доступны**, что позволяет восстановить исходный код  

### Работает ли пользователь веб-сервера в соответствии с принципом наименьших привилегий?
- [ ] Да — процесс веб-сервера запускается от имени **выделенного непривилегированного** пользователя  
- [ ] Нет — процесс веб-сервера запускается с **избыточными привилегиями** (например, `root` или `Administrator`)  

### Защищены ли лог-файлы от несанкционированного чтения или изменения?
- [ ] Да — доступ к лог-файлам **ограничен** администраторами  
- [ ] Да — лог-файлы доступны для чтения, но **не** содержат токенов сессий или персональных данных (PII)  
- [ ] Нет — лог-файлы **общедоступны для чтения** и раскрывают конфиденциальные данные транзакций или информацию о пользователях
