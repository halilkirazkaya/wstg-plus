## WSTG-BUSL-09 — Тестирование загрузки вредоносных файлов (Test Upload of Malicious Files)

Функциональность загрузки файлов позволяет пользователям отправлять данные на сервер, предоставляя прямой вектор для внедрения вредоносного контента в среду приложения. Злоумышленники используют слабую логику валидации для загрузки веб-шеллов (web shells), вредоносного ПО (malware) или специально сформированных файлов — таких как SVG или HTML — для достижения удаленного выполнения кода (Remote Code Execution, RCE) или межсайтового скриптинга (Cross-Site Scripting, XSS). Эта уязвимость обычно встречается в таких функциях, как настройки профиля, системы управления документами и обработчики вложений, где сервер не может должным образом проверить типы файлов, содержимое и права на выполнение. Успешная эксплуатация часто приводит к полной компрометации системы, эксфильтрации данных (data exfiltration) или использованию сервера в качестве точки перехода (pivot point) для атак на внутреннюю сеть.

### Предоставляет ли приложение функциональность для загрузки файлов?
- [ ] Нет — функциональность загрузки файлов отсутствует  
- [ ] Да — функциональность доступна для аутентифицированных пользователей  
- [ ] Да — функциональность доступна для **неаутентифицированных** пользователей *(Критично)*  

### Применяется ли проверка расширений файлов на стороне сервера?
- [ ] Да — **применяется** строгий белый список (allowlist) расширений, и обход **невозможен**  
- [ ] Да — используется белый список, но обход **возможен** через чувствительность к регистру или двойные расширения  
- [ ] Да — используется черный список (blocklist), который **можно** обойти с помощью альтернативных расширений (например, `.phtml`, `.asa`)  
- [ ] Нет — проверка расширений на стороне сервера **не применяется**  

### Проверяется ли содержимое файла помимо расширения или MIME-типа?
- [ ] Да — приложение проверяет магические байты (magic bytes) и выполняет глубокую инспекцию содержимого  
- [ ] Да — приложение проверяет только заголовок `Content-Type`, который **может** быть легко подделан  
- [ ] Нет — приложение полностью полагается на расширение файла для валидации  

### Хранятся ли загруженные файлы в директории с правами на выполнение?
- [ ] Нет — файлы хранятся в специализированном сервисе хранения (например, S3) или в директории без прав на выполнение  
- [ ] Да — файлы хранятся на веб-сервере, но выполнение **отключено** через конфигурацию  
- [ ] Да — файлы хранятся в директории, доступной через веб, и выполнение **разрешено** *(Критично)*  

### Можно ли обойти фильтры безопасности путем манипуляции именем файла?
- [ ] Нет — имена файлов очищаются или случайным образом перегенерируются сервером  
- [ ] Да — фильтры **можно** обойти, используя нулевые байты (null bytes) (например, `shell.php%00.jpg`)  
- [ ] Да — символы обхода директории (path traversal) (например, `../../`) **могут** быть использованы для загрузки файлов в произвольные директории
