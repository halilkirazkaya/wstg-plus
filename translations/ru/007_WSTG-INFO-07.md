## WSTG-INFO-07 — Картирование путей выполнения в приложении (Map Execution Paths Through Application)

Картирование путей выполнения включает систематическую идентификацию всех достижимых эндпоинтов (endpoints), функциональных потоков и ветвей принятия решений внутри веб-приложения. Этот процесс жизненно важен для обеспечения того, чтобы никакая «скрытая» функциональность — такая как устаревший код, интерфейсы отладки (debug interfaces) или недокументированные маршруты API — не оставалась вне рамок проверки безопасности, так как в них часто отсутствуют современные средства контроля безопасности. Злоумышленники используют сочетание автоматизированного сканирования (spidering) и ручного исследования для визуализации структуры приложения, часто обнаруживая в процессе такие уязвимости, как несанкционированный административный доступ или недостатки бизнес-логики (business logic). Сопоставляя входные данные с конкретными ответами на стороне сервера, тестировщики могут выявлять критическую логику обработки, требующую целевого углубленного тестирования для обеспечения полного покрытия.

### Было ли приложение полностью проиндексировано с использованием автоматизированного и ручного сканирования (crawling)?
- [ ] Да — полное картирование всех видимых и связанных ресурсов **завершено**  
- [ ] Да — автоматизированное сканирование **завершено**, но ручное исследование ожидается  
- [ ] Нет — приложение **не было** проиндексировано  

### Были ли выявлены скрытые или недокументированные эндпоинты с помощью анализа JS или перебора директорий (directory brute-forcing)?
- [ ] Нет — недокументированные пути не обнаружены с помощью анализа по сторонним каналам (side-channel analysis)  
- [ ] Да — скрытые пути обнаружены, но доступ к ним **невозможен** без авторизации  
- [ ] Да — недокументированные эндпоинты **доступны** и предоставляют критическую функциональность *(Critical / High)*  

### Можно ли изменить пути выполнения с помощью нестандартных параметров или заголовков (headers)?
- [ ] Нет — такие параметры, как `debug`, `test` или `admin`, **не влияют** на поток выполнения  
- [ ] Да — параметры изменяют вывод, но **не позволяют** обойти механизмы контроля безопасности  
- [ ] Да — применяются параметры, изменяющие путь, которые **позволяют** обходить намеченную логику  

### Полностью ли закартирован поток многоэтапной бизнес-логики (например, многофакторная аутентификация, оформление заказа)?
- [ ] Да — все состояния и переходы в многоэтапных процессах **идентифицированы**  
- [ ] Нет — переходы в сложных конечных автоматах (state machine) **не могут** быть полностью закартированы  

### Доступны ли файлы документации API (Swagger/OpenAPI) или карты клиентской части (Source Maps)?
- [ ] Нет — архитектурные карты или файлы документации не находятся в открытом доступе  
- [ ] Да — документация присутствует, но **защищена** аутентификацией  
- [ ] Да — Swagger UI, OpenAPI JSON или JS Source Maps **включены** и общедоступны
