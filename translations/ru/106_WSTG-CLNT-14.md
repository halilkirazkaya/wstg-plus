## WSTG-CLNT-14 — Тестирование Обратного Табнаббинга (Testing for Reverse Tabnabbing)

Обратный Табнаббинг (Reverse Tabnabbing) — это уязвимость на стороне клиента (Client-side vulnerability), при которой страница, открытая через `target="_blank"`, сохраняет функциональную ссылку на родительскую страницу через объект `window.opener`. Это позволяет контролируемому злоумышленником целевому сайту программно перенаправить исходную доверенную вкладку на вредоносный URL-адрес, обычно на фишинговую страницу (Phishing page), предназначенную для кражи учетных данных. Атака эксплуатирует доверие пользователя к исходной вкладке, так как он вряд ли заметит, что страница, которую он только что покинул, перешла на другой домен. Современные практики безопасности требуют использования атрибутов `rel="noopener"` или `rel="noreferrer"` в тегах привязки (Anchor tags) или установки свойства `opener` в значение null в JavaScript для предотвращения этого межоконного взаимодействия.


| Поле | Значение |
|---|---|
| **Идентификатор WSTG** | WSTG-CLNT-14 |
| **CWE** | CWE-1022 |
| **Статус теста** | Не выполнен |
| **Уровень критичности** | Низкий / Средний* |

> *Уровень критичности является низким (Low) в большинстве современных браузеров, так как теперь они по умолчанию применяют `rel="noopener"` для ссылок с `target="_blank"`. Критичность становится средней (Medium), если приложение ориентировано на устаревшие браузеры или использует `window.open()` без установки свойства `opener` в значение null.

### Присутствуют ли ссылки, которые открываются в новом окне или вкладке?
- [ ] Нет — ссылки не используют `target="_blank"` или эквивалентное перенаправление на основе JavaScript  
- [ ] Да — `target="_blank"` используется исключительно для внутренних доверенных ссылок  
- [ ] Да — `target="_blank"` используется для внешних ссылок или URL-адресов, предоставленных пользователями  

### Ограничена ли связь через `window.opener` в тегах привязки?
- [ ] Да — `rel="noopener"` или `rel="noreferrer"` **последовательно применяется** ко всем ссылкам с `target="_blank"` *(Наиболее безопасно)*  
- [ ] Да — атрибуты применяются, но **отсутствуют** в ссылках с высоким уровнем риска или созданных пользователями  
- [ ] Нет — атрибуты **не применяются**, что позволяет дочерней странице получить доступ к `window.opener`  

### Уязвимо ли приложение к Обратному Табнаббингу через JavaScript `window.open()`?
- [ ] Нет — `window.open()` не используется или явно устанавливает свойство `opener` в значение null  
- [ ] Да — `window.open()` используется, и ссылка `opener` **остается доступной** для дочернего окна  

### Может ли злоумышленник контролировать место назначения ссылки, открывающейся в новой вкладке?
- [ ] Нет — все адреса ссылок жестко закодированы и указывают на доверенные домены  
- [ ] Да — адреса ссылок предоставляются пользователями (например, профили в социальных сетях, ссылки в биографии) и **не** проходят очистку (sanitization) для защиты от табнаббинга
