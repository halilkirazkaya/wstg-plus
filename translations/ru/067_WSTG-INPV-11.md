## WSTG-INPV-11 — Внедрение кода (Code Injection)

Внедрение кода (Code Injection) происходит, когда приложение интерпретирует предоставленные пользователем фрагменты кода в своей среде выполнения, как правило, через небезопасные функции языка, такие как `eval()`, `exec()` или `system()`. Эта уязвимость отличается от внедрения команд ОС (Command Injection), поскольку она нацелена на контекст выполнения языка программирования (например, PHP, Python, Node.js), а не на оболочку операционной системы. Злоумышленники используют эти точки входа для выполнения произвольной логики, эксфильтрации переменных окружения или достижения полного удаленного выполнения кода (Remote Code Execution, RCE) на сервере. Пентестеры должны исследовать функции, которые обрабатывают математические выражения, серверные шаблоны или динамические параметры конфигурации, где ввод может быть интерпретирован как исполняемый код.

### Оценивают ли какие-либо функции приложения динамический ввод через функции интерпретации кода, специфичные для языка?
- [ ] Нет — функции динамической интерпретации **отсутствуют** в кодовой базе  
- [ ] Да — функции, такие как `eval()`, `exec()` или `include()`, используются, но **не принимают** пользовательский ввод  
- [ ] Да — функции используются и обрабатывают данные, предоставленные пользователем  

### Применяются ли механизмы валидации или очистки (Sanitization) ввода перед выполнением?
- [ ] Да — ввод строго проверяется по **жестко заданному белому списку** (Whitelist)  
- [ ] Да — ввод очищается с помощью черных списков, но возможен **обход** (Bypass)  
- [ ] Нет — ввод передается напрямую в функцию интерпретации, и элементы контроля **не применяются**  

### Является ли среда выполнения изолированной (Sandboxed) или ограниченной для предотвращения доступа на системном уровне?
- [ ] Да — выполнение происходит в строго ограниченной "песочнице" (Sandbox), где системные вызовы **не могут** быть выполнены  
- [ ] Да — существуют некоторые ограничения, но возможен **выход из песочницы** (Sandbox Escape)  
- [ ] Нет — среда обеспечивает полный доступ к API языка и функциям ОС *(Критично)*  

### Может ли внедрение кода быть использовано для эксфильтрации конфиденциальной информации?
- [ ] Нет — выполнение происходит вслепую (Blind), и внеполосная связь (Out-of-band communication) **невозможна**  
- [ ] Да — переменные окружения или локальные файлы **могут** быть извлечены через HTTP/DNS-запросы  
- [ ] Да — результаты выполненного кода **напрямую отображаются** в ответе приложения  

### Позволяет ли приложение удаленное включение файлов или динамическую загрузку библиотек?
- [ ] Нет — могут быть выполнены только локальные, предопределенные пути кода  
- [ ] Да — приложение **может** быть принуждено к загрузке и выполнению кода из удаленного или контролируемого злоумышленником источника
