## WSTG-SESS-02 — Тестирование атрибутов файлов куки (Cookies)

Управление сессиями (Session Management) часто полагается на HTTP-куки (Cookies) для поддержания состояния, что делает защитные атрибуты этих куки основной целью для злоумышленников. Из-за отсутствия или неправильной настройки таких атрибутов, как `HttpOnly`, `Secure` и `SameSite`, приложения подвергают сессионные токены риску кражи через межсайтовый скриптинг (Cross-Site Scripting (XSS)), перехвату по незашифрованным каналам или атакам подделки межсайтовых запросов (Cross-Site Request Forgery (CSRF)). Пентестеры (Pentesters) изучают эти флаги, чтобы определить, может ли злоумышленник извлечь идентификаторы сессий из объектной модели документа (Document Object Model (DOM)) браузера или заставить браузер отправлять их в кросс-доменных запросах. Правильная настройка атрибутов является фундаментальной мерой эшелонированной защиты (Defense-in-Depth), гарантирующей, что конфиденциальные токены остаются в рамках авторизованных безопасных контекстов.

### Применяется ли флаг `HttpOnly` к конфиденциальным сессионным куки?
- [ ] Да — применен ко **всем** конфиденциальным сессионным куки *(Наиболее безопасно)*  
- [ ] Да — применен только к некоторым сессионным куки  
- [ ] Нет — флаг **не применен**, что позволяет получить доступ через клиентские скрипты *(Критично)*  

### Принудительно ли используется флаг `Secure` для куки, передаваемых по HTTPS?
- [ ] Да — применен ко **всем** куки, передаваемым через TLS  
- [ ] Нет — флаг **не применен**, что позволяет отправлять куки через открытый HTTP  

### Используется ли атрибут `SameSite` для снижения рисков CSRF?
- [ ] Да — установлено значение `Strict` или `Lax` для сессионных куки  
- [ ] Да — установлено значение `None`, но с присутствующим флагом `Secure`  
- [ ] Нет — атрибут **отсутствует** или установлен в значение `None` **без** флага `Secure`  

### Ограничены ли атрибуты `Domain` и `Path` минимально необходимой областью действия?
- [ ] Да — ограничены **конкретным** поддоменом и путем приложения  
- [ ] Нет — область действия слишком **широкая** (например, установлена для родительского домена или корня `/`), что увеличивает поверхность атаки  

### Установлен ли корректный срок действия для конфиденциальных сессионных куки с помощью атрибутов `Expires` или `Max-Age`?
- [ ] Да — установлены соответствующие даты истечения срока действия  
- [ ] Нет — куки являются постоянными с чрезмерно **длительным** временем жизни  
- [ ] Нет — куки являются сессионными, но **отсутствует** принудительное завершение сессии на стороне сервера по тайм-ауту
