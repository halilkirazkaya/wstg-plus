## WSTG-INFO-02 — Определение типа и версии веб-сервера (Fingerprinting)

Определение типа и версии веб-сервера (Web server fingerprinting) — это процесс идентификации конкретного типа программного обеспечения, версии и базовой операционной системы целевого веб-сервера. Злоумышленники проводят такую разведку для поиска известных уязвимостей, таких как неисправленные CVE (Common Vulnerabilities and Exposures) или недостатки конфигурации, характерные для определенных версий Apache, Nginx, IIS или других серверных технологий. Обычно это достигается путем анализа заголовков HTTP-ответов (например, `Server`, `X-Powered-By`), изучения уникального поведения протоколов или наблюдения за информацией, передаваемой через страницы ошибок по умолчанию и служебные файлы. Успешная идентификация позволяет атакующему адаптировать стратегию эксплуатации (Exploit), переходя от общего сканирования к атакам с высокой вероятностью успеха против известных архитектурных недостатков.

| Поле | Значение |
|---|---|
| **WSTG ID** | WSTG-INFO-02 |
| **CWE** | CWE-200 |
| **Статус теста** | Не выполнялось |
| **Степень риска** | Информационный / Низкий* |

> *Степень риска становится Высокой, если идентификация выявляет устаревшую или снятую с поддержки (EOL) версию сервера с известными критическими уязвимостями.

### Присутствуют ли идентифицирующие строки в заголовках HTTP-ответов?
- [ ] Нет — заголовки `Server` и `X-Powered-By` **отсутствуют** или содержат общие (generic) значения  
- [ ] Да — заголовки раскрывают ПО сервера, но **не** конкретные номера версий  
- [ ] Да — заголовки раскрывают конкретное ПО сервера и **точные** номера версий  

### Содержат ли страницы ошибок по умолчанию или системные ответы сведения о сервере?
- [ ] Нет — используются кастомные страницы ошибок, которые **не** раскрывают информацию о сервере  
- [ ] Да — страницы ошибок по умолчанию допускают утечку названия ПО сервера и/или версии  
- [ ] Да — страницы ошибок раскрывают данные о сервере, версию и **базовую операционную систему**  

### Раскрывается ли версия сервера через файлы по умолчанию или структуру каталогов?
- [ ] Нет — файлы установки по умолчанию, руководства и тестовые скрипты были **удалены**  
- [ ] Да — файлы по умолчанию (например, `info.php`, `manual/`, `test.html`) **присутствуют** и раскрывают данные о версии  

### Можно ли точно определить тип сервера с помощью анализа уникального поведения?
- [ ] Нет — ответы сервера нормализованы, и идентификация на основе поведения (Fingerprinting) **невозможна**  
- [ ] Да — уникальное поведение (например, порядок заголовков, специфические коды ошибок или особенности стека TCP/IP) **позволяет** идентифицировать сервер  

### Является ли выявленная версия сервера актуальной и поддерживаемой?
- [ ] Да — выявленная версия актуальна и **поддерживается** вендором  
- [ ] Нет — выявленная версия является **устаревшей** или **снятой с поддержки (EOL)** и имеет известные уязвимости
