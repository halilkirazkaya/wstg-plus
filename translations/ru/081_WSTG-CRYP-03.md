## WSTG-CRYP-03 — Тестирование передачи конфиденциальной информации по незашифрованным каналам

Передача конфиденциальных данных по незашифрованным каналам, таким как обычный протокол HTTP, подвергает информацию риску перехвата и модификации посредством атак типа «Человек посередине» (Man-in-the-Middle, MITM). Данная уязвимость обычно возникает, когда веб-приложения не обеспечивают обязательное использование протокола защиты транспортного уровня (Transport Layer Security, TLS) на всех конечных точках, особенно в устаревших компонентах, формах входа или во время первоначального перехода с HTTP на HTTPS. С точки зрения злоумышленника, это позволяет осуществлять пассивный перехват (sniffing) учетных данных аутентификации, токенов сессий и персональных данных (Personally Identifiable Information, PII) в скомпрометированных или ненадежных сегментах сети, таких как публичные сети Wi-Fi. Обеспечение инкапсуляции всех конфиденциальных данных внутри надежного TLS-туннеля является фундаментальным условием поддержания конфиденциальности и целостности пользовательских взаимодействий, а также предотвращения захвата сессий (session hijacking).

### Разрешает ли приложение обмен данными по незашифрованному протоколу HTTP для чувствительных конечных точек?
- [ ] Нет — весь трафик приложения строго принудительно направляется через HTTPS *(наиболее безопасно)*  
- [ ] Да — некритичные страницы доступны по HTTP, но чувствительные конечные точки — нет  
- [ ] Да — чувствительные конечные точки (вход в систему, оформление заказа, профиль) доступны по незашифрованному протоколу HTTP *(высокий риск)*  

### Настроено ли глобальное перенаправление с HTTP на HTTPS?
- [ ] Да — приложение выполняет перенаправление 301/302 на HTTPS для всех HTTP-запросов  
- [ ] Да — перенаправление выполняется только для определенных чувствительных конечных точек  
- [ ] Нет — перенаправление с HTTP на HTTPS не применяется  

### Внедрен ли механизм HTTP Strict Transport Security (HSTS)?
- [ ] Да — HSTS включен с длительным значением `max-age` и включает директивы `includeSubDomains` и `preload`  
- [ ] Да — HSTS включен, но отсутствуют директивы `includeSubDomains` или `preload`  
- [ ] Нет — HSTS не включен на сервере приложения

### Передает ли приложение конфиденциальные данные сторонним API по незашифрованным каналам?
- [ ] Нет — все внешние вызовы API выполняются через зашифрованные (HTTPS) туннели  
- [ ] Да — некоторая нечувствительная телеметрия отправляется по HTTP
