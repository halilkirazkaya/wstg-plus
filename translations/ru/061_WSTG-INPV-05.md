## WSTG-INPV-05 — SQL-инъекция (SQL Injection)

SQL-инъекция (SQL Injection) возникает, когда вводимые пользователем данные включаются в SQL-запросы без надлежащей параметризации (parameterization) или очистки (sanitization), что позволяет злоумышленникам манипулировать логикой запроса. Успешная эксплуатация может привести к несанкционированной эксфильтрации данных (data exfiltration), обходу аутентификации (authentication bypass), модификации данных и, в некоторых случаях, к удаленному выполнению кода (Remote Code Execution, RCE) через функции базы данных, такие как `xp_cmdshell` или `LOAD_FILE()`. Эта уязвимость обычно затрагивает формы входа, функции поиска, параметры REST API, HTTP-заголовки и любые конечные точки, которые формируют динамические SQL-запросы на основе данных, контролируемых пользователем. С точки зрения злоумышленника, идентификация точки внедрения является основным вектором для полной компрометации базы данных и потенциального горизонтального перемещения (lateral movement) внутри инфраструктуры.

### Обрабатываются ли параметры, контролируемые пользователем, с использованием безопасных методов доступа к базе данных?
- [ ] Да — приложение использует исключительно ORM или параметризованные запросы, и обход **невозможен**  
- [ ] Да — параметризованные запросы используются, но обход **возможен** в пограничных случаях (например, в предложениях `ORDER BY`)  
- [ ] Нет — для построения SQL-запросов используется прямая конкатенация строк **без** параметризации  

### Можно ли обойти механизм аутентификации с помощью SQL-инъекции?
- [ ] Нет — параметры входа **не** подвержены инъекции  
- [ ] Да — обход аутентификации **возможен** с использованием полезных нагрузок на основе тавтологии (например, `' OR 1=1 --`)  

### Возможна ли эксфильтрация данных с использованием внутриканальных или внеканальных методов?
- [ ] Нет — эксфильтрация данных не выявлена  
- [ ] Да — **возможна** эксфильтрация на основе UNION или на основе ошибок (error-based)  
- [ ] Да — **возможна** слепая (blind) эксфильтрация (Boolean или Time-based)  
- [ ] Да — **возможна** внеканальная (out-of-band) эксфильтрация (DNS/HTTP)  

### Проявляются ли в функциях приложения уязвимости SQL-инъекций второго порядка (second-order)?
- [ ] Нет — сохраненные данные обрабатываются безопасно при их извлечении для последующих запросов  
- [ ] Да — ввод пользователя сохраняется и позже используется в небезопасном SQL-запросе **без** проверки  

### Ограничены ли привилегии учетной записи службы базы данных до минимально необходимых?
- [ ] Да — учетная запись службы имеет **минимальные привилегии** (например, доступ ограничен конкретными таблицами/действиями)  
- [ ] Нет — учетная запись службы имеет повышенные привилегии (например, права `DROP TABLE`, `FILE` или **включен** `xp_cmdshell`)
