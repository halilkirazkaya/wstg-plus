## WSTG-CLNT-13 — Тестирование внедрения межсайтовых сценариев (Cross-Site Script Inclusion, XSSI)

Внедрение межсайтовых сценариев (Cross-Site Script Inclusion, XSSI) возникает, когда веб-приложение экспортирует конфиденциальные данные внутри динамически генерируемого JavaScript-файла или JSONP-ответа, который подконтрольный злоумышленнику сайт может затем включить через тег `<script>`. Поскольку включение скриптов обходит политику одного источника (Same-Origin Policy, SOP), злоумышленник может выполнить скрипт в своем собственном контексте и переопределить глобальные объекты или переменные для эксфильтрации (извлечения) конфиденциальной информации. Эта уязвимость обычно проявляется в аутентифицированных эндпоинтах (конечных точках), которые возвращают специфические для пользователя данные, такие как адреса электронной почты, API-ключи или метаданные сессии в формате скрипта. С точки зрения злоумышленника, эксплуатация включает создание вредоносной страницы, которая ссылается на уязвимый скрипт и фиксирует результирующие изменения глобальных переменных или вызовы функций.

| Поле | Значение |
|---|---|
| **WSTG ID** | WSTG-CLNT-13 |
| **CWE** | CWE-200 |
| **Статус теста** | Не выполнено |
| **Уровень серьезности** | Средний / Высокий |

> *Уровень серьезности становится высоким, если утекшие данные включают CSRF-токены, идентификаторы сессий или персональные данные (Personally Identifiable Information, PII) высокой степени конфиденциальности.

### Возвращают ли какие-либо аутентифицированные эндпоинты динамический JavaScript или JSONP, содержащий конфиденциальную информацию?
- [ ] Нет — все скрипты являются статическими и **не могут** содержать пользовательские данные  
- [ ] Да — динамические скрипты существуют, но **не содержат** конфиденциальную информацию  
- [ ] Да — динамические скрипты содержат PII или токены и **доступны** для разных источников (cross-origin)  

### Реализован ли заголовок `X-Content-Type-Options: nosniff` для ответов с конфиденциальными скриптами?
- [ ] Да — заголовок **применен** и предотвращает MIME-sniffing  
- [ ] Нет — заголовок **отсутствует**, что потенциально позволяет исполнять нескриптовый контент как скрипт  

### Защищены ли конфиденциальные скрипты механизмами противодействия XSSI, такими как неисполняемые префиксы или кастомные заголовки?
- [ ] Да — для скриптов требуется кастомный заголовок или токен, который **невозможно** отправить через стандартный тег `<script>`  
- [ ] Да — в скриптах используются неисполняемые префиксы (например, `)]}'`), которые **невозможно** легко обойти  
- [ ] Нет — скрипты доступны напрямую через GET-запросы с использованием неявных учетных данных (Ambient credentials) *(Критично)*  

### Возможна ли эксфильтрация конфиденциальных данных путем переопределения глобальных переменных или прототипов?
- [ ] Нет — область видимости конфиденциальных данных ограничена локально или они защищены современными функциями браузера  
- [ ] Да — конфиденциальные данные присваиваются глобальным переменным и **могут** быть перехвачены  
- [ ] Да — данные утекают через JSONP-callback-функции, которые **могут** быть перехвачены
