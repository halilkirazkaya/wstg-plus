## WSTG-INPV-05 — حقن لغة الاستعلامات البنيوية (SQL Injection)

يحدث حقن لغة الاستعلامات البنيوية (SQL Injection) عندما يتم دمج المدخلات التي يوفرها المستخدم في استعلامات SQL دون استخدام المعاملات البرمجية (Parameterization) أو التطهير (Sanitization) بشكل صحيح، مما يسمح للمهاجمين بالتلاعب بمنطق الاستعلام. يمكن أن يؤدي الاستغلال الناجح إلى استخراج البيانات (Data Exfiltration) بشكل غير مصرح به، وتجاوز المصادقة (Authentication Bypass)، وتعديل البيانات، وفي بعض الحالات تنفيذ الأوامر عن بُعد (Remote Code Execution) عبر ميزات قاعدة البيانات مثل `xp_cmdshell` أو `LOAD_FILE()`. تؤثر هذه الثغرة الأمنية بشكل شائع على نماذج تسجيل الدخول، ووظائف البحث، ومعاملات واجهة برمجة التطبيقات (REST API parameters)، وترويسات HTTP (HTTP headers)، وأي نقطة نهاية (Endpoint) تنشئ استعلامات SQL ديناميكية من مدخلات يتحكم فيها المستخدم. من منظور المهاجم، يعد تحديد نقطة الحقن هو البوابة الرئيسية للاختراق الكامل لقاعدة البيانات والتحرك الجانبي (Lateral Movement) المحتمل داخل البنية التحتية.

### هل يتم معالجة المعاملات التي يتحكم فيها المستخدم باستخدام طرق وصول آمنة لقاعدة البيانات؟
- [ ] نعم — يستخدم التطبيق ORM أو الاستعلامات ذات المعاملات البرمجية (Parameterized Queries) حصرياً والتجاوز **غير ممكن**  
- [ ] نعم — يتم استخدام الاستعلامات ذات المعاملات البرمجية ولكن التجاوز **ممكن** عبر الحالات الاستثنائية (Edge cases) (مثل عبارات `ORDER BY`)  
- [ ] لا — يتم استخدام دمج السلاسل النصية (String Concatenation) الخام لبناء استعلامات SQL **بدون** استخدام المعاملات البرمجية  

### هل يمكن التحايل على آلية المصادقة عبر SQL Injection؟
- [ ] لا — معاملات تسجيل الدخول **ليست** عرضة للحقن  
- [ ] نعم — تجاوز المصادقة (Authentication Bypass) **ممكن** باستخدام حمولات قائمة على الحشو المنطقي (Tautology-based payloads) (مثل `' OR 1=1 --`)  

### هل يمكن استخراج البيانات عبر تقنيات داخل النطاق (In-band) أو خارج النطاق (Out-of-band)؟
- [ ] لا — لم يتم تحديد أي استخراج للبيانات  
- [ ] نعم — استخراج البيانات القائم على UNION أو القائم على الأخطاء (Error-based) **ممكن**  
- [ ] نعم — الاستخراج الأعمى (Blind) (سواء Boolean أو Time-based) **ممكن**  
- [ ] نعم — استخراج البيانات خارج النطاق (DNS/HTTP) **ممكن**  

### هل تظهر وظائف التطبيق ثغرات حقن SQL من الدرجة الثانية (Second-order)؟
- [ ] لا — يتم التعامل مع البيانات المخزنة بأمان عند استردادها للاستعلامات اللاحقة  
- [ ] نعم — يتم تخزين مدخلات المستخدم واستخدامها لاحقاً في استعلام SQL غير آمن **بدون** تحقق  

### هل صلاحيات حساب خدمة قاعدة البيانات مقتصرة على الحد الأدنى المطلوب؟
- [ ] نعم — يمتلك حساب الخدمة (Service Account) **الحد الأدنى من الصلاحيات** (Least Privilege) (مثل قصرها على جداول/إجراءات محددة)  
- [ ] لا — يمتلك حساب الخدمة صلاحيات مرتفعة (مثل `DROP TABLE` أو صلاحيات `FILE` أو تفعيل `xp_cmdshell` **ممكن**)
