## WSTG-CONF-04 — مراجعة ملفات النسخ الاحتياطي القديمة والملفات غير المشار إليها بحثاً عن المعلومات الحساسة (Review Old Backup and Unreferenced Files for Sensitive Information)

تتضمن مراجعة ملفات النسخ الاحتياطي القديمة والملفات غير المشار إليها (Unreferenced files) تحديد الملفات المنسية أو المؤقتة أو المخفية على خادم الويب (Web server) والتي لا يُقصد الوصول إليها علنياً. غالباً ما تشمل هذه الملفات نسخاً احتياطية للكود المصدري (Source code) مثل (`.zip`, `.bak`) أو ملفات التبديل الخاصة بمحررات النصوص (Swap files) مثل (`.swp`, `~`) أو البيانات الوصفية لأنظمة التحكم في الإصدار (Source control metadata) مثل (`.git`, `.svn`) والتي قد تسرب معلومات حساسة. يستخدم المهاجمون قوائم كلمات مؤتمتة وأدوات اكتشاف للقيام بهجوم القوة الغاشمة (Brute Force) على اتفاقيات التسمية الشائعة، بهدف استخراج بيانات اعتماد قاعدة البيانات (Database credentials)، أو مفاتيح API المضمنة برمجياً (Hardcoded API keys)، أو المنطق البرمجي الذي يسهل عملية الاستغلال اللاحقة. تنشأ هذه الثغرة عادةً من عمليات صيانة الخادم اليدوية أو سلاسل التطوير والدمج المستمر (CI/CD pipelines) المعيبة التي تفشل في تنظيف المجلد الرئيسي للويب (Web root) في بيئة الإنتاج.

### هل خاصية سرد محتويات المجلدات (Directory listings) مفعّلة على خادم الويب؟
- [ ] لا — سرد المجلدات **معطل** ويعيد رمز الحالة 403 Forbidden أو خطأ مخصص  
- [ ] نعم — سرد المجلدات **مفعّل** في المجلدات غير الحساسة  
- [ ] نعم — سرد المجلدات **مفعّل** في المجلدات التي تحتوي على كود مصدري أو ملفات حساسة *(حرجة)*  

### هل يمكن اكتشاف ملفات النسخ الاحتياطي ذات الامتدادات الشائعة (مثل .bak، .old، .save)؟
- [ ] لا — الامتدادات الشائعة للنسخ الاحتياطي **غير موجودة** أو محظورة بواسطة إعدادات الخادم  
- [ ] نعم — ملفات النسخ الاحتياطي موجودة ولكنها **لا** تحتوي على معلومات حساسة  
- [ ] نعم — ملفات النسخ الاحتياطي الخاصة بالإعدادات أو الكود المصدري **متاحة** للوصول *(عالية)*  

### هل المجلدات الخاصة بأنظمة التحكم في الإصدار (مثل .git، .svn) أو بياناتها الوصفية مكشوفة؟
- [ ] لا — البيانات الوصفية للتحكم في الإصدار **غير موجودة** أو محظورة بشكل صحيح  
- [ ] نعم — البيانات الوصفية موجودة ولكن **لا يمكن** إعادة بناء المستودع (Repository) بالكامل  
- [ ] نعم — مستودع الكود المصدري بالكامل **يمكن** استخراجه عبر البيانات الوصفية المكشوفة  

### هل توجد أرشيفات مضغوطة (مثل .zip، .tar.gz، .rar) للتطبيق في المجلد الرئيسي للويب (Web root)؟
- [ ] لا — لم يتم اكتشاف ملفات أرشيف حساسة عبر القوة الغاشمة أو التعداد (Enumeration)  
- [ ] نعم — تم العثور على أرشيفات ولكنها محمية بكلمة مرور أو تحتوي على أصول عامة  
- [ ] نعم — أرشيفات غير مشفرة تحتوي على كود التطبيق أو بياناته **متاحة** للوصول العلني  

### هل يسرب التطبيق ملفات مؤقتة تم إنشاؤها بواسطة محررات النصوص أو بيئات التطوير المتكاملة (IDE)؟
- [ ] لا — الملفات المؤقتة مثل `.swp` أو `~` أو `.DS_Store` **غير موجودة**  
- [ ] نعم — ملفات مؤقتة غير حساسة **موجودة**  
- [ ] نعم — الملفات المؤقتة تكشف عن مقتطفات من الكود المصدري أو مسارات داخلية
