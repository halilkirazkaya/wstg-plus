## WSTG-CLNT-03 — حقن لغة ترميز النص التشعبي (HTML Injection)

يحدث حقن لغة ترميز النص التشعبي (HTML Injection) عندما يفشل التطبيق في تطهير (Sanitize) المدخلات التي يوفرها المستخدم قبل تصييرها (Rendering) داخل المتصفح، مما يسمح للمهاجم بحقن وسوم HTML عشوائية في نموذج كائن المستند (DOM) الخاص بصفحة الويب. وبالرغم من تشابهه مع البرمجة عبر المواقع (XSS)، فإن الهدف الأساسي من حقن HTML هو التلاعب بالعرض المرئي للصفحة أو تسهيل هجمات التصيد الاحتيالي (Phishing) من خلال حقن نماذج خبيثة ومحتوى مضلل. يستهدف المهاجمون المعلمات المنعكسة (Reflected) في واجهة المستخدم، مثل مصطلحات البحث، أو حقول ملف تعريف المستخدم، أو رسائل الخطأ، لخداع الضحايا للكشف عن بيانات اعتماد حساسة أو التفاعل مع روابط خارجية غير مصرح بها. تمثل هذه الثغرة خطراً جسيماً على سلامة واجهة مستخدم التطبيق ويمكن أن تكون مقدمة لهجمات هندسة اجتماعية أو هجمات متعلقة بالجلسات أكثر تعقيداً.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-CLNT-03 |
| **CWE** | CWE-80 |
| **حالة الاختبار** | لم يتم التنفيذ |
| **الخطورة** | منخفضة / متوسطة |

### هل تنعكس المدخلات التي يتحكم فيها المستخدم في DOM دون ترميز HTML مناسب؟
- [ ] لا — يتم ترميز جميع المدخلات المنعكسة بدقة بنظام ترميز HTML قبل تصييرها  
- [ ] نعم — يتم ترميز بعض الأحرف ولكن يمكن تجاوز ذلك لبعض الوسوم  
- [ ] نعم — تنعكس المدخلات بشكلها الخام، وحقن HTML ممكن  

### هل يمكن حقن عناصر HTML هيكلية لتغيير تنسيق الصفحة؟
- [ ] لا — يقوم التطبيق بتصفية أو إزالة الوسوم الهيكلية مثل `<div>` أو `<table>` أو `<iframe>`  
- [ ] نعم — يمكن حقن العناصر الهيكلية ولكنها لا تؤثر بشكل كبير على واجهة المستخدم  
- [ ] نعم — يمكن تشويه واجهة المستخدم بشكل كبير عبر الوسوم المحقونة  

### هل من الممكن حقن عناصر تصيد وظيفية مثل النماذج أو الروابط الخبيثة؟
- [ ] لا — يتم حظر أو تطهير وسوم `<form>` و `<input>` و `<a>` بشكل فعال  
- [ ] نعم — يمكن حقن روابط خبيثة لإعادة توجيه المستخدمين إلى مواقع خارجية  
- [ ] نعم — يمكن حقن عناصر `<form>` وظيفية لسرقة بيانات الاعتماد *(متوسطة)*  

### هل تعمل رؤوس الأمان أو سياسات أمان المحتوى (CSP) على التخفيف من أثر الحقن؟
- [ ] نعم — توجد سياسة أمان المحتوى (CSP) تقييدية ويتم تطبيق `form-action` أو `frame-src`  
- [ ] لا — رؤوس الأمان موجودة ولكن CSP معطلة أو تحتوي على unsafe-inline أو رموز عامة (wildcards)  
- [ ] لا — لم يتم تفعيل أي سياسة CSP أو رؤوس أمان ذات صلة
