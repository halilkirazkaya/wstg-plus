## WSTG-SESS-10 — اختبار رموز ويب جيسون (JSON Web Tokens - JWT)

تُستخدم رموز ويب جيسون (JSON Web Tokens - JWT) بشكل شائع لإدارة الجلسات عديمة الحالة (Stateless Session Management) ونشر الهوية، لكن أمنها يعتمد كلياً على التنفيذ الصحيح لخوارزميات التوقيع والتحقق الصارم من المطالبات (Claims). غالباً ما يحاول المهاجمون تجاوز المصادقة (Authentication) عن طريق استغلال ثغرات خوارزمية "none"، أو استخدام الهجوم بالقوة الغاشمة (Brute Force) ضد مفاتيح السر الضعيفة من نوع HS256، أو تنفيذ هجمات تبديل الخوارزمية (Algorithm-switching) حيث يتم استخدام مفتاح عام غير متماثل كمفتاح HMAC متماثل. تظهر هذه الثغرات الأمنية عادةً في ملفات تعريف ارتباط الجلسة (Session Cookies) أو ترويسات المصادقة (Authorization headers)، مما قد يسمح للمهاجم بتصعيد الصلاحيات (Privilege Escalation) أو انتحال شخصية أي مستخدم عن طريق تعديل المطالبات مثل `role` أو `user_id` دون إبطال السلامة التشفيرية للرمز.

### هل تقبل الخادم خوارزمية "none"؟
- [ ] لا — الخادم **يرفض** الرموز التي تستخدم `alg: none` في الترويسة  
- [ ] نعم — الخادم **يقبل** الرموز التي تحتوي على `alg: none` وتوقيع فارغ *(حرجة)*  

### كيف يتم التحقق من توقيع JWT وحمايته من الهجوم بالقوة الغاشمة (Brute Force)؟
- [ ] نعم — يتم استخدام مفاتيح غير متماثلة قوية (RS256/ES256) أو أسرار HMAC ذات اعتلاج (Entropy) عالٍ  
- [ ] نعم — يتم استخدام HS256 ولكن **يمكن** كسر السر بالقوة الغاشمة بسبب ضعف الاعتلاج أو القيم الافتراضية  
- [ ] لا — التحقق من التوقيع **لا يتم تطبيقه** أو **يمكن** تجاوزه عبر تبديل الخوارزمية (من RS256 إلى HS256)  

### هل يتم فرض المطالبات القياسية (exp, nbf, iat) بصرامة بواسطة الواجهة الخلفية (Backend)؟
- [ ] نعم — مطالبة `exp` (انتهاء الصلاحية) موجودة ولا **يمكن** تجاوزها  
- [ ] نعم — مطالبة `exp` موجودة ولكن الخادم **لا يفرضها** أثناء عملية التحقق  
- [ ] لا — مطالبات انتهاء الصلاحية **مفقودة** أو يتم تجاهلها، مما يسمح بإعادة تشغيل الجلسة (Session Replay) إلى أجل غير مسمى  

### هل يمنع التنفيذ حقن المفاتيح المستند إلى الترويسة (Header-based key injection) مثل (kid, jku, x5u)؟
- [ ] نعم — يتم تنقية الترويسات ويُسمح فقط بالمفاتيح أو المصادر الداخلية الموثوقة  
- [ ] لا — ترويسة `kid` (معرف المفتاح) عرضة لثغرة تخطي المسار (Directory Traversal) أو حقن SQL للإشارة إلى ملف معروف  
- [ ] لا — ترويسات `jku` أو `x5u` **يمكن** توجيهها إلى عناوين URL يتحكم فيها المهاجم لتحميل مفاتيح خبيثة
