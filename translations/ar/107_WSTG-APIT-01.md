## WSTG-APIT-01 — استطلاع واجهات برمجة التطبيقات (API Reconnaissance)

يُعد استطلاع واجهات برمجة التطبيقات (API Reconnaissance) عملية منهجية لتحديد ورسم خرائط لسطح الهجوم (Attack Surface) الخاص بواجهة برمجة التطبيقات للكشف عن نقاط النهاية (Endpoints)، والطرق المدعومة، وهياكل البيانات الأساسية. يقوم المهاجمون بتنفيذ هذه المرحلة لاكتشاف واجهات برمجة التطبيقات غير الموثقة أو الخفية (Shadow APIs)، والإصدارات المهجورة التي تحتوي على ثغرات قديمة، وملفات التوثيق المكشوفة مثل تعريفات Swagger أو OpenAPI التي تكشف عن منطق العمل (Business Logic) الداخلي. من خلال تحليل أنماط عناوين URL المتوقعة، وملفات JavaScript من جانب العميل، ونتائج الهجوم الشامل (Brute Force) للمجلدات، يمكن للمهاجم بناء خريطة شاملة لواجهة برمجة التطبيقات (API) لتحديد الأهداف لهجمات كسر تفويض مستوى الكائن (Broken Object Level Authorization - BOLA) أو التعيين الجماعي (Mass Assignment). يستهدف هذا الاستطلاع عادةً المسارات الشائعة مثل `/api/v1/` أو `/swagger.json` أو `/graphql` للحصول على خارطة طريق لوظائف الخلفية (Backend) للتطبيق.

### هل ملفات توثيق واجهة برمجة التطبيقات (Swagger, OpenAPI, WSDL) متاحة للعامة؟
- [ ] لا — توثيق واجهة برمجة التطبيقات (API) غير متاح أو مقيد بالمصادقة (Authentication)  
- [ ] نعم — التوثيق متاح ولكنه يتطلب بيانات اعتماد (Credentials) صالحة  
- [ ] نعم — توثيق واجهة برمجة التطبيقات (مثل `/swagger-ui.html`) متاح للعامة دون مصادقة  

### هل يمكن اكتشاف نقاط نهاية واجهة برمجة التطبيقات غير الموثقة أو "الخفية" (Shadow) عبر الفحص العشوائي (Fuzzing)؟
- [ ] لا — لم يسفر الفحص العشوائي (Fuzzing) للمجلدات ونقاط النهاية عن أي مسارات غير موثقة  
- [ ] نعم — تم العثور على نقاط نهاية غير موثقة ولكن لا يمكن الوصول إليها دون تصريح (Authorization)  
- [ ] نعم — تم العثور على نقاط نهاية غير موثقة ويمكن الوصول إليها دون تصريح صالح  

### هل يكشف التطبيق عن إصدارات متعددة لواجهة برمجة التطبيقات (مثل /v1/، /v2/، /beta/)؟
- [ ] لا — الإصدار الحالي والمحصن من واجهة برمجة التطبيقات هو الوحيد المتاح للوصول  
- [ ] نعم — توجد إصدارات قديمة ولكنها تطبق نفس ضوابط الأمان (Security Controls) الخاصة بالإصدار الحالي  
- [ ] نعم — الإصدارات القديمة (مثل `/v1/`) متاحة للوصول وتفتقر إلى ضوابط الأمان الموجودة في الإصدارات الأحدث  

### هل تسرب موارد جانب العميل (JavaScript/تطبيقات الجوال) هياكل نقاط النهاية أو مفاتيح واجهة برمجة التطبيقات؟
- [ ] لا — كود جانب العميل لا يحتوي على مسارات API ثابتة (Hardcoded) أو مفاتيح حساسة  
- [ ] نعم — كود جانب العميل يحتوي على خرائط لنقاط نهاية واجهة برمجة التطبيقات ولكن بدون مفاتيح حساسة  
- [ ] نعم — مفاتيح واجهة برمجة التطبيقات، أو الأسرار (Secrets)، أو عناوين URL لنقاط النهاية الداخلية فقط مشفرة بشكل ثابت (Hardcoded) في موارد جانب العميل  

### هل يمكن اكتشاف بارامترات (Parameters) أو ترويسات (Headers) مخفية في نقاط النهاية المعروفة؟
- [ ] لا — لم يكشف الفحص العشوائي للبارامترات (مثلاً عبر `Arjun`) عن أي مدخلات مخفية  
- [ ] نعم — تم اكتشاف بارامترات مخفية (مثل `debug=true` أو `admin=1`) ولكنها غير فعالة  
- [ ] نعم — تم اكتشاف بارامترات مخفية ويمكن استخدامها لتغيير سلوك التطبيق
