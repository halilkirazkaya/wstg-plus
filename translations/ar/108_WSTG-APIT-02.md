## WSTG-APIT-02 — ضعف صلاحيات الوصول على مستوى الكائنات (Broken Object Level Authorization - BOLA)

يحدث ضعف صلاحيات الوصول على مستوى الكائنات (Broken Object Level Authorization - BOLA)، والمعروف أيضًا باسم المرجع المباشر غير الآمن للكائنات (Insecure Direct Object Reference - IDOR)، عندما تفشل واجهة برمجة التطبيقات (API) في التحقق مما إذا كان المستخدم يمتلك الأذونات المناسبة للوصول إلى مورد معين محدد بواسطة معرف (ID) أو التلاعب به. يستغل المهاجمون ذلك عن طريق التعداد المنهجي أو تخمين المعرفات — مثل المعرفات الرقمية أو معرفات UUID — في مسارات الطلبات (Request Paths)، أو معلمات الاستعلام (Query Parameters)، أو أجسام JSON للوصول إلى البيانات التابعة لمستخدمين آخرين. تعتبر هذه الثغرة الأمنية هي المشكلة الأكثر شيوعًا وتأثيرًا في أمن واجهات برمجة التطبيقات (API Security)، مما قد يؤدي إلى تسريب البيانات بكميات كبيرة، أو التعديل غير المصرح به، أو الاستيلاء الكامل على الحساب (Account Takeover). من منظور المهاجم، الهدف هو تحديد نقاط النهاية (Endpoints) التي تقبل معرفات الكائنات واختبار ما إذا كان منطق المصادقة من جانب الخادم (Server-side authorization logic) مفقودًا أو يمكن تجاوزه من خلال التلاعب بتلك المعرفات.

### هل معرفات الكائنات قابلة للتنبؤ أو التعداد ضمن طلبات واجهة برمجة التطبيقات (API)؟
- [ ] لا — المعرفات طويلة، عشوائية، وآمنة تشفيرياً (مثل UUIDv4)  
- [ ] نعم — المعرفات هي أرقام صحيحة متسلسلة (مثل `101` ، `102`)  
- [ ] نعم — تتبع المعرفات نمطاً قابلاً للتنبؤ أو يتم اشتقاقها من معلومات عامة  

### هل تقوم واجهة برمجة التطبيقات (API) بإجراء تحقق من جانب الخادم لملكية الكائن لكل طلب؟
- [ ] نعم — يتم تطبيق فحوصات الصلاحيات لكل طلب والتجاوز **غير ممكن**  
- [ ] نعم — يتم تطبيق الفحوصات ولكن التجاوز **ممكن** عبر تلوث المعلمات (Parameter Pollution) أو نفق الأساليب (Method Tunneling)  
- [ ] لا — يعتمد التطبيق فقط على كون المستخدم موثقاً دون التحقق من ملكية المورد المحدد  

### هل من الممكن الوصول إلى مورد مستخدم آخر أو تعديله عن طريق تغيير المعرف؟
- [ ] لا — الوصول غير المصرح به لموارد المستخدمين الآخرين **غير ممكن**  
- [ ] نعم — الوصول للقراءة (Horizontal BOLA) غير المصرح به **ممكن**  
- [ ] نعم — التعديل أو الحذف (Horizontal BOLA) غير المصرح به **ممكن**  

### هل تسمح واجهة برمجة التطبيقات (API) بالوصول إلى كائنات إدارية أو على مستوى النظام من خلال استبدال المعرفات (IDs)؟
- [ ] لا — الموارد الإدارية محمية بطبقات صلاحيات ثانوية  
- [ ] نعم — الوصول إلى موارد مستوى النظام أو تعديلها (Vertical BOLA) **ممكن**  

### هل يمكن تجاوز فحص الصلاحيات عن طريق نقل المعرف إلى جزء مختلف من الطلب؟
- [ ] لا — منطق الصلاحيات متسق بغض النظر عن موقع المعلمة  
- [ ] نعم — التجاوز **ممكن** عند نقل المعرف من مسار URL إلى جسم JSON أو الرؤوس (Headers)  
- [ ] نعم — التجاوز **ممكن** عند توفير معرفات متعددة ومعالجة الخادم للمعرف غير المصرح به
