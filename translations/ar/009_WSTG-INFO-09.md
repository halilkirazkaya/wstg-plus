## WSTG-INFO-09 — تحديد البصمة التقنية لتطبيق الويب (Fingerprint Web Application)

تحديد البصمة التقنية (Fingerprinting) لتطبيق الويب هو عملية التعرف على إطار العمل (Framework) المحدد، أو نظام إدارة المحتوى (CMS)، أو الحزمة التقنية (Technology Stack) الأساسية وإصداراتها المرتبطة. تُعد مرحلة الاستطلاع (Reconnaissance) هذه حيوية لأنها تتيح للمهاجم مطابقة المكونات المحددة مع الثغرات الأمنية (CVEs) المعروفة والثغرات المستغلة (Exploits) العامة، مما يقلل بشكل كبير من سطح الهجوم (Attack surface). يتم جمع المعلومات عادةً من ترويسات استجابة HTTP (HTTP response headers)، ومسارات الملفات الفريدة، وأسماء ملفات تعريف الارتباط (Cookie names)، والبيانات الوصفية للكود المصدري HTML، ورموز التجزئة التشفيرية (Cryptographic hashes) للأصول الثابتة (Static assets). من خلال التحديد الدقيق للحزمة التقنية، يمكن للمهاجم الانتقال من الفحص الآلي (Automated scanning) العام إلى الاستغلال المستهدف للغاية لنقاط الضعف الخاصة بإصدار معين.

### هل تكشف ترويسات استجابة HTTP عن الحزمة التقنية أو أرقام الإصدارات؟
- [ ] لا — يتم تجريد الترويسات الحساسة أو أنها تحتوي على قيم عامة  
- [ ] نعم — الترويسات الافتراضية مثل `X-Powered-By` أو `Server` أو `X-AspNet-Version` موجودة  
- [ ] نعم — تكشف الترويسات عن إصدارات محددة وقديمة لخادم الويب أو إطار عمل التطبيق  

### هل يمكن تحديد إطار عمل التطبيق أو نظام إدارة المحتوى (CMS) من خلال مسارات ملفات فريدة أو اتفاقيات التسمية؟
- [ ] لا — مسارات الملفات مخفية ولا تكشف عن التقنية الأساسية  
- [ ] نعم — هياكل الأدلة الشائعة (مثل `/wp-content/` أو `/_next/` أو `/node_modules/`) مرئية  
- [ ] نعم — تحتوي الملفات الفريدة مثل `robots.txt` أو `humans.txt` أو `sitemap.xml` على بيانات وصفية خاصة بالتقنية  

### هل تظهر أرقام إصدارات محددة داخل الكود المصدري HTML أو الأصول الثابتة؟
- [ ] لا — لا توجد معلومات عن الإصدار في التعليقات أو معلمات الأصول  
- [ ] نعم — توجد تعليقات HTML أو علامات وصفية (مثل `<meta name="generator" content="...">`)  
- [ ] نعم — يتم إلحاق سلاسل الإصدار بأسماء ملفات CSS/JS (مثل `jquery.js?v=1.12.4`) ولا يمكن تعطيلها  

### هل تكشف صفحات الخطأ الافتراضية أو واجهات الإدارة عن بيئة البرامج؟
- [ ] لا — يستخدم التطبيق صفحات خطأ مخصصة وعامة لجميع رموز الحالة  
- [ ] نعم — صفحات الخطأ الافتراضية لخادم الويب أو إطار العمل مفعلة وتسرب بيانات الإصدار  
- [ ] نعم — بوابات تسجيل الدخول الإدارية (مثل `/wp-admin` أو `/phpmyadmin`) يمكن الوصول إليها وتحديدها  

### هل تكشف ملفات تعريف الارتباط (Cookies) عن إطار عمل إدارة الجلسة (Session management)؟
- [ ] لا — تستخدم ملفات تعريف ارتباط الجلسة أسماء عامة أو مخصصة  
- [ ] نعم — يتم استخدام أسماء ملفات تعريف ارتباط خاصة بالتقنية (مثل `PHPSESSID` أو `JSESSIONID` أو `ASPSESSIONID`)
