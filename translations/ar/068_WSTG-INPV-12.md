## WSTG-INPV-12 — حقن الأوامر (Command Injection)

يحدث حقن الأوامر (Command Injection) عندما يمرر التطبيق بيانات غير آمنة مقدمة من المستخدم، مثل مدخلات النماذج، أو ترويسات (HTTP Headers)، أو ملفات تعريف الارتباط (Cookies)، إلى شل النظام (System Shell). تسمح هذه الثغرة للمهاجم بتنفيذ أوامر نظام تشغيل عشوائية على الخادم، وعادةً ما يكون ذلك بصلاحيات عملية تطبيق الويب. من منظور المهاجم، يتم استغلال ذلك باستخدام محارف الشل الوصفية (Shell Metacharacters) مثل الفواصل المنقوطة، أو الأنابيب (Pipes)، أو علامات الاقتباس الخلفية لربط الأوامر الضارة بتدفق التنفيذ المقصود. غالباً ما يكون الأثر كارثياً، مما يؤدي إلى اختراق كامل للنظام، أو استخراج البيانات (Data Exfiltration) بشكل غير مصرح به، أو التحرك الجانبي (Lateral Movement) داخل الشبكة الداخلية.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-INPV-12 |
| **CWE** | CWE-78 |
| **حالة الاختبار** | لم يتم التنفيذ |
| **الخطورة** | حرجة (Critical) |

### هل يستدعي التطبيق أوامر على مستوى النظام أو ملفات شل تنفيذية؟
- [ ] لا — التطبيق **لا** يتفاعل مع شل نظام التشغيل (OS Shell)  
- [ ] نعم — يستخدم التطبيق واجهات برمجة تطبيقات (APIs) داخلية أو مكتبات عالية المستوى لمهام النظام  
- [ ] نعم — يستدعي التطبيق أوامر الشل مباشرة عبر استدعاءات النظام مثل `system()`، أو `exec()`، أو `popen()`  

### هل يتم التحقق من صحة مدخلات المستخدم أو تنقيتها قبل تمريرها إلى استدعاءات النظام؟
- [ ] نعم — يتم تطبيق القائمة البيضاء (Allow-listing) والتحقق من المدخلات بشكل صارم  
- [ ] نعم — يتم استخدام التنقية (Sanitization) أو الهروب (Escaping) ولكن التجاوز **ممكن**  
- [ ] لا — يتم تمرير مدخلات المستخدم مباشرة إلى الشل **دون** التحقق من صحتها  

### هل يمكن استخدام محارف الشل الوصفية (Shell Metacharacters) للتلاعب بتدفق تنفيذ الأوامر؟
- [ ] لا — يتم تصفية المحارف الوصفية أو الهروب منها بشكل صحيح  
- [ ] نعم — التنفيذ داخل النطاق (In-band) حيث تنعكس مخرجات الأوامر في الاستجابة **أمر ممكن**  
- [ ] نعم — التنفيذ الأعمى (Blind) حيث لا تنعكس أي مخرجات **أمر ممكن**  

### هل استخراج البيانات خارج النطاق (OOB Exfiltration) ممكن من المضيف؟
- [ ] لا — لا يوجد للخادم حركة مرور صادرة (Egress) أو فشل إشارات OOB عبر (DNS/HTTP)  
- [ ] نعم — استخراج البيانات عبر إشارات OOB المستندة إلى DNS أو HTTP **أمر ممكن**  

### هل يتم تشغيل عملية التطبيق بصلاحيات نظام مرتفعة؟
- [ ] لا — يعمل التطبيق بحساب خدمة مخصص ومنخفض الصلاحيات  
- [ ] نعم — يعمل التطبيق بصلاحية `root` أو `Administrator` أو مستخدم ذو صلاحيات عالية *(حرجة)*
