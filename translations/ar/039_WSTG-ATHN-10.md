## WSTG-ATHN-10 — اختبار ضعف المصادقة في القنوات البديلة (Testing for Weaker Authentication in Alternative Channel)

يتضمن اختبار ضعف المصادقة في القنوات البديلة (Testing for Weaker Authentication in Alternative Channel) تحديد وتحليل المسارات الثانوية للوصول إلى الحساب — مثل واجهات برمجة التطبيقات للجوال (Mobile APIs)، أو سير عمل استرداد كلمة المرور، أو واجهات مكتب المساعدة، أو النطاقات الفرعية القديمة (Legacy Subdomains) — التي قد لا تفرض نفس الصرامة الأمنية التي تفرضها بوابة الويب الأساسية. غالباً ما تفتقر هذه القنوات البديلة إلى المصادقة متعددة العوامل (Multi-Factor Authentication - MFA) القوية، أو تحديد معدل الطلبات (Rate Limiting) الصارم، أو متطلبات كلمة المرور المعقدة، مما يخلق "حلقة ضعيفة" تقوض نظام المصادقة بالكامل. يستهدف المهاجمون تحديداً نقاط الدخول المهملة هذه لتنفيذ هجمات حشو الاعتمادات (Credential Stuffing) أو تجاوز MFA من خلال استغلال التباينات في كيفية معالجة الواجهات المختلفة للتحقق من الهوية. يعد ضمان التكافؤ عبر جميع أسطح المصادقة أمراً ضرورياً لمنع الوصول غير المصرح به والحفاظ على سلامة جلسة المستخدم وبياناته.

### هل توجد قنوات مصادقة بديلة وتم تحديدها؟
- [ ] لا — توجد قناة مصادقة واحدة فقط  
- [ ] نعم — تم تحديد قنوات متعددة (مثل API تطبيق الجوال، عميل سطح المكتب، البوابة القديمة، خدمات SOAP)  

### هل تفرض القنوات البديلة تكافؤاً أمنياً مع القناة الأساسية؟
- [ ] نعم — تفرض جميع القنوات متطلبات **مطابقة** لتعقيد كلمة المرور و MFA  
- [ ] نعم — تفرض القنوات تعقيد كلمة المرور ولكن MFA **غير مطلوب** أو **يمكن** تجاوزه  
- [ ] لا — تمتلك القنوات البديلة متطلبات مصادقة **أضعف بكثير**  

### هل يتسق تحديد معدل الطلبات (Rate Limiting) وإغلاق الحساب عبر جميع القنوات؟
- [ ] نعم — يتم **فرض** تحديد معدل الطلبات وإغلاق الحساب بشكل متسق عبر جميع نقاط النهاية  
- [ ] نعم — الضوابط موجودة ولكن التجاوز (Bypass) **ممكن** في قنوات معينة (مثل Mobile API)  
- [ ] لا — لا يتم **تطبيق** تحديد معدل الطلبات أو إغلاق الحساب على مسارات المصادقة الثانوية  

### هل يمكن تجاوز MFA بالانتقال إلى قناة بديلة؟
- [ ] لا — المصادقة متعددة العوامل (MFA) إلزامية و**لا يمكن** تجاوزها بغض النظر عن نقطة الدخول  
- [ ] نعم — المصادقة متعددة العوامل (MFA) مطلوبة على بوابة الويب ولكن **لا يتم فرضها** على API الجوال أو الأنظمة القديمة  

### هل مسارات استرداد كلمة المرور أو "نسيت كلمة المرور" أضعف من تسجيل الدخول القياسي؟
- [ ] لا — تستخدم مسارات الاسترداد تحققاً قوياً خارج النطاق (Out-of-band) ولا تسرب المعلومات  
- [ ] نعم — تستخدم مسارات الاسترداد أسئلة أمان ضعيفة **يمكن** تخمينها بسهولة أو البحث عنها  
- [ ] نعم — مسارات الاسترداد عرضة للتعداد (Enumeration) أو **لا تتطلب** نفس مستوى الإثبات مثل تسجيل الدخول القياسي
