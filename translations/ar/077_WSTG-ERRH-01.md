## WSTG-ERRH-01 — اختبار المعالجة غير الصحيحة للأخطاء (Testing for Improper Error Handling)

تحدث المعالجة غير الصحيحة للأخطاء (Improper Error Handling) عندما يكشف تطبيق الويب عن تفاصيل تقنية حساسة - مثل تتبع المكدس (Stack Traces)، أو معلومات مخطط قاعدة البيانات (Database Schema)، أو مسارات الملفات الداخلية (Internal File Paths) - من خلال استجابات الخطأ الخاصة به. يقوم المهاجمون بتحفيز هذه الأخطاء عن طريق إرسال مدخلات غير صالحة (Malformed Input)، أو الوصول إلى موارد غير موجودة، أو إحداث استثناءات من جهة الخادم (Server-side Exceptions) لرسم مخطط للبنية التحتية الأساسية للتطبيق وتحديد المتجهات المحتملة لمزيد من الاستغلال (Exploit). غالبًا ما يكون تسريب المعلومات (Information Leakage) هذا مقدمة لهجمات أكثر خطورة، بما في ذلك حقن SQL (SQL Injection) أو تخطي المسارات (Path Traversal)، من خلال تزويد المهاجم بمواصفات دقيقة للبيئة. يجب أن يوفر التنفيذ الآمن رسائل عامة سهلة الاستخدام للمستخدمين مع التقاط تفاصيل التشخيص فقط في سجلات آمنة من جهة الخادم.

| **CWE** | CWE-209 |

### هل يستخدم التطبيق معالج أخطاء عام وشامل (Global Error Handler) للاستثناءات غير المعالجة؟
- [ ] نعم — صفحات الخطأ العامة **مفعلة** ولا تكشف عن **أي** تفاصيل تقنية  
- [ ] نعم — يتم استخدام صفحات خطأ مخصصة ولكن التسريب **ممكن** عبر ترويسات (Headers) محددة  
- [ ] لا — صفحات خطأ الخادم الافتراضية (مثل Tomcat، IIS، Nginx) **مرئية**  

### هل يمكن استخراج معلومات تقنية من خلال المدخلات غير الصالحة أو الحالات الحدية (Boundary Cases)؟
- [ ] لا — يتم التعامل مع الأخطاء بسلاسة مع وجود معرفات مرجعية فريدة لتسجيل البيانات  
- [ ] نعم — يتم **الكشف** عن تتبع المكدس (Stack Traces) أو استعلامات قاعدة البيانات في الاستجابات  
- [ ] نعم — يتم **الكشف** عن مسارات الملفات الداخلية، أو متغيرات البيئة، أو سلاسل إصدار الخادم  

### هل تسرب استجابات واجهة برمجة التطبيقات (API) كائنات خطأ مسهبة أو معلومات تصحيح الأخطاء؟
- [ ] لا — تعيد واجهات برمجة التطبيقات (API) أكواد خطأ موحدة ورسائل JSON منقحة  
- [ ] نعم — تعيد واجهات برمجة التطبيقات كائنات تصحيح أخطاء (Debug) مسهبة أو تفاصيل استثناء كاملة في جسم الاستجابة  
- [ ] نعم — يتم تضمين تتبع المكدس في حقول `details` أو `exception` ضمن استجابة JSON  

### هل يتصرف التطبيق بشكل مختلف (بناءً على الوقت أو المحتوى - Time-based or Content-based) عند حدوث أخطاء؟
- [ ] لا — تواقيع الاستجابة (Response Signatures) والتوقيتات متسقة بغض النظر عن نوع الخطأ  
- [ ] نعم — **يمكن** استخدام الاستجابات المتباينة لتعداد الحالات الصالحة مقابل غير الصالحة (مثلاً، تعداد المستخدمين User Enumeration)
