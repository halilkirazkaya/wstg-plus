## WSTG-INFO-10 — تخطيط بنية التطبيق (Map Application Architecture)

يتضمن تخطيط بنية التطبيق (Mapping Application Architecture) تحديد التقنيات الأساسية، ومكونات البنية التحتية، ومسارات تدفق البيانات التي تدعم تطبيق الويب. من خلال تحليل رؤوس استجابة HTTP (HTTP response headers)، ورسائل الخطأ، وتنسيقات ملفات تعريف الارتباط (Cookie formats)، وامتدادات الملفات، يمكن للمختبرين إعادة بناء مخطط لخوادم الويب (Web servers)، وخوادم التطبيقات (Application servers)، وقواعد البيانات، وعمليات التكامل مع الأطراف الثالثة المستخدمة. تعد هذه المعرفة أساسية لتخصيص الهجمات اللاحقة، حيث تتيح اختيار برمجيات الاستغلال (Exploits) الخاصة بالمنصة وتحديد الاختناقات المحتملة أو الأجهزة الوسيطة سيئة التكوين مثل موازنات الحمل (Load Balancers) وجدران حماية تطبيقات الويب (WAFs). يستغل المهاجم هذا المخطط الهيكلي للانتقال من المسح العام إلى الاستغلال المستهدف لحزمة البرمجيات (Software Stack) المحددة ومكوناتها المترابطة.

| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-INFO-10 |
| **CWE** | CWE-200 |
| **حالة الاختبار** | لم يتم تنفيذه (Not Performed) |
| **الخطورة** | معلوماتي / منخفضة* |

> *تصبح الخطورة متوسطة (Medium) إذا تم الكشف عن طوبولوجيا الشبكة الداخلية أو عناوين البروتوكول الإنترنت (IP) الخاصة.

### هل يمكن تحديد تقنيات خادم الويب وخادم التطبيقات بدقة؟
- [ ] لا — لا يمكن التحديد بسبب التحصين (Hardening) الفعال أو التعمية (Obfuscation)  
- [ ] نعم — نوع التقنية معروف ولكن لا يمكن تحديد الإصدارات (Versions) المحددة  
- [ ] نعم — يتم الإفصاح عن التقنية والإصدارات المحددة بالكامل عبر الرؤوس أو تواقيع الملفات *(معلوماتي)*  

### هل يمكن اكتشاف الأجهزة الوسيطة (مثل WAFs، وموازنات الحمل، والوكلاء) في مسار الاتصال؟
- [ ] لا — لم يتم اكتشاف أي أجهزة وسيطة أو أنها شفافة تماماً  
- [ ] نعم — يُشتبه في وجودها من خلال التوقيت أو السلوك، ولكن لم يتم تأكيد هويتها  
- [ ] نعم — تم تحديد منتجات معينة (مثل Cloudflare، Nginx، F5) عبر رؤوس فريدة أو سلوك محدد  

### هل يسرب التطبيق معلومات حول هيكل المجلدات الداخلي أو طوبولوجيا الشبكة؟
- [ ] لا — لم يتم الإفصاح عن المسارات الداخلية أو عناوين IP  
- [ ] نعم — يتم تسريب المسارات الداخلية في رسائل الخطأ أو تتبع المكدس (Stack Traces) ولكن لا يتم تسريب عناوين IP الداخلية  
- [ ] نعم — يتم الإفصاح عن كل من هيكل المجلدات الداخلي وعناوين IP الخاصة  

### هل يمكن استنتاج نوع وإصدار قاعدة بيانات الخلفية (Backend Database) من سلوك التطبيق؟
- [ ] لا — لا يمكن تحديد تفاصيل قاعدة البيانات  
- [ ] نعم — تم استنتاج نوع قاعدة البيانات (مثلاً PostgreSQL، MSSQL) عبر رسائل الخطأ أو السلوك  
- [ ] نعم — يتم الإفصاح عن نوع قاعدة البيانات وإصدارها بشكل صريح في جسم الاستجابة أو الرؤوس  

### هل التطبيق مستضاف لدى مزودي خدمات سحابية (Cloud Service Providers) معروفين أو شبكات توصيل محتوى (CDNs) محددة؟
- [ ] لا — لا يمكن تحديد البنية التحتية للاستضافة  
- [ ] نعم — تم تحديد مزود الخدمة السحابية (AWS، Azure، GCP) ولكن لم يتم تحديد خدمات معينة  
- [ ] نعم — تم تخطيط خدمات سحابية محددة (مثل S3 buckets، Lambda، Azure Functions) ويمكن الوصول إليها
