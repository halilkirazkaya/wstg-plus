## WSTG-ATHN-01 — اختبار نقل بيانات الاعتماد عبر قناة مشفرة (Testing for Credentials Transported over an Encrypted Channel)

يضمن اختبار نقل بيانات الاعتماد عبر قناة مشفرة حماية بيانات المصادقة الحساسة (Sensitive Authentication Data)، مثل أسماء المستخدمين وكلمات المرور ورموز الجلسات (Session Tokens)، من الاعتراض أثناء الانتقال (Interception during transit). يمكن للمهاجمين المتواجدين على الشبكة — مثل هجمات "رجل في المنتصف" (Man-in-the-Middle - MitM) على شبكات الواي فاي العامة أو الشبكات الداخلية المخترقة — استخدام أدوات التنصت على الحزم (Packet Sniffing) لالتقاط بيانات الاعتماد في نص صريح (Cleartext) إذا لم يتم فرض بروتوكول TLS/SSL بشكل صحيح. تحدث هذه الثغرة الأمنية عادةً في نقاط نهاية تسجيل الدخول (Login Endpoints)، ونماذج إعادة تعيين كلمة المرور، ورؤوس مصادقة واجهة برمجة التطبيقات (API Authentication Headers) حيث يفشل التطبيق في فرض HTTPS أو يقوم بإعادة التوجيه من HTTP بشكل غير صحيح. يمنح الاستغلال الناجح للمهاجم وصولاً كاملاً إلى حسابات المستخدمين، مما يؤدي إلى اختراق كامل لبيانات المستخدم واحتمالية التحرك الجانبي (Lateral Movement) داخل التطبيق.

### هل يتم فرض بروتوكول HTTPS لعملية المصادقة بالكامل؟
- [ ] نعم — تم تفعيل HSTS (HTTP Strict Transport Security) ويتم فرض جميع حركة المرور بصرامة عبر HTTPS  
- [ ] نعم — تحدث إعادة التوجيه إلى HTTPS، ولكن HSTS غير مفعل  
- [ ] لا — يمكن إرسال بيانات الاعتماد عبر بروتوكول HTTP غير المشفر  

### هل يتم تقديم صفحات ونماذج تسجيل الدخول عبر قناة مشفرة؟
- [ ] نعم — الصفحة التي تحتوي على نموذج تسجيل الدخول يتم تسليمها عبر HTTPS  
- [ ] لا — يتم تقديم صفحة تسجيل الدخول عبر HTTP، مما يسمح باختطاف إجراء النموذج (Form-action hijacking) أو التنصت على بيانات الاعتماد  

### هل يتم تطبيق خاصية `Secure` على جميع ملفات تعريف ارتباط الجلسة الحساسة؟
- [ ] نعم — خاصية `Secure` (Secure flag) مطبقة على جميع ملفات تعريف ارتباط المصادقة والجلسة (Session Cookies)  
- [ ] نعم — خاصية `Secure` مطبقة على بعض ملفات تعريف الارتباط فقط  
- [ ] لا — خاصية `Secure` غير مطبقة، مما يسمح بتسريب ملفات تعريف الارتباط عبر الطلبات غير المشفرة  

### هل يدعم الخادم إصدارات TLS ضعيفة أو مجموعات تشفير غير آمنة؟
- [ ] لا — تم تفعيل إصدارات TLS الحديثة فقط (+1.2) ومجموعات التشفير (Cipher Suites) القوية  
- [ ] نعم — الإصدارات القديمة (TLS 1.0/1.1) أو التشفيرات الضعيفة (RC4, DES, CBC) مدعومة  

### هل يمنع التطبيق إرسال بيانات الاعتماد إلى نطاقات خارجية عبر قنوات غير مشفرة؟
- [ ] نعم — لا يتم إرسال بيانات حساسة إلى نطاقات خارجية (Third-party domains) أو يتم فرض جميع الاستدعاءات الخارجية عبر HTTPS  
- [ ] لا — يتم إرسال رؤوس المصادقة أو بيانات الاعتماد إلى نقاط نهاية تابعة لجهات خارجية (مثل أدوات التحليل أو SSO) عبر HTTP
