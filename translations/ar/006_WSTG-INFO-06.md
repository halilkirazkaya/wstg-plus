## WSTG-INFO-06 — تحديد نقاط دخول التطبيق (Identify Application Entry Points)

تتضمن عملية تحديد نقاط دخول التطبيق (Identify Application Entry Points) رسم خارطة لسطح الهجوم (Attack Surface) بالكامل من خلال تعداد جميع الطرق الممكنة التي يمكن للمستخدم أو النظام الخارجي من خلالها التفاعل مع التطبيق. تشمل هذه العملية اكتشاف جميع عناوين (URLs)، ونقاط نهاية واجهة برمجة التطبيقات (API Endpoints)، والمعاملات (Parameters) سواء كانت (GET، POST، أو المستندة إلى ملفات تعريف الارتباط Cookie-based)، وترويسات (HTTP Headers)، والبروتوكولات المتخصصة مثل (WebSockets) أو (gRPC). بالنسبة لمختبر الاختراق (Penetration Tester)، تُعد هذه المرحلة حيوية لأن الثغرات غالباً ما توجد في واجهات برمجة التطبيقات الخفية (Shadow APIs) غير الموثقة، أو نقاط النهاية القديمة (Legacy Endpoints)، أو هياكل المعاملات المعقدة التي تم التغاضي عنها أثناء التطوير. يضمن التعداد الشامل ألا يظل أي مكون من مكونات التطبيق بمثابة "صندوق أسود" (Black Box) غير مختبر، مما يمنع المهاجمين من العثور على مسارات غير مراقبة للوصول إلى النظام.

### هل تم تعداد جميع معاملات GET و POST عبر التطبيق؟
- [ ] نعم — تم الانتهاء من التعداد الشامل عبر الزحف (Crawling) الآلي واليدوي  
- [ ] نعم — تم تحديد المعاملات الشائعة فقط عبر الزحف القياسي  
- [ ] لا — المعاملات غير محددة (Unidentified) بشكل كبير أو لم تُختبر  

### هل يتم البحث عن المعاملات غير الموثقة أو المخفية (مثل debug، admin) باستخدام تقنيات (Fuzzing)؟
- [ ] لا — اكتشاف المعاملات المخفية ليس ضرورياً لهذا النطاق المحدد  
- [ ] نعم — تم إجراء فحص المعاملات (Parameter Fuzzing) (على سبيل المثال، باستخدام `Arjun`) دون العثور على نتائج  
- [ ] نعم — تم اكتشاف معاملات مخفية ورسم خريطتها لمزيد من الاختبارات  
- [ ] لا — لم يتم إجراء عملية اكتشاف المعاملات المخفية  

### هل تم تحديد جميع طرق HTTP المدعومة (PUT، DELETE، PATCH، إلخ) لكل نقطة نهاية؟
- [ ] نعم — يتم تطبيق اكتشاف الطرق (Method Discovery) عبر جميع نقاط النهاية المكتشفة  
- [ ] نعم — يتم تطبيق اكتشاف الطرق فقط على نقاط النهاية ذات الأهمية العالية أو الحساسة  
- [ ] لا — تم تحديد طرق GET و POST القياسية فقط  

### هل تم تخطيط نقاط الدخول غير القياسية مثل WebSockets أو gRPC أو الترويسات المخصصة (Custom Headers)؟
- [ ] لا — التطبيق لا يستخدم بروتوكولات غير قياسية أو ترويسات مخصصة  
- [ ] نعم — تم تحديد جميع نقاط الدخول الخاصة بالبروتوكولات والترويسات المخصصة  
- [ ] لا — توجد نقاط دخول غير قياسية ولكن لم يتم تخطيطها  

### هل تم تخطيط سطح واجهة برمجة التطبيقات (API Surface) بالكامل (بما في ذلك نقاط النهاية ذات الإصدارات مثل /v1/ أو /v2/)؟
- [ ] لا — التطبيق لا يوفّر واجهة برمجة تطبيقات (API)  
- [ ] نعم — تم تحديد وتوثيق جميع إصدارات ونقاط نهاية (API)  
- [ ] نعم — تم تحديد إصدار (API) الحالي للإنتاج فقط  
- [ ] لا — لا تزال نقاط نهاية (API) غير مخططة أو تم اكتشافها جزئياً فقط
