## WSTG-CRYP-01 — اختبار أمن طبقة النقل الضعيفة (Weak Transport Layer Security)

تتضمن عملية اختبار أمن طبقة النقل الضعيفة (Weak Transport Layer Security) تحليل تنفيذ وإعداد بروتوكولات (SSL/TLS) لضمان سرية البيانات وسلامتها أثناء النقل. يستهدف المهاجمون الإعدادات الخاطئة مثل البروتوكولات القديمة (SSLv2، TLS 1.0)، ومجموعات التشفير (Cipher suites) الضعيفة (مثل NULL أو RC4 أو المجهولة)، والشهادات غير الصالحة أو الموقعة بشكل ضعيف لتنفيذ هجمات "رجل في المنتصف" (Man-in-the-Middle - MitM). يستهدف هذا الاختبار عادةً نقاط نهاية خادم الويب الخاص بالتطبيق أو موزعات الحمل (Load balancers) حيث يتم إنشاء الاتصال المشفر. يسمح الاستغلال (Exploit) الناجح للخصم باعتراض البيانات الحساسة، أو اختطاف جلسات المستخدمين، أو خفض مستوى الاتصالات إلى حالات غير آمنة من خلال تقليل إصدار البروتوكول أو فك تشفير حركة المرور.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-CRYP-01 |
| **CWE** | CWE-326 |
| **حالة الاختبار** | لم يتم التنفيذ |
| **الخطورة** | عالية / متوسطة* |

> *الخطورة تكون "عالية" في حال نقل بيانات حساسة (معلومات الهوية الشخصية PII، بيانات الاعتماد، رموز الجلسة)؛ و"متوسطة" لحركة مرور المعلومات العامة.

### هل يدعم الخادم بروتوكولات TLS/SSL قديمة أو غير آمنة؟
- [ ] لا — يتم **تمكين** بروتوكولات TLS 1.2 و TLS 1.3 فقط  
- [ ] نعم — تم **تمكين** بروتوكولات TLS 1.0 أو TLS 1.1  
- [ ] نعم — تم **تمكين** بروتوكولات SSLv2 أو SSLv3 *(حرج)*  

### هل يقبل الخادم مجموعات تشفير (Cipher suites) ضعيفة أو غير آمنة؟
- [ ] لا — يتم **دعم** التشفيرات القوية والحديثة فقط (مثل AES-GCM، ChaCha20)  
- [ ] نعم — يتم **دعم** تشفيرات ذات تعمية ضعيفة (مثل 3DES، RC4) أو ذات طول بت منخفض  
- [ ] نعم — يتم **دعم** تشفيرات NULL، أو مجهولة (ADH)، أو تشفيرات التصدير (Export ciphers) *(حرج)*  

### هل شهادة الخادم صالحة وموثوقة؟
- [ ] نعم — الشهادة صالحة، وتطابق النطاق (Domain)، وموقعة من قبل **مرجع مصدق موثوق (Trusted CA)**  
- [ ] لا — الشهادة **منتهية الصلاحية** أو تستخدم **خوارزمية توقيع ضعيفة** (مثل SHA-1)  
- [ ] لا — الشهادة **موقعة ذاتياً (Self-signed)** أو يوجد **عدم تطابق** في اسم النطاق  

### هل يدعم الخادم "إعادة التفاوض الآمن" (Secure Renegotiation) ويمنع هجمات خفض الإصدار (Downgrade attacks)؟
- [ ] نعم — إعادة التفاوض الآمن **مدعومة** وخاصية TLS Fallback SCSV **ممكنة**  
- [ ] لا — إعادة التفاوض الآمن **غير مدعومة**، مما قد يسمح بحقن (MitM injection)  
- [ ] لا — الخادم عرضة لهجمات **POODLE** أو **ROBOT**  

### هل تم تنفيذ أمن النقل الصارم لـ HTTP (HSTS) بشكل صحيح؟

| الترويسة (Header) | موجودة | مفقودة |
|--------|:-------:|:-------:|
| `Strict-Transport-Security` |  |  |

- [ ] نعم — ترويسة HSTS موجودة مع `max-age` طويل وخيار `includeSubDomains` **مُفعل**  
- [ ] نعم — ترويسة HSTS موجودة ولكن يفتقد لخيار `includeSubDomains` أو لديه **max-age قصير**  
- [ ] لا — ترويسة HSTS **غير موجودة**
