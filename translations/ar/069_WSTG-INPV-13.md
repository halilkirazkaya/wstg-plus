## WSTG-INPV-13 — حقن سلسلة التنسيق (Format String Injection)

يحدث حقن سلسلة التنسيق (Format String Injection) عندما يقوم تطبيق الويب بتمرير مدخلات يتحكم بها المستخدم مباشرة إلى وسيط سلسلة التنسيق لدالة متغيرة المعاملات (variadic function)، مثل عائلة دوال `printf` في لغة C أو مغلفات التسجيل منخفضة المستوى (low-level logging wrappers). وبالرغم من أن هذه الثغرة أقل شيوعاً في أطر عمل الويب الحديثة ذات الأكواد المدارة (managed-code web frameworks)، إلا أنها تظل حرجة في تطبيقات CGI القديمة، أو الإضافات الأصلية (native extensions)، أو أنظمة التسجيل في الحالات الاستثنائية التي تتفاعل مع مكتبات النظام منخفضة المستوى. يستغل المهاجمون ذلك عن طريق تزويد محددات تنسيق (format specifiers) مثل `%x` أو `%p` لتسريب عناوين ذاكرة حساسة وبيانات من المكدس (stack)، أو محدد `%n` لإجراء عمليات كتابة عشوائية في الذاكرة (arbitrary memory writes). وعادةً ما يؤدي الاستغلال الناجح إلى اختراق كامل للنظام عبر تنفيذ التعليمات البرمجية عن بُعد (Remote Code Execution - RCE) أو، كحد أدنى، حالة حجب الخدمة (Denial-of-Service - DoS) مؤثرة من خلال إيقاف العمليات.

### هل يقوم التطبيق بمعالجة مدخلات المستخدم عبر كود أصلي (native code) أو واجهات CGI القديمة؟
- [ ] لا — التطبيق مبني بالكامل على أطر عمل الأكواد المدارة (مثل JVM أو CLR)  
- [ ] نعم — يستخدم التطبيق JNI، أو إضافات C/C++ أصلية، أو واجهات CGI ثنائية قديمة حيث **يمكن** أن توجد ثغرات سلسلة التنسيق  

### هل يتم تمرير مدخلات المستخدم كوسيط أساسي للدوال التي تدعم التنسيق؟
- [ ] لا — يتم تمرير المدخلات كوسطاء بيانات، وسلاسل التنسيق **ثابتة ومضمنة برمجياً** (static and hardcoded)  
- [ ] نعم — يتم دمج مدخلات المستخدم مباشرة في وسيط سلسلة التنسيق، ولكن **يتم تطبيق** عملية التطهير (sanitization)  
- [ ] نعم — يتم استخدام مدخلات المستخدم مباشرة كسلسلة تنسيق و**لا يتم تطبيق** أي عملية تطهير  

### هل من الممكن تسريب محتويات الذاكرة باستخدام محددات التنسيق؟
- [ ] لا — يتم التحقق من صحة المدخلات ولا يتم معالجة محددات مثل `%p` أو `%x` أو `%s`  
- [ ] نعم — يؤدي تزويد محددات `%p` أو `%x` إلى انعكاس عناوين ذاكرة المكدس (stack) أو الكومة (heap)  
- [ ] نعم — **يمكن** استخراج بيانات حساسة (مثل المؤشرات أو ملفات تعريف المكدس "stack cookies") عبر محددات التنسيق  

### هل يمكن إيقاف التطبيق أو التلاعب به باستخدام محدد `%n`؟
- [ ] لا — المحددات التي تسمح بالكتابة في الذاكرة **معطلة** أو محظورة بواسطة المترجم/البيئة  
- [ ] نعم — يتوقف التطبيق عن العمل (DoS) عند تزويد سلاسل مثل `%s%s%s%s` أو ما شابهها  
- [ ] نعم — عمليات الكتابة العشوائية في الذاكرة عبر `%n` **ممكنة**، مما قد يؤدي إلى تنفيذ التعليمات البرمجية عن بُعد (RCE)  

### هل يمكن تجاوز حمايات الملفات الثنائية الحديثة (ASLR, DEP, Stack Canaries) من خلال هذا الحقن؟
- [ ] لا — البيئة محصنة ولا يمكن استخدام تسريب الذاكرة لتجاوز الحمايات  
- [ ] نعم — **يمكن** استخدام حقن سلسلة التنسيق لتسريب العناوين الأساسية، مما يجعل تجاوز ASLR/DEP **ممكناً**
