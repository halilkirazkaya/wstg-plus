## WSTG-INFO-07 — رسم مسارات التنفيذ عبر التطبيق (Map Execution Paths Through Application)

تتضمن عملية رسم مسارات التنفيذ (Mapping execution paths) تحديداً منهجياً لجميع نقاط النهاية (endpoints) القابلة للوصول، والتدفقات الوظيفية، وفروع اتخاذ القرار داخل تطبيق الويب. تكتسب هذه العملية أهمية حيوية لضمان عدم بقاء أي وظائف "مظلمة" - مثل الأكواد القديمة (legacy code)، أو واجهات تصحيح الأخطاء (debug interfaces)، أو مسارات واجهة برمجة التطبيقات (API) غير الموثقة - مخفية عن التدقيق الأمني، حيث تفتقر هذه المسارات غالباً إلى الضوابط الأمنية الحديثة. يستخدم المهاجمون مزيجاً من الزحف الآلي (automated spidering) والاستكشاف اليدوي لتصور هيكل التطبيق، وغالباً ما يكتشفون ثغرات مثل الوصول الإداري غير المصرح به أو عيوب منطق الأعمال (business logic flaws) خلال هذه العملية. ومن خلال ربط المدخلات باستجابات محددة من جهة الخادم (server-side)، يمكن للمختبرين تحديد منطق المعالجة الحساس الذي يتطلب اختباراً معمقاً ومستهدفاً لضمان تغطية أمنية قوية.

### هل تمت فهرسة التطبيق بالكامل باستخدام الزحف الآلي واليدوي؟
- [ ] نعم — تم إكمال خريطة شاملة لجميع الموارد المرئية والمرتبطة.
- [ ] نعم — تم إكمال الزحف الآلي ولكن الاستكشاف اليدوي لا يزال قيد الانتظار.
- [ ] لا — لم يتم فهرسة التطبيق.

### هل تم تحديد نقاط نهاية مخفية أو غير موثقة عبر تحليل ملفات JS أو التخمين على المجلدات (directory brute-forcing)؟
- [ ] لا — لم يتم اكتشاف مسارات غير موثقة عبر تحليل القنوات الجانبية (side-channel analysis).
- [ ] نعم — تم اكتشاف مسارات مخفية ولكن لا يمكن الوصول إليها بدون تفويض.
- [ ] نعم — نقاط النهاية غير الموثقة قابلة للوصول وتوفر وظائف حساسة *(حرجة / عالية)*.

### هل يمكن تغيير مسارات التنفيذ عبر معاملات (parameters) أو ترويسات (headers) غير قياسية؟
- [ ] لا — المعاملات مثل `debug` أو `test` أو `admin` لا تؤثر على تدفق التنفيذ.
- [ ] نعم — المعاملات تغير المخرجات ولكنها لا تتجاوز ضوابط الأمن.
- [ ] نعم — يتم تطبيق معاملات تغيير المسار وتسمح بتجاوز المنطق المقصود.

### هل تم رسم تدفق منطق الأعمال متعدد الخطوات (مثل المصادقة متعددة العوامل (multi-factor auth) أو عملية الدفع) بالكامل؟
- [ ] نعم — تم تحديد جميع الحالات والانتقالات في العمليات متعددة الخطوات.
- [ ] لا — لا يمكن رسم انتقالات آلة الحالة (state machine) المعقدة بالكامل.

### هل ملفات توثيق API (مثل Swagger/OpenAPI) أو خرائط جهة العميل (Source Maps) مكشوفة؟
- [ ] لا — لا توجد خرائط معمارية أو ملفات توثيق مكشوفة علناً.
- [ ] نعم — التوثيق موجود ولكنه محمي بواسطة المصادقة.
- [ ] نعم — واجهة `Swagger UI` أو ملفات `OpenAPI JSON` أو `JS Source Maps` مفعلة ويمكن الوصول إليها علناً.
