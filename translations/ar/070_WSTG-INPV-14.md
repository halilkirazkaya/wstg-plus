## WSTG-INPV-14 — اختبار الثغرات المحضنة (Testing for Incubated Vulnerabilities)

تحدث الثغرات المحضنة (Incubated Vulnerabilities)، والتي يُشار إليها أيضاً بالثغرات المستمرة (Persistent) أو ثغرات الدرجة الثانية (Second-order vulnerabilities)، عندما يتم تخزين حمولة خبيثة (Malicious Payload) بواسطة التطبيق في حالة "خاملة" ثم يتم تنفيذها أو معالجتها لاحقاً في سياق مختلف. يستهدف نمط الهجوم هذا عادةً الأنظمة الخلفية (Back-end systems)، أو وحدات التحكم الإدارية (Administrative consoles)، أو أدوات إعداد التقارير الآلية التي تستخرج البيانات من قاعدة بيانات مشتركة أو نظام ملفات دون إعادة التحقق (Re-validation) الكافي أو ترميز المخرجات (Output encoding). يجب على مختبري الاختراق (Pentesters) تتبع دورة حياة مدخلات المستخدم من نقطة الإدخال ("الحاضنة") إلى كل موقع يتم فيه عرض تلك البيانات أو استخدامها في النهاية بواسطة مكونات أخرى أو تطبيقات ثانوية. غالباً ما يؤدي الاستغلال الناجح إلى نتائج عالية التأثير مثل اختطاف جلسة الإدارة (Administrative session hijacking) عبر ثغرة حقن البرمجيات عبر المواقع المخزنة (Stored XSS) أو تسريب البيانات من خلال حقن لغة الاستعلامات البنيوية من الدرجة الثانية (Second-order SQL Injection) في وحدات التقارير الداخلية.

### هل توجد نقاط نهاية (Endpoints) يتم فيها تخزين مدخلات يتحكم بها المستخدم لاسترجاعها لاحقاً بواسطة مستخدمين أو عمليات أخرى؟
- [ ] لا — التطبيق **لا** يقوم بتخزين مدخلات المستخدم لاستخدامها لاحقاً  
- [ ] نعم — المدخلات **يتم** تخزينها في حقول قاعدة البيانات، أو ملفات السجلات (Log files)، أو إعدادات التكوين (Configuration settings)  

### هل يتم تطبيق التحقق من صحة المدخلات (Input validation) أو التطهير (Sanitization) قبل كتابة البيانات في طبقة التخزين المستمر (Persistent storage layer)؟
- [ ] نعم — يتم تطبيق تحقق صارم بناءً على قائمة السماح (Allow-list) عند نقطة الإدخال  
- [ ] نعم — يتم تطبيق تطهير عام ولكن التخطي (Bypass) **ممكن** عبر الترميز أو الأحرف غير القياسية  
- [ ] لا — يتم تخزين المدخلات في شكلها الخام وغير المسبوق التحقق  

### هل يتم ترميز البيانات المخزنة أو تطهيرها بشكل صحيح عندما يتم عرضها في الواجهات الإدارية أو الثانوية؟
- [ ] نعم — يتم تطبيق ترميز مخرجات مدرك للسياق (Context-aware output encoding) في كل مكان يتم فيه عرض البيانات  
- [ ] نعم — الترميز **مطبق** في بعض المواقع ولكنه **مفقود** في مواقع أخرى (مثل لوحة تحكم المسؤول الداخلية)  
- [ ] لا — يتم عرض البيانات دون أي ترميز أو تطهير، مما يسمح بتنفيذ الحمولة (Payload)  

### هل يمكن للحمولات المخزنة في قاعدة البيانات أن تؤثر على عمليات الدفعات الخلفية (Back-end batch processes) أو أنظمة المراقبة خارج النطاق (Out-of-band monitoring systems)؟
- [ ] لا — تستخدم العمليات الخلفية طرق تحليل آمنة أو استعلامات بارامترية (Parameterized queries)  
- [ ] نعم — الحمولات المخزنة **يمكن** أن تحفز تفاعلات في المنطق الخلفي (مثل CSV injection، أو حقن الأوامر (Command Injection) في السجلات)  
- [ ] نعم — الحمولات المخزنة **يمكن** أن تحفز طلبات خارج النطاق (OOB) إلى خوادم يسيطر عليها المهاجم
