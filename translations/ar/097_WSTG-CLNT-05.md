## WSTG-CLNT-05 — اختبار حقن ملفات التنسيق المتعاقبة (CSS Injection)

يحدث حقن ملفات التنسيق المتعاقبة (CSS Injection) عندما يسمح التطبيق للمدخلات التي يوفرها المستخدم بالتأثير على صفحات التنسيق المتعاقبة (Cascading Style Sheets - CSS) لصفحة الويب دون إجراء تطهير (Sanitization) أو هروب (Escaping) مناسب. وعلى الرغم من أنه يُنظر إليها غالباً على أنها مشكلة تجميلية، إلا أنه يمكن للمهاجمين الاستفادة من حقن CSS لتسريب البيانات الحساسة، مثل رموز الحماية من تزوير الطلبات عبر المواقع (CSRF tokens) أو معرفات الجلسة (Session IDs)، وذلك باستخدام محددات السمات (Attribute selectors) وخصائص صور الخلفية (background-image) لتحفيز طلبات خارجية. تحدث هذه الثغرة عادةً في نقاط النهاية حيث تنعكس تفضيلات المستخدم (مثل السمات المخصصة أو ألوان الخطوط) داخل قوالب التنسيق `<style>` أو سمات التنسيق المضمنة (inline style attributes). من منظور المهاجم، يمكن أن يؤدي الاستغلال الناجح إلى إعادة توجيه واجهة المستخدم (UI redressing)، أو التصيد الاحتيالي (Phishing) من خلال تعديل التصميم بشكل غير مصرح به، أو تسريب البيانات بشكل خفي في البيئات التي قد تمنع فيها سياسات أمن المحتوى (Content Security Policies - CSP) الصارمة الهجمات القائمة على جافا سكريبت.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-CLNT-05 |
| **CWE** | CWE-74 |
| **حالة الاختبار** | لم يُنفذ |
| **الخطورة** | متوسطة / عالية* |

> *تصبح الخطورة "عالية" إذا كانت نقطة الحقن تسمح بتسريب سمات حساسة مثل رموز CSRF أو إذا كان من الممكن استخدامها لإعادة توجيه واجهة المستخدم (UI redressing) في تدفقات عمل حساسة.

### هل يقوم التطبيق بعكس مدخلات يتحكم فيها المستخدم ضمن سياقات CSS؟
- [ ] لا — لا يتم عكس المدخلات في وسوم التنسيق أو السمات أو ملفات CSS
- [ ] نعم — يتم عكس المدخلات ولكنها تقتصر بشكل صارم على القيم الأبجدية الرقمية الآمنة
- [ ] نعم — يتم عكس المدخلات داخل سمة `style` أو قالب `<style>`

### هل يتم تطهير المحارف الوصفية (Metacharacters) والوظائف الخاصة بـ CSS بشكل صحيح؟
- [ ] نعم — يتم **الهروب بشكل صحيح** من المحارف مثل `{` و `}` و `:` والوظائف مثل `url()`
- [ ] نعم — التطهير مفعل ولكن **من الممكن** تجاوزه عبر الترميز أو تعليقات CSS
- [ ] لا — لا يتم تطبيق أي تطهير على المحارف الوصفية لـ CSS

### هل من الممكن تسريب البيانات باستخدام محددات CSS؟
- [ ] لا — المحدِّدات **لا يمكنها** تحفيز طلبات خارجية أو تسريب البيانات
- [ ] نعم — تسريب البيانات الجزئي **ممكن** عبر محددات السمات و `background-image`
- [ ] نعم — التسريب الكامل للرموز الحساسة **ممكن** باستخدام تقنيات التخمين (Brute Force) المؤتمتة

### هل تخفف سياسة أمن المحتوى (CSP) من مخاطر حقن CSS؟
- [ ] نعم — يتم تقييد `style-src` إلى 'self' ويتم **تقييد** `img-src` أو `connect-src`
- [ ] نعم — سياسة CSP موجودة ولكنها تستخدم 'unsafe-inline' أو تسمح بنطاقات خارجية عامة (wildcard)
- [ ] لا — لا توجد سياسة CSP مطبقة لمنع تحميل الموارد الخارجية عبر CSS

### هل يمكن استخدام الثغرة لتنفيذ إعادة توجيه واجهة المستخدم (UI redressing) أو التصيد الاحتيالي؟
- [ ] لا — نطاق الحقن محدود للغاية بحيث لا يسمح بتعديل التصميم بشكل كبير
- [ ] نعم — تعديل واجهة المستخدم **ممكن**، مما يسمح بتركيب عناصر خبيثة
- [ ] نعم — التحكم الكامل في تصميم الصفحة **ممكن**، مما يسهل هجمات التصيد الاحتيالي المقنعة للغاية
