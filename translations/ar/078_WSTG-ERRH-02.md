## WSTG-ERRH-02 — اختبار تتبع المكدس (Stack Traces)

يتم إنشاء تتبعات المكدس (Stack Traces) عندما يفشل التطبيق في معالجة استثناء (Exception) بشكل سليم، مما يؤدي إلى الكشف عن حالة التنفيذ الداخلية وتسلسل الاستدعاءات (Call Hierarchy) للمستخدم النهائي. بالنسبة لمختبر الاختراق، تعتبر هذه التتبعات ذات قيمة عالية لأنها تكشف عن إطار عمل التطبيق (Application Framework)، وإصدارات المكتبات، ومسارات نظام الملفات الداخلية، وتدفق المنطق (Logic Flow)، مما يقلل بشكل كبير من الجهد المطلوب لعملية الاستطلاع (Reconnaissance). تتضمن عملية الاستغلال (Exploit) إثارة الأخطاء عمدًا عبر مدخلات مشوهة (Malformed Input)، أو أنواع بيانات غير متوقعة، أو انتهاكات لشروط الحدود (Boundary Condition Violations) لإجبار التطبيق على الدخول في حالة غير مستقرة واستخراج معلومات حول بيئة التكنولوجيا الأساسية. غالبًا ما يوفر هذا التسريب السياق اللازم لتحديد المكونات الخارجية (Third-party Components) المعرضة للخطر أو لتصميم استغلالات محددة لعيوب المنطق المكتشفة ضمن مسارات الأكواد المكشوفة.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-ERRH-02 |
| **CWE** | CWE-209 |
| **حالة الاختبار** | لم يتم تنفيذه |
| **الخطورة** | منخفضة / متوسطة* |

> *تزداد الخطورة إلى "متوسطة" إذا كشف تتبع المكدس عن تفاصيل معمارية حساسة، أو استعلامات قاعدة بيانات (Database Queries)، أو بارامترات التكوين الداخلية.

### هل يتم إرجاع تتبعات المكدس الكاملة إلى العميل عند حدوث أخطاء في التطبيق؟
- [ ] لا — يعيد التطبيق صفحات خطأ عامة أو رسائل خطأ مخصصة  
- [ ] نعم — تتبعات المكدس **مفعلة** ولكن فقط لوحدات برمجية محددة وغير حساسة  
- [ ] نعم — تتبعات المكدس الكاملة **مفعلة** ومرئية في جسم استجابة HTTP  

### هل تكشف رسائل الخطأ عن معلومات بيئية حساسة؟
- [ ] لا — لا تحتوي رسائل الخطأ على أي تفاصيل تقنية أو بيئية  
- [ ] نعم — تكشف الرسائل عن **مسارات ملفات داخلية** أو **هياكل المجلدات** من جانب الخادم  
- [ ] نعم — تكشف الرسائل عن **مخططات قواعد البيانات (Database Schemas)**، أو **استعلامات SQL**، أو **إصدارات المكتبات الخارجية**  

### هل يمكن إثارة تتبعات المكدس عن طريق التلاعب ببارامترات المدخلات؟
- [ ] لا — يتم التعامل مع المدخلات المشوهة بشكل سليم أو يتم رفضها عبر التحقق من الصحة (Validation)  
- [ ] نعم — يمكن إثارة التتبعات بواسطة **أنواع بيانات غير متوقعة** (على سبيل المثال، إرسال مصفوفة "Array" في مكان يتوقع فيه نص "String")  
- [ ] نعم — يتم إثارة التتبعات بواسطة **Null Bytes**، أو **الأحرف الخاصة**، أو **القيم الحدودية (Boundary Values)**  

### هل يتم تطبيق معالج أخطاء عام بشكل متسق عبر التطبيق؟
- [ ] نعم — يتم **تطبيق معالج استثناءات عام** بشكل متسق عبر جميع نقاط النهاية (Endpoints) التي تم اختبارها  
- [ ] نعم — يوجد معالج ولكنه **غير مطبق** على الأنظمة القديمة (Legacy)، أو واجهات برمجة التطبيقات (API)، أو نقاط النهاية المضافة حديثًا  
- [ ] لا — لم يتم **اكتشاف** آلية معالجة أخطاء عامة، مما يؤدي إلى ظهور أخطاء الخادم الافتراضية
