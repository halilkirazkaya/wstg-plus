## WSTG-CRYP-03 — اختبار المعلومات الحساسة المرسلة عبر قنوات غير مشفرة

يؤدي إرسال البيانات الحساسة عبر قنوات غير مشفرة، مثل بروتوكول HTTP العادي، إلى تعرض المعلومات للاعتراض والتعديل من خلال هجمات "رجل في المنتصف" (Man-in-the-Middle - MITM). تحدث هذه الثغرة عادةً عندما تفشل تطبيقات الويب في فرض بروتوكول طبقة النقل (Transport Layer Security - TLS) عبر جميع النهايات الطرفية (Endpoints)، لا سيما في المكونات القديمة، أو نماذج تسجيل الدخول، أو أثناء الانتقال الأولي من HTTP إلى HTTPS. من وجهة نظر المهاجم، يسمح ذلك بالتنصت (Sniffing) السلبي على بيانات الاعتماد (Credentials)، ورموز الجلسة (Session Tokens)، ومعلومات التعريف الشخصي (Personally Identifiable Information - PII) عبر قطاعات الشبكة المخترقة أو غير الموثوقة مثل الشبكات اللاسلكية العامة. إن ضمان تغليف جميع البيانات الحساسة داخل نفق TLS قوي يعد أمراً أساسياً للحفاظ على سرية ونزاهة تفاعلات المستخدم ومنع اختطاف الجلسة (Session Hijacking).

### هل يسمح التطبيق بالاتصال عبر بروتوكول HTTP غير المشفر للنهايات الطرفية الحساسة؟
- [ ] لا — يتم فرض جميع حركات مرور التطبيق بدقة عبر HTTPS *(الأكثر أماناً)*  
- [ ] نعم — الصفحات غير الحساسة يمكن الوصول إليها عبر HTTP ولكن النهايات الطرفية الحساسة **ليست كذلك**  
- [ ] نعم — النهايات الطرفية الحساسة (تسجيل الدخول، الدفع، الملف الشخصي) **يمكن** الوصول إليها عبر HTTP غير المشفر *(خطورة عالية)*  

### هل توجد إعادة توجيه شاملة من HTTP إلى HTTPS؟
- [ ] نعم — يقوم التطبيق بإجراء إعادة توجيه 301/302 إلى HTTPS لجميع طلبات HTTP  
- [ ] نعم — يتم إجراء إعادة التوجيه فقط لنهايات طرفية حساسة محددة  
- [ ] لا — **لا يتم تطبيق** أي إعادة توجيه من HTTP إلى HTTPS  

### هل تم تطبيق سياسة أمن النقل الصارمة (HSTS)؟
- [ ] نعم — تم **تفعيل** HSTS مع قيمة `max-age` طويلة وتتضمن توجيهات `includeSubDomains` و `preload`  
- [ ] نعم — تم **تفعيل** HSTS ولكنها تفتقر إلى توجيهات `includeSubDomains` أو `preload`  
- [ ] لا — سياسة HSTS **غير مفعلة** على خادم التطبيق

### هل يقوم التطبيق بنقل بيانات حساسة إلى واجهات برمجة التطبيقات (APIs) التابعة لجهات خارجية عبر قنوات غير مشفرة؟
- [ ] لا — يتم إجراء جميع استدعاءات API الخارجية عبر أنفاق مشفرة (HTTPS)  
- [ ] نعم — يتم إرسال بعض بيانات القياس عن بُعد (Telemetry) غير الحساسة عبر HTTP  
- [ ] نعم — يتم إرسال مفاتيح API، أو معلومات PII، أو بيانات الاعتماد إلى جهات خارجية عبر HTTP غير مشفر
