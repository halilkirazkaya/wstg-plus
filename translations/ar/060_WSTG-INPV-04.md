## WSTG-INPV-04 — اختبار تلوث معلمات HTTP (HTTP Parameter Pollution)

يحدث تلوث معلمات HTTP (HTTP Parameter Pollution - HPP) عندما يتلقى التطبيق معلمات HTTP متعددة تحمل الاسم نفسه ويعالجها بطريقة غير متسقة أو غير آمنة. من خلال توفير معلمات مكررة، يمكن للمهاجم التلاعب بالمنطق الداخلي للتطبيق، مما قد يؤدي إلى تجاوز جدران حماية تطبيقات الويب (Web Application Firewalls - WAF)، أو مرشحات التحقق من المدخلات، أو آليات التحكم في الوصول. تعتمد هذه الثغرة بشكل كبير على خادم الويب الأساسي أو إطار عمل التطبيق، والذي قد يختار المعلمة الأولى، أو الأخيرة، أو نسخة مدمجة من المعلمات المتكررة. يستهدف الاستغلال عادةً نقاط النهاية التي تمرر المعلمات إلى واجهات برمجة التطبيقات (API) الخلفية، أو استعلامات قاعدة البيانات، أو آليات إعادة التوجيه، مما يؤدي إلى آثار تتراوح بين برمجة المواقع المتقاطعة (Cross-Site Scripting - XSS) إلى تسريب البيانات غير المصرح به.

### كيف يتعامل التطبيق مع معلمات HTTP المتعددة التي تحمل الاسم نفسه؟
- [ ] لا — يتم **رفض** المعلمات المكررة أو تجاهلها من قبل الخادم  
- [ ] نعم — يختار الخادم باستمرار التواجد **الأول** أو **الأخير** فقط  
- [ ] نعم — يقوم الخادم **بدمج** القيم، مما قد يسمح بعمليات الحقن (Injection)  

### هل يمكن استغلال HPP لتجاوز ضوابط الأمان مثل WAF أو مرشح الإدخال؟
- [ ] لا — تقوم ضوابط الأمان بفحص **جميع** تواجدات المعلمة بشكل صحيح  
- [ ] نعم — يقوم الـ WAF أو المرشح بفحص التواجد **الأول** فقط، مما يسمح بمرور تواجد **ثانٍ** خبيث  
- [ ] نعم — يسمح الدمج للمهاجم **بتقسيم** الحمولة (Payload) عبر معلمات متعددة لتجنب الاكتشاف  

### هل يعكس التطبيق المعلمات الملوثة في الاستجابة دون تنقية مناسبة؟
- [ ] لا — يتم تنقية (Sanitized) المعلمات أو ترميزها قبل عكسها في واجهة المستخدم (UI)  
- [ ] نعم — يؤدي التلوث إلى محتوى منعكس، ولكن يتم **تطبيق** التنقية  
- [ ] نعم — يؤدي التلوث إلى محتوى منعكس و**لا يتم** تطبيق التنقية، مما يؤدي إلى XSS أو أخطاء منطقية  

### هل المعلمات التي يتم تمريرها إلى استدعاءات API الداخلية أو الأنظمة الخلفية عرضة للتلوث؟
- [ ] لا — تستخدم طلبات الأنظمة الخلفية (Back-end) طرق بناء آمنة وغير ديناميكية  
- [ ] نعم — يسمح HPP للمهاجم **بحقن** أو **تجاوز** المعلمات في هيكل طلب النظام الخلفي  

### هل يظهر التطبيق سلوكًا مختلفًا عند تلوث المعلمات في طلبات GET مقابل طلبات POST؟
- [ ] لا — السلوك متسق عبر جميع طرق HTTP  
- [ ] نعم — معلمات GET فقط هي العرضة للتلوث  
- [ ] نعم — معلمات POST (الجسم/Body) فقط هي العرضة للتلوث
