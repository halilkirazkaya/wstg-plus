## WSTG-CRYP-04 — اختبار البدائيات التشفيرية الضعيفة (Testing for Weak Cryptographic Primitives)

تتضمن البدائيات التشفيرية الضعيفة استخدام خوارزميات قديمة أو غير آمنة، مثل MD5 أو SHA-1 أو DES أو RC4، والتي تكون عرضة لهجمات التصادم (Collision Attacks)، أو التحليل التكراري، أو فك التشفير بالقوة الغاشمة (Brute Force). تحدث هذه نقاط الضعف عادةً في تجزئة كلمات المرور (Password Hashing)، أو تشفير البيانات أثناء السكون (Data Encryption at Rest)، أو إنشاء رموز الجلسة (Session Token Generation)، أو التحقق من التوقيع الرقمي داخل الخلفية البرمجية للتطبيق (Application Backend). من وجهة نظر المهاجم، يسمح تحديد هذه البدائيات باختراق سلامة البيانات وسريتها، مثل كسر التجزئات (Hashes) التي تم اعتراضها لاستعادة كلمات المرور النصية الصريحة أو تزوير رموز المصادقة (Authentication Tokens). يجب على التطبيقات الحديثة بدلاً من ذلك استخدام بدائيات قياسية، مقاومة للتصادم، وعالية الاعتلاج (High-entropy) مثل Argon2 أو Bcrypt أو AES-GCM لضمان حماية قوية ضد تحليل التشفير (Cryptanalysis).

### هل تُستخدم خوارزميات تجزئة (Hashing Algorithms) مهجورة مثل MD5 أو SHA-1 لتخزين البيانات الحساسة؟
- [ ] لا — يستخدم التطبيق خوارزميات حديثة مقاومة للتصادم مثل Argon2 أو Bcrypt  
- [ ] نعم — تُستخدم الخوارزميات المهجورة ولكن **فقط** لفحوصات السلامة غير المتعلقة بالأمن  
- [ ] نعم — **تُستخدم** الخوارزميات المهجورة لكلمات المرور أو الرموز الحساسة *(عالية)*  

### هل يتم توظيف شفرات تشفير متماثل (Symmetric Encryption) ضعيفة مثل DES أو 3DES أو RC4 حاليًا؟
- [ ] لا — يتم استخدام AES (بـ 128 بت أو أعلى) في وضع آمن مثل GCM أو CBC  
- [ ] نعم — الشفرات القديمة **مُمكنة** للتوافق مع الإصدارات السابقة ولكنها **ليست** الافتراضية  
- [ ] نعم — الشفرات القديمة هي الطريقة الأساسية لحماية البيانات أثناء السكون أو الانتقال  

### هل يطبق التطبيق منطقًا تشفيريًا "محلي الصنع" (Roll-your-own) أو غير قياسي؟
- [ ] لا — يلتزم التطبيق تمامًا بمكتبات التشفير القياسية والخاضعة لمراجعة الأقران  
- [ ] نعم — توجد أغلفة (Wrappers) مخصصة ولكن البدائيات الأساسية **هي** معايير صناعية قياسية  
- [ ] نعم — **يتم تطبيق** خوارزميات تشفير ملكية أو منطق مخصص على البيانات الحساسة *(حرجة)*  

### هل تُدار أملاح التشفير (Salts) ومتجهات التهيئة (Initialization Vectors - IVs) وفقًا لأفضل الممارسات؟
- [ ] نعم — الأملاح فريدة لكل مستخدم ومتجهات التهيئة (IVs) غير متوقعة لكل عملية  
- [ ] نعم — تُستخدم الأملاح ولكنها **ليست** فريدة عبر جميع السجلات  
- [ ] لا — الأملاح ثابتة، أو مضمنة برمجياً، أو غائبة، كما **يتم إعادة استخدام** متجهات التهيئة عبر جلسات متعددة  

### هل طول البت للمفاتيح غير المتماثلة (Asymmetric Keys) مثل RSA أو Diffie-Hellman كافٍ للمواصفات الحديثة؟
- [ ] نعم — مفاتيح RSA/DH هي 2048 بت أو أعلى، أو يتم استخدام تشفير المنحنى الإهليلجي (ECC)  
- [ ] لا — المفاتيح أقل من 2048 بت ولكن تجاوزها **ليس** تافهًا  
- [ ] لا — المفاتيح 1024 بت أو أصغر، مما يجعلها **عرضة** للتحليل إلى عوامل أو الهجمات الحسابية
