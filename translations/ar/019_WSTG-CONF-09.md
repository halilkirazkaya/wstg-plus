## WSTG-CONF-09 — اختبار أذونات الملفات (Test File Permission)

يتضمن اختبار أذونات الملفات (File Permission) تدقيق مستويات التحكم في الوصول (Access Control) المخصصة للملفات والمجلدات على خادم الويب (Web Server) لضمان عدم تعرض الموارد الحساسة للمستخدمين أو العمليات غير المصرح لها بشكل مفرط. يمكن أن تسمح الأذونات التي تم تكوينها بشكل غير صحيح للمهاجمين بقراءة ملفات التكوين الحساسة التي تحتوي على بيانات اعتماد قاعدة البيانات (Database Credentials)، أو عرض الكود المصدري (Source Code) المملوك، أو تعديل البرامج النصية من جانب الخادم لتحقيق تنفيذ التعليمات البرمجية عن بعد (Remote Code Execution - RCE). تحدث هذه الثغرة عادةً خلال مرحلة النشر (Deployment) عندما تُترك علامات "قابلة للقراءة للجميع" (World-readable) أو "قابلة للكتابة للجميع" (World-writable) على مجلدات التطبيق الحيوية، مثل مسارات التكوين، أو مجلدات السجلات (Logs)، أو مستودعات الرفع. من وجهة نظر المهاجم، غالبًا ما يكون اكتشاف ملف غير مؤمن بشكل صحيح بمثابة المحفز الأساسي لتصعيد الامتيازات (Privilege Escalation) أو الاختراق الكامل للنظام.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-CONF-09 |
| **CWE** | CWE-732 |
| **حالة الاختبار** | لم يتم تنفيذه |
| **الخطورة** | متوسطة / عالية* |

> *تصبح الخطورة "عالية" إذا كانت ملفات التكوين الحساسة (مثل `.env` أو `web.config`) قابلة للقراءة أو إذا كان الوصول للكتابة إلى جذر الويب (Web Root) ممكنًا.

**المراجع:**
* https://owasp.org/www-project-web-security-testing-guide/latest/4-Web_Application_Security_Testing/02-Configuration_and_Deployment_Management_Testing/09-Test_File_Permission  
* https://hacktricks.wiki/en/linux-hardening/privilege-escalation/index.html  

**الأدوات:** `ls`, `find`, `LinPeas`, `WinPeas`, `ffuf`, `dirsearch`, `Nmap`

### هل ملفات تكوين التطبيق الحساسة محمية من الوصول غير المصرح به؟
- [ ] نعم — ملفات التكوين (مثل `.env` و `config.php`) **لا يمكن الوصول إليها** عبر طلبات الويب  
- [ ] نعم — الملفات موجودة ولكن الوصول **مقتصر** على المستخدمين المحليين المصرح لهم فقط  
- [ ] لا — ملفات التكوين الحساسة **يمكن الوصول إليها** وتكشف عن بيانات الاعتماد أو الأسرار  

### هل يمكن للمهاجم تعديل الملفات أو المجلدات داخل جذر الويب (Web Root)؟
- [ ] لا — جميع مجلدات جذر الويب هي للقراءة فقط (**Read-only**) لمستخدم خادم الويب  
- [ ] نعم — توجد مجلدات قابلة للكتابة ولكن تنفيذ البرامج النصية **معطل**  
- [ ] نعم — توجد مجلدات قابلة للكتابة للجميع و**تسمح** برفع وتنفيذ البرامج النصية *(حرجة)*  

### هل البيانات التعريفية لنظام التحكم في الإصدار (Version Control Metadata) أو ملفات النسخ الاحتياطي مكشوفة بسبب الأذونات المتساهلة؟
- [ ] لا — ملفات `.git` و `.svn` وملفات النسخ الاحتياطي (مثل `.bak` و `~`) **غير موجودة** أو محمية  
- [ ] نعم — مجلدات البيانات التعريفية موجودة ولكن عرض محتويات المجلد (Directory Listing) **معطل**  
- [ ] نعم — البيانات التعريفية الحساسة أو النسخ الاحتياطية **يمكن الوصول إليها بالكامل**، مما يسمح باستعادة الكود المصدري  

### هل يعمل مستخدم خادم الويب وفق مبدأ الامتيازات الأقل (Principle of Least Privilege)؟
- [ ] نعم — تعمل عملية خادم الويب كمستخدم **مخصص ذو امتيازات منخفضة**  
- [ ] لا — تعمل عملية خادم الويب **بامتيازات غير ضرورية** (مثل `root` أو `Administrator`)  

### هل ملفات السجلات (Log Files) مؤمنة ضد القراءة أو التعديل غير المصرح به؟
- [ ] نعم — ملفات السجلات **مقتصرة** على المستخدمين الإداريين  
- [ ] نعم — ملفات السجلات قابلة للقراءة ولكنها **لا** تحتوي على رموز الجلسة (Session Tokens) أو معلومات تحديد الهوية الشخصية (PII)  
- [ ] لا — ملفات السجلات **قابلة للقراءة علنًا** وتكشف عن بيانات المعاملات الحساسة أو معلومات المستخدم
