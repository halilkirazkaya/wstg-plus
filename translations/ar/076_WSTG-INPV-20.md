## WSTG-INPV-20 — التخصيص الجماعي (Mass Assignment)

تحدث ثغرة التخصيص الجماعي (Mass Assignment)، والمعروفة أيضاً باسم الإرسال الزائد (Overposting) أو إلغاء التسلسل غير الآمن للمعاملات (Insecure Deserialization of parameters)، عندما يقوم تطبيق الويب بربط معاملات طلبات HTTP الواردة تلقائياً بخصائص الكائنات الداخلية دون تصفية مناسبة للسمات (Attribute filtering). تنتشر هذه الثغرة في أطر عمل MVC الحديثة وواجهات برمجة تطبيقات RESTful (RESTful APIs) حيث يتم إلغاء تسلسل بيانات JSON أو XML أو البيانات المشفرة بالنماذج مباشرة إلى نماذج البيانات في جانب التطبيق. يستغل المهاجمون هذا السلوك عن طريق حقن معاملات إضافية غير مدرجة في الطلب — مثل `isAdmin` أو `role` أو `account_balance` — لتعديل حقول حساسة لم يقصد المطورون تعريضها لتحكم المستخدم. يؤدي الاستغلال الناجح عادةً إلى تصعيد الامتيازات (Privilege escalation)، أو تعديل البيانات غير المصرح به، أو تجاوز مسارات عمل منطق الأعمال (Business logic) الحرجة.

### هل يستخدم التطبيق الربط التلقائي لمعاملات الطلبات الواردة؟
- [ ] لا — يتم تعيين المعاملات يدوياً لمتغيرات محددة أو كائنات آمنة  
- [ ] نعم — يتم استخدام الربط التلقائي ولكن يتم فرض قوائم بيضاء (White-lists) صارمة أو كائنات نقل البيانات (DTOs - Data Transfer Objects)  
- [ ] نعم — يتم استخدام الربط التلقائي ويمكن تعديل السمات الداخلية الحساسة  

### هل يمكن حقن السمات المتعلقة بالإدارة أو الأذونات بنجاح؟
- [ ] لا — يتم تجاهل المعاملات المحقونة، أو إزالتها، أو تتسبب في خطأ في التحقق من الصحة  
- [ ] نعم — يتم قبول المعاملات الإضافية ولكنها لا تؤثر على حالة كائن الواجهة الخلفية (Backend)  
- [ ] نعم — يؤدي حقن معاملات مثل `is_admin` أو `role` أو `permissions` بنجاح إلى تصعيد الامتيازات *(حرج)*  

### هل من الممكن تعديل منطق الأعمال الحساس أو الحقول المالية عبر الإرسال الزائد؟
- [ ] لا — الحقول مثل `price` أو `balance` أو `status` محمية من التخصيص الجماعي  
- [ ] نعم — يمكن تعديل سمات الأعمال الحساسة عن طريق إضافتها إلى جسم طلب JSON أو النموذج (Form request body)  

### هل يكشف التطبيق عن هياكل الكائنات الداخلية مما يسهل عملية تخمين المعاملات؟
- [ ] لا — تتضمن الاستجابات فقط الحقول العامة المقصودة والوثائق مقيدة  
- [ ] نعم — حقول الكائنات الداخلية مرئية في استجابات GET، مما يجعل اكتشاف المعاملات ممكناً  
- [ ] نعم — توثيق واجهة برمجة التطبيقات (Swagger/OpenAPI) يدرج صراحة الخصائص الداخلية التي يمكن تخصيصها
