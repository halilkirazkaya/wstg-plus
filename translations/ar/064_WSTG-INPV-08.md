## WSTG-INPV-08 — اختبار حقن تضمينات جانب الخادم (SSI Injection)

يحدث هجوم حقن تضمينات جانب الخادم (Server-Side Includes Injection - SSI Injection) عندما يفشل التطبيق في تطهير مدخلات المستخدم قبل دمجها في ملفات HTML التي يعالجها محرك SSI الخاص بالخادم. يستغل المهاجمون ذلك عن طريق حقن توجيهات SSI، مثل `<!--#exec cmd="id" -->` لتنفيذ أوامر القشرة (Shell Commands) العشوائية، أو قراءة الملفات المحلية، أو تسريب متغيرات البيئة (Environment Variables). توجد هذه الثغرة عادةً في التطبيقات التي تخدم ملحقات ملفات قديمة مثل `.shtml` أو `.shtm` أو `.stm` أو عندما يتم تكوين خادم الويب (Web Server) صراحةً لمعالجة SSI ضمن ملفات HTML القياسية. يمنح الاستغلال الناجح للمهاجم نفس صلاحيات عملية خادم الويب، مما يؤدي غالباً إلى اختراق كامل للنظام أو كشف البيانات الحساسة.

| **حالة الاختبار** | لم يتم إجراؤه |

### هل يدعم خادم الويب ويعالج توجيهات SSI؟
- [ ] لا — الخادم **لا** يدعم SSI أو ملحقات الملفات مثل `.shtml` **معطلة**  
- [ ] نعم — SSI **مفعل** ولكن مدخلات المستخدم **لا** تنعكس في الصفحات المعالجة  
- [ ] نعم — SSI **مفعل** ومدخلات المستخدم **تنعكس** في الصفحات المعالجة  

### هل يمكن تنفيذ أوامر نظام عشوائية عبر توجيه `#exec`؟
- [ ] لا — توجيه `#exec` **معطل** أو مقيد بواسطة تكوين الخادم  
- [ ] نعم — تنفيذ الأوامر **ممكن** عبر حمولات (Payloads) `#exec cmd` أو `#exec cgi`  

### هل تسريب الملفات الحساسة أو متغيرات البيئة ممكن؟
- [ ] لا — التوجيهات مثل `#include` أو `#config` **معطلة**  
- [ ] نعم — قراءة الملفات المحلية (مثل `/etc/passwd`) **ممكنة** عبر `#include virtual`  
- [ ] نعم — متغيرات بيئة الخادم **يمكن** تسريبها عبر `#printenv` أو `#echo`  

### هل ضوابط التحقق من المدخلات أو جدار حماية تطبيقات الويب (WAF) فعالة ضد حمولات SSI؟
- [ ] نعم — يتم تطبيق تحقق صارم من المدخلات (Input Validation) والتجاوز **غير ممكن**  
- [ ] نعم — التحقق/WAF **مطبق** ولكن التجاوز **ممكن** باستخدام ترميز الأحرف أو بناء جملة SSI مختلف  
- [ ] لا — لا يوجد أي تحقق أو حماية من WAF للأحرف الخاصة بـ SSI مثل (`<`, `!`, `#`, `-`, `"`)
