## WSTG-INFO-02 — بصمة خادم الويب (Fingerprint Web Server)

تُعد عملية تحديد بصمة خادم الويب (Web Server Fingerprinting) هي العملية المتبعة لتحديد نوع البرمجيات وإصدارها ونظام التشغيل الأساسي لخادم الويب المستهدف. يقوم المهاجمون بإجراء هذا الاستطلاع لتحديد الثغرات الأمنية المعروفة، مثل الثغرات الأمنية والتعرضات الشائعة (CVEs) غير المصححة أو نقاط الضعف في الإعدادات، والتي تختص بإصدارات معينة من Apache أو Nginx أو IIS أو تقنيات الخوادم الأخرى. يتم تحقيق ذلك عادةً من خلال تحليل ترويسات استجابة HTTP (HTTP response headers) مثل (`Server` و `X-Powered-By`) أو فحص سلوكيات البروتوكول الفريدة أو مراقبة المعلومات المسربة عبر صفحات وملفات الخطأ الافتراضية. تتيح عملية تحديد البصمة الناجحة للمهاجم تخصيص استراتيجية الاستغلال (Exploitation strategy) الخاصة به، والانتقال من الفحص العام إلى الهجمات ذات الاحتمالية العالية ضد العيوب الهيكلية المعروفة.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-INFO-02 |
| **CWE** | CWE-200 |
| **حالة الاختبار** | لم يتم تنفيذه (Not Performed) |
| **الخطورة** | معلوماتي / منخفضة* |

> *تصبح الخطورة عالية (High) إذا كشفت عملية تحديد البصمة عن إصدار خادم قديم أو منتهي الدعم (EOL) يحتوي على ثغرات أمنية حرجة معروفة.

### هل توجد سلاسل نصية تعريفية في ترويسات استجابة HTTP؟
- [ ] لا — الترويسات `Server` و `X-Powered-By` **غير موجودة** أو تحتوي على قيم عامة  
- [ ] نعم — الترويسات تكشف عن برمجيات الخادم ولكن **ليس** أرقام الإصدارات المحددة  
- [ ] نعم — الترويسات تكشف عن برمجيات الخادم المحددة وأرقام الإصدارات **الدقيقة**  

### هل تسرب صفحات الخطأ الافتراضية أو الاستجابات التي يولدها النظام تفاصيل عن الخادم؟
- [ ] لا — يتم استخدام صفحات خطأ مخصصة **ولا تكشف** عن معلومات الخادم  
- [ ] نعم — صفحات الخطأ الافتراضية تسرب اسم برمجيات الخادم و/أو إصداره  
- [ ] نعم — صفحات الخطأ تسرب تفاصيل الخادم، وإصداره، و**نظام التشغيل الأساسي**  

### هل يتم الكشف عن إصدار الخادم عبر الملفات الافتراضية أو هياكل المجلدات؟
- [ ] لا — تم **حذف** ملفات التثبيت الافتراضية، والأدلة، وسكربتات الاختبار  
- [ ] نعم — الملفات الافتراضية (مثل `info.php`, `manual/`, `test.html`) **موجودة** وتكشف عن بيانات الإصدار  

### هل يمكن استنتاج نوع الخادم بدقة من خلال تحليل السلوك الفريد؟
- [ ] لا — استجابات الخادم موحدة (normalized) وتحديد البصمة بناءً على السلوك **غير ممكن**  
- [ ] نعم — السلوكيات الفريدة (مثل ترتيب الترويسات، أو أكواد خطأ محددة، أو خصائص مكدس TCP/IP) **تسمح** بتحديد هوية الخادم  

### هل إصدار الخادم الذي تم تحديده مدعوم حالياً ومحدّث؟
- [ ] نعم — الإصدار المحدد حديث و**مدعوم** من قبل الشركة المصنعة  
- [ ] لا — الإصدار المحدد **قديم** أو **منتهي الدعم (EOL)** مع وجود ثغرات أمنية معروفة
