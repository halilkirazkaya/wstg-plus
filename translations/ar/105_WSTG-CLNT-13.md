## WSTG-CLNT-13 — اختبار تضمين البرامج النصية عبر المواقع (Cross-Site Script Inclusion - XSSI)

يحدث تضمين البرامج النصية عبر المواقع (Cross-Site Script Inclusion - XSSI) عندما يقوم تطبيق الويب بتصدير بيانات حساسة داخل ملف JavaScript يتم إنشاؤه ديناميكيًا أو استجابة JSONP، والتي يمكن لموقع يسيطر عليه المهاجم تضمينها عبر وسم `<script>`. نظرًا لأن تضمين البرامج النصية يتجاوز سياسة الأصل الواحد (Same-Origin Policy - SOP)، يمكن للمهاجم تنفيذ البرنامج النصي في سياقه الخاص وتجاوز الكائنات أو المتغيرات العامة لاستخراج (Exfiltrate) المعلومات الحساسة. تظهر هذه الثغرة عادةً في نقاط النهاية الموثقة (Authenticated Endpoints) التي تعيد بيانات خاصة بالمستخدم مثل عناوين البريد الإلكتروني، أو مفاتيح واجهة برمجة التطبيقات (API Keys)، أو البيانات الوصفية للجلسة (Session Metadata) بتنسيق برنامج نصي. من منظور المهاجم، تتضمن عملية الاستغلال (Exploit) إنشاء صفحة خبيثة تشير إلى البرنامج النصي المصاب وتلتقط التغييرات الناتجة في المتغيرات العامة أو استدعاءات الدوال.


| الحقل | القيمة |
|---|---|
| **معرف WSTG** | WSTG-CLNT-13 |
| **CWE** | CWE-200 |
| **حالة الاختبار** | لم يتم تنفيذه |
| **الخطورة** | متوسطة / عالية |

> *تصبح الخطورة عالية إذا تضمنت البيانات المسربة رموز CSRF، أو معرفات الجلسة (Session Identifiers)، أو معلومات التعريف الشخصية (PII) عالية الحساسية.

### هل تقوم أي نقاط نهاية موثقة بإرجاع JavaScript ديناميكي أو JSONP يحتوي على معلومات حساسة؟
- [ ] لا — جميع البرامج النصية ثابتة (Static) و**لا يمكن** أن تحتوي على بيانات خاصة بالمستخدم  
- [ ] نعم — توجد برامج نصية ديناميكية ولكنها **لا تحتوي** على معلومات حساسة  
- [ ] نعم — تحتوي البرامج النصية الديناميكية على معلومات تعريف شخصية (PII) أو رموز (Tokens) و**يمكن** الوصول إليها عبر أصول مختلفة (Cross-origin)  

### هل تم تطبيق الترويسة `X-Content-Type-Options: nosniff` في استجابات البرامج النصية الحساسة؟
- [ ] نعم — الترويسة **مطبقة** وتمنع استنتاج نوع الميم (MIME-type sniffing)  
- [ ] لا — الترويسة **مفقودة**، مما قد يسمح بتنفيذ محتوى ليس برنامجًا نصيًا كبرنامج نصي  

### هل البرامج النصية الحساسة محمية بآليات مكافحة XSSI مثل البادئات غير القابلة للتنفيذ أو الترويسات المخصصة؟
- [ ] نعم — تتطلب البرامج النصية ترويسة مخصصة أو رمزًا **لا يمكن** إرساله عبر وسم `<script>` القياسي  
- [ ] نعم — تستخدم البرامج النصية بادئات غير قابلة للتنفيذ (مثل `)]}'`) والتي **لا يمكن** تجاوزها بسهولة  
- [ ] لا — يمكن الوصول إلى البرامج النصية مباشرة عبر طلبات GET باستخدام بيانات الاعتماد المحيطة (Ambient Credentials) *(حرجة)*  

### هل يمكن تسريب البيانات الحساسة عن طريق تجاوز المتغيرات العامة أو النماذج الأولية (Prototypes)؟
- [ ] لا — البيانات الحساسة محصورة محليًا أو محمية عبر ميزات المتصفح الحديثة  
- [ ] نعم — يتم تعيين البيانات الحساسة لمتغيرات عامة و**يمكن** التقاطها  
- [ ] نعم — يتم تسريب البيانات من خلال استدعاءات JSONP التي **يمكن** اعتراضها
