## WSTG-BUSL-08 — اختبار رفع أنواع ملفات غير متوقعة (Test Upload of Unexpected File Types)

يسمح رفع أنواع ملفات غير متوقعة للمهاجمين بتجاوز قيود منطق العمل (Business Logic) عبر إرسال ملفات لا ينوي التطبيق معالجتها، مما قد يؤدي إلى تنفيذ التعليمات البرمجية عن بُعد (Remote Code Execution - RCE)، أو حقن النص البرمجي عبر المواقع (Cross-Site Scripting - XSS)، أو حجب الخدمة (Denial of Service - DoS). يقوم المهاجمون بفحص هذه الثغرات عن طريق تعديل امتدادات الملفات (File Extensions)، وأنواع MIME (MIME types)، والأسماء السحرية للملفات (Magic Bytes) لخداع منطق التحقق من الصحة (Validation Logic) في جانب الخادم لقبول محتوى ضار. يحدث هذا عادةً في ميزات رفع الصور الشخصية، أو أنظمة إدارة المستندات، أو مرفقات تذاكر الدعم الفني حيث لا يوجد تحقق كافٍ في الجانب الخلفي (Back-end). يمكن أن يؤدي الاستغلال الناجح إلى اختراق الخادم المضيف إذا تم تنفيذ الملف المرفوع، أو يؤدي إلى هجمات جانب العميل إذا تم تقديم الملف لمستخدمين آخرين بنوع محتوى (Content-Type) غير صحيح.

### هل يقيد التطبيق عمليات رفع الملفات بمجموعة محددة من الامتدادات؟
- [ ] نعم — يتم فرض قائمة بيضاء (Whitelist) صارمة للامتدادات والتجاوز (Bypass) **غير ممكن**  
- [ ] نعم — توجد قائمة بيضاء للامتدادات ولكن التجاوز **ممكن** عبر الامتدادات المزدوجة (Double Extensions) أو بايتات القيمة الصفرية (Null Bytes)  
- [ ] لا — **يمكن** رفع أي امتداد ملف  

### هل يتم التحقق من محتوى الملف بما يتجاوز امتداد الملف؟
- [ ] نعم — يتحقق منطق جانب الخادم من الأسماء السحرية للملفات (Magic Bytes) ويجري فحصاً عميقاً للمحتوى (Deep Content Inspection)  
- [ ] نعم — يتحقق منطق جانب الخادم من نوع MIME عبر رأس الطلب `Content-Type` فقط  
- [ ] لا — **لا يتم تطبيق** أي تحقق من المحتوى بعد فحص الامتداد  

### هل يمكن للمهاجم تنفيذ كود عبر رفع ويب شيل (Web Shell) أو سكربت؟
- [ ] لا — يتم تخزين الملفات المرفوعة في دليل غير قابل للتنفيذ أو في حاوية تخزين سحابية مثل (S3/Azure Blob)  
- [ ] نعم — يتم تخزين الملفات في دليل يمكن الوصول إليه عبر الويب ولكن التنفيذ **معطل** عبر إعدادات الخادم (Server Configuration)  
- [ ] نعم — السكربتات الضارة المرفوعة **يمكن** تنفيذها مباشرة عبر مسار URL متوقع *(حرجة)*  

### هل هجمات جانب العميل (Client-side attacks) مثل XSS ممكنة عبر أنواع الملفات المرفوعة؟
- [ ] لا — يتم تقديم الملفات مع رؤوس `Content-Disposition: attachment` و `X-Content-Type-Options: nosniff`  
- [ ] نعم — ملفات SVG أو HTML **يمكن** رفعها وعرضها في المتصفح، مما يؤدي إلى XSS  
- [ ] نعم — البيانات الوصفية للصور (EXIF) **تتم معالجتها** وانعكاسها في واجهة المستخدم دون تطهير (Sanitization)
