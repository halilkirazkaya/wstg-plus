## WSTG-CONF-11 — اختبار التخزين السحابي (Test Cloud Storage)

يتضمن اختبار سوء تكوين التخزين السحابي (Cloud Storage Misconfigurations) تحديد ومراجعة خدمات تخزين الكائنات (Object Storage Services) التي يمكن الوصول إليها بشكل عام مثل Amazon S3، أو Azure Blobs، أو Google Cloud Storage. غالبًا ما تحتوي هذه الموارد على بيانات حساسة تشمل النسخ الاحتياطية (Backups)، أو الكود المصدري (Source Code)، أو معلومات الهوية الشخصية (PII)، أو ملفات التكوين (Configuration Files) التي يتم الكشف عنها بسبب قوائم التحكم في الوصول (ACLs) أو سياسات إدارة الهوية والوصول (IAM) التي تتسم بالتساهل المفرط. يقوم المهاجمون باستقراء أسماء الحاويات (Buckets) من خلال الاستطلاع (Reconnaissance) لملفات JavaScript، وأكواد HTML المصدرية، وتقنيات الهجوم الشامل (Brute Force) للعثور على نقاط دخول لتسريب البيانات (Data Exfiltration) أو تعديل الملفات غير المصرح به. يمكن أن يؤدي الاستغلال (Exploit) إلى تأثير كبير، بما في ذلك اختراقات البيانات الكاملة أو القدرة على تقديم ملفات ضارة من نطاق موثوق.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-CONF-11 |
| **CWE** | CWE-922 |
| **حالة الاختبار** | لم يتم تنفيذه (Not Performed) |
| **الخطورة** | عالية / حرجة* |

> *تصبح الخطورة "حرجة" في حال كان من الممكن الوصول إلى معلومات الهوية الشخصية (PII)، أو بيانات الاعتماد، أو النسخ الاحتياطية الخاصة ببيئة الإنتاج، أو في حال تم تمكين الوصول للكتابة غير المصادق عليه.

### هل تم تحديد واستقراء نقاط نهاية التخزين السحابي (Buckets/Containers)؟
- [ ] لا — لا يتم استخدام نقاط نهاية تخزين سحابي أو لا يمكن اكتشافها  
- [ ] نعم — تم تحديد نقاط نهاية التخزين عبر تحليل الكود أو مراقبة حركة مرور البيانات (Traffic Monitoring)  
- [ ] نعم — تم اكتشاف نقاط نهاية التخزين عبر الهجوم الشامل على اتفاقيات التسمية (Naming Convention Brute-force)  

### هل يمكن للمستخدمين غير المصرح لهم (Unauthenticated) سرد محتويات التخزين السحابي؟
- [ ] لا — ميزة سرد محتويات الحاوية (Bucket Listing) **معطلة** وتعيد استجابة 403 Forbidden أو 404 Not Found  
- [ ] نعم — ميزة السرد **ممكنة** ولكن لا توجد ملفات حساسة  
- [ ] نعم — ميزة السرد **ممكنة** وأسماء الملفات الحساسة/البيانات الوصفية (Metadata) مرئية  

### هل من الممكن تحميل ملفات حساسة من الحاويات التي تم تحديدها؟
- [ ] لا — الملفات محمية بواسطة سياسات IAM أو تتطلب مصادقة صالحة  
- [ ] نعم — يمكن الوصول إلى الملفات ولكنها تتطلب رابطًا موقعًا (Signed URL) **لا يمكن** تخمينه بسهولة  
- [ ] نعم — الملفات الحساسة (النسخ الاحتياطية، ملفات `.env`، معلومات PII) **متاحة للعامة** للتحميل *(حرجة)*  

### هل يسمح برفع الملفات أو تعديلها دون مصادقة؟
- [ ] لا — عمليات الرفع أو التعديل غير المصادق عليها **غير ممكنة**  
- [ ] نعم — المصادقة مطلوبة للرفع ولكن **يمكن تجاوزها** عبر شروط سياسات ضعيفة  
- [ ] نعم — الكتابة، أو الاستبدال، أو حذف الملفات دون مصادقة **ممكن** *(حرجة)*  

### هل سياسات مشاركة الموارد عبر الأصول (CORS) على نقطة نهاية التخزين تقييدية؟
- [ ] نعم — سياسة CORS **معطلة** أو مقتصرة على أصول موثوقة محددة  
- [ ] لا — سياسة CORS **ممكنة** باستخدام علامة النجمة `*` كأصل، مما يسمح بالوصول غير المصرح به عبر الويب
