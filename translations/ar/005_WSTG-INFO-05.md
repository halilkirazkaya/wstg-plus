## WSTG-INFO-05 — مراجعة محتوى صفحات الويب بحثاً عن تسريب المعلومات (Information Leakage)

تتضمن عملية مراجعة محتوى صفحات الويب الفحص اليدوي والآلي لاستجابات التطبيق، بما في ذلك ملفات HTML و JavaScript و CSS، لتحديد المعلومات الحساسة التي قد تُكشف للمستخدمين عن غير قصد. يقوم المهاجمون بفحص الكود المصدري (Source Code) من جانب العميل بحثاً عن تعليقات المطورين (Developer Comments)، وحقول النماذج المخفية (Hidden Form Fields)، ومسارات الخادم الداخلية (Internal Server Paths)، ومفاتيح واجهة برمجة التطبيقات المضمنة برمجياً (Hardcoded API Keys)، ومعلومات تصحيح الأخطاء (Debugging Information) التي يمكن أن تساعد في مراحل الاستغلال اللاحقة. يحدث هذا التسريب غالباً عندما ينسى المطورون إزالة البيانات الوصفية (Metadata) أو كود تصحيح الأخطاء قبل الانتقال إلى مرحلة الإنتاج، مما قد يكشف عن عيوب منطقية (Logic Flaws) أو تفاصيل البنية التحتية الخلفية (Backend Infrastructure). من منظور المهاجم، يوفر هذا وسيلة منخفضة الجهد لرسم خريطة للهيكل الداخلي للتطبيق واكتشاف نقاط الدخول (Entry Points) التي تم التغافل عنها دون إطلاق تنبيهات أمنية أو التفاعل مع المنطق البرمجي للخادم.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-INFO-05 |
| **CWE** | CWE-200 |
| **حالة الاختبار** | لم يتم تنفيذه |
| **الخطورة** | منخفضة / متوسطة* |

> *تصبح الخطورة "عالية" (High) في حال اكتشاف بيانات اعتماد مضمنة برمجياً، أو مفاتيح خاصة، أو متغيرات بيئة داخلية حساسة.

### هل توجد تعليقات للمطورين تحتوي على معلومات حساسة في الكود المصدري؟
- [ ] لا — لم يتم العثور على تعليقات أو وجدت تعليقات عامة غير حساسة فقط  
- [ ] نعم — توجد تعليقات ولكنها لا تحتوي على أي منطق أو بيانات حساسة  
- [ ] نعم — التعليقات **تكشف** عن مسارات داخلية، أو استعلامات SQL، أو تعليمات إدارية  
- [ ] نعم — التعليقات **تكشف** عن بيانات اعتماد أو ملاحظات مطورين حساسة *(عالية)*  

### هل تحتوي حقول إدخال HTML المخفية على بيانات حساسة أو معلومات عن الحالة (State Information)؟
- [ ] لا — لم يتم العثور على حقول مخفية أو أنها تحتوي على رموز (Tokens) غير حساسة فقط  
- [ ] نعم — تُستخدم الحقول المخفية للحالة ولكنها **مشفرة** (Encrypted) أو **موقعة** (Signed)  
- [ ] نعم — تحتوي الحقول المخفية على كلمات مرور **بنص صريح** (Plaintext)، أو أدوار المستخدمين، أو معرفات داخلية  

### هل توجد أسرار مضمنة برمجياً (Hardcoded Secrets)، أو مفاتيح API، أو عناوين IP داخلية في ملفات JavaScript؟
- [ ] لا — لم يتم العثور على نصوص حساسة أو أسرار في ملفات JS  
- [ ] نعم — تحتوي ملفات JS على مفاتيح API عامة مع قيود **مناسبة**  
- [ ] نعم — تحتوي ملفات JS على مفاتيح API **غير محمية**، أو عناوين IP داخلية، أو نقاط نهاية خاصة (Private Endpoints)  
- [ ] نعم — تحتوي ملفات JS على بيانات اعتماد إدارية **مضمنة برمجياً** أو مفاتيح خاصة *(حرجة)*  

### هل يكشف التطبيق عن إصدارات إطارات العمل (Frameworks) أو مسارات الملفات الداخلية في المصدر؟
- [ ] لا — معلومات إطار العمل ومسارات الملفات **مصغرة** (Minified) أو **مموهة** (Obfuscated)  
- [ ] نعم — أسماء وإصدارات إطارات العمل **مرئية** ولكنها محدثة (Patched)  
- [ ] نعم — مسارات الملفات الداخلية أو مسارات الخادم المطلقة **مكشوفة** في الكود المصدري  

### هل الكود المصدري عرضة للتسريب بسبب الافتقار إلى التصغير (Minification) أو التمويه (Obfuscation)؟
- [ ] لا — كود الإنتاج **مصغر بالكامل** وتمت إزالة التعليقات منه  
- [ ] نعم — الكود مصغر ولكن التعليقات **لا تزال موجودة** في المصدر  
- [ ] نعم — خرائط المصدر (Source Maps - ملفات `.map`) **متاحة للعموم**، مما يسمح بإعادة بناء الكود المصدري بالكامل
