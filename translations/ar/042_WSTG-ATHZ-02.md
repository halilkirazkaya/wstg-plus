## WSTG-ATHZ-02 — اختبار تجاوز مخططات التصريح (Testing for Bypassing Authorization Schema)

يحدث تجاوز مخططات التصريح (Bypassing Authorization Schema) عندما يفشل التطبيق في فرض ضوابط الوصول، مما يسمح للمهاجم بالوصول إلى الموارد أو الوظائف خارج نطاق صلاحياته المقصودة. تظهر هذه الثغرة عادةً في شكل تصعيد الصلاحيات الأفقي (Horizontal Privilege Escalation)، حيث يصل المهاجم إلى بيانات مستخدم آخر بنفس المستوى، أو تصعيد الصلاحيات الرأسي (Vertical Privilege Escalation)، حيث يكتسب مستخدم ذو صلاحيات منخفضة قدرات إدارية. يستغل المهاجمون هذه العيوب عن طريق التلاعب بمعاملات الطلب مثل معرفات الموارد (resource IDs)، أو تعديل رموز الجلسة (session tokens)، أو استخدام ترويسات HTTP مثل `X-Original-URL` للالتفاف على القيود المستندة إلى المسار. يعد ضمان وجود تصريح قوي أمراً بالغ الأهمية للحفاظ على سرية البيانات ومنع الإجراءات الإدارية غير المصرح بها التي قد تؤدي إلى اختراق النظام بأكمله.

### هل يتم منع تصعيد الصلاحيات الأفقي بين المستخدمين الذين يمتلكون نفس الدور؟
- [ ] نعم — **يتم تطبيق** فحص التصريح على كل طلب مورد و**لا يمكن** التجاوز  
- [ ] نعم — **يتم تطبيق** فحص التصريح ولكن يمكن تجاوزه عبر المراجع المباشرة غير الآمنة للكائنات (IDOR) أو التلاعب بالمعاملات (parameter tampering)  
- [ ] لا — **يمكن** للمستخدمين الوصول إلى بيانات تعود لمستخدمين آخرين بمجرد تغيير معرف المورد *(عالية)*  

### هل يتم منع تصعيد الصلاحيات الرأسي للمستخدمين ذوي الصلاحيات المنخفضة؟
- [ ] لا — التطبيق يحتوي على مستوى صلاحيات واحد فقط  
- [ ] نعم — **لا يمكن** الوصول إلى الوظائف الإدارية من قبل المستخدمين ذوي الصلاحيات المنخفضة  
- [ ] نعم — الوظائف الإدارية مخفية في واجهة المستخدم ولكن **يمكن** الوصول إليها عبر عنوان URL مباشر  
- [ ] لا — **يمكن** للمستخدمين ذوي الصلاحيات المنخفضة القيام بإجراءات إدارية من خلال التلاعب بالأدوار أو الصلاحيات *(حرجة)*  

### هل يمكن تجاوز التصريح باستخدام التلاعب بأفعال HTTP (HTTP verb tampering)؟
- [ ] لا — يتم فرض ضوابط الوصول بغض النظر عن طريقة (method) HTTP المستخدمة  
- [ ] نعم — يؤدي تغيير الطريقة (على سبيل المثال، من `GET` إلى `POST` أو `PUT`) إلى **تجاوز** فحوصات التصريح  

### هل يمكن الوصول إلى النهايات الطرفية (endpoints) الإدارية أو المقيدة دون أي مصادقة؟
- [ ] لا — تتطلب جميع النهايات الطرفية الحساسة جلسة صالحة وصلاحيات مناسبة  
- [ ] نعم — يمكن الوصول إلى بعض النهايات الطرفية الإدارية إذا عرف المهاجم عنوان URL المباشر  
- [ ] نعم — الوصول غير المصادق عليه للوظائف الحساسة **ممكن** *(حرجة)*  

### هل تسمح ترويسات HTTP بالالتفاف على التصريح المستند إلى المسار؟
- [ ] لا — التطبيق **ليس** عرضة لعمليات التجاوز المستندة إلى الترويسات  
- [ ] نعم — ترويسات مثل `X-Original-URL` أو `X-Rewrite-URL` أو `X-Forwarded-For` **يمكن** استخدامها لتجاوز ضوابط الوصول
