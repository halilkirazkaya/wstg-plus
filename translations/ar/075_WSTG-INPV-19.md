## WSTG-INPV-19 — اختبار تزوير الطلبات عبر الخادم (Server-Side Request Forgery - SSRF)

يحدث تزوير الطلبات عبر الخادم (Server-Side Request Forgery - SSRF) عندما يتمكن المهاجم من التأثير على تطبيق ويب لإجراء طلبات غير مصرح بها من الخادم إلى موارد داخلية أو خارجية. من خلال التلاعب بالمعلمات التي تتوقع عناوين URL، أو أسماء المضيفين، أو عناوين IP، يمكن للمهاجم تجاوز ضوابط مستوى الشبكة مثل جدران الحماية (Firewalls) وقوائم التحكم في الوصول (ACLs) لاستكشاف قطاعات الشبكة الداخلية، أو الوصول إلى خدمات البيانات الوصفية للسحاب (Cloud Metadata Services - IMDS)، أو التفاعل مع واجهات البرمجيات التطبيقية (APIs) وقواعد البيانات الداخلية الضعيفة. تظهر هذه الثغرة عادةً في الميزات التي تتضمن جلب الصور، أو إنشاء ملفات PDF، أو الـ Webhooks، أو تحميل الملفات عبر عنوان URL. من منظور المهاجم، يعد SSRF أداة قوية تُستخدم للانتقال الجانبي (Pivot) إلى بيئات معزولة، أو استخراج بيانات التكوين الحساسة، أو من المحتمل تحقيق تنفيذ الأوامر عن بُعد (Remote Code Execution - RCE) من خلال التفاعل مع الخدمات الداخلية مثل Redis أو Memcached.

### هل يقوم التطبيق بمعالجة عناوين URL أو أسماء مضيفين يوفرها المستخدم؟
- [ ] لا — لا توجد ميزات تقبل عناوين URL أو أسماء مضيفين خارجية  
- [ ] نعم — الميزة موجودة ولكن يتم التحقق من المدخلات بصرامة مقابل قائمة سماح (Allow-list)  
- [ ] نعم — الميزة موجودة وتقبل عناوين URL المقدمة من المستخدم  

### هل يتم تطبيق ضوابط التحقق أو قوائم الحظر على الوجهة المطلوبة؟
- [ ] نعم — يتم فرض قائمة سماح (Allow-list) صارمة للنطاقات/عناوين IP الموثوقة  
- [ ] نعم — يتم استخدام قائمة حظر (Deny-list) (مثل حظر `127.0.0.1` و `169.254.169.254`)  
- [ ] لا — لا يتم تطبيق أي تحقق أو تصفية (Filtering) على المدخلات  

### هل من الممكن تجاوز الفلاتر عبر الترميز، أو إعادة التوجيه، أو إعادة ربط نظام أسماء النطاقات (DNS Rebinding)؟
- [ ] لا — الفلاتر قوية وتتعامل مع إعادة التوجيه (Redirects) وحل نظام أسماء النطاقات (DNS Resolution) بشكل آمن  
- [ ] نعم — التجاوز ممكن باستخدام ترميز URL أو تنسيقات IP بديلة (Hex, Octal)  
- [ ] نعم — التجاوز ممكن عبر إعادة توجيه HTTP من فئة 3xx إلى أهداف داخلية  
- [ ] نعم — التجاوز ممكن عبر هجمات إعادة ربط نظام أسماء النطاقات (DNS Rebinding) للتحليل إلى عناوين IP داخلية  

### هل يمكن للتطبيق الوصول إلى خدمات البيانات الوصفية الداخلية أو الواجهات المحلية؟
- [ ] لا — لا يمكن الوصول إلى الشبكة المحلية ونقاط نهاية البيانات الوصفية للسحاب (Cloud Metadata Endpoints)  
- [ ] نعم — الوصول إلى المضيف المحلي (`127.0.0.1`) أو خدمات الاسترجاع (Loopback) ممكن  
- [ ] نعم — الوصول إلى خدمات البيانات الوصفية للسحاب (مثل AWS/GCP/Azure IMDS) ممكن *(حرجة - Critical)*  

### ما هي طبيعة استجابة SSRF؟
- [ ] أعمى (Blind) — لا يتم إرجاع أي استجابة أو بيانات وصفية للمستخدم  
- [ ] جزئي (Partial) — يتم إرجاع البيانات الوصفية للاستجابة (Headers, Size, Timing) للمستخدم  
- [ ] كامل (Full) — يتم عرض جسم الاستجابة الكامل (Response Body) من المورد الداخلي للمستخدم
