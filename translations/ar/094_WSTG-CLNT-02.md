## WSTG-CLNT-02 — اختبار تنفيذ الجافا سكريبت (JavaScript Execution)

يركز اختبار تنفيذ الجافا سكريبت (JavaScript execution testing) على تحديد الحالات التي يتعامل فيها التطبيق بشكل غير صحيح مع البيانات المدخلة من المستخدم (User-supplied data)، مما يؤدي إلى تنفيذ نصوص برمجية غير مصرح بها داخل سياق متصفح الضحية. تؤدي هذه الثغرة عادةً إلى حقن البرامج النصية عبر المواقع (Cross-Site Scripting - XSS)، مما يسمح للمهاجمين باستخراج ملفات تعريف ارتباط الجلسة (Session cookies)، أو التلاعب بنموذج كائن المستند (Document Object Model - DOM)، أو تنفيذ إجراءات نيابة عن المستخدم. يفحص مختبرو الاختراق المصبات (Sinks) مثل `innerHTML` و `document.write()` و `eval()` حيث يمكن معالجة البيانات غير المعقمة القادمة من مصادر (Sources) مثل أجزاء عنوان URL (URL fragments) أو `localStorage` أو `window.name`. يؤدي الاستغلال الناجح إلى المساس بسلامة وسرية جلسة المستخدم ويمكن أن يعمل كنقطة انطلاق لهجمات أكثر تعقيداً على جانب العميل (Client-side attacks).

### هل يقوم التطبيق بمعالجة البيانات المدخلة من المستخدم في مصبات (Sinks) جافا سكريبت خطيرة؟
- [ ] لا — يتم تعقيم جميع البيانات أو استخدامها في مصبات آمنة مثل `textContent`  
- [ ] نعم — يتم معالجة البيانات في مصبات خطيرة ولكن **يتم تطبيق** التعقيم أو الترميز (Encoding)  
- [ ] نعم — يتم معالجة البيانات في مصبات خطيرة و**لا يتم تطبيق** التعقيم  

### هل توجد سياسة أمان المحتوى (Content Security Policy - CSP) للتخفيف من تنفيذ النصوص البرمجية؟
- [ ] نعم — تم **تفعيل** سياسة CSP تقييدية وتجاوزها (Bypass) **غير ممكن**  
- [ ] نعم — تم **تفعيل** سياسة CSP ولكن **يمكن تجاوزها** عبر `unsafe-inline` أو شبكات توصيل المحتوى (CDNs) غير الآمنة  
- [ ] لا — لم يتم **تفعيل** أي سياسة CSP  

### هل يمكن تنفيذ الجافا سكريبت عبر معاملات URL أو الأجزاء (DOM-based XSS)؟
- [ ] لا — يتم ترميز المدخلات بشكل صحيح والتنفيذ **غير ممكن**  
- [ ] نعم — تنعكس المدخلات في DOM ولكن يتم **منع التنفيذ** بواسطة حمايات إطارات العمل الحديثة  
- [ ] نعم — يتم تنفيذ المدخلات مباشرة في المتصفح، والاستغلال **ممكن**  

### هل معالجات الأحداث (Event handlers) أو مخططات URI عرضة لحقن النصوص البرمجية؟
- [ ] لا — يتم تعقيم المدخلات لإزالة سمات الأحداث ومخططات `javascript:`  
- [ ] نعم — **يمكن** حقن معالجات الأحداث ولكن الفلاتر تحد من تعقيد الحمولة (Payload)  
- [ ] نعم — تنفيذ جافا سكريبت عشوائي عبر سمات `onerror` أو `onload` أو `href` **أمر ممكن**
