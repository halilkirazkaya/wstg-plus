## WSTG-INPV-03 — اختبار التلاعب بأفعال بروتوكول نقل النص الفائق (HTTP Verb Tampering)

يستغل التلاعب بأفعال بروتوكول نقل النص الفائق (HTTP Verb Tampering) نقاط الضعف في كيفية قيام خوادم الويب وأطر عمل التطبيقات بتفويض الوصول إلى موارد معينة بناءً على طريقة HTTP (HTTP method) المستخدمة. يحاول المهاجمون تجاوز القيود الأمنية من خلال استبدال الطرق القياسية مثل `GET` أو `POST` ببدائل مثل `HEAD` أو `PUT` أو `OPTIONS` أو حتى سلاسل نصية عشوائية غير قياسية قد تعالجها الواجهة الخلفية بشكل غير متسق. تحدث هذه الثغرة عادةً عندما تقوم التكوينات الأمنية - مثل مرشحات Java EE web.xml أو قواعد تفويض .NET - بإدراج الطرق المسموح بها صراحةً ولكنها تفشل في مراعاة الطرق الأخرى، أو عند استخدام نهج "الرفض حسب الطريقة" (deny-by-method) بدلاً من "رفض الكل" (deny-all). يمكن أن يؤدي الاستغلال (Exploit) الناجح إلى الوصول غير المصرح به إلى الوظائف الإدارية، أو تعديل البيانات، أو الكشف عن معلومات تتعلق بالتكوين الداخلي للخادم.


| الحقل | القيمة |
|---|---|
| **WSTG ID** | WSTG-INPV-03 |
| **CWE** | CWE-288 |
| **حالة الاختبار** | لم يتم تنفيذه |
| **الخطورة** | متوسطة / عالية* |

> *تصبح الخطورة "عالية" إذا كان من الممكن تنفيذ وظائف إدارية أو تعديل البيانات (PUT/DELETE) بدون مصادقة (Authentication).

**المراجع:**

**الأدوات:** `Burp Suite (Repeater/Intruder)`, `curl`, `nmap`, `ffuf`

### هل يستجيب التطبيق لطرق HTTP غير القياسية أو العشوائية؟
- [ ] لا — يرفض التطبيق جميع الطرق باستثناء تلك المطلوبة صراحةً  
- [ ] نعم — يستجيب التطبيق للطرق القياسية (OPTIONS, TRACE) ولكن **لا يمكن** تجاوز الأمان  
- [ ] نعم — يقبل التطبيق سلاسل عشوائية (مثلاً: FOO, TEST) ويعالجها كطلبات `GET` أو `POST`  

### هل يمكن تجاوز المصادقة/التفويض (Authentication/Authorization) عن طريق تغيير طريقة HTTP؟
- [ ] لا — يتم تطبيق الضوابط الأمنية بشكل شامل بغض النظر عن فعل HTTP المستخدم  
- [ ] نعم — الضوابط موجودة ولكن التجاوز **ممكن** باستخدام طريقة `HEAD`  
- [ ] نعم — الضوابط الأمنية **لا يتم تطبيقها** على الطرق البديلة، مما يسمح بالوصول غير المصرح به إلى النهايات الطرفية (Endpoints) المقيدة  

### هل الطرق الخطرة مثل `PUT` أو `DELETE` مُمكّنة على الخادم؟
- [ ] لا — الطرق الخطرة **معطلة** أو تعيد رمز الحالة 405 Method Not Allowed  
- [ ] نعم — الطرق مُمكّنة ولكنها تتطلب مصادقة صالحة وذات صلاحيات عالية  
- [ ] نعم — الطرق **مُمكّنة** وتسمح برفع ملفات غير مصرح بها أو حذف الموارد *(حرجة)*  

### هل تكشف طريقة `OPTIONS` عن معلومات حساسة حول الأفعال المسموح بها؟
- [ ] لا — طريقة `OPTIONS` **معطلة** أو تعيد استجابة عامة  
- [ ] نعم — تعيد `OPTIONS` رأس `Allow` ولكنها لا تكشف عن طرق داخلية مقيدة  
- [ ] نعم — تكشف `OPTIONS` عن طرق داخلية فقط أو إدارية تساعد في المزيد من الاستغلال  

### هل طريقة `TRACE` مُمكّنة، مما قد يسمح بهجوم تتبع المواقع المشترك (Cross-Site Tracing - XST)؟
- [ ] لا — طريقتا `TRACE` و `TRACK` **معطلتان**  
- [ ] نعم — طريقة `TRACE` **مُمكّنة** وتقوم بعكس رؤوس HTTP، بما في ذلك ملفات تعريف الارتباط (Cookies) أو الرموز المميزة (Tokens) الحساسة
