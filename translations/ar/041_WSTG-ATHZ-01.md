## WSTG-ATHZ-01 — اختبار التنقل بين المجلدات وتضمين الملفات (Directory Traversal File Include)

تنشأ ثغرات التنقل بين المجلدات (Directory Traversal) وتضمين الملفات (File Inclusion) عندما يستخدم التطبيق مدخلات يمكن للمستخدم التحكم بها لإنشاء مسارات للملفات أو المجلدات دون التحقق من الصحة (Validation) أو التطهير (Sanitization) الكافي. يقوم المهاجمون باستغلال هذه الثغرات عن طريق حقن سلاسل مثل `../` للتنقل خارج المجلد المقصود، مما قد يؤدي إلى الوصول إلى ملفات النظام الحساسة، أو بيانات التكوين، أو الكود المصدري للتطبيق. في الحالات الأكثر خطورة التي تشمل تضمين الملفات المحلية (LFI) أو تضمين الملفات عن بعد (RFI)، قد يتمكن المهاجم من تحقيق تنفيذ الأوامر البرمجية عن بعد (Remote Code Execution - RCE) من خلال تضمين برامج نصية خبيثة أو الاستفادة من تسميم سجلات النظام (Log Poisoning) ومغلفات PHP (PHP wrappers). توجد هذه الثغرات عادةً في المعاملات (Parameters) المستخدمة لتحميل المحتوى الديناميكي، أو محركات القوالب (Template Engines)، أو نقاط نهاية استرداد الصور حيث يتعامل المنطق البرمجي من جانب الخادم مع مسارات نظام الملفات.

| **CWE** | CWE-22 |
| **حالة الاختبار** | لم يتم تنفيذه |
| **الخطورة** | عالية / حرجة |

### هل تقبل المعاملات أسماء ملفات أو مسارات للمعالجة من جانب الخادم؟
- [ ] لا — لا تظهر أي معاملات تتفاعل مع نظام الملفات  
- [ ] نعم — توجد معاملات ولكنها تستخدم قائمة سماح (Allowlist) صارمة لمعرفات الملفات  
- [ ] نعم — تقبل المعاملات أسماء ملفات أو مسارات مباشرة  

### هل يتم تطهير المدخلات لمنع سلاسل التنقل بين المجلدات؟
- [ ] نعم — يتم التحقق من صحة المدخلات مقابل قائمة سماح صارمة وسلاسل التنقل **غير ممكنة**  
- [ ] نعم — يتم تطهير المدخلات عن طريق إزالة سلاسل `../` ، ولكن التجاوز التكراري (Recursive Bypass) **ممكن**  
- [ ] لا — لا يتم تطبيق أي تطهير أو تحقق من الصحة على المدخلات المتعلقة بالمسار  

### هل من الممكن الوصول إلى ملفات خارج المجلد المقيد عبر سلاسل التنقل؟
- [ ] لا — يمنع التطبيق أو نظام التشغيل الوصول إلى الملفات خارج النطاق المحدد  
- [ ] نعم — الوصول إلى الملفات داخل جذر الويب (Web Root) **ممكن**  
- [ ] نعم — الوصول إلى ملفات النظام الحساسة (مثل `/etc/passwd` ، `C:\Windows\win.ini`) **ممكن** *(حرجة)*  

### هل يسمح الخادم بتضمين عناوين URL عن بعد (RFI)؟
- [ ] لا — تضمين الملفات عن بعد **معطل** على مستوى الخادم/التطبيق  
- [ ] نعم — **يمكن** تضمين الملفات عن بعد ولكن التنفيذ **غير ممكن**  
- [ ] نعم — **يمكن** تضمين الملفات عن بعد وتنفيذها على الخادم *(حرجة)*  

### هل يمكن تجاوز الفلاتر باستخدام الترميز أو الأحرف الخاصة؟
- [ ] لا — تتعامل الفلاتر بفعالية مع مختلف الترميزات والـ Null Bytes  
- [ ] نعم — التجاوز **ممكن** باستخدام ترميز URL، أو ترميز URL المزدوج (Double URL Encoding)، أو Unicode بـ 16 بت  
- [ ] نعم — التجاوز **ممكن** باستخدام حقن البايت الفارغ (Null Byte Injection - `%00`) أو مغلفات نظام الملفات (مثل `php://filter`)
